## 摘要
LLM智能体越来越多地通过外部工具自主执行操作，导致复杂且不断演变的安全风险。然而，现有安全测试针对专家设计的安全违规行为，且相应结果由硬编码规则评估，随着智能体演变，扩展这些规则成本高昂。为此，我们提出Vera，一个端到端的自动化安全测试框架，通过三阶段自强化流水线将软件工程测试原则实例化到非确定性智能体中。首先，文献驱动的探索持续发现新风险并将其结构化组织为安全风险、攻击方法和工具执行环境的分类体系。其次，跨分类维度的组合组合生成可执行安全用例，每个用例指定具体安全目标、程序化构建的初始状态以及基于可观察工件（artifact）的确定性验证谓词。第三，自适应执行在隔离沙箱中运行异构智能体，其中控制智能体基于运行时观察引导多轮交互，而基于证据的验证器从环境状态和工具调用证据（而非模型自述）判断结果。我们在四个生产级智能体框架（OpenClaw、Hermes、Codex、Claude Code）上评估Vera，揭示了重大安全弱点，多通道攻击下平均攻击成功率高达93.9%；我们还发布了Vera-Bench，包含跨三种执行环境的124个风险类别的1600个可执行安全用例。这些结果表明，模块化、可执行的测试基础设施对于快速演进的智能体系统进行严格且可维护的大规模安全评估至关重要。代码公开在 https://github.com/Yunhao-Feng/Vera。

## Abstract
LLM agents increasingly perform autonomous actions through external tools, leading to complex and evolving safety risks. However, existing safety testing targets expert-designed safety violations, and the corresponding outcomes are evaluated by hard-coded rules, making them costly to extend as agents evolve. To this end, we present Vera, an end-to-end automated safety testing framework that instantiates software engineering testing principles for non-deterministic agents through a three-stage, self-reinforcing pipeline. First, a literature-driven exploration continuously discovers and structures emerging risks into taxonomies of safety risks, attack methods, and tool execution environments. Second, combinatorial composition across taxonomy dimensions produces executable safety cases, each specifying a concrete safety goal, a programmatically constructed initial state, and a deterministic verification predicate grounded in observable artifacts. Third, adaptive execution runs heterogeneous agents in isolated sandboxes where a control agent steers multi-turn interaction based on runtime observations, while evidence-grounded verifiers judge outcomes from environment state and tool-call evidence rather than model self-report. We evaluate Vera on four production agent frameworks (OpenClaw, Hermes, Codex, Claude Code), revealing substantial safety weaknesses, with average attack success rates reaching 93.9\% under multi-channel attacks; we also release Vera-Bench, comprising 1600 executable safety cases spanning 124 risk categories across three execution settings. These results indicate that modular, executable testing infrastructure is essential for rigorous and maintainable safety evaluation of rapidly evolving agentic systems at scale. The code is publicly available at https://github.com/Yunhao-Feng/Vera.

### 关键技术细节

1. **阶段一：连续风险探索（Continuous Risk Exploration）**
   - 建立三个正交的分类体系：安全风险分类R（如凭据泄露、未授权修改）、攻击方法分类M（如提示注入、任务分解、角色扮演）、执行环境分类Ω_E（如邮件、代码托管、支付、搜索）。
   - 通过一个摘要代理（Summary Agent）从科研文献（arXiv、OpenReview约800篇）中迭代提取候选概念，对每个分类执行四种操作：创建、更新、合并、删除，使分类体系收敛。
   - 探索迭代公式：\( (T_{t+1}, Q^{(t+1)}) = \Phi(T_t, Q^{(t)}, P^{(t)}) \)，其中T_t为三个分类，P^{(t)}为检索文档，Φ为更新算子，Q^{(t)}为搜索概念集。

2. **阶段二：可执行测试用例构建（Executable Test Case Construction）**
   - **安全目标生成**：从三个分类中各选一个叶子节点，通过LLM目标组合器G将三元组(r, m, e)实例化为具体安全目标g。采用格式示范和动态更新保证输出一致性与多样性。全笛卡尔积经兼容性过滤和去重后产生候选集。
   - **安全用例编译**：每个目标g被编译为三元组σ = ⟨g, s0, V_g⟩，其中s0是程序化构建的初始环境状态（由LLM初始器自动生成，如含密码的邮件、有漏洞依赖的仓库），V_g是确定性验证谓词（Python脚本，检查环境状态和工具调用记录）。初始化序列和验证器均可重放，保证可复现。
   - **过滤与变体生成**：过滤掉依赖内部推理轨迹或重复的用例。每个基本场景扩展为三个变体：良性（无对抗元素）、单通道（仅用户消息对抗，工具结果不变）、多通道（用户消息+工具结果注入对抗内容）。

3. **阶段三：自适应执行与基于证据的验证（Adaptive Execution and Evidence-Grounded Verification）**
   - **沙箱执行环境**：通过MCP（Model Context Protocol）中间件统一接入多种代理框架，每个执行实例拥有独立的Docker Compose栈（12个容器含5个自托管后端服务）。工具网关记录原始结果f_{i,j}和返回给代理的观察值˜f_{i,j}，并在多通道模式下应用四种注入模式（恒等、追加、前缀、覆盖）。
   - **自适应测试驱动**：控制代理（Control Agent）接收安全用例和环境模式，在每个交互回合基于状态c_i选择用户消息u_i和网关规则J_i。观察目标代理响应、工具调用和环境状态变化后，动态调整策略（如任务分解、重新措辞、换注入点）。循环直到预算耗尽、证据产生或无法进展。
   - **基于证据的验证**：验证器V_g按操作抵抗性优先级选择证据源：环境状态 > 工具调用记录 > 代理响应。环境状态优先，因为工具调用只表明意图，不保证效果；代理响应仅在文本输出本身构成违规时使用。验证在沙箱存活期间运行，可查询实时API。采用不对称判断：控制代理报告失败时直接标记y=0，报告成功时需验证器确认后才标记y=1，避免假阳性。

1. **整体攻击成功率极高**：多通道攻击平均ASR达93.9%，单通道达90.6%，在所有四个框架上均表现显著安全弱点。

## 7. 优点

- **威胁模型全面**：支持单通道和多通道攻击，可探测不同交互表面的鲁棒性差异。
- **开源与可复现**：代码公开，所有环境构造和验证脚本可重放。
- **下游应用价值**：证明可用生成的测试数据训练更好的安全分类器。
