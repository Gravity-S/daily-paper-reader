## 摘要
自主代理，无论是基于LLM的软件代理还是机器人物理代理，在缺乏持续人类监督的情况下运行时都会面临一类常见的故障模式：来自未经验证动作的安全违规、来自无约束循环的行为不稳定，以及来自未处理错误状态的连续性丢失。我们开发了\system{}，一个离散时间控制系统，它结合了五个执行档位（\Gobs{}、\Gsug{}、\Gplan{}、\Gexec{}、\Gint{}）与效用门控调度和事件驱动的回退。对于单代理情况，我们证明了单调稳定性、执行安全性、最终稳定、回退完备性，以及等价于档位约束的马尔可夫决策过程。对于多代理信息物理系统（CPS），我们应用已建立的\smart{}管理自主生命周期，并将运行时证据映射到其四个治理状态（\Stable{}/\Meta{}/\Assisted{}/\Regulated{}）。共识门控、群体级Lyapunov分析、每个代理的档位权限和会合控制提供了分布式安全性和稳定性保证，包括在所述假设下的零碰撞。我们在一个三代理UR5机器人装配单元上评估了所得运行时，使用从NIST\emph{机器人手臂位置精度退化测量}数据集校准的故障幅度，跨越10,000个蒙特卡洛情节。它实现了99.6%的异常检测率，而单代理基线为2.1%，检测延迟降低了$3.5\times$，并提供了正式物理工作空间安全证书。执行档位充当\smart{}运行时治理状态下的微级权限，将动作控制与自主治理分离。

## Abstract
Autonomous agents, whether LLM-driven software agents or robotic physical agents, face a common class of failure modes when operating without continuous human oversight: safety violations from unverified actions, behavioral instability from unconstrained loops, and continuity loss from unhandled error states. We develop \system{}, a discrete-time control system that combines five execution gears (\Gobs{}, \Gsug{}, \Gplan{}, \Gexec{}, \Gint{}) with utility-gated dispatch and event-driven fallback. For the single-agent case, we prove monotonic stability, execution safety, eventual stabilization, fallback completeness, and equivalence to a gear-constrained Markov decision process. For multi-agent cyber-physical systems (CPS), we apply the established \smart{} managed-autonomy lifecycle and map runtime evidence into its four governance states (\Stable{}/\Meta{}/\Assisted{}/\Regulated{}). Consensus gating, swarm-level Lyapunov analysis, per-agent gear authority, and rendezvous control provide distributed safety and stability guarantees, including zero collision under the stated assumptions. We evaluate the resulting runtime on a three-agent UR5 robotic assembly cell using fault magnitudes calibrated from the NIST \emph{Degradation Measurement of Robot Arm Position Accuracy} dataset across 10,000 Monte Carlo episodes. It achieves a 99.6\% anomaly detection rate versus 2.1\% for the single-agent baseline, reduces detection latency by $3.5\times$, and supplies a formal physical-workspace safety certificate. The execution gears act as micro-level permissions beneath the \smart{} runtime governance states, separating action control from autonomy governance.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：自主代理（包括LLM驱动的软件代理和机器人物理代理）在缺乏持续人类监督时面临三类常见故障模式：
  - **安全违规**：未经验证的行动导致不可逆副作用。
  - **行为不稳定**：代理可能在策略间振荡、无法收敛或进入退化循环。
  - **连续性丢失**：代理意外停止，丢失累积上下文，需要昂贵的重启。
  - 对于多代理信息物理系统（CPS），还出现**协调盲视**：一个代理的传感器故障影响所有邻居，但逐代理控制层无法检测或响应。

- **整体含义**：现有自主代理的授权是二元的、静态的，缺乏根据观察到的安全信号动态调整允许行动范围的机制。论文旨在通过设计一个运行时控制系统，将动作控制与自主治理分离，提供可验证的安全和稳定性保证。

## 2. 论文提出的方法论

### 2.1 核心思想
- **EntropyRuntime**：一个离散时间闭环控制系统，位于LLM代理及其执行环境之间。包含五个执行档位（G0~G4），结合效用门控调度和事件驱动回退，动态限制代理的行动范围。

- **多代理扩展**：使用SMARt管理自主生命周期，将运行时证据映射到四个治理状态（STABLE/META-COGNITIVE/ASSISTED/REGULATED），并引入共识效用门、群体Lyapunov函数、每代理档位权限和会合策略三元组。

### 2.2 关键技术细节
- **档位状态空间**：G0（观察/安全保持）、G1（建议，无外部副作用）、G2（规划，允许有界可逆恢复动作）、G3（执行，独立选择有副作用的动作）、G4（集成，系统级协调）。行动空间严格嵌套：A0 ⊂ A1 ⊂ A2 ⊂ A3 ⊂ A4 = A。
- **效用门控**：U(s,a) = α·Δ_task + β·safety - γ·cost。行动需满足U(s,a) ≥ θ才能执行，否则触发回退。
- **事件驱动回退**：行动被拒绝后，记录拒绝并增加σ，生成替代行动；若k次失败则降档；连续m次失败进入G0等待人类审查。
- **多代理共识门**：GATE_c = 1当且仅当所有代理的局部效用U_i ≥ θ。任何代理低于阈值都会阻止整体调度，无需通信。

- **会合策略三元组**：指定下降策略（完成当前周期/立即下降）、持有策略（严格依赖/独立继续）和返回策略（专家明确/自动继续[δ]/重启复位）。

### 2.3 公式与算法流程（文字说明）
- **单代理控制循环**：读取环境状态st和当前档位gt → LLM生成候选动作at ∈ A_gt → 效用门评估GATE(st, at) → 若接受则执行并更新σ递减；若拒绝则触发回退：寻找替代行动，成功则执行，失败则σ递增、档位降级并设置错误标志 → 更新档位gt+1。

## 3. 实验设计

### 3.1 数据集与场景

- **检测延迟降低**：平均检测延迟从43.1周期降至12.2周期，降低3.5倍。
- **碰撞率为零**：两种条件下物理碰撞均为零，但基线零碰撞是架构假象（UR5编码器与摄像头隔离）。治理运行时在反馈耦合系统中提供可证明的碰撞避免。

- **单代理理论保证**：证明了单调稳定性（期望σ不增）、执行安全性（仅正效用行动执行）、最终稳定（到达固定档位）、回退完备性（最多4步降档至G0）及MDP表示定理。

## 7. 优点
