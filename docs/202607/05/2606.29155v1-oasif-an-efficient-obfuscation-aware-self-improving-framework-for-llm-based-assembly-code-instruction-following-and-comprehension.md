## 摘要
大型语言模型（LLM）最近在自动化二进制分析中展现出潜力，但在商业级混淆下仍然脆弱。我们提出了OASIF，一种用于混淆汇编代码理解的可感知混淆的自演化指令遵循框架。OASIF将一个标记高效的汇编编码器与一个轻量级投影器相结合，在有限的上下文预算下将长混淆代码暴露给预训练的代码LLM，并采用三阶段训练：（i）特征空间对齐，（ii）监督指令微调，以及（iii）具有混合奖励的在线自演化强化学习，从而在最小人工验证下实现持续适应。在VMISA-Bench（一个包含三种基于虚拟机的商业混淆器的具有挑战性的分布外测试集）上，OASIF持续改进开源基座模型；Qwen2.5-Coder-Instruct-14B在Code Virtualizer、Themida (v3.0.7)和VMProtect (v3.5)上分别实现了+15.9、+5.8和+16.9个百分点的成功率提升，并将OASIF-Bench的平均值提升了+9.8个百分点。OASIF在七个标准BCSD基准测试上进一步提供了稳定的增益，同时在HumanEval、VulBench和HumanEval-Decompile上保持了通用和领域相关能力。

## Abstract
Large Language Models (LLMs) have recently shown promise in automated binary analysis, yet they remain brittle under commercial-grade obfuscation. We present OASIF, an Obfuscation-Aware Self-evolving Instruction-Following framework for obfuscated assembly comprehension. OASIF couples a token-efficient assembly encoder with a lightweight projector to expose long obfuscated code to a pretrained code LLM under a bounded context budget and follows a three-phase training: (i) feature-space alignment, (ii) supervised instruction fine-tuning, and (iii) online self-evolving reinforcement learning with hybrid rewards, enabling continual adaptation with minimal manual verification. On VMISA-Bench, a challenging out-of-distribution suite featuring three commercial VM-based obfuscators, OASIF consistently improves open-source backbones; Qwen2.5-Coder-Instruct-14B attains Success Rate gains of +15.9, +5.8, and +16.9 percentage points (pp) on Code Virtualizer, Themida (v3.0.7), and VMProtect (v3.5), respectively, and improves the OASIF-Bench average by +9.8. OASIF further delivers stable gains across seven standard BCSD benchmarks while preserving general and domain-relevant capabilities on HumanEval, VulBench, and HumanEval-Decompile.

# OASIF 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

大型语言模型（LLM）在自动化二进制分析中展现出潜力，但在面对商业级代码混淆，尤其是基于虚拟机的混淆（VM-based obfuscation）时，表现脆弱。商用混淆技术（如 Code Virtualizer、Themida、VMProtect）会将原生指令转换为专有虚拟指令集，导致汇编代码急剧膨胀（可达数十万指令），控制流复杂、上下文切换频繁，远超开源 LLM 的上下文窗口。现有方法多依赖静态监督，难以获得高质量的标注数据，且无法适应未见过的混淆模式。因此，迫切需要一种能高效处理长序列、可自适应的框架，以提升 LLM 对混淆汇编的指令遵循与理解能力。

## 2. 方法论：核心思想、关键技术细节、算法流程

**核心思想**：OASIF 结合一个标记高效的汇编编码器、一个轻量级投影器和一个预训练代码 LLM，通过三阶段训练实现混淆汇编的鲁棒理解。第三阶段采用在线自演化强化学习，利用混合奖励不断自我改进，减少人工验证依赖。

**关键技术细节**：
- **Token-Efficient Assembly Representation**：使用专用汇编编码器（CLAP-ASM，110M 参数）将任意长度的混淆汇编代码压缩为固定维度的特征向量，再通过 MLP 投影器（30M 参数）映射到 LLM 的语义空间，从而在有限上下文窗口内处理长序列。
- **特殊标记**：引入 `<inst_start>` 和 `<inst_end>` 分隔指令，结构化输入。
- **三阶段训练**：
  - **Phase I：特征空间对齐** — 冻结 LLM，仅优化投影器，使用简化摘要的生成损失，对齐汇编特征与 LLM 语义空间。
  - **Phase II：指令微调** — 冻结编码器，联合训练投影器和 LLM，使用多种任务数据（详细描述、对话、推理）进行监督微调。
  - **Phase III：在线自演化强化学习** — 采用 GRPO 算法，模型对每个混淆样本采样多个候选回答，利用混合奖励打分，迭代优化策略。奖励函数为：
    - **结构奖励** \(R_{struc}\)：正确识别混淆类型得 1，仅检测出是否存在混淆得 0.2，否则为 0。
    - **语义奖励** \(R_{sem}\)：模型自评估生成回答在帮助性、相关性、准确性、详细性、全面性五维度的得分（基于原始未混淆代码）。
    - **总奖励** \(R_{total} = \lambda R_{struc} + (1-\lambda) R_{sem}\)，\(\lambda=0.5\)。
- **数据引擎**：从 BinaryCorp-3M 和 Juliet Test Suite 编译原始汇编，使用 OLLVM 生成混淆变体（SUB、BCF、FLA、ALL），再通过 LLM 生成合成问答对，经筛选得到三个阶段的数据集：\(D_{align}\)（430K 简化摘要）、\(D_{sft}\)（11.75K 多任务）、\(D_{rl}\)（4.244K 挑战级推理，经人工验证）。

## 6. 主要结论与发现

- **OASIF 持续改进开源 LLM 在混淆汇编理解上的性能**：在 VMISA-Bench 上，Qwen2.5-Coder-Instruct-14B 的 SR 提升 +15.9（Code Virtualizer）、+5.8（Themida）、+16.9（VMProtect）；OASIF-Bench 平均 +9.8。

- **Agentic 管道中 OASIF 可增强前端理解**：将 OASIF 14B 作为前端与 Claude-Opus-4-6 后端结合，在 VMProtect 上 SR 达 62.4%，优于直接用 Qwen3-Coder-Plus 前端（50.6%）。

## 7. 优点

- **三阶段训练框架设计精巧**：逐步从对齐到微调再到自演化，充分利用合成数据与奖励信号，减少人工标注成本。
- **混合奖励机制**：结构奖励提供显式分类信号，语义奖励提供密集、多维的生成质量反馈，避免稀疏奖励问题。
- **专用汇编编码器**：解决长混淆序列的上下文窗口瓶颈，使模型能处理数十万指令。
- **跨模型、跨基准的一致性增益**：验证了方法的通用性，且对较小基座提升更显著。
- **Agentic 管道实验**：展示了 OASIF 作为前端分析模块的实用性，可集成到更大的分析流水线中。
