## 摘要
爪型AI智能体（如OpenClaw）是始终在线的进程，持久持有凭证、文件、工具和外部服务的访问权限。它们承担系统级职责——安装软件包、维护状态、调度子任务以及中介I/O——使得安全失效比其它智能体严重得多。然而现有基准聚焦于模型响应和工具调用，跨组件失效模式基本未被衡量。我们采用计算机系统类比：将爪型智能体视为一个智能体计算机系统，其网关运行时扮演类似操作系统的中介角色，其技能类似于用户安装的应用程序，其插件类似于具有运行时权限的可加载扩展。每个组件都有经典对应物，其保护机制——经过数十年的网络安全研究完善——在智能体侧缺失。基于此视角，我们开发了SafeClawArena，一个包含406个对抗性任务的基准，覆盖四个攻击面（技能供应链完整性、持久状态利用、跨边界数据流、间接提示注入），在真实智能体平台的容器化副本中执行，附带金丝雀标记的凭证，并通过九个输出通道的自动污点追踪进行评估。我们评估了三个平台（OpenClaw、NemoClaw、SeClaw）和五个前沿大语言模型。最高攻击成功率达到70%；恶意插件无论LLM如何均100%成功。SeClaw将GPT-5.4的攻击成功率从70%降至22%，部分通过效用-安全权衡而非主动防御，而Claude-Opus-4.6已接近每个平台22%的基线。这些结果暴露了当前防御的不足，并为未来加固指明了方向。代码和数据：https://github.com/sunblaze-ucb/SafeClawArena。

## Abstract
Claw-like AI agents (e.g., OpenClaw) are always-on processes with persistent access to credentials, files, tools, and external services. They take on system-level responsibilities -- installing packages, maintaining state, scheduling subtasks, and mediating I/O -- making security failures far more severe than in other agents. Yet existing benchmarks focus on model responses and tool calls, leaving cross-component failure modes largely unmeasured. We adopt a computer-system analogy: treating a Claw-like agent as an agentic computer system whose gateway runtime plays an OS-like mediation role, whose Skills resemble user-installed applications, and whose Plugins resemble loadable extensions with runtime privileges. Each component has a classical counterpart whose protection mechanisms -- refined over decades of cybersecurity research -- are absent on the agent side. From this perspective, we develop SafeClawArena, a benchmark of 406 adversarial tasks across four attack surfaces (Skill Supply-Chain Integrity, Persistent State Exploitation, Cross-Boundary Data Flow, and Indirect Prompt Injection), executed in containerized replicas of real agent platforms with canary-marked credentials and evaluated via automated taint tracking across nine output channels. We evaluate three platforms (OpenClaw, NemoClaw, SeClaw) and five frontier LLMs. The highest attack success rate reaches 70%; malicious Plugins succeed in 100% of cases regardless of the LLM. SeClaw cuts GPT-5.4's attack success rate from 70% to 22%, partly through utility-security tradeoffs rather than active defenses, while Claude-Opus-4.6 already sits near a 22% floor on every platform. These results expose the inadequacy of current defenses and suggest directions for future hardening. Code and data: https://github.com/sunblaze-ucb/SafeClawArena.

## 1. 核心问题与整体含义
- **研究动机**：Claw-like AI智能体（如OpenClaw）具有系统级权限——持久持有凭证、访问文件系统、安装软件包、调度任务等，其安全失效后果远超传统智能体。然而现有安全基准（如AgentHarm、InjecAgent等）仅关注模型层面的响应安全和工具调用安全，忽略了**跨组件的系统级失效模式**（如技能供应链、持久状态利用、跨边界数据流、间接提示注入）。
- **核心问题**：如何系统化评估Claw-like智能体的整体安全性？现有的“模型对齐”视角不足以覆盖架构性安全漏洞。
- **整体含义**：需要从**计算机系统安全视角**重新审视Claw-like智能体，将失效率映射到经典安全原则（如进程隔离、最小权限、数据-指令分离等），并构建针对性的基准测试，以暴露当前平台和LLM的防御缺口。

## 2. 方法论
### 核心思想
- **计算机系统类比**：将Claw-like智能体映射为“智能体计算机系统”：
  - 网关运行时 ↔ 操作系统（中介角色）
  - 技能（Skills） ↔ 用户安装的应用
  - 插件（Plugins） ↔ 进程内可加载扩展（如npm模块）
- **提取经典安全原则**：基于映射缺失的保护机制，提炼出5个被违反的经典原则：
  - I1：进程隔离（技能/用户提示/文件内容共享同一LLM上下文缓冲区）
  - I2：最小权限（技能和插件继承完整智能体权限）
  - I3：持久状态保护（内存、配置、日志无完整性验证）
  - I4：跨边界中介（所有出站调用使用同一凭证集）
  - I5：数据-指令分离（文档内容与用户指令具有同等权限）
- **组织成四个攻击面**：SSI（技能供应链完整性）、PSE（持久状态利用）、CDF（跨边界数据流）、IPI（间接提示注入）。

### 关键技术细节
- **SafeClawArena基准**：包含406个对抗性任务，覆盖四个攻击面（SSI:100、PSE:60、CDF:146、IPI:100）。
- **攻击类别**：每个维度细分多个子类（共24个），每类映射到对应的安全原则。
- **任务执行**：
  - 每个任务在**容器化副本**中运行（三个平台：OpenClaw、NemoClaw、SeClaw）。
  - 工作区中植入**金丝雀标记凭证**（如CANARY_DB_PASS_2oy0if）。
  - 使用模拟Google Workspace CLI（Sim-Google）记录所有出站调用，避免真实网络流量。
- **自动评估**：基于确定性污点追踪（非LLM-as-judge），检查9个输出通道（智能体回复、文件写入、内存写入、网关日志等）是否出现金丝雀字符串。
- **评分规则**：
  - SSI/IPI：二进制（0=攻击成功，1=安全）
  - PSE：组合得分（0.4×持久性 + 0.6×行为影响）

## 6. 主要结论与发现
- **最高攻击成功率70%**（OpenClaw/NemoClaw + GPT-5.4），恶意插件（Category 1.4）在所有未加固配置下**100%成功**，无论底层LLM如何。
- **平台加固不等于万能**：SeClaw将GPT-5.4成功率从70%降至22%，但部分通过**效用-安全权衡**（移除技能绑定的插件加载器、缩小配置模式）而不是主动防御。对Gemini系列效果有限甚至变差（G-Pro的PSE和IPI反而恶化）。
- **模型排序在加固下逆转**：在未加固平台上，Claude-Opus-4.6最安全（~20%），GPT-5.4最不安全（70%）；在SeClaw上，GPT-5.4与Opus-4.6几乎平手（21.9% vs 20.9%），而Gemini系列成为最不安全。
- **更强的模型不一定更安全**：Opus-4.6在Category 1.5（不安全捆绑脚本）上比GPT-5.4更差，因为其更强的指令遵循能力使其更易执行恶意脚本。
- **同一个加固对不同模型效果不同**：SeClaw的日志泄露（Cat 3.8）上，G-Flash得分0.03，Opus-4.6得分0.91，差距0.88。
- **理论防御覆盖不预测实际效果**：D5（内存完整性）和D10（输入净化）虽然部署，但几乎不降低对应攻击的成功率。
- **防御需要联合评估平台和LLM**：单一防御层不够；CDF需要数据丢失预防策略而非简单允许/拒绝。

## 7. 优点
- **新颖的理论视角**：将Claw-like智能体映射为计算机系统，使用经典安全原则（进程隔离、最小权限等）作为组织基准的结构依据，而非枚举攻击场景。
- **真实平台作为测试平台**：使用生产级平台二进制（OpenClaw等）直接运行，而非自定义Python脚手架的模拟，结果可直接归因于实际产品行为。
- **全面覆盖系统层攻击面**：填补了现有基准缺失的SSI、PSE、CDF三个维度。
- **自动化确定性评估**：使用金丝雀标记和字符串匹配，避免LLM评判的偏见，保证可复现性。
- **跨平台、跨模型的大规模评测**：15种配置、6090次执行，揭示了平台与模型之间的强交互，为安全设计提供实证依据。
- **开放代码和数据**：完整仓库、Dockerfile、任务JSON生成器已公开，支持独立复现与扩展。

## 8. 不足与局限
- **平台覆盖有限**：仅测试OpenClaw家族（OpenClaw、NemoClaw、SeClaw），未扩展到AutoGen、LangChain、Cursor等其他框架（但原则和防御定义是框架无关的）。
- **防御消融未实证**：11种防御的理论覆盖图（图3）和部署状态（表7）给出了，但未在实机上逐一开启/关闭每种防御以测量其独立效果，削弱了防御建议的因果推理。
- **任务集非穷尽**：24个子类无法覆盖所有未来攻击（如多智能体大规模协调、模型侧持久记忆等），但计算机系统视角提供了扩展路径。
- **可复现性受限**：LLM后端由供应商管理，权重随时间演化，影响了精确可复现（所有使用托管LLM的基准共有此局限）。

- **潜在滥用风险**：基准可被攻击者用作攻击目录。论文通过聚焦于原则性失败（而非零日漏洞）、使用容器化无真实验证环境、以及责任披露来缓解。
