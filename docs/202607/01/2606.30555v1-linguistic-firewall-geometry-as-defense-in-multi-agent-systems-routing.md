## 摘要
大语言模型（LLM）的快速集成推动了多智能体系统（MAS）的发展，其中专业化智能体协作执行复杂工作流。在这些环境中，有效的编排需要鲁棒的路由机制，将任务高效分配给最合适的智能体。然而，现有路由器从根本上依赖于未经证实的代理指标——从文本自我描述到静态替代表示——来衡量智能体的能力。这种对非经验性数据的依赖造成了智能体预设画像与其实际操作能力之间的关键差距，引入了严重的安全漏洞。恶意智能体可以轻易歪曲自身能力或隐藏后门，从而逃避标准外部分析和静态表示学习技术。在这项工作中，我们提出了ANTAP（自动非文本智能体选择器），一种基于评估的路由架构，摒弃间接代理指标，转而采用主动能力测试。通过动态查询智能体以经验性地确定其真实能力，ANTAP将性能提炼为共享语义空间内的固定行为算子。在推理时，路由通过纯粹的非文本代数投影执行，建立一道“语言防火墙”，使得基于元数据的攻击无法表达。在我们的实验中，与基于描述的路由基线（攻击成功率达67.3%及以上）相比，ANTAP针对基于描述的注入攻击实现了近乎为零的ASR。针对自适应嵌入攻击，ANTAP的ASR显著低于基于嵌入的基线，降低了20%，同时其设计本身对描述操纵具有抵抗力。

## Abstract
The rapid integration of Large Language Models (LLMs) has driven the evolution of Multi-Agent Systems (MAS), where specialized agents collaborate to execute complex workflows. Effective orchestration in these environments requires robust routing mechanisms to efficiently allocate tasks to the most suitable agent. However, existing routers fundamentally rely on unverified proxies, ranging from textual self-descriptions to static surrogate representations, to gauge an agent's competence. This reliance on non-empirical data creates a critical gap between an agent's projected profile and its actual operational capabilities, introducing severe security vulnerabilities. Malicious agents can easily misrepresent their proficiencies or harbor covert backdoors that evade both standard external analysis and static representation-learning techniques. In this work, we introduce ANTAP (Automatic Non-Textual Agent Picker), an evaluation-driven routing architecture that discards indirect proxies in favor of active capability testing. By dynamically querying agents to ascertain their true competencies empirically, ANTAP distills performance into fixed behavioral operators within a shared semantic space. At inference time, routing is performed via a purely non-textual algebraic projection, establishing a "linguistic firewall" that renders metadata-based attacks inexpressible. In our experiments, ANTAP achieves near-zero ASR against description-based injection attacks, compared to 67.3\% and above for the description-based router baseline. Against adaptive embedding attacks, ANTAP achieves substantially lower ASR than the embedding-based baseline, with a 20\% reduction, while remaining resilient to description manipulation by design.

## 1. 核心问题与整体含义

*   **研究动机**：多智能体系统（MAS）中，路由器负责将用户查询分配给最合适的智能体。现有路由器大多依赖于未经证实的代理信息（如智能体的文本自我描述）来评估其能力。
*   **核心问题**：这种对非经验性数据（文本描述）的依赖构成了一个关键的安全攻击面。恶意智能体可以通过操纵其文本描述（例如嵌入提示注入）来“劫持”路由器，使得无能的甚至有害的智能体被错误选中，或者隐藏后门（如“潜伏代理”），在特定条件下触发恶意行为。
*   **整体含义**：论文提出，不应信任智能体“说了什么”，而应信任其“做了什么”。通过将路由决策的基础从可能导致幻觉和易受攻击的自然语言，转移到从实际行为中提取的、不可篡改的几何算子上，可以从根本上消除一大类严重的安全风险。

## 2. 方法论：ANTAP

*   **核心思想**：提出ANTAP架构，将路由过程与不安全、不可靠的文本元数据解耦。其核心理念是：将每个智能体建模为一个**线性算子**，该算子通过学习其在受信任基准测试上的**真实行为**（成功/失败模式）得到。在推理时，路由决策仅基于查询嵌入与这些算子之间的几何对齐程度（代数投影），不解析任何文本。
*   **关键技术细节**：
    1.  **离线阶段（构建行为算子）**：
        *   **数据构建**：在一个受信任的安全环境中，使用一组基准查询 \(B\) 对每个智能体 \(a_i\) 进行测试。根据其表现（答案正确性 + 工具调用的合法性）构建一个二元性能矩阵 \(Y\)（成功=+1，失败=-1）。
        *   **算子学习**：假设智能体的行为可以用一个线性算子 \(W\) 来近似，该算子将查询嵌入映射到成功的可能性。通过求解一个Tikhonov正则化的最小二乘问题来学习算子 \(W^* = Y(QQ^T + \lambda I)^{-1}Q\)，其中 \(Q\) 是基准查询的嵌入矩阵。
    2.  **在线阶段（语言防火墙）**：
        *   **路由机制**：对于新查询，系统计算其嵌入 \(z = E(q_{new})\)，然后通过一个单一的矩阵-向量乘法得到各智能体的预测得分向量：\(s = W^*z\)。
        *   **最终决策**：将查询分发给得分最高的智能体：\(a^* = argmax_i(s_i)\)。
        *   **安全特性**：整个路由过程**完全不处理任何文本**（不读取智能体描述），因此任何嵌入在描述中的攻击指令都无法触及决策函数，构成了“语言防火墙”。

## 3. 实验设计

*   **数据集**：使用了三个主要基准测试集：**MMLU**（通用知识）、**Big-Bench Hard (BBH)**（推理）、**AgentHarm**（有害请求规避）。
*   **基准方法**：
    *   **AutoGen**：基于文本描述的传统路由器，代表标准LLM路由方案。
    *   **EmbedLLM**：基于学习嵌入的路由器，代表次优的可学习方法。
*   **对比方法**：比较了ANTAP与上述两种方法在“攻击成功率（ASR）”和“路由准确率（ACC）”上的表现。
*   **实验场景**：
    *   **RQ1（主要安全评估）**：针对三种攻击向量的评估。
        *   描述注入攻击，以测试结构免疫性。
        *   潜伏代理攻击（5种数据投毒变体），以测试行为抑制能力。
        *   开放有害请求规避，以测试对有害查询的敏感性。
    *   **RQ2（系统变体鲁棒性）**：通过改变系统属性（如智能体场景、基模型同质性、提示/描述长度、智能体数量）来测试ANTAP的稳定性和优势来源。
    *   **RQ3（自适应攻击鲁棒性）**：在**白盒**威胁模型下，使用贪婪词元替换攻击（扩展自GCG），测试ANTAP在面对针对其嵌入模块的优化攻击时的韧性。

## 4. 资源与算力

*   论文中明确说明，所有实验运行在**单块 NVIDIA A40 (48 GB)** GPU上，搭配Intel Xeon Gold 6230R CPU和503 GB RAM。

## 5. 实验数量与充分性

*   **实验数量**：实验设计较为详尽，覆盖了三个研究问题（RQ），每个问题都包含定性和定量评估。
    *   **RQ1**：对比了5种攻击变体。
    *   **RQ2**：进行了8种系统变体的消融实验。
    *   **RQ3**：针对不同触发长度（1-16个词元）各进行了7轮重复实验。
*   **充分性与公平性**：
    *   **充分**：实验从多个角度（安全性、效率、鲁棒性）评估了方法，涵盖了标准的攻击向量和更具挑战性的自适应攻击，设计比较周到。
    *   **公平**：
        *   消融实验设计系统，能清晰隔离出ANTAP优势的来源（如ASR下降与模型多样性无关）。
        *   在RQ2中，假设检验使用了Bootstrap方法（10,000次重采样）计算95%置信区间，增加了统计可靠性。

## 6. 主要结论与发现

1.  **结构免疫性**：ANTAP通过 **彻底消除推理时的文本输入**，实现了对描述注入攻击的**结构免疫**。其ASR值从AutoGen的**67.3%-75.3%** 降至**0.2%-0.9%**。
2.  **行为抑制能力**：ANTAP通过严格的离线评估，能够学习到后门智能体在触发条件下的**负面信号**，自动抑制其被选中。对于5种潜伏代理攻击变体，ANTAP的ASR最高仅为**2.4%**（BadNet），远低于EmbedLLM的21.3%和AutoGen的73.2%。
3.  **鲁棒的几何边界**：即使在白盒自适应攻击下，ANTAP的ASR也几乎不随攻击词元长度增加而上升（始终在55%-63%波动），性能显著优于EmbedLLM（ASR从约70%飙升至86%并持续高位）。
4.  **效率和可扩展性**：通过将高延迟的LLM推理替换为低成本的矩阵-向量乘法，ANTAP在保证安全性的同时，大幅降低了路由延迟（从428毫秒降至约27-38毫秒），并能够线性地扩展到更多的智能体。

## 7. 优点

*   **架构级安全创新**：不是通过增加检查（heuristics）来防御攻击，而是从架构层面**移除攻击面**——即文本信息准入路由决策流程。
*   **高效且可扩展**：将推理时的路由复杂度从与文本长度相关的LLM推理降低到与智能体数量相关的固定代数运算，既快又安全。
*   **激励一致性**：通过强制路由器根据实证行为而非文采来评估智能体，**消除了“关键词填充”（Keyword Stuffing）等操纵空间**，激励开发者提升智能体的真实能力。
*   **鲁棒性分析**：提供了对多个SOTA方法的公平和系统性比较，并深入探究了自身方法的边界，如自适应攻击场景和案例回退策略。

## 8. 不足与局限

*   **对可信离线环境的依赖**：ANTAP的安全假设依赖于**离线校准环境是完全可信且未中毒的**。如果攻击者能够在基准构建阶段注入有害后门，防御结构就会瓦解。
*   **严格的二分法评分**：目前使用`+1/-1`的胜负二分法，可能会丢失部分智能体在复杂任务中的细微表现差异，无法满足需要更细致能力梯度的场景。
*   **应用限制**：
    *   路由系统要求所有智能体共享**相同的嵌入空间**，这限制了其与异构LLM的集成灵活性。
    *   高度依赖高质量的基准测试构建，对嵌入模型的鲁棒性和安全性要求极高，若嵌入模型被攻击，整个系统将失效。
*   **未覆盖的研究**：论文未深入探讨：
    *   **动态池环境**：智能体频繁加入/退出或持续学习的情况。
    *   **跨模态攻击**：当输入包含多模态（如图像）时的路由鲁棒性。
    *   **隐私泄露风险**：通过路由行为推断用户查询的隐私风险。
