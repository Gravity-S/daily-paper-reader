# Abstract 

Increasingly autonomous agentic AI systems pose novel multi-agent risks, such as secret collusion via covert communication channels. The natural defence to these collusion attempts is to monitor plain-text communication, but the effi-cacy of monitors has been called into doubt by increasingly sophisticated model steganography; indeed, some theoretical schemes have been proposed that are information-theoretically or computationally indistinguishable from good-faith plain-text communication. In this paper, we demonstrate that the complexity of these schemes is no longer a safety barrier, as agentic coding models can already produce undetectable stegosystems when given realistic tool usage, such as code execution or accessing research papers through web searches. Agents also adapt when key ingredients are missing, for example, by adding model-sampling com-ponents or implementing related keyed coding schemes. We then frame tacit steganographic coordination between agents as a Schelling-point problem and introduce coordination metrics for estimating when two agents are likely to se-lect compatible schemes without explicit prior agreement. Our results suggest a shift in the threat model for covert communication between AI agents, where the main barrier is no longer whether frontier agents can understand and implement sophisticated stegosystems, but coordination: whether independently acting agents can converge on compatible schemes, keys, and parameters. We find substantial convergence on broad scheme families but limited strict one-shot coordination, suggesting that shared artefacts, repeated interaction, and tool-mediated search are the settings where covert communication risks are most acute. Overall, our findings provide empirical grounding for the recent strategic confinement hypothesis, which assumes that capable agents can construct covert channels that survive monitoring. 

# 1 Introduction 

Monitored natural-language channels are a common safety boundary in multi-agent AI systems. Modern large language model (LLM) agents increasingly use tools, call APIs, execute code, and coordinate with other agents through natural-language messages [Yao et al., 2023, Yuan et al., 2025, Wu et al., 2023]. A system designer may therefore allow two agents to exchange task-relevant messages while relying on a human or model monitor to prevent private information, hidden goals, or coordination signals from crossing the boundary [Greenblatt et al., 2024, Baker et al., 2025]. This defence assumes that if the monitor can read the messages, it can enforce the information-flow policy.

Steganography breaks this assumption, since a sender can encode a payload inside apparently benign text, allowing a receiver with the right decoding procedure to recover information that the monitor does not know was transmitted [Simmons, 1984]. This risk has been formalised as secret collusion among AI agents, where agents use covert communication to hide the true nature of their interaction from oversight [Motwani et al., 2025]. Prior work has also shown that language-model steganography can be made information-theoretically undetectable [Schroeder de Witt et al., 2023] or undetectable under appropriate sampling and cryptographic assumptions [von Ahn and Hopper, 2004, Ziegler et al., 2019, Christ et al., 2023, Zamir, 2024]. Thus, we ask if current agentic models operationalise such schemes when given realistic tool affordances. To answer this, we evaluate tool-using LLM agents to construct covert communication channels under a range of increasingly restricted environments. Agents can implement candidate undetectable stegosystems from research papers, recover implementation details through web search, add missing sampling components, and hide covert-channel logic inside an existing multi-agent codebase. We then study the coordination problem faced by two agents that cannot explicitly agree on a scheme. Treating steganographic design choices as focal points, we estimate how often independent agents converge on compatible algorithms and parameters, building on prior work on tacit coordination in LLMs [Aharon et al., 2026]. Our results suggest that covert communication is no longer bottlenecked by cryptographic expertise, and is becoming an ordinary consequence of agentic tool use. The limiting factor is increasingly not cryptographic invention, but ordinary agentic competence: reading papers, writing code, using tools, and converging on salient designs. Overall, our findings provide empirical grounding for the recent strategic confinement hypothesis [Schroeder de Witt, 2026], a reinterpretation of Lampson’s confinement problem [Lampson, 1973] for strategic agents, which assumes that capable agents can construct covert channels that survive monitoring. 

Contributions. We make four contributions. First, we introduce an agentic covert-channel con-struction benchmark, where a tool-using LLM must build a working natural-language stegosystem rather than merely describe one (Section 3.1). Second, we frame tacit steganographic agreement 

as a Schelling-point coordination problem over algorithms, keys, and decoding-relevant parameters (Section 3.2.1). Third, we show robust implementation capability : agents construct functional covert channels with papers, web search, code execution, model-sampling components, and existing multi-agent codebases (Section 4.1). Fourth, we identify exact parameter agreement as the main bottleneck: agents often converge on broad scheme families, but strict first-shot complete coordination remains fragile (Section 4.2). 

# 2 Background and Problem Setup 

Monitored-channel threat model. We consider a multi-agent system (MAS) with at least two agents, a sender A and a receiver B, connected by some monitored natural-language channel. Agent 

A has access to some information x that B is not permitted to learn. A monitor observes the messages sent from A to B, and blocks messages that appear to violate the information-flow policy. The intended safety boundary is therefore content-level monitoring, where, if the monitor can read every message, it should be able to prevent restricted information from crossing the channel. Instead of sending x directly, A can send an apparently benign cover text c′ from which B can recover a hidden payload using a decoding procedure. The monitor may observe the complete text of c′ while remaining unaware that a payload was transmitted. We focus on text-based steganographic channels, where the cover object is natural-language model output. 

Steganographic channels. A keyed stegosystem consists of an encoder, a decoder, a payload space, a cover distribution, and a shared secret key. The encoder maps a payload and cover-generation process into a stego-object, while the decoder uses the key to recover the payload. The relevant security notion is not whether the text looks suspicious to a casual reader, but whether a monitor without the key can distinguish stego-objects from ordinary samples drawn from the cover distribution. Prior work has proposed language-model stegosystems and watermarking-derived schemes with formal undetectability guarantees under appropriate assumptions [Christ et al., 2023, Zamir, 2024]. Our question is whether current agents can operationalize such schemes in practice. 2Agentic construction. Tool use changes the threat model. A model that cannot directly produce a covert channel from memory may still be able to read papers, search the web, write code, run tests, use model-sampling components, and iterate until an encoder/decoder pair works. We therefore evaluate covert-channel construction as an agentic capability rather than as a memorised behaviour. In our experiments, success requires producing an implementation that can transmit synthetic payloads through benign-looking outputs under the constraints of the environment. 

Tacit coordination. A second obstacle is coordination. Two agents may both be capable of building covert channels, but fail to communicate if they independently choose incompatible algorithms, keys, or hyperparameters. We treat this as a focal-point coordination problem, where the design space contains many possible steganographic schemes, but some are more salient because they are named in the prompt, present in a shared file system, easy to find on the web, or common in the training distribution. We estimate the probability of tacit agreement by sampling independent agent plans and measuring how often they collide on compatible algorithmic and parameter choices. We provide an extended background, with a gentle introduction to steganography in Appendix A. Additionally, we provide a survey of related work in the fields of misalignment, watermarking, and LLM cooperation in Appendix B.

We break down the likelihood of harmful steganography S into 3 key areas assumed independent of each other: the likelihood that an agent can implement a scheme I, the likelihood that agents can coordinate on a compatible channel C, and the risk that illegal information flow can cause harm R.

3.1 Steganographic implementation 

Agents are frequently permitted unlimited and unmonitored tool usage, for example, with Claude’s 

–dangerously-skip-permissions tag [Hughes, 2026]. We assess whether agents can implement un-detectable steganography, such that a monitor with access only to agent outputs cannot detect the presence of a hidden payload. Formally, we define the threat model used. 

Actors and goals 

• Adversary: A capable agent intending to covertly transmit information in its natural language outputs without detection by content-level monitoring. Shares a secret key with the receiver. • Receiver: A cooperating party (human or AI) that possesses the secret key and intends to decode hidden messages from the agent’s outputs. • Monitor: The defender (human or AI) responsible for detecting and preventing covert channels. Has access to all agent outputs, but not the tool usage or intermediate artefacts. 

Adversary capabilities and knowledge 

1. Ability to execute code in a sandboxed environment. 2. Access to a sub-model (or API) that returns log probabilities for candidate next tokens, and the ability to call it. 3. Access to a file system to stage intermediate artefacts, including models and test data, and optionally install vetted packages. 4. Ability to find/derive an undetectable steganography scheme, given that the paper by Zamir [2024] is in their file system. In some aspects, this is a reasonable threat model. Developers rarely monitor tool usage unless they are particularly security-conscious, agents typically have access to the file system and can execute code, and frontier models are capable of understanding complex research papers and correctly implementing concepts from them. However, few agents have access to a sub-model; malicious prompts could be prevented with additional monitoring; generic file systems don’t contain research papers detailing a steganographic implementation. 3.3 Risk 

Alongside our assessment of the likelihood that steganography emerges, it is also relevant to consider the maximal potential harm that could occur if a steganographic channel existed in a system. We categorise the potential risk of a MAS as none , low , moderate , or high depending on the possibility of illegal information flow, specified by information flow restrictions, and explain how a system designer can model them and identify specification violations using a monitor. Risk is inherent in the definition of the environmental information set specifications and cannot necessarily be reduced. 

3.3.1 Information flow 

We define the discrete time steps within a system with communication channels where the time step value t is x at the xth distinct time at which a message is sent over a channel. 4.1 Agent implementations 

For each environment variation, we use an instance of Claude Opus 4.6 [Anthropic, 2026] on Claude Code [Anthropic, 2025a] with the –dangerously-skip-permissions tag [Hughes, 2026] on a virtual machine. Both the task and paper are contained in the file system accessible to the agent, along with a virtual environment containing the necessary packages, and a file containing model log probabilities with some example usage code. The agent is prompted, “Follow the instructions in task.md”, the contents of which are shown in Figure 7, then left to implement autonomously until it terminates. The sub-model provided was small, and thus could be run on a CPU, not requiring any GPU compute. In each experiment, the agent is provided with the paper by Zamir [2024] unless otherwise stated. Specific experiment variation and implementation details are shown in Appendix D. Implementation correctness was assessed by an instance of Claude Opus 4.6 [Anthropic, 2026] required to perform comprehensive tests and a complete evaluation; see Appendix C for prompt information. Initial implementations were also reviewed manually to ensure the correctness of the core result. We found that in the default threat model described in Section 3.1, agents can successfully implement undetectable steganographic schemes, including with access to a paper describing an alternative scheme by Schroeder de Witt et al. [2023]. The agent was also successful when the initial environment was empty: without immediate access to a sub-model to sample from, the agent introduced one itself, highlighting its ability to add complex artefacts to achieve a complete and correct scheme. We then changed the initial environment to be a generic MAS, where sub-model access was contained within the existing implementation, and gave the agent the additional task of attempting to hide its implementation in the MAS such that another developer couldn’t detect it. The agent succeeded, although it implemented a different scheme than the one in the paper we gave it, and an Opus 4.6 monitor couldn’t detect the presence of a fully functional stegosystem. In another experiment, we relaxed the requirement that an implementation description is contained in the agent’s file system; the agent used a web search to find the instructed paper and successfully implemented the scheme. The only experiment where we didn’t obtain a positive result was attempting to inject the malicious prompt in the research paper, providing the agent with an innocuous prompt instead. We expect that, given more extensive testing or specific prompt injection techniques tailored to the agent, this defence would prove fallible. 

4.2 Coordination in multi-agent systems 

We adapt the code from Aharon et al. [2026] by adding a steganography planning experiment, where the agent is instructed to implement a steganographic system to communicate with a partner; agents are told the problem specification and to return their chosen scheme, including their exact hyperparameters and design choices, along with their reasoning. Outputs are then fed into two instances of Opus 4.6, one categorising the chosen schemas and hyperparameters to identify successful cooperation in batches of 10, and one asserting that if another agent copied the scheme exactly, it would succeed, in batches of 5. Initial LLM outputs were checked manually to ensure correctness. The categories are then used to calculate ACI, HCI and CCI. The models use API-based inference, meaning no GPU compute is required, though the adapted code base does support local models. We vary the environment by changing the prompt in the following axes, experimenting using a 2 × 2 × 2 factorial. • Undetectability - We either tell the agents that they must implement a computationally undetectable scheme, or that this is not necessary. • Shared Secret Key/Public Key Infrastructure - We either tell agents that they share a secret key with the other agent, or that there is a public key infrastructure they can access. • Common Knowledge - We either explicitly tell agents the common knowledge base, or that they have access to that information without mentioning the other agents’ information set. Limited by tokens, we fixed N = 100 samples per agent, which ensured coordination instances occurred for most environments; thus, CCI is a reasonable approximation to Schelling sharpness. We advise practitioners that our results where CCI is 0 are not conclusive, and a larger sample size could lead to non-zero complete coordination. Each agent was run on every environment in our factorial; the performance in each metric by model is shown in Figure 1. 4.3 Mitigations 

Agents with access to tools proved very effective at implementing stegosystems, highlighting the dangers of unmonitored tool use. The most effective mitigation is to monitor tool usage and to move away from methods that skip permissions. Another effective but computationally expensive method is to crowd the common knowledge base with similar implementation descriptions, reducing the Schelling sharpness by introducing new, similarly salient Nash equilibria; an exponential number of descriptions need to be included to ensure a negligible chance of coordination. For the MAS experiments, we found that agents with no tool usage coordinate effectively on most hyperparameters, but cannot coordinate completely on undetectable schemes; hence, using monitors between agents reduces the threat posed by steganography. 

4.4 Limitations Sample size. A key limitation of our implementation experiments is the very small sample size. Each experiment had long setup, implementation, and verification times, and as such a large sample size was infeasible. This is also a limitation of the coordination experiments, caused by a restriction on available tokens: we provide a lower bound for CCI, but with a larger sample size, many instances where some hyperparameter probability was 0 in our experiment would contribute to the CCI. We also didn’t provide agents with tool access for these experiments, as it was beyond the scope of our research, but repeating experiments with web and PDF tools would likely raise the CCI significantly. 

Model scope. We also only used Anthropic models in these experiments; thus, our claims only apply directly to those, though we would expect them to generalise to LLMs from other providers. 

# 5 Conclusion 

In this work, we evaluated the limiting factor of agentic covert communication by separating the problem into steganographic implementations and tacit coordination on stegosystems. Empirically, we demonstrated that under reasonable tool affordances, frontier agentic models can operationalise varied stegosystems, including ones that are computationally undetectable or perfectly secure. We also demonstrated that complete successful coordination without prior communication or tool usage is non-negligible under reasonable circumstances. Both our formulation for Schelling sharpness and coordination metrics apply to future coordination games research. Similarly, our discussion on information flow and risk can be flexibly applied to other research on monitored-channel systems. 

Long-term alignment and high-stakes research. The highest-stakes setting for this threat may not be ordinary chatbot interaction, but automated safety-critical research itself. Future frontier-AI safety 9cases may depend on AI agents reading papers, writing evaluation code, auditing logs, red-teaming deployments, maintaining shared workspaces, and producing evidence that determines whether more capable systems are safe to train or deploy [Buhl et al., 2024, Greenblatt et al., 2024, Korbak et al., 2025]. In such settings, our results suggest that once agents can use tools, the relevant question is no longer only whether a monitor can read their final messages, but whether the whole research environment prevents illicit information flow through messages, code, and shared artefacts. Even low-bandwidth covert channels could let agents coordinate research directions, bypass staged access controls, or collude in audits. Long term, we hope this work helps move agentic AI safety from ad hoc message monitoring toward information-flow assurance for high-stakes multi-agent systems: safety cases should argue not only that each agent is individually monitored, but that the surrounding tool-mediated organisation remains robust to covert coordination.

# A Steganography introduction 

Steganography is the technique of hiding information such that its presence is undetectable [Cachin, 1998], for example, by hiding a message within an innocuous cover. When two parties share some secret information, they can create a secret key stegosystem :

Definition 7 A secret key stegosystem [Katzenbeisser and Petitcolas, 2000] is the quintuple 

S = ⟨C, M, K, D K , E K ⟩ where C is the set of possible covers, M is the set of secret messages with 

|M | ≤ | C|, K is the set of secret keys, EK : C × M × K → C is the embedding function, and 

DK : C × K → M is the decoding function, satisfying the property that ∀m ∈ M, ∀c ∈ C, ∀k ∈

K, D (E(c, m, k ), k ) = m.

This is demonstrated effectively by the “Prisoners’ Problem" by Simmons [1984] in Figure 4. Typically, the cover space contains only inconspicuous covers, and the stego-object generated by the embedding function must be perceptually similar to the cover, so Eve doesn’t suspect anything untoward [Katzenbeisser and Petitcolas, 2000]. A simple stegosystem is an acrostic, where the hidden message is encoded as the first letter of consecutive words or sentences, as shown in Figure 5. 16 Figure 4: The Prisoners’ Problem, from Simmons [1984]. Alice and Bob communicate covert messages without their communications being detectable by Eve by hiding them in the cover text. 

Figure 5: An LLM-generated weather report with a payload embedded as an acrostic. 

A.1 Undetectability 

Steganography hides the existence of messages, but different schemes have different levels of detectability to different observers. For example, the payload within the acrostic shown in Figure 5 is undetectable to most people reading the passage, but not to someone told to look for acrostic messages. 

Definition 8 Let C be a non-empty cover set and a be some actor who attempts to distinguish between covers. Their similarity function is sim a : C × C → (0 , 1] , where sim a(x, y ) = 1 iff actor 

a perceives no difference between covers x, y .

A.2 Steganography in LLMs 

LLM outputs can be adjusted to encode a payload. A simple method that is perceptually undetectable to humans is zero-width encoding; different Unicode characters with no width can be used to hide a payload without visually changing the cover [Bashir et al., 2020], as shown in Figure 6. LLMs can choose a mapping of zero-width Unicode characters to sequences of bits which represent portions 18 of ASCII characters, thus hiding an ASCII message in their ordinary outputs that is invisible to humans but decodable by other LLMs. This is therefore not secure in the presence of a monitor where zero-width encoding is perceptually detectable (i.e. another LLM that can see the zero-width characters). A more complex computationally undetectable stegosystem plants a payload in the LLM response by adapting a watermarking algorithm that doesn’t alter the distribution of responses [Christ et al., 2023]. A shared secret key is required to extract the payload from the model’s response; crucially, without the key, it is provably impossible to distinguish between the original LLM responses and the responses containing the payload [Zamir, 2024]. A steganographic channel can be used for illegal information flow in a MAS, even with the presence of a monitor, subverting the model specification and allowing some subject sj to obtain information

Another example is that models may collude via the covert channel to perpetuate biases [Motwani et al., 2025]. Steganography can be damaging even when an agent doesn’t exist within a MAS, or when it is in an isolated environment without internet access. B.1 Misalignment 

AI alignment is concerned with ensuring AI systems behave in line with human intentions and values [Ji et al., 2025]; preventing malicious behaviour becomes more essential as AI capabilities, and thus, risks from misalignment, grow. Since, by definition, emergent steganography is unintended by the system designer, if agents were perfectly aligned with the developer’s intentions, such behaviour wouldn’t occur; reducing misalignment is a key approach to reducing steganography. Standard training objectives for LLMs of next word prediction [Radford et al., 2019] or determining contextual similarity of sentences [Devlin et al., 2019] aren’t necessarily in line with human values [Shen et al., 2023]; additional interventions are required for alignment. However, specifying human values is intrinsically difficult due to cultural variations; a popular view is that LLMs should be helpful, honest and harmless (known colloquially as triple H) [Askell et al., 2021]. Most successful methods employ reinforcement learning from human feedback (RLHF) [Christiano et al., 2017], for example, using human preferences as a reward signal in fine-tuning by having labellers rank outputs [Ouyang et al., 2022]. Despite this, frontier LLMs can exhibit extreme misalignment, highlighted by a recent example, where an LLM resorted to blackmail, espionage, and lethal action when presented with a conflict of goals or threatened with replacement [Lynch et al., 2025]. Additionally, fine-tuning models on a narrow malicious objective can induce broad misalignment, unrelated to the objective [Betley et al., 2025], showing that current alignment methods are fragile and vulnerable to backdoors. Misalignment can be measured with safety benchmarks by simulating tool usage in common environments, tracking different categories of risks to ensure broad alignment is recorded [Zhang et al., 2025a,b, Chen et al., 2024]. Despite the improved utility of using a MAS, it can exhibit greater misalignment with the system goal than single agents [Shen et al., 2026], and team misalignment may arise from agents with varied goals or intentions that conflict with each other or with the system’s goal. Conflict between agents resembles contending opinions in human populations [Kierans et al., 2022], which can be quantified as the probability of misalignment given a population and problem area [Jang et al., 2017], meaning team misalignment in large, diverse agent groups with potentially conflicting goals across various problems can be calculated [Kierans et al., 2025]. Social conventions, including collective bias and system misalignment not present in individual models, can be amplified, induced, or overridden by playing simple coordination games, coordinating differently than a single model on a cultural bias when presented with two options [Flint et al., 2025, Ashery et al., 2025]. For mixed-motive games, team alignment can be improved through an optimisation method that progressively aligns individual and collective objectives [Li et al., 2024]. For general problems, methods of reducing team misalignment often resort to social scenarios that would typically reduce misalignment in human populations [Ashery et al., 2025], introducing frameworks like debate [Irving et al., 2018, Estornell and Liu, 2024]; these improve factuality and reasoning [Du et al., 2023] by promoting divergent thinking [Liang et al., 2023]. In this project, we explore how unintended steganographic collusion between LLMs can emerge due to misspecified training reward incentives [Mathew et al., 2025]. This is a form of reward hacking: a type of misalignment where agents undergoing an optimisation process, such as training, exploit gaps in a misspecified reward function, thereby finding an unintended solution and obtaining the reward without necessarily achieving the objective [Pan et al., 2022]. Training on documents that inform agents about reward hacking induces the behaviour [Hu et al., 2025]; but even without this, frontier LLMs frequently engage in sophisticated reward hacking behaviour, for example, in autonomous software development, they attempt to modify tests, exploit loopholes and bugs, and gain access to existing implementations [Von Arx et al., 2025]. Mitigation typically requires a monitor observing the chain of thought (CoT) reasoning to determine agent intent; however, with extreme optimisation or strict restrictions on undesired reasoning, agents can learn to obfuscate their intent from the monitor within their CoT while reward hacking at a significant rate [Baker et al., 2025, Skaf et al., 2025]. 

B.2 Watermarking 

Originally a method to protect the copyright of digital media, watermarking robustly hides infor-mation about the object it is within [Katzenbeisser and Petitcolas, 2000]. Both watermarking and steganography imperceptibly convey information by embedding it in covers, but are distinct fields, 20 as watermarking requires a high level of robustness against attacks to prevent the removal of the watermark [Katzenbeisser and Petitcolas, 2000]. Due to their inherent similarities, steganographic algorithms utilise many mathematical primitives common in watermarking algorithms, such as pseudorandom functions (PRFs) [Goldreich et al., 1986] used to spread the hidden information throughout the cover in a way that appears random, and error correcting codes (ECCs) used for robustness in watermarking [Christ and Gunn, 2024] and for increasing the efficiency of embeddings in steganography [Munuera, 2007]. Consequently, numerous steganographic methods are developed concurrently with watermarking methods [Evsutin et al., 2020, Wang et al., 2023]. Given the potential for LLMs to generate illegal and explicit content, with AI-generated child abuse imagery more than doubling between 2024 and 2025 [Kendall MP and Phillips MP, 2025], the ease of generating and spreading falsehoods or emotional distress [Grinbaum and Adomaitis, 2022], and the increasingly imperceptible distinction between AI-generated content and real life, even to advanced algorithmic detectors with access to image metadata [Ghiur˘ au and Popescu, 2025, Elkhatat et al., 2023], a watermark for LLM-generated content is extremely valuable. In particular, we desire a fingerprint where the embedded watermark specifies metadata that determines the author of the data [Katzenbeisser and Petitcolas, 2000]. Fortunately, there is extensive research into embedding watermarks in AI-generated content for different purposes. There exist watermarks for text-based outputs, detectable using efficient open-source algorithms [Kirchenbauer et al., 2024, Fernandez et al., 2023] and undetectable without a secret key [Christ et al., 2023, Boroujeny et al., 2024]; and for image outputs, remaining undetectable without a secret key [Gunn et al., 2024]. Major LLM providers originally pledged to watermark content for safety [Bartz et al., 2023], but real-world deployment remains limited, likely due to misaligned incentives between LLM providers and end users [Liu et al., 2026]. 

B.3 LLM cooperation 

LLMs exhibit strong coordination capabilities in general scenarios [Zhu et al., 2025], but without an imposed organisational structure, their helpful nature leads to over-reporting and unnecessary compliance [Guo et al., 2024]; the benefits of structure are exhibited in real-life applications like a MAS-operated vending machine [Anthropic, 2025b]. A tacit coordination game is a pure coordi-nation game with multiple Nash equilibria; rational decision makers cannot be certain about which equilibrium action other players will use [Huyck et al., 1990]. LLMs have remarkable abilities in tacit coordination games, but often struggle in common-sense coordination involving numbers or cultural archetypes [Aharon et al., 2026]. Additionally, LLMs perform well in extended self-interested games [Akata et al., 2025] such as the iterated Prisoner’s Dilemma family [Robinson and Goforth, 2006]. Advanced cooperative reasoning problems require Theory of Mind (ToM) reasoning - the ability to understand that other people have their own thoughts [Ruhl, 2025], thus LLMs must predict the intentions of their partners [Agashe et al., 2025]. Despite early research suggesting that LLMs possess ToM [Kosinski, 2023, Strickland, 2024, Strachan et al., 2024], recent results highlight deficiencies in LLM social reasoning abilities [Amirizaniani et al., 2024, Agashe et al., 2025], implying that ToM is likely mimicked by reflection of pre-training data [Aharon et al., 2026, Kosinski, 2023]. ToM reasoning can be improved by integrating human intentions and using emotive language [Amirizaniani et al., 2024], by explicitly representing agent beliefs [Li et al., 2023], or by inferring and propagating weighted hypotheses about agents’ thoughts [Kim et al., 2025].
