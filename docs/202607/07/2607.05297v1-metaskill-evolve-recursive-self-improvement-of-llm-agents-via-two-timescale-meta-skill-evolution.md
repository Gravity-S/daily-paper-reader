## 摘要
近期的大语言模型智能体能够处理越来越长程、开放式的任务，而外部技能——提供给智能体的可复用过程性知识——进一步扩展了这一能力。然而，固定的、手工编写的技能很少是最优的，也无法适应智能体遇到的任务多样性。自我改进智能体通过从执行轨迹中重写自己的技能文件来解决这个问题，在具有挑战性的基准测试中取得了显著的改进。然而，这种自我进化仍然是非递归的：它只改进了任务技能（智能体做什么），而改进过程（如何改进）是一次性编写并保持固定的。我们提出了MetaSkill-Evolve，一个双时间尺度框架，使得智能体技能改进成为递归的：每个分支同时携带一个任务技能$s$和一个分支局部元技能$m=(ψ,σ,α,π,\varepsilon)$，其五个组件分别参数化改进流水线中的分析器、检索器、分配器、提议器和进化器智能体。任务技能在快速循环中进化，而元技能在较慢的循环中进化，且在同一流水线应用于自身的情况下，无需额外模型或目标。所有五个流水线智能体共享一个冻结的主干网络，MetaSkill-Evolve在三个智能体基准测试（OfficeQA、SealQA、ALFWorld）上优于无技能、静态技能和单层进化基线，分别在保留的测试准确率上比原始主干网络提高了23.54、16.09和1.92个百分点。

## Abstract
Recent LLM agents tackle increasingly long-horizon, open-ended tasks, and external skills, reusable procedural knowledge supplied to the agent, further extend this capability. However, a fixed, hand-authored skill is rarely optimal, and cannot adapt to the diversity of tasks an agent encounters. Self-improving agents address this by rewriting their own skill files from execution traces, yielding meaningful gains on challenging benchmarks. Yet such self-evolution remains non-recursive: it improves only the task skill (what the agent does) while the improvement procedure (how it improves) is authored once and held fixed. We introduce MetaSkill-Evolve, a two-timescale framework that makes agentic skill improvement recursive: every branch carries both a task skill $s$ and a branch-local meta-skill $m=(ψ,σ,α,π,\varepsilon)$ whose five components parameterise the Analyzer, Retriever, Allocator, Proposer, and Evolver agents of the improvement pipeline. Task skills evolve on a fast loop while the meta-skill evolves on a slower one under the same pipeline applied to itself, with no additional model or objective. With all five pipeline agents sharing a single frozen backbone, MetaSkill-Evolve outperforms no-skill, static-skill, and single-level evolution baselines on three agentic benchmarks (OfficeQA, SealQA, ALFWorld), improving held-out test accuracy over the raw backbone by +23.54, +16.09, and +1.92 points respectively.

## 1. 核心问题与研究背景

- **核心问题**：现有的LLM智能体在执行长期、开放式任务时依赖于外部可复用知识（技能），但这些技能通常是手工编写的、固定的，无法随任务多样性自适应改进。已有的自改进系统仅能优化任务技能本身（what the agent does），而改进过程（how it improves）是作者一次编写后固定不变的，缺乏递归性。这导致改进效果容易停滞，无法根据分支历史动态调整诊断、编辑、搜索等策略。
- **研究动机**：将改进过程本身也作为可进化的第一类对象，与它产生的任务技能共同进化，实现一种有界的递归自改进。直觉上，智能体不仅需要学习更好的任务行为，还需要学习更好的学习策略（元技能）。论文通过分离任务技能进化（快时间尺度）和元技能进化（慢时间尺度）来回答这一研究问题。

## 2. 方法论

### 核心思想
- **双时间尺度框架**：每个分支状态包含任务技能 $s$ 和分支级元技能 $m = (\psi, \sigma, \alpha, \pi, \varepsilon)$。任务技能在每个迭代（快循环）中进化；元技能每 $H$ 个迭代（慢循环）进化一次，进化信号来源于过去 $H$ 个后代的平均改进量（元生产力 $\hat{P}$）。
- **递归自改进**：五个组件（分析器 $\psi$、检索器 $\sigma$、分配器 $\alpha$、提议器 $\pi$、进化器 $\varepsilon$）本身也是Markdown格式的技能文件，与任务技能格式相同。因此，相同的五智能体流水线既可以改进任务技能，也可以递归地改进元技能本身，无需额外模型或目标函数。所有五个智能体共享一个冻结的骨干（Gemma-4 31B）。
- **前沿选择**：每个迭代从存档中选择下一个要扩展的父节点，评分公式为：
  $$v^* = \arg\max_{v \in \mathcal{F}} \left[ \eta_1 U_v + \eta_2 \hat{P}_v + \eta_3 N_v \right]$$
  其中 $U_v$ 是节点任务技能效用，$\hat{P}_v$ 是元生产力（后代平均提升），$N_v$ 是访问冷却因子（$1/(1+\text{times\_selected}_v)$）。三部分分别对应：利用高效用、关注产生有用后代的节点、防止单个分支垄断预算。
- **进化图**：整个搜索历史记录在SQLite有向无环图中，每个节点保存完整分支状态、效用、元生产力等信息。严格只将 $\Delta U > 0$ 的节点加入存档（可成为后代父节点），但所有节点（包括失败）都保存在图中，可供检索器作为灵感来源。

### 关键技术细节
- **五智能体流水线**：
  1. 分析器（$\psi$）：基于最差训练样例的执行轨迹，输出故障标签 $\phi$ 和自由分析 $a$；
  2. 检索器（$\sigma$）：根据标签 $\phi$ 检索同分支和跨分支的灵感节点，经过过度获取和LLM重排序；
  3. 分配器（$\alpha$）：根据历史改进情况自适应选择子代预算 $K \in [1, K_{\max}]$；
  4. 提议器（$\pi$）：生成具体的编辑提议 $\delta$，当 $K>1$ 时使用多样性提示避免重复；
  5. 进化器（$\varepsilon$）：将编辑写入磁盘并验证文件哈希一致性。
- **慢循环元技能更新**：每 $H$ 个快循环，将过去 $H$ 个后代的标签、诊断、结果、$\hat{P}$ 组合成合成元失败轨迹，用同样的五智能体流水线对所有五个元技能文件进行联合编辑（Proposer串行，Evolver并行），确保跨组件一致性。
- **分支隔离**：每个分支启动前从SQLite恢复其快照，防止分支间文件状态泄露。

## 6. 主要结论与发现

1. **递归自改进有效**：在OfficeQA和SealQA（骨干有较大提升空间）上，从No-Skill → Static → Single-Level → MetaSkill-Evolve呈现单调递增趋势，最终提升分别为+23.54和+16.09个百分点。在ALFWorld（骨干已接近天花板92.31%）上仍带来+1.92个百分点的提升，且慢循环是唯一带来正向增益的成分。
2. **元技能进化贡献独立**：相比Single-Level Evolution（冻结元技能），MetaSkill-Evolve额外提升OfficeQA +6.38、SealQA +8.05、ALFWorld +1.92，证明改进过程的进化具有独立价值。
3. **组件重要性因场景而异**：OfficeQA上分配器$\alpha$最重要（移除后-19.7pt），SealQA上提议器$\pi$最重要（-8.4pt），ALFWorld上$\pi$主导（-7.7pt）。
4. **元更新周期越短越好**：H=2（最频繁）在所有基准上表现最佳，过长的周期导致元技能滞后于任务技能，可能覆盖有用变化。
