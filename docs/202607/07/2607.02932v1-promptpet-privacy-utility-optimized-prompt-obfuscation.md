## 摘要
隐私是用户与AI聊天机器人交互时面临的重要挑战，因为用户可能显式或隐式地分享敏感信息，而AI聊天机器人可以利用这些信息进行用户画像。本文旨在通过用户端机制保护用户隐私，该机制转换用户提示中的敏感信息，同时保留足够的信息以从聊天机器人获取有用响应。这种方法面临保护隐私（即避免画像）与保持效用（即获得个性化和任务特定的响应）之间的固有权衡。为此，我们考虑、评估并比较了四种不同的混淆操作，即删除、抽象、替换以及我们引入的一种新颖的噪声/去噪方案。额外的创新见解包括：利用数据类型分类法来识别和混淆敏感信息，并明确考虑聊天响应的效用以做出混淆决策。首先，我们系统性地独立优化和评估每种混淆操作在隐私-效用权衡方面的表现。其次，我们提出了PROMPTPET，一个基于LLM的智能体，它使用强化学习启发的规则优化器（首次在此背景下应用）为提示的每个敏感部分选择最佳混淆操作。使用真实世界的聊天数据集，我们展示了PROMPTPET达到了任何单一混淆操作所能达到的最佳隐私-效用权衡，并且显著优于之前的最先进方法。

## Abstract
Privacy is an important challenge when users interact with AI chatbots, since users may share sensitive information, explicitly or implicitly, and AI chatbots can use this information for user profiling. In this paper, we aim to protect user privacy via a user-side mechanism that transforms sensitive information in a user prompt, while preserving enough information to elicit a useful response from the chatbot. This approach faces an inherent tradeoff between protecting privacy (i.e., avoiding profiling) and preserving utility (i.e., getting personalized and task-specific responses). To that end, we consider, evaluate, and compare four different obfuscation actions, namely redaction, abstraction, replacement, and a novel noising/denoising scheme that we introduce. Additional novel insights include: utilizing a data type taxonomy to both identify and obfuscate sensitive information and explicitly taking into account the utility of chat responses in making the obfuscation decision. First, we systematically optimize and evaluate each obfuscation action independently in terms of the privacy-utility tradeoff it achieves. Second, we propose PROMPTPET, an LLM-based agent that selects the best obfuscation action for each sensitive part of the prompt, using a reinforcement-learning inspired rule optimizer, applied for the first time in this context. Using a real-world chat dataset, we show that PROMPTPET matches the best privacy-utility tradeoff attainable by any single obfuscation action and significantly outperforms prior state-of-the-art approaches.

### 1. 核心问题与研究动机

-   **核心问题**：当用户与AI聊天机器人交互时，不可避免地会泄露敏感个人信息（如健康状况、财务状况等），这些信息可能被服务提供商用于用户画像、定向广告等用户不希望的用途。如何在保护用户隐私（避免被画像）的同时，又不损害聊天机器人的回答效用（即生成个性化、有用的响应），是一个关键挑战。
-   **研究动机**：现有方法（如PII脱敏、数据最小化）要么不够全面，无法保护非PII的敏感信息；要么会严重损害响应效用。本文旨在设计一个**用户端**的、无需服务提供商配合的提示混淆机制，在信息泄露之前就对其进行保护，并能灵活平衡隐私与效用。

### 2. 方法论

-   **核心思想**：将用户提示中的敏感信息识别为“敏感单元”，并为每个单元智能选择不同的混淆操作。关键在于，针对不同类型和重要性的信息，采用差异化的混淆策略，以达到全局最优的隐私-效用权衡。
-   **关键技术细节**：
    -   **阶段一：敏感信息检测与重要性评分**
        1.  **敏感单元检测**：利用指令微调的本地LLM（Gemma3:12B），基于一个用户定义的敏感信息分类法（本文采用IAB受众分类法的“人口统计与兴趣”分支，含36个类别和656个数据类型），识别并标注出提示中的“敏感单元”及其对应的“类别”和“数据类型”。
        2.  **重要性评分**：为每个敏感单元计算一个[0,1]区间的重要性分数，评估其对于生成有用响应的贡献。分数越高，单元越关键，也就越“脆弱”，需要更小心地处理。该方法基于实体显著性预测技术。
    -   **阶段二：混淆操作与规则优化**
        1.  **四种混淆操作**：
            -   **Redact (删除)**：替换为占位符 `[REDACTED]`，隐私增益最大但效用损失也最大。
            -   **Abstract (抽象)**：用该敏感单元在分类法中的更高级别节点（如类别名）替换，如“抑郁症” -> “健康”。
            -   **Replace (替换)**：替换为同一类别下另一个数据类型的合理替代值，如“山景城” -> “帕罗奥图”。
            -   **Noise (加噪/去噪)**：**【核心创新】** 保留原始敏感单元，并注入一些来自其他不相关数据类型的、上下文合理的“诱饵”信息。提示聊天机器人分别回答原始问题和诱饵问题。最后在客户端通过一个简单的过滤机制，去除诱饵部分，只保留对用户有用的原始回答。这能在几乎不影响效用的前提下显著提升隐私。
        2.  **规则优化器**：这是PROMPTPET的第二个核心创新。受“优化即提示”启发，它使用一个**强化学习**模式的框架，由两个LLM智能体协作完成：
            -   **混淆动作决策器**：根据当前的一套“决策规则集”，为提示中的每个敏感单元分配合适的动作。
            -   **规则优化器**：根据决策器在验证集上的表现（隐私与效用权衡得分），以及每个规则的失败总结，迭代地提出、修正规则集，目标是最大化 `α * 隐私 + (1-α) * 效用 - λ * 规则集长度` 的优化目标。这个过程相当于让网络自己“学会”在什么情况下应该使用什么混淆策略。
    -   **整体流程**：用户的原始提示 → 敏感单元检测与重要性评分 → 规则优化器决策每个单元的动作 → 应用混淆操作 -> 生成混淆后的提示 -> 发送给在线AI代理 → 接收响应（可能包含诱饵内容） → 过滤掉诱饵部分 → 返回最终有用响应给用户。

### 3. 实验设计

-   **数据集**：基于 **WildChat-1M** 数据集（一个真实用户-ChatGPT交互日志）。作者进一步使用了 **Trust-No-Bot** 方法从中提取了3,319个包含敏感信息的对话，并只保留每个对话的第一轮查询，最终筛选出包含7,911个敏感单元的2,182个查询。
-   **Benchmark与对比方法**：
    1.  **单动作上界**：将Redact, Abstract, Replace, Noise四种动作单独视为一种策略，分析它们各自的隐私-效用权衡曲线。
    2.  **先前工作**：
        -   **Ngong等人的方法**：一种基于上下文重写的提示混淆框架。
        -   **Zhou等人的方法**：一种零样本的LLM最小化器，保留、删除或抽象每个敏感单元。
-   **评估指标**：
    -   **隐私增益**：比较从原始提示和混淆后提示中推断出的用户画像的语义相似度。使用了一种基于IAB分类法的、考虑语义相似度和类别的细粒度度量。
    -   **效用保持**：比较原始响应和过滤后的响应。采用两个互补指标：句子嵌入的余弦相似度（衡量全局语义）和基于NLI的原子知识主张保留度（衡量细节信息保留）。

### 6. 主要结论与发现

1.  **噪声/去噪是最优单动作**：在所有单一混淆动作中，新提出的Noise动作在大部分操作点上主导了隐私-效用权衡。在最大混淆强度下，它实现了77%的隐私和76%的效用，显著优于删除（59%/63%）和抽象（70%/65%）。在`成功区`（高隐私、高效用）的比例达到73.7%，是其他动作的近两倍。
2.  **混合策略胜过单一策略**：每个查询的最优混淆动作是不同的。Noise最常是最优解（58.3%），但其余动作在41.7%的查询上是唯一最优解，说明“一刀切”的策略不可取。
3.  **PROMPTPET超越先前方法**：通过学习到的决策规则集，PROMPTPET的隐私-效用权衡曲线完全贴合了最佳单动作的上界，并**全面超越了**Ngong和Zhou等人的方法。具体地，它比最强的先前工作多出**3.3倍**的隐私增益（每单位效用成本），总体隐私水平高出**1.4倍**。
4.  **规则优化的有效性**：一个更好的初始规则集（例如Oracle种子）能引导出更好的最终规则集。优化过程能显著提升质量，例如将从空种子开始的规则集提升至接近最优水平。
5.  **失败分析**：PROMPTPET的主要失败模式是“保护不足”（17.5%），即未能完全掩盖上下文中的敏感信号；其次是“过度混淆”（7.6%），主要是由于对重要敏感单元错误地使用了删除操作。

### 7. 优点

1.  **创新的噪声/去噪机制**：这是论文最核心的亮点。通过保留原始信息并注入诱饵，它几乎完美地避免了删除操作带来的效用灾难，同时有效降低了隐私风险。其结构化的模板和简单的过滤机制也使其高效可行。
2.  **系统性的方法论**：将问题分解为检测、评分、混淆和规则优化四个环节，每个环节都有清晰的动机和设计。特别是**重要性评分**的引入，让混淆操作可以根据具体情况“量身定制”，非常巧妙。
3.  **首次应用强化学习优化提示混淆规则**：将提示混淆的决策过程建模为一个可优化的规则学习问题，使得策略能够从数据中自动习得，而非依赖人工编写的固定规则，这显著提升了适应性和性能。这是该领域的一个主要创新。
4.  **用户端、无需合作的解决方案**：整个机制完全运行在用户设备上，只依赖一个有能力的本地LLM和简单的后处理，不依赖AI服务提供商的合作，这符合“信任最小化”的隐私保护原则。
5.  **全面的评估体系**：使用了多维度的指标（隐私增益、两种效用指标）和分布分析（KDE图）来全面衡量性能，而非仅依赖平均值，结果呈现更具说服力。

### 8. 不足与局限

1.  **对手模型假设的局限性**：假设对手是“诚实但好奇”的，即它会正确执行任务，但会从收到的提示中被动学习。未考虑更强大的对手，如：
    -   **跨提示去混淆**：对手通过观察用户多个混淆后的提示，通过关联分析推断真实信息。
    -   **主动诱导**：故意生成某种响应以诱使用户暴露更多敏感信息。
2.  **依赖本地LLM的性能和质量**：系统的效果高度依赖于本地LLM（Gemma3:12B）的检测、评分和生成能力。在论文测试中，检测器会漏掉很多弱信号或隐式信号，这导致了“保护不足”的失败模式。一个性能更强的本地模型可能会改善这一点，但也会增加计算成本。
3.  **评估场景的单一性**：仅在单轮对话的初始提示上进行了评估。虽然论文提到了处理多轮对话是未来工作，但目前的方法可能无法很好地扩展到有上下文依赖的多轮场景，因为诱饵信息的注入可能破坏对话的连续性或引入逻辑矛盾。
4.  **延迟问题**：尽管在同级GPU上可行，但每个查询额外增加的11.6秒延迟（主要来自检测的5.5秒）对于追求实时交互的用户体验来说可能是不小的负担。虽然作者提到检测是主要瓶颈，且可以通过更小的专用模型缓解，但这在文中并未实现。
5.  **效用评估的局限性**：虽然使用了两个互补指标，但它们并非完美。余弦相似度可能无法捕捉到细微信息变化，而NLI的准确性高度依赖于NLI模型本身的质量。此外，论文对“效用”的定义主要基于信息保留，没有考虑用户对响应“有用性”的主观判断。
