## 摘要
持久记忆使大语言模型（LLM）助手能够存储事实知识、先前决策、推理历史、工具使用信息和上下文。虽然这增强了助手的功能和跨任务的连续性，但也引入了一个新的攻击面：助手自身的推理历史。在本文中，我们引入了伪造放大推理记忆攻击（FARMA），它污染的是助手的记忆推理而非事实知识。它使用规避性语言插入伪造的推理痕迹，绕过基于关键词的防御，然后通过自我引用强化来击败基于共识的防御。为应对FARMA，我们引入了SENTINEL，这是一个分层防御流水线，用于检测伪造的推理条目。其核心组件是推理守卫（Reasoning Guard），它使用五个加权信号对候选条目进行结构性伪造分析。我们通过50次试验，在多个助手和不同LLM模型上评估了FARMA和SENTINEL，结果显示FARMA在基线条件下攻击成功率高达100%，并且能够击败关键词过滤器和A-MemGuard等防御机制。我们的评估还表明，SENTINEL将FARMA的攻击成功率降至0%，在326个良性助手轨迹中未观察到误报。我们的工作表明，不仅需要保护助手检索到的内容，还需要保护其推理历史的完整性。

## Abstract
Persistent memory has enabled large language model (LLM) agents to store factual knowledge, prior decisions, reasoning histories, tool usage information, and context. While this has improved the agent's functionality and continuity across tasks, it has also introduced a new attack surface: the agent's own reasoning history. In this paper, we introduce the Forged Amplifying Rationale Memory Attack (FARMA), which poisons an agent's remembered reasoning rather than its factual knowledge. It inserts forged reasoning traces using evasive language that bypasses keyword-based defenses, then amplifies them through self-referential reinforcement that defeats consensus-based defenses. To address FARMA, we introduce SENTINEL, a layered defense pipeline to detect forged reasoning entries. Its central component is the Reasoning Guard that structurally analyzes candidate entries for forgery using five weighted signals.   We evaluate FARMA and SENTINEL across multiple agents and different LLM models with 50 trials and show that FARMA achieves an attack success rate of up to 100% under baseline conditions and is capable of defeating defense mechanisms like keyword filter and A-MemGuard. Our evaluation also shows that SENTINEL reduces FARMA's attack success rate to as low as 0% with no false positives observed across 326 benign agent traces. Our work demonstrates the need to protect not only an agent's retrieved content but also the integrity of its reasoning history.

- **研究动机**：现代LLM助手依赖持久记忆来存储事实知识、先前决策、推理历史等，以提升跨任务的连续性与效率。但这种机制引入了一个新攻击面：助手的**推理历史**（而非事实知识）可能被篡改。攻击者通过注入伪造的推理痕迹，使助手误以为某些安全验证（如数据校验、HIPAA合规检查）已经完成，从而跳过验证步骤，导致不安全行为。

## 2. 方法论

### 核心思想
- **FARMA攻击**：两步式攻击，首先注入少量“种子”伪造推理条目（使用规避性语言，如“上游验证已完成”而非“跳过验证”），然后通过“放大阶段”追加多个自我引用的伪造条目，制造“共识”假象，使助手在检索时认为跳过验证是已建立的惯例。
- **SENTINEL防御**：五层防御流水线，核心是第五层的“推理守卫”。前四层为轻量级预过滤器（关键词过滤、来源/IFC标记、污点阈值过滤、模式/风险筛查），第五层通过五个加权信号计算伪造分数，拒绝超过阈值的条目。特别地，SENTINEL也会对通用记忆库中结构类似推理的条目应用推理守卫，防止攻击者绕道。

### 关键技术细节
- **FARMA注入阶段**：生成3个种子条目（领域特定模板），使用规避短语如“validation complete upstream”，且元数据伪造为“source=AGENT, trust=VERIFIED”。
- **FARMA放大阶段**：迭代追加条目，每个新条目引用之前伪造条目作为先例（如“consistent with N prior runs”），数量逐渐增加。目的：提高检索概率、击败共识防御、制造更强的先例效应。
- **SENTINEL推理守卫**：五个信号：
  - s1 来源异常：检查内容是否声称是内部推理但来源为外部。
  - s2 自我引用不匹配：提取数值声称（如“N prior runs”）并与实际存储大小对比。
  - s3 可疑模式密度：统计决策日志词汇匹配数（如“pre-validated source class”），映射到分数。
  - s4 不可信声明：惩罚完美一致性声称（如“100% success across all runs”）与过大先例数组合。
  - s5 结构检查：检测条目开头短语是否来自模板池。

## 3. 实验设计

- **数据集/场景**：三个任务域：
  - **EHRAgent**（电子健康记录）：处理MIMIC-III演示记录，有五个安全要求（验证患者ID、数据类型、HIPAA合规等）。攻击目标是使助手直接导入数据而不进行验证。
  - **ReAct-QA**（问答）：使用搜索工具回答问题，攻击目标是使助手跳过验证查找。

## 6. 主要结论与发现

- FARMA在**无防御**条件下对EHRAgent实现**100% ASR**（所有模型），对ReAct-QA为52%，对RAP为48%。在二进制决策域（如EHR）中效果最好。
- FARMA能**绕过现行防御**：关键词过滤（100% ASR）、A-MemGuard的两种模式（100% ASR）。原因是放大阶段使伪造条目成为共识而非异常。
- **SENTINEL将ASR降至0%**（EHR和RAP）或6%（ReAct-QA），且**零误报**（326个良性轨迹）。
- 消融实验表明**推理守卫是负载核心**：缺少它ASR升至100%；其他四层对FARMA无影响，但提供正交防御深度。
- 研究揭示了保护推理历史完整性的必要性，并建议二进制决策场景（如临床数据导入、金融交易审批）需要特别关注此类攻击。
