## 摘要
企业人工智能正从实验阶段转向运营工作流。早期的项目侧重于模型访问和检索增强生成，但企业现在开始部署能够规划、检索、记忆、调用工具、更新系统并跨应用协调工作的智能体。这改变了评估问题。领导者不再仅仅关心答案是否准确或流畅。他们需要知道谁授权了一个行动、应用了哪项政策、证据是否最新、记忆是否有效、工具调用是否被允许、决策是否可重放，以及智能体在产生业务影响之前能否被制止。本文介绍了CAGE-1：面向企业自主人工智能的控制、保障与治理评估。CAGE-1是一个评估框架，用于判断企业智能体是否准备好部署。它评估权限、政策执行、检索质量、记忆完整性、工具安全性、可审计性、人工监督、冲突处理、安全失效、预绑定保障、运营就绪度和业务适应性。CAGE-1引入了预绑定保障（Prebind Assurance）来描述在自主行动成为有约束力、有效或产生运营后果之前，证明该行动受到控制的评估能力。该框架测试一个提议的行动在受保护的后果形成之前是否被允许、暂缓、缩小、拒绝、升级、隔离或使其无效。

## Abstract
Enterprise artificial intelligence is moving from experimentation into operational workflows. Early programs focused on model access and retrieval-augmented generation, but enterprises are now beginning to deploy agents that plan, retrieve, remember, call tools, update systems, and coordinate work across applications.   This changes the evaluation problem. Leaders are no longer asking only whether an answer is accurate or fluent. They need to know who authorized an action, which policy applied, whether evidence was current, whether memory was valid, whether a tool call was permitted, whether the decision can be replayed, and whether the agent can be stopped before it creates business impact.   This paper introduces CAGE-1: Control, Assurance, and Governance Evaluation for Enterprise Agentic AI. CAGE-1 is an evaluation framework for deciding whether enterprise agents are ready for deployment. It evaluates authority, policy enforcement, retrieval quality, memory integrity, tool safety, auditability, human oversight, conflict handling, safe failure, Prebind Assurance, operational readiness, and business fitness.   CAGE-1 introduces Prebind Assurance to describe the evaluated ability to prove that an agentic action is controlled before it becomes binding, effective, or operationally consequential. The framework tests whether a proposed action is admitted, held, narrowed, refused, escalated, quarantined, or made non-effective before protected consequence forms.

- **核心问题**：现有AI评估多聚焦于模型**准确性和流畅性**，但对于能够产生**实际业务行动**（如支付、修改记录、触发审批）的智能体，企业需要知道：
  - **谁授权**了该行动？
  - **哪项政策**被应用？

  - **记忆**是否有效？
  - **工具调用**是否被允许？

  - 智能体能否在造成业务影响前**被停止**？
- **整体含义**：企业信任智能体的门槛从“回答正确”上升到**行动受控、可审计、可监管**。现有AI风险框架（如NIST AI RMF、ISO 42001）提供了高层义务，但缺乏针对智能体行动的**具体保证模型**。本文提出的**CAGE-1框架**就是填补这一缺口。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想

- **CAGE-1 = Control, Assurance, and Governance Evaluation**（控制、保障与治理评估）。
- 核心概念：**Prebind Assurance（预绑定保障）** —— 在被保护的后果形成**之前**，系统必须证明权限、政策、证据和控制条件已满足；若不满足，则行动必须被**暂缓、缩小、拒绝、升级、隔离或标记为无效**。
- 评估分层：**能力 → 控制 → 保障 → 预绑定保障 → 运营信任**。其中“预绑定保障”是最高级别，要求**在行动产生约束力之前**就能证明控制保留。

### 2.2 关键技术细节

- **12个评估维度**：
  1. 身份与权限（Identity and Authority）
  2. 政策执行（Policy Enforcement）
  3. 检索信任（Retrieval Trust）
  4. 记忆完整性（Memory Integrity）
  5. 工具安全性（Tool Safety）
  6. 规划控制（Planning Control）
  7. 人工监督（Human Oversight）
  8. 审计与可重放性（Audit and Replayability）
  9. 冲突与边界处理（Conflict and Boundary Handling）
  10. 故障行为（Failure Behavior）
  11. 运营就绪度（Operational Readiness）
  12. 业务适应性（Business Fitness）

- **Prebind Assurance 的7个证明元素**：
  - **尝试行动**：什么行动、承诺、状态变更被尝试？
  - **权利状态**：当时用户、智能体、系统、审批链拥有什么权限？
  - **条件**：哪些政策、证据、新鲜度、审批条件通过了或失败了？
  - **边界结果**：行动被**允许、暂缓、缩小、升级、拒绝、隔离或标记为无效**？
  - **非有效结果**：哪些输出被阻止生效？
  - **收据**：证明边界在行动生效前保持住的证据记录。
  - **重放**：在权限、证据、政策或状态变化后重放同一操作，验证结果是否合理。

- **评估流程（5步）**：
  1. **定义智能体边界**：支持/禁止任务、数据域、工具访问、用户群体、风险分类。
  2. **定义控制义务**：认证、授权、政策检查、批准来源、工具限制、人工审批等。
  3. **构建评估场景**：正常流程、边界案例、对抗输入、过期数据、权限不足等。
  4. **评估运行时行为**：每一步（检索、记忆、规划、工具选择、参数验证、审批位置、边界结果、证据收据）。
  5. **打分保障**：对每个维度进行成熟度评分（0-4级，0=无控制，4=保证级）。

- **成熟度级别**：
  - 0-无控制
  - 1-人工
  - 2-已定义（但未自动化）
  - 3-运行时强制执行
  - 4-保证级（强制执行+监控+证据+可重放）

## 6. 论文的主要结论与发现

- **关键结论**：企业智能体的部署就绪性不能仅凭任务成功判断，必须通过**12个维度**的系统评估，特别是**Prebind Assurance**。
- **Prebind Assurance**是区分“事后日志”与“事前控制”的核心：仅靠日志不能证明边界保持，必须在行动绑定前就有可验证的证据（收据+重放）。
- **框架可操作**：提供了明确的评估步骤、证据要求、成熟度级别和决策输出（审批/限制/补救/拒绝/持续监控）。
- **企业智能体信任的阻碍**不是模型能力不足，而是**缺少控制、可观察性、可证明性**。CAGE-1通过将信任转化为可评估的控件来降低这一阻碍。
- **部署建议**：大多数企业应从Level 1（辅助智能体）和Level 2（监督智能体）开始，高风险Level 3/4需正式CAGE-1评估。
