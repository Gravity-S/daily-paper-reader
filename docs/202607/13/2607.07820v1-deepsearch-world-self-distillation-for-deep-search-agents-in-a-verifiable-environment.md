## 摘要
训练工具使用智能体从自身经验中改进仍然具有挑战性，因为监督微调依赖于固定的教师蒸馏轨迹，而稀疏奖励强化学习为长期交互提供的监督较弱。我们提出了DeepSearch-Evolve，一个基于DeepSearch-World构建的网络智能体自蒸馏框架，DeepSearch-World是一个确定性的、可验证的环境，具有可复现的搜索和页面阅读工具。DeepSearch-World包含从实体级随机游走构建的42万个多跳问答任务，并支持自进化中关键的主体认知行为，包括进度验证、基于事实的反思和失败恢复。DeepSearch-Evolve迭代地进行轨迹生成、过滤、数据混合和微调，以训练更强的智能体。无需从更强大的模型蒸馏，DeepSearch-World-9B在与开源智能体相比时表现出竞争性性能，在BrowseComp上达到31.2%，在GAIA上达到61.5%，在HotpotQA上达到93.4%，表明可验证环境使得长期网络智能体能够实现可扩展的自进化。我们将发布环境、42万训练池、验证集、模型和代码，以促进未来对自我改进的深度搜索智能体的研究。

## Abstract
Training tool-use agents to improve from their own experience remains challenging, as supervised fine-tuning relies on fixed teacher-distilled trajectories, while sparse-reward reinforcement learning provides weak supervision for long-horizon interactions. We present DeepSearch-Evolve, a self-distillation framework for web agents built on DeepSearch-World, a deterministic and verifiable environment with reproducible search and page-reading tools. DeepSearch-World contains 420K multi-hop QA tasks constructed from entity-level random walks and supports key agentic cognitive behaviors useful for self-evolving, including progress verification, grounded reflection, and failure recovery. DeepSearch-Evolve iteratively performs trajectory generation, filtering, data mixing, and fine-tuning to train stronger agents. Without distillation from more capable models, DeepSearch-World-9B achieves competitive performance compared with open-source agents, reaching 31.2% on BrowseComp, 61.5% on GAIA, and 93.4% on HotpotQA, showing that verifiable environments enable scalable self-evolution for long-horizon web agents. We will release the environment, 420K training pool, validation set, model, and code to facilitate future research on self-improving deep search agents.

## 1. 论文的核心问题与整体含义（研究动机和背景）

**核心问题**：工具使用智能体（tool-use agents）在执行多步任务（如网页搜索、信息检索）时，难以从自身交互经验中自发改进。现有方法要么依赖**固定教师蒸馏轨迹**（监督微调 SFT），受限于教师能力上限和轨迹多样性；要么依赖**稀疏最终奖励**（强化学习 RL），在长程交互中难以定位失败环节（如查询构建、工具选择、证据提取等）。

**研究背景**：近期大语言模型（LLM）已能进行多步规划和工具调用，但如何让模型**自主进化**（self-evolving）仍是关键挑战。论文指出，现有自改进管线受限于静态模仿、稀疏奖励或不可靠的密集监督，缺乏一个**可验证的环境**来提供过程级监督。

**整体含义**：论文提出一种新型自蒸馏框架 DeepSearch-Evolve，构建在确定性、可验证的环境 DeepSearch-World 之上。该环境支持进度验证、基于事实的反思和故障恢复，使智能体能从自身生成的**已验证轨迹**中提取规划、记忆、纠错等行为，无需外部更强模型蒸馏，实现可扩展的自演进。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

**核心思想**：通过一个**确定性的离线环境**（DeepSearch-World）提供过程级验证信号，然后使用**脚手架教师（scaffold teacher）** 生成高质量轨迹，最后将这些轨迹转换为标准 ReAct 格式，通过**迭代 SFT**（自蒸馏）训练学生模型。整个过程无需在线 RL 或外部教师。

**关键技术细节**：

1. **环境构建（DeepSearch-World）**：
   - 基于 Wikipedia 知识图谱，通过实体级随机游走（entity-level random walks）构建 42 万个多跳 QA 任务。
   - 离线语料库包含约 1000 万个条目（标题、摘要、全文），使用 BM25 索引（Pyserini）实现确定性搜索，SQLite 偏移索引实现确定性页面访问。
   - 环境支持**实体级进度验证**：存储每个问题对应的目标实体集合，每次工具调用后检查是否匹配未解决实体，实现无需 LLM 评判的客观进度追踪。
   - 失败时触发阶段式**基于规则的反思**（grounded reflection），第一次失败给通用修订信号，重复失败则提供规范实体名或模糊描述，鼓励查询重写和恢复。

2. **脚手架教师智能体（scaffold teacher agent）**：
   - 分为三个阶段：**Plan**（初始化结构化的进度状态，含 `completed_list`、`todo_list`、`experience`、`information` 四个可写字段）、**Act**（最多 Tmax=30 步的迭代工具调用，每次调用后根据环境反馈和反射更新进度状态）、**End**（基于验证的工作记忆生成简洁答案）。
   - 该脚手架仅在轨迹生成阶段使用，用于显式监督规划、记忆、错误恢复等行为。

3. **自进化训练循环（DeepSearch-Evolve）**：
   - **迭代过程**：第 R 轮，当前模型 π_θR 作为教师，在DeepSearch-World中生成脚手架轨迹 → 保留答案正确的轨迹 → 用 Qwen3.5-9B 进行**轨迹质量过滤**（去除冗余证据、弱目标对齐、不一致推理）→ **脚手架到 ReAct 转换**（移除阶段特定提示，将进度状态和反射重写为学生推理痕迹）→ **SFT 更新**学生模型 π_θR+1，作为下一轮教师。
   - **数据混合**：由于生成和训练异步，引入重要性采样（decay factor γ=0.5），优先使用最新验证数据同时保留早期轨迹以防灾难性遗忘。
   - **更新公式**（式 3）：LSFT(θ) = E_{τ~D(R)} ∑_{t=1}^T KL(δ_yt || π_θ(·|x, y<t))，等价于负对数似然。论文讨论其与 on-policy self-distillation (OPSD) 的关系，指出 OPSD 可能不稳定，而 evolving SFT 在长程工具使用中更稳定。

## 6. 论文的主要结论与发现

1. **可验证环境使能自演进**：DeepSearch-World 的确定性验证信号（实体级进度追踪、反思）使智能体能够从自身经验中学习规划、记忆、纠错等高级认知行为，无需外部更强模型蒸馏。
2. **DeepSearch-World-9B 性能优异**：在 BrowseComp（31.2%）、GAIA（61.5%）、HotpotQA（93.4%）等基准上，超越大多数同等规模开源智能体，且与依赖更强教师蒸馏的竞争方法（如 Marco-DR 31.4% on BrowseComp）接近。
3. **骨干模型大幅提升**：相比于 Qwen3.5-9B-Instruct，DeepSearch-World-9B 在全部基准上提升显著（+23.8~+48.1 个百分点）。
4. **数据池规模至关重要**：420K 训练池比 100K 能提供更持久的增益，避免过早饱和和过拟合（图 5）。
5. **组件消融验证**：拒绝采样（正确答案验证）是主要增益来源，质量过滤进一步去除冗余；状态内化和反思重写均贡献正向效果。
6. **长程交互能力增强**：训练后的模型平均工具调用轮数从 4.7 增至 18.0，访问调用从 0.9 增至 5.4，高级能力评分从 19% 升至 70%。

## 7. 优点：方法或实验设计上的亮点

- **环境设计新颖**：DeepSearch-World 是首个提供确定性、可重放、实体级验证的离线 Wikipedia 环境，支持过程监督和受控实验。
- **自蒸馏框架简洁有效**：使用脚手架教师生成高质量轨迹，然后转换为标准 ReAct 格式，避免了复杂 RL 的超参调优和不稳定性。
- **迭代进化机制**：异步生成和训练，重要性采样混合历史数据，防止灾难性遗忘。
- **大规模数据自动构建**：通过实体随机游走+特征模糊化生成 42 万真实多跳问题，覆盖广泛知识性和推理路径。
- **丰富的消融和可视化分析**：不仅报告最终性能，还分析了轮次趋势、工具调用行为、高级能力评分，揭示内部分析机制。
- **开放性**：承诺发布环境、数据、模型和代码，有利于社区复现和后续研究。

## 8. 不足与局限

1. **环境领域有限**：当前环境仅基于 Wikipedia，覆盖知识范围受限，无法处理实时新闻、专业数据库、多模态内容等扩展场景。
2. **训练算法单一**：仅使用 SFT（自蒸馏）进行更新，未探索 RL 或 OPSD 更新，论文自身也承认“如何将高级规划、纠错能力注入 RL 训练仍待探索”。
3. **离线到实时 Gap**：尽管使用 SerpAPI/Jina 进行对齐，但 Wikipedia 环境的结构化工具（固定 URL、确定返回）与真实搜索引擎（动态、噪声）存在差异，可能影响迁移性。
4. **评估维度有限**：未在实时动态任务（如 WebArena、SWE-bench）或需要多模态输入的基准上测试。
5. **计算资源依赖**：虽然 9B 模型相对轻量，但 11 轮迭代训练仍需多 GPU 天数，对于资源有限的研究者可能存在门槛。
6. **潜在偏差**：来自 Wikipedia 的数据可能包含社会偏见，模型有可能继承这些偏见，论文也提及了这一点。
