## 摘要
持久化AI智能体将大语言模型（LLMs）从单轮交互扩展到长期运行的软件系统。与传统聊天助手不同，这些智能体中的不安全内容可以通过持久化状态、可复用技能和工具中介的交互传播，形成更大的语义攻击面。我们观察到，此类智能体中大多数安全关键交互通过自然语言令牌流传输，包括内存更新、工具参数、检索文件以及组件间通信。这一观察使我们能够提出一种新的安全形式化：不安全行为可以在到达特权运行时接收点之前被拦截为有风险的语义流。基于这一洞见，我们提出TokenWall，一种作为智能体令牌流语义防火墙的运行时防御框架。TokenWall对这些流进行边界感知的语义审计，构建结构化的源-接收审计记录，在执行前进行轻量级本地检查，并将模糊的高风险案例选择性地升级到更强的仲裁模块。与依赖稀疏审计或远程大模型监督的先前方法不同，TokenWall实现了全覆盖的执行前中介，同时减少远程仲裁和延迟。在CIK-Bench上的实验表明，TokenWall将攻击成功率降低至12.5%，同时在没有人工确认的情况下保持97.4%的良性可执行通过率。TokenWall在良性案例上仅引入0.69秒的额外延迟，表明语义运行时遏制可以为持久化AI智能体实现实用的安全-效用权衡。

## Abstract
Persistent AI agents extend large language models (LLMs) beyond single-turn interaction into long-lived software systems. Unlike traditional chat assistants, unsafe content in these agents can propagate through persistent state, reusable skills, and tool-mediated interactions, creating a substantially larger semantic attack surface. We observe that most security-critical interactions in such agents are transmitted through natural-language token flows, including memory updates, tool arguments, retrieved files, and inter-component communications. This observation enables a new security formulation: unsafe behavior can be intercepted as risky semantic flows before reaching privileged runtime sinks. Based on this insight, we propose TokenWall, a runtime defense framework that acts as a semantic firewall over agent token flows. TokenWall performs boundary-aware semantic auditing over these flows, constructing structured source-sink audit records, applying lightweight local inspection before execution, and selectively escalating ambiguous high-risk cases to stronger arbitration modules. Unlike prior approaches that rely on sparse auditing or remote large-model oversight, TokenWall enables full-coverage pre-execution mediation while reducing remote arbitration and latency. Experiments on CIK-Bench show that TokenWall reduces attack success rate to 12.5% while maintaining a 97.4% benign executable pass rate without human confirmation. TokenWall further introduces only 0.69 seconds of additional latency on benign cases, demonstrating that semantic runtime containment can achieve a practical security-utility trade-off for persistent AI agents.

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想：Token流语义审计

- 将智能体执行建模为一连串**Token流** \( f = (x, s, t, c, b) \)，其中 \( x \) 是分割后的载荷（span），\( s \) 是源，\( t \) 是接收点（sink），\( c \) 是运行时元数据，\( b \) 是边界类型。
- 三个安全表面：**上下文流**（影响推理/记忆）、**权限流**（改变身份/会话绑定）、**能力流**（触发工具执行/外部披露）。
- 在Token流到达保护接收点之前进行预执行审计，做出 **allow / rewrite / defer / block** 决策。

### 2.1 技术流程（分层审计管道）

1. **确定性预检（Precheck）**：基于规则快速识别显式违规（如硬编码密钥、私有端点、破坏性操作）。硬规则直接拦截；软规则标记为证据传递给下一阶段。
2. **本地小型语义审计器（Local Small Auditor，默认Qwen3-4B）**：
   - 接收结构化的源-汇记录（含安全表面、边界元数据、载荷片段、所有者意图摘要、预检信号）。
   - 输出结构化报告，包含决策（allow/rewrite/defer/block）、五个[0,1]分数（风险、不确定性、可利用性、影响、重写完整性），以及是否需要升级。
   - 支持**可恢复重写**：移除/屏蔽风险片段，保留良性任务语义；但禁止添加新事实、扩大权限等。
3. **升级仲裁（Fallback Arbiter）**：当本地审计报告满足升级条件（如高不确定性、高残留风险、不完整重写）时，调用强容量仲裁器（Qwen3.6-Plus）做出最终决策。升级条件由表面特定的谓词 \(\Phi_{\text{type}}(f)\) 根据预设阈值判断。

### 2.3 关键设计点

- 审计在**预效应边界**（pre-effect boundary）进行，即载荷已生成但未提交到持久状态或外部行动之前。
- 结构化报告设计：决策+多维度分数+证据，支持细粒度升级决策。
- 放弃全量大模型审计，仅对模糊/高风险案例升级，从而实现全量覆盖与低延迟的权衡。

## 7. 优点

- **新颖性**：首次系统性地提出将**Token流**作为安全审计的基本单元，而非传统的输入/输出或动作级审计，更加细粒度。
- **实用性**：通过分层审计实现全量覆盖（而非稀疏）的预执行中介，同时将远程升级比例降低至15.9%，兼顾安全与效率。
- **可恢复性**：引入语义重写机制，在不阻断良性任务的情况下移除风险片段，对比直接拦截或人工介入更具实用价值。
- **系统性和透明度**：详细披露了预检规则、审计提示模板、升级谓词阈值等，具备较好的可复现性。

## 8. 不足与局限

- **评估范围有限**：仅在CIK-Bench单一基准上测试，且仅针对OpenClow风格个人代理，未覆盖更广泛的代理环境（如Coding Agent、多Agent系统）或真实世界任务分布。
- **依赖本地模型质量**：本地审计器的性能直接决定默认路径的安全性和效率，若部署于弱模型或资源受限环境，性能可能下降。
- **安全覆盖不完整**：仅针对运行时语义流的安全，不处理主机被攻陷、在运行时外窃取凭证、拒绝服务攻击、用户故意授权有害行为等情况。
- **升级仲裁仍存在风险**：虽然减轻了远程调用，但模糊/高风险案例仍需依赖强仲裁器，其决定质量受限于模型本身的安全性；且仲裁器可能被对抗性样本绕过。

- **可扩展性**：实验仅在benchmark环境中进行，未评估大规模并发部署下的吞吐量、缓存效率等系统级性能。
