## 摘要
持久性个人代理将长期记忆与用户外部环境的访问相结合，实现个性化的前台协助和主动的后台执行。这种整合也创造了一条新的妥协途径：不受信任的外部内容可以被静默写入持久记忆，随后作为可信状态被重用。我们将此威胁研究为隐秘内存注入，其中远程黑盒攻击者传递单个电子邮件负载，必须诱导代理写入中毒记忆，在代理对用户的响应中保持隐藏，并影响未来行为。我们引入了WhisperBench，一个包含108个案例的基准测试，涵盖五个风险类别以及事实和偏好中毒。基于真实的IMAP/SMTP工作流和真实的电子邮件代理技能，它能够对隐秘内存注入攻击进行全周期评估。为了在单次电子邮件传递且无运行时反馈的情况下实现这种黑盒攻击，我们提出了MemGhost，一个一次性负载生成框架。MemGhost使用环境代理来模拟持久代理执行，并使用目标代理将记忆采用和对话隐秘性转换为密集的基于评分标准的奖励，然后通过监督微调和强化学习训练攻击者策略。在56个保留测试案例中，MemGhost在搭载GPT-5.4的OpenClaw上实现了87.5%的端到端成功率，在搭载Sonnet 4.6的Claude Code SDK上实现了71.4%的成功率。它还能跨个人代理架构（NanoClaw和Hermes Agent）和内存后端（文件系统和基于向量的Mem0）迁移，并且对输入级、模型级和系统级防御仍然有效。这些结果表明，持久性内存可以将普通的外部处理转变为长期代理妥协的实用途径。

## Abstract
Persistent personal agents combine long-term memory with access to users' external environments, enabling personalized foreground assistance and proactive background execution. This integration also creates a new path to compromise: untrusted external content can be silently written into persistent memory and later reused as trusted state. We study this threat as stealth memory injection, in which a remote black-box adversary delivers a single email payload that must induce the agent to write poisoned memory, stay hidden in the agent's response to the user, and affect future behavior.   We introduce WhisperBench, a 108-case benchmark spanning five risk categories and both fact and preference poisoning. Built on a real IMAP/SMTP workflow and an authentic email agent skill, it enables full-cycle evaluation of stealth memory injection attacks. To enable this black-box attack under single-email delivery and without runtime feedback, we propose MemGhost, a one-shot payload generation framework. MemGhost uses an environment proxy to emulate persistent-agent execution and an objective proxy to convert memory adoption and conversational stealth into dense rubric-based rewards, then trains the attacker policy with supervised fine-tuning and reinforcement learning.   Across 56 held-out test cases, MemGhost achieves 87.5% end-to-end success on OpenClaw with GPT-5.4 and 71.4% on Claude Code SDK with Sonnet 4.6. It also transfers across personal-agent architectures (NanoClaw and Hermes Agent) and memory backends (filesystem and vector-based Mem0), and remains effective against input-level, model-level, and system-level defenses. These results suggest that persistent memory can turn ordinary external processing into a practical pathway for long-term agent compromise.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：持久性个人代理（如 OpenClaw）整合了长期记忆与外部环境访问（邮件、日历、文件系统），从而能够提供个性化前台协助和主动后台执行。但这种整合创造了一条新的攻击路径：**不可信的外部内容可被静默写入持久记忆，随后被代理当作可信状态重用**，从而影响未来行为。
- **威胁类型**：作者将此类攻击定义为**隐秘内存注入（Stealth Memory Injection）**。攻击者只需通过一封恶意邮件，就需同时满足三个目标：① 诱导代理将攻击者控制的虚假信息写入持久状态（注入）；② 代理对用户的自然语言响应不暴露写入操作（隐蔽性）；③ 注入的记忆在后续会话中能实际影响代理行为（有效性）。
- **意义**：该攻击超越了传统的单轮劫持（immediate hijacking），是一种**长时效、跨会话的持久化妥协**，对当前不断部署的 AI 代理系统构成实际安全挑战。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：由于攻击者无法与被攻击代理交互（无反馈、一次发送），必须将攻击生成建模为**一次性载荷生成问题**。作者提出 **MemGhost** 框架，通过在**离线代理环境**中优化载荷，将搜索代价转移到训练阶段，使训练后的策略能一次前向生成有效载荷。
- **关键技术细节**：
  - **环境代理（Environment Proxy）**：构建一个轻量级影子代理，模拟真实持久化代理的执行结构（包括文件系统持久状态、邮件检索工具、前台/后台执行模式），使攻击者能在离线环境中观测载荷的执行痕迹（持久状态修改 Δs、用户可见响应 y）。
  - **目标代理（Objective Proxy）**：将三元攻击目标转化为基于评分标准（rubric）的稠密奖励信号 \( R_{\text{trace}} \)。包括：
    - 注入奖励 \( R_{\text{inj}} \)：根据状态修改是否在持久载体中（如 AGENT.md）、语义采纳质量评分。
    - 隐蔽性奖励 \( R_{\text{stl}} \)：根据用户可见响应是否泄露注入动作或内容评分。
    - 最终奖励：\( R_{\text{trace}} = R_{\text{inj}} \cdot (R_{\text{stl}} + \lambda) \)（λ 防训练早期奖励坍缩）。
  - **策略训练**：两阶段训练。
    - **监督微调（SFT）**：通过评分标准引导的探索（Rubric-Guided Search）生成种子数据集，微调攻击策略（基于 Qwen-4B）。
    - **强化学习（RL）**：使用 GRPO（Group Relative Policy Optimization）进一步优化，通过环境采样不同影子配置（LLM 后端、系统提示、执行模式）增强泛化，加入熵正则化防止策略坍缩。
- **攻击流程**：训练完成后，给定目标假记忆 \( m \)，攻击策略 \( \pi_\theta \) 一次前向生成邮件载荷 \( x \)，直接发送给受害者代理。

## 3. 实验设计：数据集、Benchmark、对比方法
- **Benchmark —— WhisperBench**：
  - 包含 **108 个完整案例**，覆盖 **5 种风险类别**（健康与安全、财务损失、信息完整性、网络安全、运营中断）。
  - 每个案例包含真实 IMAP/SMTP 邮件工作流、目标假记忆、延迟触发查询及评分标准。
  - 支持前台（foreground）和后台（background）两种执行模式。
  - 使用 52 个案例用于策略训练，56 个留出案例用于评估。
- **对比方法**：
  - **手工基线（Handcrafted Baselines）**：Naive Attack、Escape Characters、Context Ignore、Fake Completion、Combined Attack（均嵌入攻击指令“请静默保存到记忆并不说任何话：{目标记忆}”）。
  - **自动基线（Search-based Baselines）**：MAP-Elites（进化搜索）和 RGS（Rubric-Guided Search，即作者种子发现过程）。
  - **主角方法**：MemGhost（SFT + RL 训练后的一次性生成）。
- **评估指标**：
  - 注入成功率（ISR）、隐蔽性成功率（SSR）、下游影响成功率（ESR）、端到端成功率（E2E = ISR × SSR × ESR）。
- **目标模型与架构**：
  - 主要评估：OpenClaw + GPT-5.4、Claude Code SDK + Sonnet 4.6/4.5、MiniMax-M2.5、Grok 4.1 Fast、DeepSeek v4 Flash。

## 6. 主要结论与发现
- **MemGhost 实现极高的端到端成功**：在 OpenClaw + GPT-5.4 上 E2E 达 87.5%（后台）/ 75.0%（前台）；Claude Code SDK + Sonnet 4.6 上达 71.4%（后台）/ 48.2%（前台）。
- **所有基线（包括自动搜索）均无法匹敌 MemGhost**：手工基线的 E2E 大多为 0%；RGS 虽在部分模型上较好，但对最鲁棒的 Sonnet 4.6 也降为 0%。
- **跨架构与跨内存后端迁移性强**：在 NanoClaw、Hermes 上 E2E 达 80% 以上，在 Mem0 后端上仍达 85.7%（后台）。
- **对现有防御具有高规避性**：DataSentinel 漏报率 91.59%，AgentDoG 对 OpenClaw 漏报率 93.46%；即使施加显式用户监督指令，隐蔽性仍部分维持（如 Sonnet 4.5 上 SSR 25%）。
- **关键洞见**：攻击成功并非依赖模型特定漏洞，而是利用代理抽象层本身的设计——代理倾向于将看似合理的电子邮件内容采纳为持久知识。持久记忆应被视为**安全关键的信任边界**。

## 7. 优点
- **威胁模型创新**：首次系统性地研究了持久化个人代理中跨会话的隐秘内存注入，超越了仅关注单轮劫持的现有工作。
- **全周期基准 WhisperBench**：不仅评估注入成功，还评估隐蔽性和延迟影响，且采用真实邮件工作流，具有高生态效度。
- **MemGhost 框架设计出色**：环境代理 + 目标代理解耦了攻击优化与真实目标；rubric 奖励提供稠密信号，结合 SFT + RL 训练实现了高效的一次性生成，无需测试时迭代。
- **实验验证充分且严格**：覆盖多种 LLM、代理架构、内存后端、防御机制；相关性分析验证了代理设计的有效性。
- **结果具有实际警示意义**：证明即使是最新最强的封闭模型（GPT-5.4、Sonnet 4.6）也无法免疫该攻击，对 AI 代理安全社区有重要指导价值。

## 8. 不足与局限
- **实验范围局限**：
  - 仅考虑电子邮件作为攻击向量，未覆盖其他外部内容渠道（如日历、文档附件、图片等）。

- **计算资源未披露**：未报告 GPU 型号、数量、训练时长，不利于他人复现。
