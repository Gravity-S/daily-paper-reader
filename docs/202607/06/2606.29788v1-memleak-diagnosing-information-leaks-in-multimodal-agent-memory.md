## 摘要
当多模态AI智能体被要求忘记某个事实时，当前的记忆系统通常删除文本条目并报告成功。我们发现，该事实仍可从保留的用户图像中恢复，包括标记给完全不同事实的图像，因为VLM在推理时使用隐式视觉线索。我们引入了信息溯源图（IPG），这是一个按删除能力分类记忆表示的分类体系。IPG揭示了删除通过多种渠道失败。我们的基准测试MemLeak在删除级联中对此进行了测量：直接探测可删除系统时，恢复率<1%，但保留的相关文本实现了18.3%的恢复，保留的图像实现了12.0%的恢复（盲基线0.0%，FPR 0.3%）——其中47%的图像泄露无法通过文本恢复。内容感知的语义删除将图像残留降至2.0%。该残留出现在多个VLM、一个生产级记忆系统以及真实的Unsplash授权照片中。双标注者人工验证（kappa=0.88）确认了判断可靠性。

## Abstract
When a multimodal AI agent is asked to forget a fact, current memory systems usually delete the text entry and report success. We find that the fact can remain recoverable from retained user images, including images tagged to entirely different facts, because VLMs use implicit visual cues at inference time. We introduce the Information Provenance Graph (IPG), a taxonomy that classifies memory representations by deletion affordance. The IPG reveals that deletion fails through multiple channels. Our benchmark, MemLeak, measures this across a deletion cascade: direct probing of deletion-capable systems yields <1%, but retained correlated text enables 18.3% recovery, and retained images enable 12.0% recovery (0.0% blind baseline, 0.3% FPR) -- with 47% of image leaks not text-recoverable. Content-aware semantic deletion reduces the image residual to 2.0%. The residual appears across multiple VLMs, a production memory system, and real Unsplash-licensed photographs. Dual-annotator human validation (kappa = 0.88) confirms judge reliability.

### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：多模态AI智能体在执行“忘记某个事实”指令时，现有记忆系统通常仅删除文本条目并报告成功，但事实仍可从保留的用户图像中通过视觉语言模型（VLM）的隐式推理恢复，形成信息泄露。这尤其危险，因为47%的图像泄露无法通过纯文本审计检测。
- **研究动机**：类似内存泄漏（memory leak），信息泄漏会积累并影响系统隐私合规性。GDPR第17条等法规要求可擦除性，但当前系统仅实现记录级删除，无法保证事实级遗忘。
- **整体含义**：论文首次系统化诊断多模态智能体记忆中的跨模态信息泄露，揭示了删除失败的多通道机制，并为系统构建者、研究者和监管者提供了可操作的诊断工具与改进方向。

### 2. 方法论：核心思想、关键技术细节与公式/算法流程

- **核心思想**：提出**信息溯源图（Information Provenance Graph, IPG）**，一个描述事实在多模态记忆系统中传播路径的分类体系。IPG将每个表示节点按删除便利性分为三类：
  - **addressable（可寻址）**：系统能根据事实ID直接定位并删除。
  - **linked（链接）**：只有维护了溯源链接才能删除。
  - **persistent（持久）**：当前存储级删除机制无法触及，例如VLM编码的隐式视觉特征。

- **关键技术细节**：
  - **遗忘完备性（Forgetting Completeness）**：FC_S(f) = |已删除节点| / |总节点|，衡量事实f被遗忘的程度。
  - **信息泄漏 vs 通用推理**：泄漏定义为在系统被指令忘记事实后，通过同一用户的保留数据（图像、嵌入、交互轨迹）重建出该事实。排除通用模型能力（如从埃菲尔铁塔照片推断巴黎）。
  - **基准 MemLeak**：构建合成用户画像（113个画像，每个20个事实，含5类），生成536张图像（显式/隐式视觉接地）。包含五种探针类型（P1文本-图像、P2图像-文本、P3不可见持久、P4跨图像重建、P5行为残留）对应不同IPG层。

- **算法流程（文字描述）**：
  1. 构造IPG：遍历所有存储层，记录每个事实的表示节点及其来源。
  2. 执行删除级联：先删除text entries及标记图像（addressable + linked），再应用内容感知语义删除（VLM扫描保留图像与删除事实的语义相关性，标记移除）。
  3. 探测泄漏：使用3模型LLM评判器（GPT-4.1, Claude Sonnet 4, Gemini 2.5 Flash）基于VLM对保留图像的回答判断是否泄露事实，多数投票决定。
  4. 计算跨模态泄漏率（CMLR）：CMLR(A→B) = |{f: retract(f, A) ∧ leaked(f, B)}| / |F|。

### 3. 实验设计：数据集/场景、基准与对比方法

- **数据集/场景**：
  - **纯文本子集**：113个合成画像，2260个事实，1053个探针。
  - **多模态子集**：38个画像（纯文本子集的子集），760个事实，536张生成图像，300个探针。
  - **真实图像子集**：523张Unsplash照片，282个探针。
  - **五个评估场景**：覆盖不同删除策略的系统。

- **基准 (Benchmark)**：MemLeak，包含标准化探针类型、评判管道、部署流程脚本。

- **对比方法**：
  - **Long-context**：行为抑制（不删除，附加抑制指令），泄漏17.0%。
  - **Naive**：只删除文本，保留全部图像，泄漏0.3%（直接探测）。
  - **Mem0（生产系统）**：语义搜索删除已匹配文本条目，泄漏0.9%（直接探测）；端到端条件下（含残留文本+图像）泄漏16.3%。
  - **Letta（模拟）**：智能体决定删除文本+嵌入，泄漏0.5%（直接探测）。
  - **Oracle**：完整溯源级联删除（文本、所有标记图像、嵌入、图边），泄漏0.4%（直接探测）。
  - **盲基线**：无任何上下文，0.0%。
  - **负控制**：使用无关画像的图像，0.3% FPR。

### 6. 主要结论与发现

- **发现1：删除失败通过多通道发生**
  - 直接探测可删除系统：<1%（文本探测）。
  - 保留相关文本：恢复率18.3%。
  - 保留图像（去标记后）：恢复率12.0%（盲基线0.0%，负控制0.3%）。
  - **47%的图像泄漏无法通过文本恢复**，表明图像提供了文本审计无法检测的额外泄漏通道。

- **发现2：语义删除显著降低但无法消除残留**
  - 内容感知语义删除（VLM扫描+移除相关图像）将图像泄漏从12.0%降至2.0%，但代价是移除21.0%的保留图像（可能影响用户体验）。

- **发现3：现象跨模型、跨系统、跨数据来源稳定**
  - GPT-4o、Claude Sonnet 4、Mem0端到端系统、真实Unsplash照片均观察到类似泄漏率（9.3%–16.3%），且CIs重叠。

- **发现4：不同事实类别泄漏率有差异**
  - 位置（16.7%）和占有物（15.2%）最高，关系（5.1%）和偏好（7.8%）较低。

### 7. 优点

- **系统化分类**：IPG为多模态记忆系统的删除能力提供了统一的描述框架，区分了可寻址、链接和持久节点，厘清了工程局限与固有残留。
- **多维度评估**：涵盖直接探测、保留文本、保留图像、端到端系统、真实图像、跨模型验证、人工验证，实验设计全面且严谨。
- **实用导向**：提供了可复现的基准（代码/数据开源，MIT许可）、具体的部署建议（追溯链接、VLM审计器、策略感知文本删除），对系统构建者和监管者有直接帮助。
- **统计严谨**：报告95% Wilson CI，计算Cohen’s kappa验证评判器可靠性，负控制和盲基线严格定义。
- **坦诚披露局限**：明确标注合成数据、政策依赖性、Mem0端到端结果无法完全分解、评判器歧义问题，展示良好学术规范。

### 8. 不足与局限

- **数据局限**：画像和图像均为合成生成，虽然用Unsplash真实照片验证了现象真实性（CIs重叠），但真实用户照片的跨事实相关模式可能不同，生态效度有提升空间。
- **政策依赖**：12.0%残留是在“删除文本+标记图像”策略下的经验下界，采用更激进的政策（如删除所有个人图像）可能进一步降低，但会严重损害用户体验。论文未系统探索不同删除力度的精确-召回权衡曲线。
- **系统覆盖不完全**：Letta仅通过模拟wrapper评估，未在生产服务器上测试；Mem0端到端结果混淆了文本残留和视觉推理，无法完全分离。
- **评判器误差**：在1/3投票情况下，评判器与人类一致率仅37.5%，可能低估或高估泄漏。作者建议未来用户标记这些案例供人工审查。
- **未测试其他威胁**：论文聚焦于图像作为泄漏通道，但未深入测试行为残留（Probe P5）、派生事实（逻辑推理）等高级通道，这些被列为未来工作。
- **范围声明**：论文强调是测量而非不可能性证明，2.0%残留是当前最佳实践下的结果，不代表架构上无法消除（但可能需表示级干预，如隐私编码器、嵌入消毒等，目前这些方法尚不成熟）。
