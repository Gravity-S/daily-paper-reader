## 摘要
人工智能智能体越来越被期望像数字员工一样运作：访问企业数据、做出决策、自主采取行动。但智能体同时比人类更不可预测——容易产生幻觉、误解和对抗性操纵——并且技术上更有能力：具有深度系统知识和高吞吐量接口，以机器速度级联损害。这种组合使得依赖智能体忠实地解释或传播安全关键元数据（如访问策略、数据分类和行为约束）变得不安全。我们提出Redpanda智能数据平面（ADP），一种围绕带外元数据通道构建的架构：基础设施路径，确定性地携带安全上下文、策略信号和审计跟踪，完全在智能体的读写路径之外，跨越异构基础设施。这些通道在智能体生命周期的每个阶段强制执行治理——在数据进入时限定数据访问范围，在执行期间约束操作，在数据输出时捕获防篡改记录。我们通过一个多智能体投资组合再平衡系统演示ADP，其中自主智能体监控市场、做出交易决策并在隔离的客户账户间执行订单——每个客户的数据范围、交易批准阈值和防篡改审计跟踪都由智能体既不能看到也不能绕过的带外通道强制执行。

## Abstract
AI agents are increasingly expected to operate as digital employees: accessing enterprise data, making decisions, and taking actions autonomously. But agents are simultaneously less predictable than humans -- prone to hallucination, misinterpretation, and adversarial manipulation -- and more technically capable: with deep system knowledge and high-throughput interfaces cascading damage at machine speed. This combination makes it unsafe to rely on agents to faithfully interpret or propagate security-critical metadata such as access policies, data classifications, and behavioral constraints.   We present the Redpanda Agentic Data Plane (ADP), an architecture built around out-of-band metadata channels: infrastructure pathways that carry security context, policy signals, and audit trails deterministically, entirely outside the agent's read and write path and across heterogeneous infrastructure. These channels enforce governance at every stage of the agent lifecycle -- scoping data access on the way in, constraining actions during execution, and capturing tamper-proof transcripts on the way out.   We demonstrate ADP with a multi-agent portfolio rebalancing system in which autonomous agents monitor markets, make trade decisions, and execute orders across isolated client accounts -- with per-client data scoping, trade approval thresholds, and tamper-proof audit trails all enforced by out-of-band channels the agents can neither see nor bypass.

- **核心问题**：人工智能自主智能体（AI agents）被期望像数字员工一样访问企业数据、做出决策并自主行动，但它们在两个方面与传统软件和人类员工存在根本差异：

  - **技术上更强大**：拥有深度系统知识，通过高吞吐量、低延迟接口与生产系统交互，能以机器速度级联损害。
- **根本矛盾**：依赖智能体自身忠实地解释或传播安全关键元数据（如访问策略、数据分类、行为约束）是不安全的。即便在系统提示或工具参数中嵌入安全规定，智能体也可能忽略、误解或被操纵规避。
- **本文含义**：提出**带外（out-of-band）元数据通道**作为统一架构原语，将安全上下文、策略执行和审计捕获完全移出智能体的操作边界，由基础设施确定性地强制执行，从而构建安全可信的自主智能体系统。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **带外元数据通道（Out-of-Band Metadata Channels）**：基础设施层面的路径，携带安全上下文、策略信号和审计信息，**完全在智能体的读写路径之外**，且智能体无法读取或写入。
- **三个定义属性**：
  - **智能体不可访问**：通道内容仅由基础设施组件（网关、代理、消息代理）处理，智能体无法读取或修改。
  - **确定性**：通道行为由配置定义，而非推理（例如速率限制由网关计数器执行，而非智能体理解指令）。
  - **可互操作**：通道跨系统边界和智能体边界传播，无需所有组件处于同一平台或由智能体中继上下文。
- **应用阶段**：在智能体生命周期的每个阶段起作用：
  - **入站（Inbound）**：通过身份和授权上下文限定数据访问范围，数据源基于上下文执行行级或资源级过滤。
  - **执行（Execution）**：通过策略约束（如速率限制、价值阈值、审批要求）由基础设施强制执行，智能体只发出动作，平台决定是否执行。
  - **出站（Outbound）**：每个交互由基础设施记录为防篡改审计跟踪，智能体无法参与记录或篡改。

### 关键技术细节：Redpanda Agentic Data Plane (ADP) 架构
- **四层基础设施**：
  - **访问控制层**：AI 网关（负责 LLM 路由、令牌预算、故障转移）和 MCP 网关（强制执行工具级策略，如 PII 脱敏、资源过滤）。两者与企业身份提供者集成，传播每智能体/每用户授权上下文，智能体无法看到。
  - **数据连接层**：通过适配器连接异构数据源（MCP 服务器、REST API、数据库、对象存储、流平台），带外元数据跨系统边界传播。
  - **智能体计算层**：提供沙盒，具有网络隔离、资源限制和身份限定访问，网关管理的 MCP 工具投放到沙盒中作为可执行动作。
  - **问责层**：每个智能体-数据交互自动记录为结构化跟踪，由基础设施收集（非智能体自报告），保证防篡改，且跟踪访问本身受带外元数据通道控制。

### 威胁模型
- 智能体本身（包括 LLM、智能体运行时进程、外部工具）被视为**不可信**，可能混淆或主动被攻陷。
- 平台基础设施（网关、消息代理、身份提供者、跟踪存储）被视为**可信**，管理员配置策略正确。

## 6. 主要结论与发现

- **核心结论**：带外元数据通道是构建安全自主智能体的有效架构原语，能够在不牺牲智能体广泛数据访问价值的前提下，提供确定性治理保障。
- **具体发现**：
  - 通过MCP网关注入身份范围，智能体无法通过参数伪造访问其他客户数据。
  - 通过AI网关进行模型无关的治理（预算控制、速率限制、输入输出护栏），智能体无法绕过。
  - 自主阈值作为基础设施路由决策，智能体无法观察或影响，防止绕过审批。
  - 基础设施收集的分布式跟踪（W3C Trace Context）提供端到端防篡改审计。
- **重要性**：该架构适用于异构基础设施，无需将企业数据集中到单一围墙花园平台。

## 7. 优点

- **架构创新**：将服务网格模式（如Istio/Envoy）应用于智能体-数据交互，系统性地将安全移出智能体控制范围，统一了访问控制、动作约束和审计。
