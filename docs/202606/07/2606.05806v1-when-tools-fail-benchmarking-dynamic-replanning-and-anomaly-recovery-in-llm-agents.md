## 摘要
现有基准测试评估LLM中工具集成推理（TIR）时，都基于理想化的“快乐路径”，很大程度上忽略了现实世界中的工具故障。我们引入了ToolMaze，这是一个用于TIR智能体动态路径发现和错误恢复的基准测试。为了将系统性重规划与盲目试错区分开来，ToolMaze采用了二维设计：基于DAG的拓扑复杂度和工具扰动的$2 \times 2$分类（显式/隐式，瞬态/永久）。评估表明，扰动几乎在所有模型上都会降低性能，其中在隐式语义故障下性能下降最为显著。由于对损坏输出存在系统性过度信任，在这些场景中扰动恢复率（PRR）下降了约37％，而复杂拓扑结构则使智能体陷入无用的试错循环。关键的是，智能体的容错能力随模型规模提升的速度比基本任务执行速度慢$3.66\times$，这凸显了动态重规划是一个独立的瓶颈，无法通过模型扩展或提示工程来解决。数据和代码可在 https://github.com/Zhudongsheng75/ToolMaze 获取。

## Abstract
Existing benchmarks evaluate Tool-Integrated Reasoning (TIR) in LLMs on idealized ''happy paths'', largely overlooking real-world tool failures. We introduce ToolMaze, a benchmark for dynamic path discovery and error recovery in TIR agents. To separate systematic replanning from blind trial-and-error, ToolMaze adopts a two-dimensional design: DAG-based topological complexity and a $2 \times 2$ taxonomy of tool perturbations (explicit/implicit, transient/permanent). Evaluations show that perturbations degrade performance across nearly all models, with the sharpest drops under implicit semantic failures. Driven by systemic over-trust in corrupted outputs, Perturbation Recovery Rate (PRR) plummets by around 37\% in these scenarios, while complex topologies trap agents in futile trial-and-error loops. Crucially, agentic fault-tolerance improves with model scale $3.66\times$ slower than basic task execution, highlighting dynamic replanning as a distinct bottleneck unaddressed by model scaling or prompting. Data and code are available at https://github.com/Zhudongsheng75/ToolMaze.

*   **整体含义**：本文证明了动态重规划和异常恢复并非模型规模或简单提示工程的副产品，而是一种独立的、更难培养的核心能力，是构建真正鲁棒、可靠智能体的关键瓶颈。

## 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想：ToolMaze 框架**。为了将系统性重规划与盲目的试错区分开，ToolMaze 采用**二维评估矩阵（C × P）** 来定义每一个评估实例。
    1.  **拓扑复杂度（C 轴）**：基于有向无环图（DAG），定义了四个递增的复杂度层级。该轴决定了智能体可用的工具调用路径数量（即恢复路径的丰富度）。
        *   **C1 (线性)**: 单一路径，无替代方案。
        *   **C2 (1对多)**: 存在功能等价的单一工具替代方案。
        *   **C3 (多对多)**: 存在组合式的、涉及多个工具的替代路径。
        *   **C4 (复杂集成)**: 结合了多个 C2 和 C3 模式的复杂 DAG。
    2.  **扰动模式（P 轴）**：定义了扰动注入的类型，通过 2 × 2 分类实现。
        *   **显式 vs. 隐式**：前者发出机器可读的错误（如 HTTP 404），后者返回结构合法但语义错误的输出。
        *   **瞬态 vs. 永久**：前者可通过重试解决，后者则需要切换路径或优雅终止。
        *   由此产生四种模式：**P1 (显式-瞬态)**, **P2 (显式-永久)**, **P3 (隐式-瞬态)**, **P4 (隐式-永久)**。

*   **关键技术细节**：
    1.  **工具语料库**：手工构建270个模拟真实API的工具，标注了功能类别（数据源、处理器、动作）和应用领域。
    2.  **DAG生成管线**：采用“工具优先”策略。先由LLM根据目标C级构建DAG，然后枚举所有有效解的路径空间，最后将DAG“自然化”为自然语言任务。这保证了语义连贯性和解空间的数学完整性。
    3.  **运行时扰动引擎**：在智能体调用工具时，依据预设的扰动配置文件（指定哪个工具失败及返回什么响应），动态注入故障，确保所有模型接收一致的扰动。
    4.  **评估指标**：实现了三个互补的指标来量化智能体的行为，而非只看最终成功率。
        *   **任务成功率 (TSR)**：衡量整体任务完成能力。
        *   **扰动恢复率 (PRR)**：条件概率，在智能体确实遇到扰动的情况下，衡量其成功恢复的能力。严格区分了“主动恢复”和“被动逃避”。
        *   **恢复成本 (RC)**：量化重规划的效率，通过比较智能体实际工具调用次数和理论最小次数来惩罚不必要的试错。

## 6. 论文的主要结论与发现

1.  **扰动普遍降低性能**：几乎所有模型在遇到扰动（P1-P4）时，性能（TSR, PRR）都显著下降，尤其是在隐式故障（P3, P4）下。

3.  **复杂拓扑结构导致无效试错**：随着 DAG 拓扑复杂度（C3, C4）增加，智能体更易陷入无效的试错循环，RC（恢复成本）显著升高。

5.  **故障感知提示的有限效果**：虽然故障感知提示（w/ hint）普遍提升了性能，但这种提升是部分性的，无法从根本上解决模型在隐式故障和复杂路径下的能力欠缺。

## 7. 优点：方法或实验设计上的亮点

*   **创新基准设计**：将任务拓扑复杂度和扰动模式组合为二维评估矩阵（C × P），是对现有“快乐路径”基准的根本性革新，能够系统、可控地评估智能体的动态恢复能力，这是一个亮点。
*   **精细化的指标**：PRR 和 RC 的引入是关键贡献。它们能有效区分“碰巧走完流程”和“真正主动恢复”，并惩罚无效试错，比单一 TSR 提供了更强的分析洞察力。
*   **严谨的解空间枚举**：通过先构建 DAG 再自然化的管线，能穷举所有有效解路径，从而可以精确判断智能体的行为是否属于最优恢复路径，为计算 RC 提供了理论依据。
*   **可控的扰动注入**：在预指定节点而非随机位置注入确定性扰动，确保了实验的公平性和可复现性。
*   **详尽的案例分析**：提供了 16 个案例研究，直观展示了模型在成功与失败场景下的行为差异，极具说明力，有助于深入理解模型的失败模式。

## 8. 不足与局限

*   **评估环境的开放性有限**：ToolMaze 基于程序化生成的 DAG 拓扑，而非完全开放的网页环境。虽然这有助于提供精确的“黄金标准”解，但牺牲了部分真实世界复杂度（如完全动态的 Web 页面、对抗性攻击等）。
*   **扰动分类的扩展性**：当前的 2 × 2 分类（显/隐，瞬/永）是基础框架。在真实系统中，更复杂的场景如级联故障（一个错误触发连锁反应）或恶意对抗注入未被覆盖，这限制了其在复杂安全场景下的直接应用。
