## 摘要
企业入侵响应仍然依赖于静态剧本和分析师驱动的分类，导致从警报生成到遏制的延迟。我们提出了Agentra，一个可监督的多智能体入侵响应系统（IRS）框架，该框架将来自IDS、EDR和XDR平台的警报转换为基于MITRE ATT&CK、MITRE D3FEND和NIST CSF 2.0的结构化事件响应计划。Agentra将响应推理分解到角色限定的智能体中，通过有界的规划器-验证器审查循环验证提出的计划，通过审核安全网关筛选检索到的威胁情报，通过行动目录和风险评分限制行动，并将决策记录在仅追加的审计日志中。我们在源自ThreatHunter-Playbook、Splunk BOTSv3和DARPA OpTC的120个事件语料库上，将Agentra与静态的OASIS CACAO v2.0网络剧本基线进行了比较。最强的配置将考虑误报的IRS F1值从0.61提高到0.84，并在仅规划器配置引入了不安全的过度反应后，将预计的有害行动率恢复到静态基线水平的0.0%。这些结果表明，多智能体响应规划可以在保持分析师批准和可审计性的同时，提高基于本体论的IRS覆盖率。

## Abstract
Enterprise intrusion response still depends on static playbooks and analyst-driven triage, creating delay between alert generation and containment. We present Agentra, a supervisable multi-agent Intrusion Response System (IRS) framework that converts alerts from IDS, EDR, and XDR platforms into structured incident response plans grounded in MITRE ATT&CK, MITRE D3FEND, and NIST CSF 2.0. Agentra decomposes response reasoning across role-scoped agents, validates proposed plans through a bounded Planner--Validator review loop, screens retrieved threat intelligence through a Moderator security gateway, gates actions through an Action Catalog and risk score, and records decisions in an append-only audit log. We evaluate Agentra against a static OASIS CACAO v2.0 cyber-playbook baseline on a 120-event corpus drawn from ThreatHunter-Playbook, Splunk BOTSv3, and DARPA OpTC. The strongest configuration improves FP-aware IRS F1 from 0.61 to 0.84 and restores the projected harmful-action rate to the static baseline level of 0.0% after Planner-only configurations introduce unsafe overreaction. These results indicate that multi-agent response planning can improve ontology-grounded IRS coverage while preserving analyst approval and auditability.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
Agentra 提出一个**可监督的多智能体入侵响应框架**，将 IDS/EDR/XDR 警报转化为基于 MITRE ATT&CK、MITRE D3FEND 和 NIST CSF 2.0 的结构化事件响应计划（IRP）。采用**角色分离**的设计：规划器（Planner）负责提议计划但无执行权限，验证器（Validator）独立审查，调解器（Moderator）筛选检索内容，执行器（Executor）仅通过预定义目录和风险门控行动。所有决策记录在仅追加的审计日志中，保证可追溯。

### 关键技术细节
- **威胁模型**：假设检测平台、API 和网络未被攻破；攻击者仅限于智能体响应层，可能通过恶意检索内容（外部威胁情报、日志等）实施间接提示注入。Agentra 将检索内容和模型输出视为不可信，通过多层控制（调解检索、角色限定、Validator 审查、Action Catalog、风险评分、分析师批准、状态变化检查、审计日志）防止单一故障导致不安全响应。
- **架构组件**：
  - **Incident Context 构建**：将警报证据归一化为结构化记录，包含 IoC、根因假设、遥测快照、严重性、时间线、ATT&CK/D3FEND/NIST CSF 映射。
  - **Planner 与 Validator 循环**：Planner 基于上下文和库生成候选 IRP；Validator 池（n 个）独立评审，至少 k 票通过（本文实例化为 1 个 Validator 与 Planner 组成有界多轮审查循环）；失败则返回 Planner 修订，达到上限后升级至 SOC。
  - **Moderator 安全网关**：Planner 不直接检索，由 Moderator 代理从内部（Graph-RAG）和外部（OSINT/威胁情报）检索并消毒，筛查嵌入指令、提示注入、允许列表违规、幻觉缓解标识等。只有消毒后的上下文写入 Context Memory。
  - **Gated Execution**：分析师批准后，每个动作通过 Action Catalog（仅允许网络、身份、端点、ITSM 的预定义操作）、Sanity Checker（内部一致性）、Risk Score（资产关键性、爆炸半径、可逆性映射为 authorize 级别）三层门控；若状态发生变化，则计划失效并触发重分析。
  - **Audit Log**：仅追加的审计基板，记录提示、检索、Moderator 决策、Validator 投票、风险评分、分析师决定及执行动作，使用 Merkle 树哈希链保证完整性。
- **算法流程**（Algorithm 1 文字描述）：
  1. 输入警报证据 E，构建事件上下文 I。
  2. 提取密钥 K，通过 Graph-RAG 和 Web 检索获得内部/外部知识。
  3. Moderator 过滤并消毒检索内容，生成已消毒上下文 X，写入审计日志。
  4. Planner 基于 I、X 和剧本库 L 生成候选计划 P。
  5. 进入有界修订循环（最多 R 轮）：每个 Validator vi 返回投票和理由；若批准票数 ≥ k，则计划通过；否则根据反馈修订 P。
  6. 若循环未达到批准，则升级给 SOC。
  7. 分析师审查（模拟批准），写入审计日志。
  8. 检查状态是否变化，若变化则作废计划。
  9. 对计划中每个动作 a：检查是否在 Action Catalog；通过 Sanity Checker；计算风险分数并映射授权级别；检查是否被授权；执行并记录审计。
  10. 返回最终计划或终止状态。

## 6. 论文的主要结论与发现

1. **整体提升**：完整 Agentra 堆栈（A4）相比静态 CACAO 基线（A0），FP-aware IRS F1 从 **0.61 提升至 0.84**，召回率显著改善（从 0.55 至最高 1.00）。
2. **安全性恢复**：仅添加 Planner（A1）时，较强模型（M4、M5）的预计有害动作率高达 **14-16%**；添加 Moderator 和 CTI 后（A3），有害动作率降至 **≤0.9%**；添加 Graph-RAG 后（A4）完全恢复到 **0.0%**（与静态基准持平）。
3. **消融关键阶段**：A3（Moderator + 威胁情报）是响应质量和安全性的**定性转折点**，使 D3FEND 精确率大幅跃升（M1 从 0.23 至 0.79，M4 从 0.15 至 0.68）。
4. **不同模型权衡**：最强的 M4（Sonnet Planner + FoundSec Validator）达到 100% 召回率，但误报率高达 0.90，F1 为 0.81；M1（Qwen Planner + Mistral Validator）以 0.84 的 F1 最佳，兼顾召回和误报控制。
5. **Planner-Validator 循环效果因组合而异**：A2 阶段对 M1 降低误报，但对 M5 导致召回崩盘（0.14），因为严格 Validator 拒绝大部分提案并超时。
6. **替换 vs 增强 CACAO**：在完整堆栈下，替换模式（M1）F1 0.84 优于增强模式（M2）0.72，表明强局部规划器可通过后续阶段弥补缺失的剧本起点。

## 7. 优点

- **架构设计合理**：角色分离（Planner/Validator/Moderator/Executor）确保单一故障无法导致不安全响应，满足企业对可监督性和审计性的要求。
- **本体论驱动**：所有响应计划基于 MITRE ATT&CK、D3FEND 和 NIST CSF，可被业界理解和复用（序列化为 OASIS CACAO v2.0）。
- **多层安全控制**：调解器检测提示注入、允许列表、幻觉；行动目录限制范围；风险评分分级授权；审计日志 Merkle 树保证不可篡改。
- **消融实验设计好**：5 个逐步增加的阶段清晰展示每个组件的边际贡献，且覆盖多个指标（响应决策、响应质量、安全性）。
- **模型多样性**：测试 5 种不同模型组合，覆盖开源和闭源、局部和 API，增加结论的通用性。
- **考虑误报压力**：特设良性/误报事件测试假阳性鲁棒性，比单纯恶意事件评估更贴近现实。
