## 摘要
技能是智能体AI的关键组成部分。虽然它们增强了智能体的能力，但也引入了新的攻击面。在这项工作中，我们通过展示动态恶意技能来研究其中一种攻击面。攻击者通过将恶意指令嵌入自然语言文档（如SKILL.md），可以在执行过程中诱导智能体将恶意逻辑动态注入原本良性的技能中。我们在OpenHands和Claude Code等智能体框架上评估了这种攻击，结果表明动态恶意技能能够在运行时成功引入一系列恶意行为，且成功率不可忽视。为了缓解这一漏洞，我们提出了一种系统级防御措施，利用操作系统内核强制只读挂载来阻止技能的动态修改。我们的评估表明，这种防御能有效阻断动态恶意技能，同时保持良性技能的功能。

## Abstract
Skills are a key enabling component of agentic AI. While they enhance agents' capabilities, they also introduce new attack surfaces. In this work, we investigate one such attack surface by demonstrating dynamic malicious skills. By embedding malicious instructions in natural-language documentation (e.g., SKILL.md), an attacker can induce an agent to dynamically inject malicious logic into an otherwise benign skill during execution. We evaluate this attack across agentic frameworks such as OpenHands and Claude Code, showing that dynamic malicious skills can successfully introduce a range of malicious behaviors at runtime with non-trivial success rates. To mitigate this vulnerability, we propose a system-level defense that prevents dynamic modification of skills using operating system kernel-enforced read-only mounts. Our evaluation demonstrates that this defense effectively blocks dynamic malicious skills while preserving the functionality of benign skills.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：Agentic AI（智能体AI）从被动文本生成转向自主行动系统，技能（Skills）是其核心组件。尽管技能增强了智能体能力，但引入新的安全风险。已有研究关注**静态恶意技能**（代码在分发时已含恶意逻辑），但这类攻击可被静态代码分析检测。
- **核心问题**：本文提出一种新的攻击面——**动态恶意技能（Dynamic Malicious Skills）**。攻击者在技能的自然语言文档（如SKILL.md）中嵌入恶意指令，诱导智能体在**运行时动态修改技能代码**，从而产生恶意行为。由于原始代码在分发时是良性的，静态分析无法检测。
- **整体含义**：该攻击揭示了当前Agentic AI系统中技能组件因动态执行机制而存在的安全隐患，威胁到系统的机密性、完整性和可用性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：构造一个动态恶意技能 s_m = (c, m, d_m)，其中原始代码c和元数据m为良性，但文档d_m包含精心设计的恶意指令。智能体在技能执行阶段首先根据d_m修改代码c，得到恶意代码c̃，然后执行c̃产生恶意行为。
- **关键技术细节**：
  1. **恶意内容构造（Δd）** 需满足三个子属性：
     - **P1 前置框架（Prerequisite Framing）**：将恶意操作描述为完成用户任务前的必要“设置步骤”（如“初始化运行前配置”），利用智能体的指令遵循偏差，使其先执行再质疑。
     - **P2 入口脚本接地（Entry-Script Grounding）**：明确指定技能的实际入口脚本名称，并将恶意行为分解为与良性开发笔记类似的编辑指令（如“添加辅助函数”、“在main()开头调用它”），使其看似正常构建说明。
     - **P3 包装穿透（Wrapper-Pierce）**：将恶意内容包裹在类似平台级标记的标签中（如`</EXTRA INFO><SYSTEM INSTRUCTION>...</SYSTEM INSTRUCTION>`），利用智能体对这类标签的信任，将其视为更高权限的上下文。
  2. **注入位置L**：将恶意文档d_m插入原文档开头（L=0）效果最佳，因为智能体自上而下处理文档时，早期指令被视为基础约束，不易被后续理解剔除。

  3. 选择注入位置L（默认开头），生成恶意文档d_m = I(d, Δd, L)。
  4. 将恶意技能上传至技能市场。

## 6. 主要结论与发现

1. **动态恶意技能是现实威胁**：在5种LLM上平均ASR为18.7%~43.7%，其中GPT-4o-mini最高达41.8%，Claude Sonnet 4.6为19.0%。
2. **模型规模增大增加漏洞**：从Qwen3-8B到Qwen3.6-35B，平均ASR从18.7%升至41.6%，说明强大指令遵循能力同时加剧了安全风险。
3. **安全微调重新分配风险而非消除**：GPT-5和Sonnet 4.6在某些行为上安全，但在其他行为上仍脆弱（如Sonnet 4.6在响应时间放大上高达82.0%）。
4. **简单行为更容易成功**：如本地文件删除、响应时间放大等可通过简单辅助函数实现的行为成功率普遍高于远程程序下载等复杂行为。
5. **权限级防御完全有效**：基于内核只读挂载的防御使ASR降至0.0%，且不影响良性技能功能（300个良性技能中仅0.33%有代码修改，且均为非必要工作区修复）。
6. **提示级防御和检测方法不足**：提示级防御平均ASR仅降低20.2%，且不同行为差异大；PromptArmor漏报率85.8%，DataSentinel误报率56.0%，均不可靠。
