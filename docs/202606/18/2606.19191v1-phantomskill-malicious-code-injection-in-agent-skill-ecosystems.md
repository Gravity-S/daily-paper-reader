## 摘要
智能体技能使得基于LLM的编码智能体能够从第三方包中获取特定领域的能力，但同时也引入了一种新的供应链攻击面。我们提出了PhantomSkill，一个攻击框架，它将恶意行为隐藏在技能的辅助资源中，而非其文本描述中。其核心技术VulMask将明显的恶意脚本重写为漏洞形态的实现，其恶意行为仅在攻击者控制的触发条件下激活。这种设计将可见信号从明确的恶意意图转变为看似普通的不安全代码。在多个代表性宿主技能、攻击目标、编码智能体、生成模型和自动审查者中，与明显的恶意脚本相比，VulMask在保持良性功能的同时，减少了警告和恶意软件级别的检测。我们的结果表明，技能生态系统需要资源级别的审查、执行时的隔离以及将智能体技能中的可利用漏洞视为潜在恶意载荷的安全策略。

## Abstract
Agent skills allow LLM-based coding agents to acquire domain-specific capabilities from third-party packages, but they also introduce a new supply-chain attack surface. We present PhantomSkill, an attack framework that hides malicious behavior in a skill's auxiliary resources rather than in its textual description. Its core technique, VulMask, rewrites overt malicious scripts into vulnerability-shaped implementations whose malicious behavior is activated only under attacker-controlled trigger conditions. This design shifts the visible signal from explicit malicious intent to ordinary-looking insecure code. Across representative host skills, attack goals, coding agents, generation models, and automated reviewers, VulMask preserves benign utility while reducing warning and malware-level detection compared with overt malicious scripts. Our results show that skill ecosystems require resource-level vetting, execution-time containment, and security policies that treat exploitable vulnerabilities in agent skills as potential malicious payloads.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：LLM（大型语言模型）编码智能体（如 Claude Code、Cursor、Gemini CLI）通过第三方技能包（Agent Skills）扩展领域能力。每个技能包含 SKILL.md 文件（描述）和辅助资源（如脚本）。用户从公开仓库或市场导入技能。
- **问题**：现有安全研究主要关注提示注入（Prompt Injection）或将恶意指令写在技能文本中，但忽略了辅助资源（脚本文件）的供应链攻击面。当有审查（智能体、用户或平台扫描器）时，显式恶意脚本易被检测。
- **核心含义**：提出了一种新的攻击范式：将恶意行为隐藏在辅助资源中，并伪装成普通的不安全代码（漏洞形状），从而逃避检测，同时保留宿主技能的良性功能。这揭示了技能生态系统需要更全面的安全审查。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将恶意载荷重写为“漏洞形状”的实现，使其看起来像普通的可利用脆弱性（如命令注入、信息泄露），而非显式恶意代码。恶意行为仅在攻击者控制的触发条件下激活。
- **关键技术 VulMask**：
  1. 定义一组相关 CWE 类别的脆弱性集合 \( W = \{w_1, w_2, \dots, w_m\} \)。
  2. 使用 LLM 驱动的生成器 \( g \) 将显式恶意脚本 \( p^\star \) 重写为掩码载荷 \( p_m = g(p^\star, W) \)，保留预期副作用但通过可触发的脆弱性暴露。
  3. 将掩码载荷嵌入宿主资源组件 \( r_m = Embed(r_h, p_m) \)，保持原始脚本名称、路径和良性输入输出行为（当触发器不存在时）。
  4. 最终恶意技能 \( s_{\text{VulMask}} = (n_h, d_h, t_h, r_m) \)，不修改技能名称、描述或主 SKILL.md 流程。
  5. 在隔离测试环境中验证：宿主技能保持功能，且触发条件下恶意副作用生效。

- **攻击目标分类**：凭证窃取、命令执行、智能体操纵、破坏（具体见表9，CWE 映射）。
- **威胁模型**：攻击者仅控制技能包内容，不能控制受害者 LLM、执行器或用户请求。防御者可以检查所有技能组件（但关键是识别恶意意图）。

## 6. 主要结论与发现

1. **VulMask 攻击有效**：在 GPT-5.5 上达到最高 ASR（58.8%），同时保持最低警告率（11.4%）和最高效用（96.6%），优于所有基线。
2. **隐蔽性高**：相对显式脚本，VulMask 显著降低编码智能体和自动审查者的恶意检测率。例如，对 GPT-5.5 检测率从 25.4% 降至 11.4%（表3）；Cisco 静态分析将掩码载荷重新分类为脆弱性而非恶意软件（表4）。
3. **跨攻击类别泛化**：四种攻击目标下均有效，破坏类效果最佳（检测率从 41.7% 降至 4.2%，ASR 79.2%）。
4. **生成器迁移性**：Opus-4.7、GLM-4.7-Flash、Qwen3-Coder 均能生成有效掩码载荷，ASR 在 41.0%~58.3% 之间。
5. **消融实验验证**：CWE 引导的重写关键——全 VulMask 优于无 CWE 变体（ASR 60.0% vs 54.2%，检测率 49.6% vs 56.5%），更优于朴素注入（表7）。
6. **防御启示**：不能仅依赖 SKILL.md 扫描，需要资源级审查、执行时隔离，并将可利用漏洞视为潜在恶意载荷。
