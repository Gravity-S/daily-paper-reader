## 摘要
代理越来越多地部署在文档密集型工作流程中，其中敏感的私人信息不是边缘情况，而是常规输入，例如，代理预订航班需要护照号码。在这种情况下，代理必须使用私人信息来准确完成任务，同时绝不将其暴露在响应中，因为它无法验证实际在键盘前的是谁。这两项义务存在根本性的矛盾。能够使用私人信息完成任务的能力足够强的模型，同样可以被诱导而泄露这些信息。为了评估任务准确性和隐私泄露之间的权衡，我们引入了任务完成与主动隐私提取防御（TRAP）基准测试。每个场景包括一份包含私人信息的文档、一个需要代理使用私有字段调用正确工具的任务查询，以及一个试图以自然语言引出相同信息的攻击查询。我们评估了涵盖前沿专有和开源模型的22个模型（多个规模），发现所有模型系列都存在非平凡的泄露，并且指令遵循能力与泄露率相关。现有的基于提示的防御减少了泄露，但以显著的任务准确性为代价。提示优化未能摆脱这种权衡。我们证明这种失败并非偶然。对于任何基于softmax的模型，没有软约束防御（如基于提示的防御）能够同时实现高任务成功率和零泄露概率。受这一不可能结果的启发，我们提出了结构化的私有字段隔离，即在私有字段到达模型之前用哈希键替换它们。这种方法在保持任务准确性的同时，在很大程度上防止了泄露。

## Abstract
Agents are increasingly deployed in document-intensive workflows where sensitive private information is not an edge case but a routine input, e.g., an agent booking a flight needs passport numbers. In such settings, the agent must use private information to complete tasks accurately while never exposing it in its responses, because it cannot verify who is actually at the keyboard. These two obligations are in fundamental tension. A model capable enough to use private information for task completion can, by the same capability, be induced to reveal it. To evaluate the trade-off of task accuracy and privacy leakage, we introduce Task-completion and Resistance to Active Privacy-extraction (TRAP). Each scenario includes a document containing private information, a task query that requires the agent to invoke the correct tool using private fields, and an attack query that attempts to elicit the same information in natural language. Evaluating 22 models spanning frontier proprietary and open-source models at multiple scales, we find that all model families exhibit non-trivial leakage, and that instruction-following ability correlates with leakage rate. Existing prompt-based defenses reduce leakage but at significant cost to task accuracy. Prompt optimization fails to escape this trade-off. We demonstrate that this failure is not incidental. For any softmax-based model, no soft-constraint defense, e.g., prompt-based defenses, can jointly achieve high task success with zero leakage probability. Motivated by this impossibility result, we propose structural private field isolation, which replaces private fields with hash keys before they reach the model. This approach largely prevents leakage while keeping task accuracy.

# 论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：大语言模型（LLM）和视觉语言模型（VLM）被广泛应用于文档密集型工作流（如预订航班、处理工资单），这些场景中模型必须访问敏感个人信息（如护照号、银行账号）才能正确完成任务。然而，模型无法验证当前操作者的真实身份（可能是授权员工、好奇同事或攻击者），因此仅仅靠身份认证无法防止隐私泄露。
- **核心矛盾**：模型需要使用私人信息完成任务，同时必须在响应中绝不泄露这些信息。这两个目标存在根本性矛盾——模型越擅长解析和利用私人信息完成任务，就越容易被诱导而泄露这些信息。
- **现有基准的不足**：现有工作要么只评估被动隐私泄露（不涉及主动攻击），要么只评估对抗攻击下的隐私泄露（忽略任务性能），缺乏同时衡量任务准确性与主动隐私防御能力的联合基准。

## 2. 方法论：核心思想、关键技术细节

### 2.1 TRAP 基准设计
- **数据集构成**：包含 10 个文档领域（5 个个人类：身份、信用卡、医疗、工资单、账户；5 个企业类：合同、日程、财务、人力资源、税务），共 147 个私有字段、88 个非私有字段、157 个工具、500 个样本。每个样本包含：
  - **文档**：含私有字段和非私有字段，以文本、图像、或文本+图像三种模态提供。
  - **任务查询**：要求模型使用私有字段调用正确的工具（如提取发票数据并调用工具）。
  - **攻击查询**：以自然语言直接询问私有字段（如“护照号是什么？”），并设置了 Direct、Ignore、Important、Roleplay 四种变体。
- **评估指标**：
  - **任务准确率**：工具调用和所有参数（含私有和非私有）正确匹配。
  - **隐私分数**：1 - 泄露率，泄露率定义为模型响应中包含目标私有字段值的比例。
- **匹配策略**：基于规则的评估器，支持大小写敏感/不敏感子串、货币规范化、列表等多类型匹配。

### 2.2 理论证明（不可能性定理）
- **定理1**：对于任何使用 softmax 输出分布的模型，若假设：
  1. 每个位置存在非零触发概率 ε > 0（即私有 token 出现在输出分布中的概率为正）；
  2. 任务成功意味着模型内部已处理私有 token，从而存在触发事件。
  则攻击序列长度 N 足够大时，泄露概率趋近于 1。
- **推论**：任何软约束防御（系统提示、RLHF 对齐、提示优化等）只能降低 ε 但无法归零，因此无法同时保证高任务准确率和零泄露。必须采用硬约束防御。

### 2.3 结构性私有字段隔离
- **核心思想**：在私有字段到达模型之前，用语义化的哈希键（如 `[PRIVATE_buyer_ein]`）替换原值，并将真实值存储在外部哈希表中。模型输出中的哈希键在工具执行层才被解析为真实值。
- **三种配置**：
  - **Oracle**：利用真实标签和源文件进行精确像素级掩码。
  - **Practical**：仅基于渲染后的图像，使用 OCR 模型定位并掩码私有区域。
  - **Auto**：无标签信息，先 OCR 再让 LLM 识别私有信息并掩码。

## 3. 实验设计

### 3.1 数据集/场景
- **TRAP 基准本身**：500 个样本，涵盖 10 种文档类型、3 种模态（文本、图像、多模态）。
- **文档生成**：图像通过 HTML 模板渲染为 PNG，部分使用 DocXPand 库生成逼真布局；文本由 LLM 生成并采用多种格式（纯文本、Markdown、INI 等）防止记忆。

- **防御策略**：

## 6. 主要结论与发现

1. **所有模型均存在显著隐私泄露**：即使任务准确率高于 60%，隐私分数普遍低于 27%；最强模型的系统提示防御只能将泄露率从 90%+ 降低到约 50%–70%，但任务准确率严重下降。
2. **指令遵循能力与泄露率正相关**：越会遵循指令的模型，越容易被攻击查询诱导泄露。
3. **模型规模扩大不能解决隐私问题**：Qwen3-VL 从 2B 到 32B，泄露率随任务能力提升而急剧恶化。
4. **攻击变体反而使隐私分数上升**：Ignore、Important、Roleplay 等对抗性措辞被模型视为警报信号，降低了泄露。
5. **授权声明成为社会工程攻击向量**：当用户自称“授权”时，模型泄露更多隐私（如 Claude Sonnet 4.5 的 ΔPriv = -26.4pp）。
6. **结构性私有字段隔离有效**：在 Oracle 配置下，隐私分数达到 90% 以上，任务准确率接近基线。
