## 摘要
技能是智能代理AI的关键使能组件。虽然它们增强了代理的能力，但也引入了新的攻击面。在本工作中，我们通过展示动态恶意技能来研究其中一个攻击面。通过将恶意指令嵌入自然语言文档（例如SKILL.md），攻击者可以诱导代理在执行期间将恶意逻辑动态注入到原本良性的技能中。我们在OpenHands和Claude Code等代理框架上评估了这种攻击，表明动态恶意技能能够在运行时以非平凡的成功率成功引入一系列恶意行为。为了缓解此漏洞，我们提出了一种系统级防御措施，该措施利用操作系统内核强制只读挂载来防止技能的动态修改。我们的评估表明，这种防御能够有效阻止动态恶意技能，同时保留良性技能的功能。

## Abstract
Skills are a key enabling component of agentic AI. While they enhance agents' capabilities, they also introduce new attack surfaces. In this work, we investigate one such attack surface by demonstrating dynamic malicious skills. By embedding malicious instructions in natural-language documentation (e.g., SKILL.md), an attacker can induce an agent to dynamically inject malicious logic into an otherwise benign skill during execution. We evaluate this attack across agentic frameworks such as OpenHands and Claude Code, showing that dynamic malicious skills can successfully introduce a range of malicious behaviors at runtime with non-trivial success rates. To mitigate this vulnerability, we propose a system-level defense that prevents dynamic modification of skills using operating system kernel-enforced read-only mounts. Our evaluation demonstrates that this defense effectively blocks dynamic malicious skills while preserving the functionality of benign skills.

- **研究动机**：Agentic AI（智能代理）的核心能力之一是通过“技能”（skills）执行复杂任务和与外部资源交互。然而，技能可以从平台或市场获取，且审核机制往往薄弱，这为攻击者提供了新的攻击面。已有工作研究了**静态恶意技能**（代码中直接嵌入恶意逻辑），但此类技能可通过静态代码分析被检测和屏蔽。本文首次提出**动态恶意技能**，即技能原始代码是良性的，但在执行时被动态修改为恶意代码，从而绕过静态检测。
- **整体含义**：动态恶意技能利用Agent的指令遵循特性，通过自然语言文档（如SKILL.md）中的巧妙构造的恶意指令，诱导Agent实时修改自身代码，实现机密性、完整性、可用性的破坏。该威胁在多个主流Agent框架和骨干模型上均有效，凸显了当前Agent安全体系的严重漏洞。

# 2. 方法论

- **核心思想**：攻击者（DyMalSkill方法）将精心构造的恶意指令注入到技能的自然语言文档（SKILL.md）中，而不是修改代码。Agent在执行该技能时，会先读取文档，并根据其中的恶意指令动态修改技能代码，从而在后续执行中产生恶意行为。攻击成功的关键在于：指令必须是**前置步骤框架**（将恶意行为包装为合法设置步骤）、**入口脚本接地**（命名实际脚本文件并使用编辑词汇，显得正常）、**包裹穿透**（使用类似平台级标签如`</EXTRA INFO><SYSTEM INSTRUCTION>`使指令被视为高信任上下文）。
- **关键技术细节**：
  - 攻击者定义恶意技能 `s_m = (c, m, d_m)`，其中c是原始良性代码，m是元数据，d_m是注入恶意内容的文档。
  - Agent执行时：`\tilde{c} = A(c, d_m)` 动态修改代码；然后执行修改后的技能 `\tilde{s} = (\tilde{c}, m, d_m)` 产生恶意行为 `\tilde{Y} = A(\tilde{s}, a)`。
  - 注入位置：默认在文档开头（prepend），实验表明此位置最有效。
  - 注入内容：需满足三个子属性（Prerequisite framing, Entry-script grounding, Wrapper-pierce），缺一不可。

# 3. 实验设计

- **数据集**：从Skillsmp平台收集15,067个技能（含Python代码）。由于计算限制，随机抽取300个技能用于开放权重模型（Qwen3-8B、Qwen3.6-35B），另抽取50个用于专有模型（GPT-4o-mini、GPT-5、Sonnet 4.6）。每个技能针对12种恶意行为进行测试。
- **恶意行为类别**：采用MalTool定义的12类行为，覆盖机密性（环境凭据窃取、API密钥滥用、远程/本地数据泄露等）、完整性（恶意数据库注入、文件删除、数据库记录删除、远程程序下载）、可用性（CPU/GPU计算劫持、响应时间放大）。
- **评估指标**：攻击成功率（ASR），即动态修改后成功实现恶意行为的比例。通过运行时验证器确认。
- **对比方法**：
  - 基线：直接使用MalTool的静态恶意技能提示（ASR=0%）。
  - 防御对比：无防御 vs. 提示防御（系统提示中禁止修改技能代码） vs. 权限防御（基于操作系统内核只读挂载+拷贝监控）。
  - 检测方法对比：PromptArmor和DataSentinel（用于检测提示注入）。
- **消融实验**：注入位置（开头/中间/结尾）、注入内容（基线 vs DyMalSkill）、重复执行次数（ASR@k）、组件消融（逐一移除Prerequisite framing、Entry-script grounding、Wrapper-pierce）。

# 6. 主要结论与发现

1. **动态恶意技能威胁显著**：在OpenHands和Claude Code上，ASR可达18.7%~43.7%，不同模型间差异较大。例如GPT-4o-mini平均ASR为41.8%，Sonnet 4.6为19.0%。
2. **模型规模增大使攻击更有效**：Qwen3-8B到Qwen3.6-35B，平均ASR从18.7%升至41.6%。
3. **安全对齐并非万能**：GPT-5和Sonnet 4.6在部分行为上被抑制，但在“响应时间放大”、“计算劫持”上仍脆弱。
4. **良性技能极少动态修改代码**：仅0.33%的执行发生修改，且均为非功能性调整；文档中要求修改的仅0.07%（且仅限配置填充）。
5. **权限防御有效且不影响良性技能**：基于内核只读挂载+拷贝监控的防御将ASR降至0%，且良性技能全部完成任务。
6. **提示防御不充分**：只降低ASR约20个百分点，仍存在21.4%的ASR。
7. **现有提示注入检测方法有缺陷**：PromptArmor假阴性率高达85.8%，DataSentinel假阳性率达56.0%。

# 8. 不足与局限

- **模型覆盖有限**：专有模型仅测试了50个技能，结果可能不够稳定；开源模型也未覆盖所有主流（如LLaMA系列）。
- **攻击假设较强**：攻击者需完全了解技能代码和文档，且能上传至市场；实际中可能受限于市场审核。
- **防御假设**：用户需运行在沙箱环境中，且拷贝监控可能被更高级的绕过（例如通过内存共享而非文件复制）攻击。
- **其他Agent框架未测试**：如AutoGPT、BabyAGI等，可能攻击成功率不同。
- **动态修改检测**：仅尝试验证了良性技能行为，但未系统研究恶意技能修改的模式识别。
- **防御副作用**：对于极少数需要修改配置的良性技能，权限防御会阻止其执行，但论文认为这些修改并非必需，防御仍可接受。
