## 摘要
开源LLM代理生态系统正在快速增长，然而社区贡献的技能——扩展代理能力的模块化工具定义——的安全性在很大程度上仍未受到审查。我们填补的空白：现有扫描器在代码层运行，结构上对指令层和多代理风险——劫持代理的自然语言指令、通过编码旁通道窃取数据或跨流水线链式危害——视而不见，因此需要的是语义化的多维审查系统，而非另一个签名匹配器。我们提出了SKILLVETBENCH，这是一个在Hugging Face上实时公开的排行榜，使用LLM作为裁判来审查代理技能。创新点：SARS（技能代理风险评分），一种面向指令遵循系统的五维代理风险度量，具有原则性的加权公式。集成内容：完整的CVSS v4.0向量分解和ClawHub双视图，将我们LLM生成的评论置于官方市场裁决旁边。实证结果：基于我们的配套基准论文[1]，LLM即裁判阶段在78个已确认的恶意技能上实现了零假阴性，在22个良性控制样本上实现了零假阳性，而最佳静态基线（SKILLSIEVE）仍遗漏了15%；对于指令层类别（如提示注入和记忆投毒），传统工具遗漏了89%至100%的威胁（例如，CODEBERT未能检测到九个记忆投毒技能中的任何一个）。四个LLM评估器的检测率从35%到95%不等，这促使在生产部署中采用集成评分。

## Abstract
Open-source LLM agent ecosystems are growing rapidly, yet the security of community-contributed skills - modular tool definitions that extend agent capabilities - remains largely unvetted. The gap we fill: existing scanners operate at the code layer and are structurally blind to instruction-layer and multi-agent risk - natural-language directives that hijack an agent, exfiltrate data through encoded side channels, or chain harm across pipelines - so what is needed is a semantic, multi-dimensional vetting system rather than another signature matcher. We present SKILLVETBENCH, a live public leaderboard on Hugging Face that uses an LLM-as-Judge to vet agent skills. What is new: SARS (Skill Agentic Risk Score), a five-dimensional agentic-risk metric with a principled weighted formula for instruction-following systems. What is integrated: full CVSS v4.0 vector decomposition and a ClawHub dual-view that places our LLM-generated review beside the official marketplace verdict. What is demonstrated: drawing on our companion benchmark paper [ 1], the LLM-as-Judge stage achieves zero false negatives across 78 confirmed-malicious skills and zero false positives across 22 benign controls, while the best static baseline (SKILLSIEVE) still misses 15%; for instruction-layer categories such as Prompt Injection and Memory Poisoning, conventional tools miss between 89% and 100% of threats (e.g., CODEBERT detects none of nine memory-poisoning skills). Detection rates vary from 35% to 95% across four LLM evaluators, motivating ensemble scoring in production deployments.

- **现有缺陷**：传统安全工具（如VirusTotal、ClawScan、CodeBERT）仅检测代码层漏洞，对指令层攻击（如提示注入、记忆投毒）和多代理协同风险（如链条放大）完全盲视（检测率0%）。人工审查难以规模化。
- **研究目标**：填补这一空白，提出一种**语义化、自动化、跨维度**的技能安全评估系统，可检测传统工具无法捕捉的指令层风险，并提供标准化的公开比较基准。

## 2. 方法论：核心思想、关键技术细节、公式流程
- **核心思想**：采用**LLM-as-Judge**框架，对完整技能工件（SKILL.md、脚本、配置、工具接口）进行静态语义分析，分四阶段处理：输入 → LLM-as-Judge → 多维评分 → 输出/排行榜。
- **关键技术细节**：
  - **LLM-as-Judge**：默认使用Qwen2.5-14B-Instruct，固定提示模板（角色/任务/评分规则/结构化输出），低温度采样（T=0.2, top-p=0.9）。预处理：解析为类型化段（指令、代码、配置、工具），截断策略优先保留指令。
  - **SARS（Skill Agentic Risk Score）**：五维原创指标，每个维度0–3分，加权公式：
    ```
    SARS = (2×IFR + 1.5×DG + 1.5×AI + 2×BR + 2×CA) / 2.7
    ```
    其中：IFR（指令保真度风险）、DG（数据引力）、AI（动作不可逆性）、BR（爆炸半径）、CA（链式放大）。权重设计使IFR/BR/CA权重更高（2×），因为它们反映代码层扫描器看不到的指令驱动风险。输出分为Critical(≥9)、High(7–8.9)、Medium(4–6.9)、Low(<4)。
  - **CVSS v4.0**：完整向量分解（如AV:N/AC:L/...），由LLM分配指标值，数值由标准FIRST函数计算（非模型）。与SARS互补：CVSS衡量代码层可被利用性，SARS衡量代理层危险性。
  - **ClawHub双视图**：将LLM生成的审查报告（通过5维度：Purpose & Capability、Instruction Scope、Install Mechanism、Credentials、Persistence & Privilege）与官方市场判决并排显示，暴露检测差距。
  - **三级判决**：最终输出Benign/Suspicious/Malicious，附带置信度（0-1）。脆弱性被标记为Medium及以上。
- **算法流程**：工件摄入→解析为类型化段→LLM推理（意图提取、分类、打分）→生成SARS、CVSS、SKV记录→汇总为报告→写入排行榜。

## 8. 不足与局限
- **LLM依赖偏差**：判决受评估模型本身偏见、非确定性和未来模型漂移影响。检测率在不同LLM间差异巨大（35%–95%），单模型结果需谨慎解读。

- **置信度未校准**：置信度反映LLM自述确定性，未通过可靠性图验证。阈值未经held-out集调优，校准为未来工作。
