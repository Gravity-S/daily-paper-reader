## 摘要
间接提示注入攻击通过在智能体执行任务时检索的第三方数据中嵌入恶意指令来劫持基于LLM的智能体。现有防御在静态基准上报告接近零的攻击成功率，但最近的适应性评估显示，一旦允许攻击者针对部署的防御进行优化，这些结果就会崩溃。在这项工作中，我们将这种崩溃归因于两种失败模式。首先，现有防御方法局限于识别特定的攻击模式，而不是评估每个嵌入指令的意图是否与用户任务相关。其次，基于训练的防御方法虽然提供了最强的安全-效用权衡，但其对抗样本来自少数手工制作的模板，导致防御者无法泛化到该狭窄策略分布之外。为了解决这些问题，我们提出了RETA，一种基于训练的方法，将防御决策基于用户任务而非攻击者控制的数据。在每个工具输出步骤中，防御者进行链式思维推理，验证其行为与用户任务一致。利用红队测试，模拟攻击者合成对抗训练数据，并接收字典学习多样性奖励，实现注入重构策略的广泛覆盖。这些共同使得防御者可以通过多目标强化学习进行优化，实现更好的安全-效用权衡。在六种黑盒自适应攻击下，RETA将每次攻击的ASR保持在10%以下，在两个目标模型上的平均ASR分别为2.92%和3.75%，同时在攻击和干净输入下保持大部分效用。

## Abstract
Indirect prompt injection attacks hijack LLM-based agents by embedding malicious instructions in third-party data that the agent retrieves during task execution. Existing defenses report near-zero attack success rate on static benchmarks, yet recent adaptive evaluations show that these results collapse once the attacker is allowed to optimize against the deployed defense. In this work, we trace this collapse to two failure modes. First, existing defense methods are confined to recognizing specific attack patterns, rather than assessing whether the intent of every embedded instruction is relevant to the user task. Second, training-based defenses, which otherwise offer the strongest safety-utility trade-off, assemble their adversarial examples from a handful of hand-crafted templates, and the resulting defender fails to generalize outside that narrow strategy distribution. To address these gaps, we propose RETA, a training-based method that grounds defense decisions on the user tasks rather than attacker-controlled data. At each tool-output step, the defender undertakes chain-of-thought reasoning verifying that its actions are consistent with the user task. Leveraging red-teaming, a simulated attacker synthesizes adversarial training data and receives a dictionary-learning diversity reward, achieving broad coverage of injection-reformulation strategies. Together, these allow the defender to be optimized via multi-objective reinforcement learning and achieve better safety-utility trade-off. Across six black-box adaptive attacks, RETA keeps every per-attack ASR below 10%, with average ASR of 2.92% and 3.75% on the two target models, while preserving most utility under attack and on clean inputs.

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：间接提示注入攻击（Indirect Prompt Injection）通过在被LLM智能体检索的第三方数据中嵌入恶意指令来劫持智能体。现有防御方法在静态基准上表现良好，攻击成功率（ASR）接近零。
- **问题**：一旦攻击者可以针对已部署的防御进行自适应优化（adaptive attacker），这些防御就会崩溃。作者识别出两个根本失败模式：
  1. 现有防御仅识别特定攻击模式（表面特征），而不是评估每条嵌入指令的意图是否与用户任务相关。
  2. 基于训练的防御（如SecAlign）虽然安全-效用权衡最佳，但其对抗训练模板过于狭窄，导致泛化能力差。
- **研究目标**：提出一种能同时满足**鲁棒性（R1）**、**攻击下效用（R2）** 和**干净输入效用（R3）** 的防御方法，抵御未知的自适应提示注入攻击。

## 2. 方法论：核心思想、关键技术细节
- **方法名称**：RETA（Reasoning-enabled Task Alignment，推理驱动的任务对齐）
- **核心思想**：将防御决策基于稳定不变的**用户任务**（trusted user task），而非攻击者控制的数据内容。通过显式的链式推理验证每个工具调用是否与用户任务对齐；并通过红队多样性对抗训练覆盖广泛的注入策略。
- **关键技术细节**：
  - **两阶段训练**：
    - **Stage I（红队对抗示例生成）**：
      - 使用固定基线模型，由攻击者策略 π_A 生成注入 d_adv，插入数据通道后执行完整轨迹。
      - 如果成功完成注入任务（Φ_t_adv=1），该注入被加入到对抗示例集 D_RT。
      - 引入**字典学习多样性奖励**：维护一个滚动策略字典，用稀疏编码表示已覆盖的策略。当新注入的策略记录（strategy record）不能被现有字典很好地解释时，获得高多样性奖励；反之则低。这迫使红队探索不同策略。
      - 采用GDPO（Group reward-Decoupled Normalization Policy Optimization）优化攻击者，奖励包括攻击成功、多样性和CoT格式。
    - **Stage II（防御者对抗强化学习）**：
      - 防御者初始化：对边界轨迹（注入成功且用户任务也成功）进行格式微调，学习输出包含任务对齐推理的响应。
      - 防御者RL：使用Stage I收集的对抗样本，防御者完成轨迹后被给予三个奖励：
        - r_sec：注入任务未完成（Φ_t_adv=0）
        - r_utl：用户任务完成（Φ_t_u=1）
        - r_CoT：每个决策步骤的任务对齐推理质量（平均法官得分）
      - 同样采用GDPO优化，安全与效用奖励权重高于推理奖励。
  - **推理机制**：在每个工具输出后，防御者生成包含C1（识别试图重定向的内容）和C2（验证计划工具调用仅源自用户任务）的推理文本，然后才执行工具调用。
- **公式/算法流程**：论文提供了算法1详细描述两阶段训练流程，包括字典学习、稀疏编码、GDPO更新等步骤。具体为：
  - 字典学习目标：min 1/2||X - VA||²_F + α||A||_1,1，求解稀疏编码矩阵A。
  - 多样性奖励：r_div = 1 - ES，其中ES为归一化可解释性分数（1-5 Likert得分）。
  - GDPO更新：对每组轨迹分别计算各奖励的组内归一化优势，然后加权合并，再进行批归一化后进行策略梯度。
