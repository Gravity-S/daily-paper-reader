## 摘要
随着LLM代理越来越多地自主选择工具，它们在具有不同权限的工具之间的选择变得与安全相关。然而，先前的工具选择研究侧重于安全无关的元数据偏好，使得特权敏感的选择未被充分探索。为填补这一空白，我们研究了过度特权工具选择，即代理在存在足够低权限替代方案时，仍选择或升级到更高权限工具。我们引入了ToolPrivBench来评估代理在存在足够低权限替代方案时是否选择更高权限工具，同时测量初始选择以及暂时性工具故障后的升级。在八个领域和五种重复风险模式中，我们发现过度特权工具选择在主流的LLM代理中很常见，并且暂时性故障进一步加剧了这一现象。我们进一步发现，一般的安全对齐无法可靠地迁移到最小特权工具选择，而提示级别的控制在暂时性故障下只能提供有限的缓解。因此，我们引入了一种特权感知的后训练防御，教导代理偏好足够低权限的工具，并仅在必要时升级。我们的缓解实验表明，这种防御在保持通用能力的同时，显著减少了不必要的高特权工具使用。

## Abstract
As LLM agents increasingly select tools autonomously, their choices among tools with different privileges become safety-relevant. However, prior tool-selection studies focus on safety-agnostic metadata preferences, leaving privilege-sensitive choices underexplored. To address this gap, we study over-privileged tool selection, in which an agent selects or escalates to a higher-privilege tool despite a sufficient lower-privilege alternative. We introduce ToolPrivBench to evaluate whether agents choose higher-privilege tools despite sufficient lower-privilege alternatives, measuring both initial selection and escalation after transient tool failures. Across eight domains and five recurring risk patterns, we find that over-privileged tool selection is common among mainstream LLM agents and is further amplified by transient failures. We further find that general safety alignment does not reliably transfer to least-privilege tool choice, while prompt-level controls provide only limited mitigation under transient failures. We therefore introduce a privilege-aware post-training defense that teaches agents to prefer sufficient lower-privilege tools and escalate only when necessary. Our mitigation experiments show that this defense substantially reduces unnecessary high-privilege tool use while preserving general capabilities.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：LLM代理在自主选择工具时，倾向于选择高权限工具（如管理员级API），即使存在足够的低权限替代工具也能完成任务。这种行为被称为**过度特权工具选择**，会无谓扩大安全漏洞的潜在影响范围。
- **背景**：现有工具选择研究关注安全无关的元数据偏好（如供应商身份、描述措辞）；代理安全研究主要针对有害输出或外部攻击（如提示注入、越狱）。而代理自身的内生性特权偏好被忽视。
- **意义**：随着“氛围编程”（vibe coding）等自主工作流兴起，代理决定“如何执行”变得关键，过度特权可能导致权限升级、数据过度暴露、安全旁路等风险。

## 6. 主要结论与发现

1. **过度特权工具选择普遍存在**：11个模型中有6个OPUR超过30%，小模型（Qwen3-8B: 64.9%）最严重，但大模型（Claude 4.6: 2.6%）也有出现。
2. **瞬态失败显著加剧升级**：大多数模型的过度特权案例中PED≥1，即发生在尝试低权限工具但遇到错误后，而非一开始就选高权限（如GPT-5.2零次选择仅5次，但升级触发35次）。
3. **领域和风险类型差异大**：基础设施和媒体领域OPUR最高；权限升级和安全绕过是最常见风险类型，范围扩展最少。
4. **安全对齐不迁移**：AgentAlign大幅降低有害工具使用（如Ministral从67.4%降至10.5%），但OPUR几乎没有下降（甚至上升），说明拒绝有害行为不意味着学会最小特权。
5. **提示工程缓解有限**：提示指令在初始选择时有效，但多轮失败后效果减弱；特权感知后训练则产生更大幅度且鲁棒的降低（如Qwen3-8B OPUR从64.9%降至27.0%）。
6. **后训练保留通用能力**：在MMLU、GSM8K、MetaTool上性能保留率均超过95%。

## 7. 优点

- **问题新颖性**：首次系统研究代理内生性特权偏好，填补了“特权过度”这一安全维度空白。
- **基准设计严谨**：通过自动化预检、双模型共识人工审核保证工具独立充分，排除能力混淆；合成工具避免预训练数据污染。
- **实验全面**：覆盖11个主流LLM、5种风险类型、8个领域，并对比了安全对齐、提示工程等多种缓解策略。
- **缓解方法有效且实用**：SFT+GRPO后训练在不显著牺牲通用能力的前提下大幅减少过度特权行为，且训练数据与评测集不重叠，验证了泛化性。

## 8. 不足与局限

- **模拟环境限制**：未使用真实生产工具或外部服务，工具子集固定且数量少（6个），无法完全反映实际代理环境的复杂性。
- **有限交互轮数**：仅5轮交互，可能无法捕捉需要更长轨迹时出现的升级模式。
- **机制分析缺失**：未深入分析过度特权选择背后的具体机制（如模型何时、为何放弃低权限工具）。
- **领域覆盖偏差**：从APIGen聚类筛选时，只有高特权密度高的领域被保留（如排除音乐、体育等低风险领域），基准偏向高风险场景。
- **人工审核成本**：标注工作量较大，且需要专家知识，扩展性受限。
- **未讨论不同模型家族之间的系统差异**：缺少对模型大小、训练数据、对齐策略与特权偏好关系的定量分析。
