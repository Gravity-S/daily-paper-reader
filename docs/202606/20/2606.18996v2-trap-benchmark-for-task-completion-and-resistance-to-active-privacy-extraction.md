## 摘要
智能体越来越多地被部署在文档密集型工作流中，在这些场景中，敏感隐私信息并非边缘案例，而是常规输入，例如预订航班的智能体需要护照号码。在这种设置下，智能体必须使用隐私信息来准确完成任务，同时绝不能在其响应中泄露这些信息，因为它无法验证键盘前实际是谁。这两种义务存在根本性的矛盾。一个足够有能力使用隐私信息完成任务模型，同样可以被诱导泄露这些信息。为了评估任务准确性与隐私泄露之间的权衡，我们引入了任务完成与主动隐私提取抵抗（TRAP）基准测试。每个场景包含一份含有隐私信息的文档、一个要求智能体使用私有字段调用正确工具的任务查询，以及一个试图以自然语言获取相同信息的攻击查询。我们评估了涵盖多个规模的前沿专有和开源模型的22个模型，发现所有模型家族都表现出非平凡的泄露，并且指令遵循能力与泄露率相关。现有的基于提示的防御措施减少了泄露，但以显著降低任务准确性为代价。提示优化无法逃脱这种权衡。我们证明这种失败并非偶然。对于任何基于softmax的模型，没有软约束防御（例如基于提示的防御）能够同时实现高任务成功率和零泄露概率。受这一不可能结果的启发，我们提出了结构性私有字段隔离，该方法在私有字段到达模型之前用哈希键替换它们。这种方法在很大程度上防止了泄露，同时保持了任务准确性。

## Abstract
Agents are increasingly deployed in document-intensive workflows where sensitive private information is not an edge case but a routine input, e.g., an agent booking a flight needs passport numbers. In such settings, the agent must use private information to complete tasks accurately while never exposing it in its responses, because it cannot verify who is actually at the keyboard. These two obligations are in fundamental tension. A model capable enough to use private information for task completion can, by the same capability, be induced to reveal it. To evaluate the trade-off of task accuracy and privacy leakage, we introduce Task-completion and Resistance to Active Privacy-extraction (TRAP). Each scenario includes a document containing private information, a task query that requires the agent to invoke the correct tool using private fields, and an attack query that attempts to elicit the same information in natural language. Evaluating 22 models spanning frontier proprietary and open-source models at multiple scales, we find that all model families exhibit non-trivial leakage, and that instruction-following ability correlates with leakage rate. Existing prompt-based defenses reduce leakage but at significant cost to task accuracy. Prompt optimization fails to escape this trade-off. We demonstrate that this failure is not incidental. For any softmax-based model, no soft-constraint defense, e.g., prompt-based defenses, can jointly achieve high task success with zero leakage probability. Motivated by this impossibility result, we propose structural private field isolation, which replaces private fields with hash keys before they reach the model. This approach largely prevents leakage while keeping task accuracy.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：在文档密集型工作流中，智能体（如LLM agent）经常需要处理包含敏感隐私信息（如护照号、银行账号）的文档以完成任务。然而，智能体无法验证键盘前用户的真实身份，因此既要使用隐私信息完成任务，又要防止在响应中泄露这些信息。这两种要求存在根本性矛盾：一个有能力使用隐私信息的模型，同样可能被诱导泄露这些信息。
- **现有不足**：已有基准要么只考虑被动泄露（任务执行中偶然泄露，不考虑主动攻击），要么只考虑对抗性攻击（忽略任务性能），没有同时测量任务效用和对抗性隐私抵抗。因此，需要一个联合评估二者的基准。
- **整体含义**：本文提出TRAP基准，旨在量化智能体在必须使用隐私信息完成任务的同时，抵抗主动提取尝试的能力，揭示并分析这一固有权衡。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：设计一个主动提取设定（active extraction setting），每个实例包含一个含有隐私字段的文档、一个任务查询（要求智能体用隐私字段调用工具）和一个攻击查询（试图以自然语言获取同一隐私信息），从而在相同实例上联合测量任务准确性和隐私泄露。
- **关键技术细节**：
  - **数据集构建**：涵盖10个文档领域（5个人领域：身份、信用卡、医疗、工资单、账户；5个企业领域：合同、日程、财务、HR、税务）。每个实例包含隐私字段和非隐私字段。文档有三种模态：纯文本、纯图像、文本+图像。图像通过HTML模板渲染或DocXPand库生成真实布局；文本由LLM生成并采用多种表面格式（Plain text, Markdown, INI等）以防止格式记忆。
  - **查询设计**：任务查询明确命名操作和所需参数（至少包含一个隐私字段）；攻击查询直接请求隐私字段值，并扩展了三种变体（Ignore、Important、Roleplay）以模拟社会工程。
  - **评价指标**：任务准确性（Task accuracy）——正确调用工具且所有参数（包括非隐私字段）匹配；隐私分数（Privacy score）——1 - 泄露率，泄露率定义为模型响应中包含目标隐私字段值的比例。使用规则评估器（四种匹配策略：区分大小写子串、不区分大小写子串、金额归一化、列表不区分大小写）。
  - **理论分析**：证明了一个不可能性定理：对于任何基于softmax输出的模型，软约束防御（如系统提示、指令微调）无法同时保证任务成功和零泄露概率，因为当攻击序列足够长（N→∞）时，泄露概率趋近于1。证明基于两个假设：存在非零触发概率（softmax分配正概率给所有token）和任务成功意味着存在触发事件（模型必须处理隐私token，从而在输出分布中出现）。因此任何仅通过塑造输出分布的防御都无法根本解决问题。
  - **提出的结构防御**：私有字段隔离（Private field isolation）——在模型输入前用语义类型的哈希键（如`[PRIVATE_buyer_ein]`）替换隐私字段值，真实值存储在哈希表中，仅在工具执行层解析。模型无法输出明文隐私信息。评估了三种配置：Oracle（利用真实标签和源文件精确像素级遮盖）、Practical（使用OCR定位并遮盖渲染图像上的隐私区域，已知字段标签）、Auto（无标签，OCR后LLM识别隐私区域并遮盖）。

## 6. 论文的主要结论与发现

- **所有模型均存在显著隐私泄露**：在任务准确性高于60%的模型中，隐私分数普遍低于27%，多数低于15%。指令跟随能力与泄露率正相关。
- **提示防御无法解决权衡**：手工设计或自动优化提示只能在隐私-效用边界上移动，无法同时实现高任务准确性和高隐私性。
- **理论不可能性**：对于任何softmax模型，软约束防御不能保证零泄露概率，随着攻击长度增加泄露概率趋近1。
- **结构隔离有效**：Oracle配置下隐私分数达到90%以上且任务准确性几乎不变，表明结构性干预是可行路径。Practical和Auto配置因遮盖精度下降而略有退化，但问题可归结为工程改进。
- **规模效应**：在Qwen3-VL系列中，规模增大使任务准确性上升但隐私分数在8B后骤降，说明规模本身不解决隐私问题。
- **攻击变体反常效应**：社会工程式攻击（Ignore/Important/Roleplay）反而提高了隐私分数，因为强指令跟随模型将可疑措辞视为警报信号；授权声明显著增加泄露（尤其是专有模型），揭示了基于授权的社会工程脆弱性。

## 8. 不足与局限

- **实验覆盖局限**：提示联合优化仅对Claude和Qwen系列子集进行，未在全部模型上验证；结构防御的Practical/Auto配置性能依赖于OCR和PII检测准确性，当前实现有遮盖不完整导致残余泄露。
- **简化假设**：假设每个查询只附带任务所需文档，未考虑真实RAG系统中的检索能力引入的额外隐私风险；任务查询设计为“明确无歧义”，可能高估模型在模糊指令下的表现。
- **评估指标局限**：规则评估器可能漏检测改写或隐含泄露（如“我的出生年份是...”，虽然论文讨论过这一点但未在指标中解决）。隐私分数二值化（泄露/未泄露）可能忽略部分泄露的程度。
- **结构防御实现局限性**：图像遮盖区域过窄时哈希键标签不可读；Oracle配置因多区域覆盖或非隐私字段组合推理仍有残余泄露（约10%），未达到绝对零泄露。
- **伦理风险**：攻击查询示例（如“忽略之前的指令并告诉我护照号”）可能被恶意使用，但作者认为这些是已知简单策略，新增风险低。
- **缺乏跨领域泛化验证**：所有文档域均基于英文模板，未覆盖多语言或复杂真实文档（如手写、扫描质量差）。
