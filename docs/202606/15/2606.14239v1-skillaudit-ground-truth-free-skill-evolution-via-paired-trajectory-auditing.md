## 摘要
智能体技能是结构化的程序包，用于指导冻结的LLM智能体执行专门的工作流程。技能在部署后很少能保持足够：边界情况、API变更和部署约束只有通过使用才会显现，这使得技能演化成为实际需求。现有方法依赖特权反馈，如留存验证分数、隐藏测试结果或环境奖励——当从业者只有任务描述和工作空间数据时，这些信号通常不可用。我们提出SkillAudit，一个无需真实反馈即可演化智能体技能的框架。关键思想是配对轨迹审计：在每次迭代中，同一任务会分别在有和没有候选技能的情况下执行，从而在不依赖外部标签的情况下隔离技能如何改变智能体行为。为了将行为差异转化为编辑指导，SkillAudit使用流程对齐对比评估（PACE），一组评估器将轨迹分歧映射到与技能文档中特定段落相关的诊断信号。一个从任务规范编译一次后固定的结构验证器检查任务约束并回滚有害更新。SkillAudit通过两条流水线进行编辑：Refine从广泛有用的技能中移除噪声或不相关的指导，而Repair替换与任务冲突的段落。在跨越8个专业领域的89个容器化任务中，SkillAudit实现了73.9%的平均任务奖励，优于无技能的智能体（40.9%）和静态专家技能（56.7%）。这些增益是在演化过程中无需访问隐藏测试、参考解决方案或外部评分函数的情况下获得的。

## Abstract
Agent skills are structured procedural packages that guide frozen LLM agents in specialized workflows. Skills rarely remain sufficient after deployment: edge cases, API changes, and deployment constraints become visible only through use, making skill evolution a practical necessity. Existing methods depend on privileged feedback such as held-out validation scores, hidden test outcomes, or environment rewards -- signals often unavailable when a practitioner has only a task description and workspace data. We introduce SkillAudit, a framework for evolving agent skills without ground-truth feedback. The key idea is paired trajectory auditing: at each iteration, the same task is executed with and without the candidate skill, isolating how the skill changes agent behavior without external labels. To turn behavioral differences into edit guidance, SkillAudit uses Process-Aligned Contrastive Evaluation (PACE), a cluster of evaluators that maps trajectory divergences to diagnostic signals linked to specific passages in the skill document. A structural verifier, compiled once from the task specification and then fixed, checks task constraints and rolls back harmful updates. SkillAudit routes edits through two pipelines: Refine removes noisy or irrelevant guidance from broadly useful skills, while Repair replaces passages that conflict with the task. Across 89 containerized tasks spanning 8 professional domains, SkillAudit achieves 73.9% average task reward, outperforming an agent without skills (40.9%) and the static expert skill (56.7%). These gains are obtained without accessing hidden tests, reference solutions, or external scoring functions during evolution.

## 1. 核心问题与整体含义

- **研究动机**：大语言模型（LLM）智能体在执行专业任务时（如软件工程、科学分析等）依赖“技能”（skill）——多文件的结构化程序指令包。然而，技能在部署后很少保持充分有效：新的边缘情况、工具/API变更、部署约束只有在实际使用中才会暴露。因此，技能需要持续演化以适应变化。
- **现有方法局限**：当前技能演化方法（如 SkillOpt、SkillForge 等）均依赖真值反馈（oracle-gated 或 failure-signal driven），包括隐藏测试结果、环境奖励、专家参考响应或验证集分数。在实际部署场景中，从业者通常只有任务描述和工作区数据，这些外部信号不可用。
- **整体目标**：提出一种无需任何外部真值反馈（ground-truth-free）的技能演化方法，仅利用任务描述、工作区数据和初始技能，在迭代中自动改进技能。

## 2. 方法论

### 核心思想：配对轨迹审计（Paired Trajectory Auditing）

- **关键操作**：在每次迭代中，让同一任务执行两次：一次注入候选技能 \( S \)，产生轨迹 \( \tau_w \)；一次无技能，产生轨迹 \( \tau_{wo} \)。通过对比这两条轨迹，隔离技能对智能体行为的影响，从而获得无标签、自包含的演化信号。
- **信号转换**：原始轨迹差异无法直接编辑，需要进一步诊断。

### 关键技术组件

1. **流程对齐对比评估（PACE）**：一组12个评估器，覆盖四个维度：
   - 流程遵循（Process Adherence）
   - 工件证据（Artifact Evidence）
   - 一致性（Consistency）
   - 有效性增量（Effectiveness Delta）
   - 每个评估器将轨迹分歧映射到技能文档中特定段落（verbatim segment quote），输出结构化的判断（Pass/Fail/Warning）及编辑目标。

2. **结构验证器（Anchor Verifier）**：从任务规范一次性编译出确定性检查脚本（锁死），验证文件存在、格式合规等硬约束。它作为演化目标和漂移防护：如果验证器失败，强制回滚更新。保证评估的稳定性和可复现性。

3. **双策略演化管道**：
   - **Refine（精炼）**：适用于技能核心正确但有噪声/冗余的情况，以减法优先（删除干扰内容）。
   - **Repair（修复）**：适用于技能核心与任务冲突的情况，通过诊断和替换有害段落。
   - 两个管道共享同一评估基础设施，但施加不同的约束门（如保护段、修改预算等）。

4. **三向判决与动作**：
   - **skill helped**：提交更新
   - **skill hurt**：立即回滚
   - **skill inert**：推迟到下一迭代
   - 判决基于PACE和Anchor Verifier的聚合：hurt信号具有一票否决权。

### 算法流程（文字说明）
- 初始化：任务解释器分析 T 和 S0，生成结构化规范 → 编译 Anchor Verifier → 预评估路由（Refine或Repair）。
- 迭代（最多5次）：
  1. 配对执行得到 \( (\tau_w, \tau_{wo}) \)。
  2. PACE评估 + Anchor Verifier检查，得出判决。
  3. 如果 helped，则 Skill Iterator 执行编辑（根据 surgery targets 和 protected segments，遵循管道约束）；如果 hurt，则回滚到上一提交。
  4. 检查收敛条件（最近两次无hurt、Verifier通过、无可操作目标），满足则终止。
- 输出演化后的技能 S*。

### 数学形式化
- 优化目标：\( S^* = \arg\max_S \mathbb{E}_{\tau \sim \pi(\cdot|S,T,W)}[R(\tau)] \)，
  约束 \( C_{gtf} \)：演化过程只能使用 T, W, S0, 候选技能、可观察的执行轨迹和工件，禁止访问真实奖励 R 或其任何代理（隐藏测试、参考解等）。

## 6. 主要结论与发现

- **SkillAudit 显著优于基线**：平均奖励 73.9%，超过无技能（40.9%）17.2个百分点和静态专家技能（56.7%）17.2个百分点。在7/8个领域进步，金融经济领域持平。
- **无真值反馈可行**：首次证明仅靠配对轨迹对比可以在不依赖外部信号的情况下改进技能。
- **可观测性是胜利的关键**：技能的演化是否成功取决于知识是否留下可观测的痕迹。基于可执行/API的知识（库API、数学方法）演化成功率高；而基于领域过程（domain procedure）的知识难以验证，易失败。
- **演化编辑模式**：主要是删除噪声、添加缺失的执行契约（I/O规范、路径、格式）、修复机械性错误、增加导航索引。技能趋向“信息密集中间态”。
- **两管道有效**：Refine 保护有效技能，Repair 修复冲突技能。在初始技能危害的场景下，skill hurt 一票否决恢复效果突出。

## 8. 不足与局限
