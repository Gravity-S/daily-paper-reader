## 摘要
现代LLM驱动的自主智能体越来越依赖丰富的用户界面(UI)状态观察，在复杂的数字环境中实现可靠的动作基础。然而，许多部署将完整的UI状态传输到远程推理服务器，即使大多数元素与当前任务无关，这可能会泄露敏感但不必要的上下文，例如认证码、私人通知和后台应用程序状态。我们提出MINIM，一个可信的本地代理，在任何观察离开设备之前在客户端进行隐私感知的最小化。基于上下文完整性(CI)，MINIM通过预测每个UI元素的固有敏感性分数(s)和任务条件下的必要性分数(n)来学习双分数表示。这些分数驱动一个三元披露策略：保留基本元素，在需要时抽象敏感属性，并移除任务无关内容。我们优化了一个CI感知的目标函数，对高风险内容更强烈地惩罚必要性错误，从而在保留任务关键信息的同时实现积极剪枝。在源自WebArena的真实世界UI观察实验表明，MINIM在保留任务关键语义上下文和可靠智能体动作所需的交互能力的同时，大幅减少了任务无关的敏感泄漏。

## Abstract
Modern LLM-powered autonomous agents increasingly rely on rich user interface (UI) state observations to achieve reliable action grounding in complex digital environments. However, many deployments transmit the full UI state to remote inference servers even when most elements are irrelevant to the current task, which can leak sensitive but unnecessary context such as authentication codes, private notifications, and background application states. We propose MINIM, a trusted local broker that performs privacy-aware minimization on the client side before any observation leaves the device. Grounded in Contextual Integrity (CI), MINIM learns a dual-score representation for each UI element by predicting an inherent sensitivity score (s) and a task-conditioned necessity score (n). These scores drive a ternary disclosure policy that keeps essential elements, abstracts sensitive attributes when needed, and removes task-irrelevant content. We optimize a CI-aware objective that penalizes necessity errors more strongly on high-risk content, enabling aggressive pruning while preserving task-critical information. Experiments on real-world UI observations derived from WebArena show that MINIM substantially reduces task-irrelevant sensitive leakage while preserving task-critical semantic context and the interactive affordances required for reliable agent actions.

## 2. 方法论：核心思想、关键技术细节、算法流程
- **核心思想**：在客户端部署一个轻量级模型，为每个 UI 元素预测两个分数：
  - **敏感度** \( s \)：固有信息泄露风险（与任务无关）。
  - **任务条件必要性** \( n \)：对于完成当前用户任务的关键程度。
- **三元披露策略**：基于阈值 \( \tau_{\text{nec}} \) 和 \( \tau_{\text{sens}} \) 对每个元素执行：
  - **K（保持）**：必要性高且敏感度低 → 直接保留。
  - **A（抽象）**：必要性高且敏感度高 → 保留结构角色但替换敏感属性值（如“[REDACTED]”）。
  - **R（移除）**：必要性低 → 直接删除。
- **关键技术**：
  - **编码器**：使用图注意力网络 v2（GATv2）处理可访问性树，节点特征由 MiniLM 文本嵌入 + UI 属性/结构特征组成（512维）。
  - **训练目标**：CI 感知损失函数：
    - 敏感度损失：绝对误差 \( L_{\text{sens}} = \sum_i |s_i - \hat{s}_i| \)。
    - 必要性损失：加权绝对误差 \( L_{\text{nec}} = \sum_i \left[ 1 + \lambda \cdot \frac{s_i}{10} \cdot \left(1 - \frac{n_{i,T}}{10}\right) \right] \cdot |n_{i,T} - \hat{n}_{i,T}| \)，对高敏感低必要性节点施加更大惩罚。
    - 总损失：\( L = L_{\text{sens}} + \alpha L_{\text{nec}} \)。
- **算法流程（部署时）**：
  1. 输入原生可访问性树 \( X_t \) 和任务 \( T \)。
  2. 通过训练好的模型为每个节点输出 \( (\hat{s}_i, \hat{n}_{i,T}) \)。
  3. 对每个节点应用三元策略（阈值 \( \tau_{\text{nec}}=1.0 \), \( \tau_{\text{sens}}=5.0 \)），生成净化的树 \( Z_t \)。
  4. 将 \( Z_t \) 发送给远程 agent。

## 6. 主要结论与发现
- MINIM 在 **TCNP-I = 0.9931**、**TCNP = 0.9491** 的高保留率下，将 **TISL 降至 0.101**（仅为完整观测泄露的 10.1%）。
- 对比 LLM 基线：虽 TCNP 略高（0.951–0.966），但 TISL 大幅升高（0.194–0.312），且保留节点比例更大（26–35% vs. MINIM 的 12%）。MINIM 在隐私-效率前沿上严格更优。
- 抽象动作（A）比严格移除（A→R）或完全暴露（A→K）更优：在微小泄露增量（0.0989→0.1010）下带来 TCNP-I 提升 2.81 个百分点。
- 对注入敏感元素（如 2FA 码、密码）抑制率 >99.9%。
- CI 合规性评估接近 Oracle 的“Need-to-Know”原则。
