## 摘要
基于云的多智能体系统协调需要与中央服务器共享状态，从而在协调与隐私之间产生冲突。全同态加密在原则上解决了这一问题，但其严格的算术约束要求控制循环的每个阶段都必须从基本原理重新设计。我们提出了一种端到端加密控制管道，其中感知、状态估计、状态传播和一致性控制均在CKKS加密数据上运行，仅使用加法、乘法与循环旋转。为了克服全同态加密的计算挑战，我们采用稳态卡尔曼增益替代在线求解矩阵，并通过对角线方法以与非零循环对角线数量成正比的代价应用图拉普拉斯算子，在统一框架内适配环、环面及完全图拓扑。为量化加密噪声的累积效应，我们利用分离定理解耦控制器与观测器误差动态，并推导出周期性的自举界限，其中CKKS自举被视为脉冲扰动；由此产生的稳态误差球取决于自举精度与闭环谱半径，为隐私-精度权衡提供了直接设计方程。该管道在多智能体编队控制场景中得到验证，确认了在加密下具有有界跟踪误差的稳定闭环运行。

## Abstract
Cloud-based coordination of multi-agent systems requires sharing state with a central server, creating a conflict between coordination and privacy. Fully homomorphic encryption (FHE) resolves this in principle, but its severe arithmetic constraints demand that every stage of the control loop be redesigned from first principles. We present an end-to-end encrypted control pipeline in which sensing, state estimation, state propagation, and consensus control all operate on CKKS-encrypted data using only addition, multiplication, and cyclic rotation. In order to overcome the computational challenges of FHE, we employ steady-state Kalman gains instead of solving for the matrices online and graph Laplacians are applied via the diagonal method at a cost proportional to the number of nonzero cyclic diagonals, accommodating ring, torus, and complete-graph topologies within a unified framework. To quantify the cumulative effect of encryption noise, we use the separation principle to decouple controller and observer error dynamics and derive a periodic bootstrapping bound in which CKKS bootstrapping acts as an impulsive disturbance; the resulting steady-state error ball depends on the bootstrapping precision and the closed-loop spectral radius, providing a direct design equation for the privacy-accuracy tradeoff. The pipeline is validated on a multi-agent formation control scenario, confirming stable closed-loop operation under encryption with bounded tracking error.

- **核心问题**：基于云的多智能体系统协调需要智能体向中央服务器发送明文状态，导致隐私泄露风险，尤其在竞争组织、监管环境或对抗通信信道中不可接受。全同态加密（FHE）理论上允许在密文上执行任意计算而不暴露明文，但FHE的严格算术约束（仅支持加、乘、循环旋转；不支持比较、分支、除法等）迫使控制回路的每个阶段都必须从零重新设计。
- **整体含义**：本文旨在构建首个端到端加密控制管道，涵盖感知、状态估计、状态传播和共识控制，所有阶段均以CKKS同态加密为基础，仅使用三个原生原语，并提供可量化的稳态跟踪误差界，从而在隐私与精度之间建立可设计的权衡。

# 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：将整个控制循环表达为CKKS的原生操作集合（加法、Hadamard乘法、循环旋转），避免密文-密文乘法（使用明文-密文乘积降低噪声和开销），预计算所有矩阵和增益（离线明文），在线只对加密状态执行明文矩阵×密文向量的操作。
- **关键技术细节**：
  - **稳态卡尔曼增益**：为避免在线求解代数Riccati方程（涉及矩阵求逆，非CKKS兼容），预先计算稳态Kalman增益\(K_{ss}\)，使观测器更新简化为明文-密文乘积：\(\text{ct}(\hat{x}(k+1)) = (A-K_{ss}C)\text{ct}(\hat{x}(k)) + K_{ss}\text{ct}(y(k))\)（忽略B项）。
  - **对角线方法实现矩阵-向量乘法**：任何明文矩阵\(M\)与密文向量\(\text{ct}(v)\)的乘积可通过下式计算：\(M\text{ct}(v) = \sum_j d_j \odot \text{rot}^j(\text{ct}(v))\)，其中\(d_j\)是第\(j\)条循环对角线的明文掩码。仅需\(O(\text{非零对角线数})\)次旋转和乘法，深度为1级。
  - **图拉普拉斯的高效加密应用**：对于环拓扑，拉普拉斯只有三个非零循环对角线，可通过两个循环旋转实现（零乘法深度）。对于环面或完全图，仍使用对角线方法，成本与循环对角线数线性正比。
  - **自举调度**：每控制周期消耗2-3级乘法深度（取决于拓扑），自举周期\(T_{\text{boot}} = \lfloor d_{\text{mult}}/d_{\text{cycle}} \rfloor \approx 5\)，自举被视为一个脉冲扰动。
- **误差分析与稳定界**：利用分离原理，将加密误差动态解耦为观测器误差\(\tilde{\epsilon}\)和控制器误差\(\hat{e}\)，证明两者均渐近稳定。误差动态为\(\hat{e}(k+1)=A_{\text{cl}}\hat{e}(k)+w(k)\)，其中\(w(k)\)包含自举脉冲。推导出稳态误差球半径：\(\varepsilon_{ss} = \bar{\delta}_{\text{eff}} / (1 - \rho_{\text{cl}}^{T_{\text{boot}}})\)，其中\(\rho_{\text{cl}}\)为闭环谱半径，\(\bar{\delta}_{\text{eff}}\)为有效自举噪声幅值（包含观测器耦合）。
- **算法流程（每控制周期）**：
  1. **加密感知**：各智能体加密测量值\(\text{ct}(y_i(k))\)并发送给协调器，打包成一个密文。
  2. **加密状态估计**：协调器用稳态Kalman增益更新加密状态估计\(\text{ct}(\hat{x}(k))\)（两个明文-密文乘积，消耗1级深度）。
  3. **加密状态传播**：用系统矩阵\(A\)左乘加密状态（消耗1级深度）。
  4. **加密共识控制**：计算\(\text{ct}(u(k)) = (I\otimes K)\text{ct}(\hat{x}) - \varepsilon(L\otimes I_n)\text{ct}(\hat{x}) + (I\otimes K_r)(r-\text{ct}(\hat{x}))\)，其中拉普拉斯项消耗0-1级深度。
  5. **解密与驱动**：协调器返回密文给智能体，智能体解密并执行。
