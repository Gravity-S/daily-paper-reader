## 摘要
持续学习，即人工智能系统通过顺序经验不断改进的能力，已引起广泛关注，但目前尚无高质量基准来评估它。我们引入了持续学习基准（CL-Bench），这是第一个经过专家验证的、旨在衡量基于LLM的系统是否真正通过经验改进的困难基准。CL-Bench涵盖六个不同领域（软件工程、信号处理、疾病爆发预测、数据库查询、策略游戏和需求预测），每个领域均由领域专家验证，并且任务设计为共享可学习的潜在结构（代码库布局、疾病爆发动态、对手策略），这些结构有状态系统可以在线发现，而无状态系统则无法发现。我们评估了从简单的上下文学习（ICL）到专用记忆系统的多种智能体架构下的前沿模型，并引入增益指标以从先验能力中隔离学习。我们发现，这些系统为改进持续学习留有空间：智能体经常过度拟合即时观察或未能跨实例重用知识，而专用记忆系统并未解决这一问题——实际上，简单的ICL优于专用记忆管理系统。CL-Bench是第一个通过专家验证任务评估跨多样真实世界领域持续学习，并在线学习与底层模型能力隔离的基准，表明需要更好的持续学习系统。

## Abstract
Continual learning, the ability of AI systems to improve through sequential experience, has attracted substantial interest, but no high-quality benchmark exists to evaluate it. We introduce Continual Learning Bench (CL-Bench), the first difficult, expert-validated benchmark designed to measure whether LLM-based systems genuinely improve with experience. CL-Bench spans six diverse domains (software engineering, signal processing, disease outbreak forecasting, database querying, strategic game-playing, and demand forecasting), each validated by domain experts and designed so that tasks share a learnable latent structure (codebase layout, disease outbreak dynamics, opponent strategies) that a stateful system can discover online but a stateless one cannot. We evaluate frontier models across several agent architectures, from naive in-context learning (ICL) to dedicated memory systems, introducing a gain metric to isolate learning from prior capabilities. We find that these systems leave headroom for improved continual learning: agents frequently overfit to immediate observations or fail to reuse knowledge across instances, and dedicated memory systems do not fix this -- in fact, naive ICL outperforms systems dedicated to memory management. CL-Bench is the first benchmark to evaluate continual learning across diverse real-world domains with expert-validated tasks and isolate online learning from underlying model capability, showing a need for better continual learning systems.

# 论文《Continual Learning Bench: Evaluating Frontier AI Systems in Real-World Stateful Environments》中文总结

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：现有缺乏高质量基准来评估 AI 系统（尤其是 LLM 智能体）是否真正通过顺序经验（持续学习）提升性能。现有评估多聚焦于记忆检索、上下文压缩或单次任务准确率，无法有效隔离在线学习与模型固有能力。
- **整体含义**：构建持续学习能力对于部署长期交互的 AI 系统（如软件工程代理、数据分析代理等）至关重要。本文提出的 CL-Bench 是首个专家验证的、跨真实世界领域的困难基准，旨在衡量 LLM 系统能否从经验中学习并改进。

## 2. 方法论：核心思想、关键技术细节、算法流程
- **核心思想**：设计包含**共享潜在结构**的任务序列，该结构仅通过在线经验可被发现和利用，从而区分“有状态（stateful）”与“无状态（stateless）”系统。引入**增益（gain）** 指标，即状态系统与相同系统在无状态模式下得分的差异，以隔离学习贡献。
- **任务设计标准**：
  - **Headroom**：初始性能远低于最大值，避免强基模型直接达到天花板。
  - **共享潜在结构**：不同实例间存在可学习规律（如数据库模式、对手策略），系统需积累经验才能利用。
  - **学习机制**：早期实例提供可被后期利用的观察和反馈。
- **增益指标**：
  - 单实例增益 g_t = r_t^{sf} - r_t^{sl}
  - 累计增益 g = Σ g_t
  - 归一化增益：bg = (r̅_sf - r̅_sl) / (r_max - r̅_sl)
- **跨任务归一化**：使用统一的参考基线（GPT-5.4 的 ICL 无状态奖励）计算归一化奖励，使得评分不随新系统加入而变化。
- **稳定性-可塑性分解**：将增益分解为跨变体边界（稳定性）和变体内（可塑性）两部分，分析系统在保留旧知识与适应新信息上的差异。

## 3. 实验设计
- **数据集/场景**：CL-Bench 包含 **6 个领域**，每个领域由专家验证：
  - 盲频谱监测（BSM）
  - 代码库适应（Codebase Adaptation）
  - 队列研究（Cohort Studies）
  - 数据库探索（Database Exploration）
  - 可剥削扑克（Exploitable Poker）
  - 销售预测（Sales Prediction）
- **Benchmark**：每个任务包含数十到上百个实例，包含概念漂移变体（如数据库迁移、对手切换）。
- **对比方法**：
  - **模型**：Claude Opus 4.7、Sonnet 4.6、Gemini 3.1 Pro、Gemini 3 Flash、GPT-5.4。
  - **持续学习系统**：
    - 全上下文 ICL（保留完整历史）
    - ICL Notepad（持久化草稿本）
    - Mem0（语义记忆检索，k=10）
    - ACE（记忆剧本压缩更新）
    - Claude Code（自动上下文压缩）
    - Codex（代码代理）
- **评估设置**：每个系统在每项任务上运行 5 轮（除 Codex 仅 1 轮），报告归一化奖励和增益，以及成本（美元）。

## 4. 资源与算力
- **明确提及**：论文**未详细说明**使用的 GPU 型号、数量、训练时长等硬件资源。仅提到模型通过原生提供商 API 查询，并尽量启用提示缓存（prompt caching）。
- **成本数据**：提供了每轮运行的 API 成本（例如：ICL + Claude Sonnet 4.6 成本 $30.4，ACE + GPT-5.4 成本 $62.8），但未涉及离线训练或推理所需的算力细节。

## 5. 实验数量与充分性
- **实验数量**：
  - 12 种系统-模型组合，跨 6 个任务，每种配置 5 轮（Codex 仅 1 轮），共计约 12×6×5 = 360 个任务级运行（每轮含多个实例步骤）。
  - 包含每个任务的奖励曲线、归一化结果、稳定性-可塑性分解。
- **充分性与公平性**：
  - 实验覆盖了主流前沿模型和多种记忆范式，对比**有状态 vs 无状态**基线，并控制模型相同以比较方法。
  - 增益指标有效隔离了学习效果，避免了强模型天然优势的混淆。
  - 但未评估参数化方法（如测试时训练），且任务实例数量有限（数十到百余），对超长周期持续学习的泛化性可能不足。

## 6. 主要结论与发现
- **朴素 ICL 是强基线**：全上下文 ICL 在归一化奖励（22.3%）和增益（25.4%）上均优于专用记忆系统（如 Mem0、ACE）。即使是最佳系统，归一化增益也仅为 25.4%，说明持续学习仍有巨大提升空间。
- **专用记忆系统表现不佳**：ACE 成本最高（$62.8）但增益仅 8.6%，ICL Notepad 在某些任务上甚至降低性能。记忆模块常引入错误泛化或陈旧信念。
- **学习行为异质性**：不同任务上学习曲线差异显著。销售预测和盲频谱监测展现出明确的学习改善，而队列研究和策略扑克增益极小。
- **稳定性-可塑性权衡**：某些系统（如 ICL Notepad）稳定性高但可塑性低，而 ICL 和 Claude Code 可塑性高但稳定性低。

## 7. 优点
- **首次提出专家验证的跨领域持续学习基准**，任务设计严格遵循 headroom、共享结构、学习机制标准。
- **增益指标创新**：通过对比有状态/无状态性能，有效隔离了在线学习的效果，避免强模型天然优势的干扰。
- **全面评估前沿模型与记忆范式**，覆盖 ICL、检索式、压缩式等多种策略，并提供成本分析。
- **稳定性-可塑性分解**揭示了不同系统的学习行为差异，有助于指导未来系统设计。
- **开源且鼓励社区贡献**，提供了任务构建标准，利于基准持续扩展。

## 8. 不足与局限
- **领域覆盖有限**：仅 6 个领域，未能涵盖所有现实持续学习场景（如对话、机器人控制等）。
- **实例数较少**：每个任务数十到一百余实例，相比真实部署的数周或数月交互仍显不足，更长时间跨度可能带来不同挑战。
- **未评估参数化方法**：如测试时训练、模型微调等，而仅评估了基于上下文的记忆范式（ICL、检索、压缩）。
- **成本高昂**：评估前沿模型需要大量 API 调用，可能限制社区贡献的可重复性。
- **基线饱和风险**：某些任务（如队列研究）当前系统接近无学习，可能因任务过于困难或设计偏差，需进一步验证。
- **顺序依赖与方差**：虽通过多轮随机排列降低顺序影响，但实例顺序仍可能影响结果；论文未充分讨论离散任务之间的顺序依赖性。
- **缺乏小型模型评估**：仅使用前沿模型，未测试小型或开放模型，未能覆盖更广泛的持续学习研究群体。
