## 摘要
使用工具的大型语言模型（LLM）智能体面临两种不同的安全漏洞：未经授权的外部操作以及在最终输出检查干预之前，运行时内部敏感明文的暴露。现有防御通常只保护一个边界，要么是规划器/运行时，要么是操作目标，因此它们本身无法同时确保两个表面的安全。我们提出SecureClaw，一种双边界架构，在效果汇聚点实施授权，在读取边界实施明文隔离。敏感读取通过一个可信网关，该网关将原始值替换为不透明句柄，并在评估部署中，作为明确的解密接口替换为有边界摘要。改变外部状态的写入遵循PREVIEW→COMMIT协议，其中只有可信执行者才能提交经策略授权的确切规范请求。运行时仍然可以对摘要和符号引用进行规划，但不能直接解引用秘密或执行副作用。在AgentDojo、AgentLeak和Agent Security Bench（ASB）上，SecureClaw是我们评估中唯一一个在通用测试平台上同时保持可用任务效用且实现攻击成功率为0%的防御措施：在ASB上为0%，在AgentDojo上为0.64%，在AgentLeak的攻击对比路径上整体泄露率为3.23%，该路径衡量最终输出和内部中继泄露。

## Abstract
Tool-using large language model (LLM) agents face two distinct security failures: unauthorized external actions and exposure of sensitive plaintext inside the runtime before any final output check can intervene. Existing defenses usually protect one boundary, either the planner/runtime or the action sink, and therefore do not by themselves secure both surfaces. We present SecureClaw, a dual-boundary architecture that places authorization at the effect sink and plaintext confinement at the read boundary. Sensitive reads pass through a trusted gateway that replaces raw values with opaque handles and, in the evaluated deployment, bounded summaries as an explicit declassification interface. Writes that change external state follow a PREVIEW$\rightarrow$COMMIT protocol in which only a trusted executor may commit the exact canonical request authorized by policy. The runtime can still plan over summaries and symbolic references, but cannot directly dereference secrets or perform side effects. Across AgentDojo, AgentLeak, and Agent Security Bench (ASB), SecureClaw is the only defense we evaluate in a common harness that simultaneously retains usable task utility and achieves 0\% attack success rate (ASR) on ASB, 0.64\% ASR on AgentDojo, and 3.23\% overall leak on AgentLeak's attacked parity lane, which measures final-output and internal-relay leakage.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：工具使用型大型语言模型（LLM）智能体在运行时面临两种相互独立的**安全失效**：
  1. **未授权外部动作**：攻击者通过间接提示注入等手段，操纵智能体执行非预期的、可能造成不可逆后果的外部操作（如发送邮件、转账、修改记录）。
  2. **运行时明文泄露**：智能体在最终输出检查之前，内部信道（如同智能体间消息C2、共享内存C5）已暴露敏感明文，即使后续提交被拦截也无法挽回。
- **现有防御不足**：现有方案通常只保护单一边界——要么保护规划器/运行时（如IPIGuard、DRIFT），要么保护动作接收端（如Faramesh），缺乏同时覆盖两个攻击面的统一架构。
- **研究动机**：系统性地分离和解决这两个问题，使LLM代理在保持任务效用的同时，从根本上杜绝这两类攻击。
- **整体含义**：提出一种双边界安全架构SecureClaw，将**效果边界授权**和**明文边界隔离**作为两个不可相互替代的安全支柱，证明二者缺一不可，并通过全面实验验证了其有效性。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想

- **双边界设计**：
  - **效果边界（Effect Surface）**：在外部动作的接收端（executor）实施强制授权。只有经策略引擎批准的确切规范化请求才能被提交执行。
  - **明文边界（Plaintext Surface）**：在敏感读取操作到达运行时之前，通过可信网关将原始明文替换为**不透明句柄（handle）** 和**有界摘要（bounded summary）**，运行时无法直接接触原始敏感数据。
- **不信任运行时**：运行时（LLM驱动）被视为不可信，仅负责规划、搜索和编排工具调用，但**不能直接触发外部副作用**，**不能直接访问原始敏感值**。

### 2.2 关键技术细节

#### 读取路径：不透明句柄隔离

- 当工具返回敏感内容时，可信网关将明文存储在受保护的句柄表中，并返回一个不透明句柄（高熵符号引用，如`HMAC-SHA256(k_handle, nonce || session || object)`）。
- 运行时只能看到句柄和**有界摘要**（由固定模式感知操作符生成，带有显式的项目和字符上限）。摘要被视为**显式解密接口（explicit declassification interface）**，而非偶然的明文副本。
- 句柄可在内存、消息、后续工具调用中传递，但只能在受信任的端点（如执行器提交时）且经策略检查后方可解引用。

#### 写入路径：PREVIEW→COMMIT协议

- 运行时只能**提议**一个有效动作（intent + inputs），而不能直接执行。
- 可信网关附加经过身份验证的调用者、会话、上下文等字段，形成完整请求ρ = (intent, caller, session, inputs, ctx)。
- 网关对ρ进行**规范化（Canon(ρ)）**，计算绑定摘要h = HMAC-SHA256(k_bind, Canon(ρ))，并向策略引擎查询授权，获得授权工件π（包含h、决策结果、时间戳等）。
- 执行器（Executor）是唯一能接触外部效果汇的组件。提交时，执行器独立重算h、验证π、检查新鲜度、防止重放、验证确认令牌等。只有所有检查通过后，才解引用所需句柄并执行外部效果。
- 任何失败均**安全关闭（fail-closed）**。

#### 拒绝感知恢复（Deny-Aware Recovery）

- 当执行器拒绝提交时，返回结构化的拒绝消息（仅含粗粒度动作类、原因代码和安全下一步提示），允许智能体切换到安全的替代路径（如“改为草稿待确认”），从而在不降低安全性的前提下恢复任务效用。

#### 安全保证形式化

- **定理1（请求绑定授权完整性）**：在MAC安全、确定性规范化、新鲜度、原子重放检查等条件下，任何PPT对手成功发起未授权外部动作的概率受限于MAC伪造优势与绑定摘要碰撞概率之和。
- **定理2（句柄仅含明文隔离）**：在仅句柄模式下，运行时视图可模拟，优势仅来自幸运猜测句柄（概率≤ q_h N_live / 2^κ）或PRF区分。
- **定理3（有界摘要接口的量化暴露）**：有界摘要引入的区分优势受每个读取事件的TV距离之和以及句柄猜测概率控制。

## 6. 论文的主要结论与发现

1. **双边界架构有效性**：SecureClaw是唯一在通用测试平台上同时实现低ASR和高任务效用的防御。
   - ASB：**0% ASR**（0/2000），**88.9%效用**（恢复机制下）。
   - AgentDojo：**0.64% ASR**（4/629），**56.6%攻击效用**，**60.8%良性效用**。
   - AgentLeak攻击对比路径：**3.23%任何泄露**（C1为主，C2仅1例，C5为0），**74.6%攻击效用**，**77.0%良性效用**。

2. **两个支柱不可替代**：
   - 移除句柄导致C2/C5泄露上升（C2从0%到20.8%，C5从0%到8.3%），而移除边界对泄露无影响——表明句柄隔离负责明文保护。
   - 边界在效果面（旁路测试、支付案例）中阻止了所有未授权提交，而句柄本身不阻止执行——表明边界负责效果授权。

3. **拒绝感知恢复显著提升效用**：在ASB上将从拒绝后的70%效用提升至86%，在AgentLeak子集上从40%提升至66.7%，均无ASR增加。

4. **残余失败集中在策略允许范围内的误授权和最终输出解密压力**，而非系统绕过。

5. **延迟可接受**：执行器验证<1ms，单服务授权~15ms，分布式变体~149ms，相对于LLM推理的500-2000ms可忽略。

- **实验局限**：
  - 主比较仅使用单一LLM（gpt-4o-mini），跨模型抽查样本小且未覆盖所有基线，泛化性有限。
  - 消融实验样本量较小（48行），部分统计信号（如C5）不足以独立支撑强结论。
  - 所有实验在本地单一工作站编排运行，未评估真实分布式部署中的网络延迟、扩展性及可靠性。
- **安全假设的局限**：
  - 核心保证假定网关、句柄存储、策略引擎、执行器等受信组件不被攻破；若受信组件被攻破，所有安全保证失效（附录B.6）。
  - 需完整的介质中介（所有效果汇和敏感读取均需经过中介）、正确的保护字段分类、受限制的时钟漂移、原子重放保护等部署前提。误分类或遗漏的中介可能导致边界绕过。
  - 未声称能防御语义错误（策略允许的动作仍可能是恶意的或误用的），残余失败正集中于此。
- **应用限制**：
  - 有界摘要接口引入显式解密信道，在摘要与秘密高度相关的场景（如自由文本），泄露风险增加，需谨慎配置摘要预算。
  - 当前部署中，AgentDojo良性效用仅60.8%（vs Faramesh 3.7%？注：原文ASB中SecureClaw效用88.9%vs Faramesh 3.7%的对比应指ASB效用，AgentDojo良性效用为60.8%），仍有提升空间。作者指出需更丰富的原则-对象-阶段绑定来减少策略内误授权。
- **虚假保证风险**：若部署时保护字段误分类或效果纳入未中介，架构边界保证将不成立，需明确映射和审计。
