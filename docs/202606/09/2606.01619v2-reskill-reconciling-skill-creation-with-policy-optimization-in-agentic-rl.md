## 摘要
智能体强化学习使大语言模型代理能够从环境奖励中持续改进，但由此产生的策略无法系统性地积累可跨任务泛化的可复用策略。模块化技能可以提供此类可复用策略，然而现有的技能增强强化学习方法将技能创建与策略优化分离，导致可能采用与演化中的策略相冲突的技能。受Anthropic的Skill Creator启发，我们提出了ReSkill，一种将强化学习置于循环中的技能创建框架，实现技能演化与策略学习的协调。ReSkill利用GRPO的组内结构，自然嵌入了三种机制，仅增加微小的额外开销：（1）基于断言的技能创建器，从过往经验中诊断失败，并提出基于条件的触发式技能修改；（2）组内rollout采样，支持对技能版本进行受控比较，捕捉哪个版本最能支持策略的持续学习；以及（3）带有自适应折扣的汤普森采样，在策略演化过程中平衡技能版本选择的探索与利用。在多个领域，ReSkill持续优于现有的基于记忆和技能的强化学习方法，在未见任务上提升最大。对技能生命周期的分析表明，随着策略改进，技能被自动创建、测试、完善和修剪，展示了协调的技能-策略协同演化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

### 2. 论文提出的方法论

- **核心思想**：利用 GRPO 的组内结构（group-wise structure），在不增加额外 rollout 预算的前提下，让每个 rollout 同时承担三个角色：① 提供 GRPO 梯度进行策略优化；② 提供轨迹和奖励供技能诊断与创建；③ 作为技能版本的测试样本，通过 bandit 机制决定是否采纳新技能。
- **关键技术细节**：
  - **断言驱动的技能创建器 (Assertion-driven Skill Creator)**：借鉴 Anthropic Skill Creator 的“诊断-提议-测试”循环，但将人类反馈替换为 RL 训练中的轨迹和奖励信号。通过一个动态断言集（rule-based predicates）对经验池中的轨迹进行统一评分，定位失败模式，并基于历史版本接受/拒绝结果提出新技能版本（包括 ADD、MODIFY、DELETE 操作）。
  - **组内轨迹采样 (Within-Group Rollout Sampling)**：在 GRPO 的每组 G 条轨迹中，对每条轨迹独立分配一个技能版本（新版本或旧版本），使得组内唯一变量为技能版本，从而实现对不同技能版本的受控比较。
  - **自适应汤普森采样 (Thompson Sampling with Adaptive Discounting)**：将旧版本和新版本视为两个臂，使用 Beta 分布建模成功概率。为考虑策略演化，后验更新时引入自适应折扣因子 \( w_t = (1 + n_t/M)^{-1} \)，其中 \( n_t \) 为该版本在当前步获得的轨迹数，\( M \) 为记忆参数（通过预测似然最大化自动估计）。每步根据后验采样分配版本，每个演化周期结束时若新版本期望成功概率高于旧版本，则接受新版本。
- **算法流程（文字描述）**：
  1. 初始化策略、技能库、经验池、后验参数；
  2. 对每个训练步 \( t \)：
     - 从任务分布中采样一个任务；
     - 根据当前汤普森采样后验计算每个版本的分配概率，对组内每条轨迹独立分配版本；
     - 在对应技能库下生成轨迹，计算奖励；
     - 用 GRPO 目标更新策略（同时使用旧策略和当前策略）；
     - 将轨迹存入经验池，更新版本后验（先折扣再累加新数据）；
     - 若到达演化频率 \( K_e \)，则根据后验接受或拒绝新版本，然后通过技能创建器生成新的候选版本，重置后验。
  3. 返回优化后的策略和技能库。

### 6. 论文的主要结论与发现

- ReSkill 在所有基准上一致超越现有方法，**在未见任务上提升尤为显著**（ALFWorld 8B 未见过 95.3% vs SKILLRL 82.6%，提升 12.7%）。

- 技能生命周期显示：早期创建基础技能，后期精炼内部化技能并剪枝不相关技能，体现自动适应。
- 冻结策略后在未见领域（ALFWorld → ScienceWorld）仅通过技能进化即可实现有效迁移，表明协调训练培养了通用技能适应能力。
