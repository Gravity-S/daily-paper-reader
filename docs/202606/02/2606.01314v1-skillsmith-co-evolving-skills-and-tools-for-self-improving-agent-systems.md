## 摘要
近期自我进化智能体已展现出技能可通过执行被发现、提炼和积累的能力。然而，现有技能进化框架通常假设固定工具层，并独立评估每项技能，这限制了其修复工具级故障或推理技能间交互的能力。我们提出SkillSmith，一种协同感知的技能-工具共进化框架。SkillSmith引入统一提议空间，其中反思产生可同时修改技能与工具的原子束，允许在技能进化识别出可复用能力缺口时，对工具进行包装、编辑、组合、拆分或淘汰。为引导此联合搜索，SkillSmith维护一种受Lotka-Volterra动力学启发的生态效用模型，其中从执行轨迹估计的交互矩阵捕获技能间的互补性与冲突，并为检索、变异优先级排序和淘汰提供压力信号。此外，SkillSmith记录反模式（包括失败特征、因果归因及补救措施），以加速诊断并否决重复已知错误的提议。在包含WildClawBench的三个基准测试及五种Qwen3.5模型规模上的实验表明，SkillSmith始终优于强基线，且其增益随任务复杂度和多技能协同激活程度的增加而放大。

## Abstract
Recent self-evolving agents have shown that skills can be discovered, refined, and accumulated through execution. However, existing skill-evolution frameworks typically assume a fixed tool layer and evaluate each skill independently, limiting their ability to repair tool-level failures or reason about interactions among skills. We propose SkillSmith, a synergy-aware skill-tool co-evolution framework. SkillSmith introduces a unified proposal space in which reflection produces atomic bundles that jointly modify skills and tools, allowing tools to be wrapped, edited, composed, split, or retired when skill evolution identifies a reusable capability gap. To guide this joint search, SkillSmith maintains an ecological utility model inspired by Lotka-Volterra dynamics, where an interaction matrix estimated from execution traces captures pairwise complementarity and conflict among skills and provides pressure signals for retrieval, mutation prioritization, and retirement. Furthermore, SkillSmith records anti-patterns, including failure signatures, causal attributions, and remedies, to accelerate diagnosis and veto proposals that repeat known mistakes. Experiments on three benchmarks, including WildClawBench, and five Qwen3.5 model scales show that SkillSmith consistently outperforms strong baselines, with gains that amplify as task complexity and multi-skill co-activation increase.

当前自进化智能体系统在技能发现、提炼与积累方面取得了进展，但存在三大核心局限：
- **工具层固化**：现有进化框架将工具层视为固定不变，若失败源于工具缺陷（如接口过时、缺失功能），系统只能通过在技能层添加复杂权变策略来补偿，导致技能臃肿且易错。
- **技能独立评估**：每项技能被单独评估与变异，忽略了多技能协同执行时的交互——互补性或冲突会影响整体性能，但现有方法不建模这种依赖关系。
- **失败经验遗忘**：失败轨迹仅在反思时使用一次，保留的只是成功人工产物。系统会重复诊断同样的错误、重新生成已被证明无效的配置，浪费计算资源并拖慢进化速度。

因此，论文提出**协同而非孤立地进化技能和工具**，同时记录并利用失败模式，以实现可持续且高效的自我改进。

### 2. 方法论：核心思想、关键技术细节

SkillSmith 将智能体的外部状态表示为三元组 \(\Sigma_t = (S_t, T_t, F_t)\)，分别对应技能库、工具库和反模式记忆。每轮迭代包含四个阶段：

- **统一反思与提案规划（§3.1）**

  - 反思器 \(R\) 接收三种辅助信号：反模式记忆（检索相似失败归因）、生态动力学（提供效用变化趋势 \(\Delta u_i\) 和交互估计 \(\hat{\beta}_{ij}\)）、结构化反馈 \(\mu_f\)。  
  - 生成**原子束提案** \(L\)：一个不可分割的包，可同时包含技能更新和工具操作（WRAP、EDIT、COMPOSE、SPLIT、RETIRE）。原子性确保相互依赖的变化同时生效，避免中间无效状态。  
  - 工具变更由**Tool-Smith** \(B_\tau\) 执行，仅支持五种受限制的生命周期原语，防止搜索空间爆炸。  
  - 提案经过反模式否决检查，若匹配已知失败模式则被拒绝并强制修正。  
  - 此外实现**协同感知合并**：将两个非支配状态中各自对技能的改进合并，并检查合并后技能对的交互系数 \(\hat{\beta}_{ij}\) 是否低于冲突阈值。

- **技能生态动力学（§3.2）**  
  - 用**Lotka-Volterra方程**建模技能库的生态系统：  
    \[
    u_{i}^{(t+1)} = u_i^{(t)} + \epsilon \hat{u}_i^{(t)} u_i^{(t)} \left(1 - \frac{\sum_j w_{ij} u_j^{(t)}}{K}\right)
    \]
    其中 \(\hat{u}_i^{(t)}\) 为标准化残差观测值，\(w_{ii}=1\)，\(w_{ij} = -\hat{\beta}_{ij}\)。  
  - 交互系数 \(\hat{\beta}_{ij}\) 从执行日志中统计：若技能 \(i, j\) 共现激活的残差增益大于各自单独激活的残差增益，则为正（互补）；反之为负（冲突）。  
  - 动态效用指导技能检索、变异优先级和淘汰：效用持续低于阈值的技能被淘汰，其记录转入反模式记忆作为“墓志铭”。  
  - 维护实例级帕累托前沿：每个训练实例保存达到最高分数的系统状态，采样时以每个状态“独有最优”的实例数加权。

- **渐进式验证与状态更新（§3.3）**  
  - 候选状态 \(\Sigma'\) 依次经过工具单元测试、端到端集成测试、回归检查。  
  - 通过后加入帕累托前沿，更新技能/工具库，执行一步生态动力学更新，存储新的失败模式到反模式记忆。

- **反模式记忆**：每个条目 \(\phi = (p, a, c)\) 包含失败特征签名 \(p\)、因果归因 \(a\)、补救措施 \(c\)。  
  - 加速诊断：反思前检索相似失败，直接将历史归因注入上下文。  
  - 提案否决：若提案的操作与已有反模式相似度超过阈值 \(\theta_{\text{veto}}\)，则被否决并要求重新生成，防止重复已知错误。

### 6. 主要结论与发现
1. **性能显著提升**：在Qwen3.5-397B上，OfficeQA准确率达到80.1%（+18.3%），SealQA准确率达到49.5%（+18.9%），且增益随模型规模单调增长。WildClawBench上SkillSmith能持续进化至第6天，而基线在第2–4天即停滞。
