## 摘要
虽然基于大语言模型的多智能体系统(LLM-MAS)通过编排专用智能体和外部工具在解决复杂任务方面展现出显著能力，但对工具输出的隐式信任却构成了关键攻击面。现有的工具攻击受限于领域特殊性或固定的静态模板。为应对这些挑战，我们提出Evo-Attacker，将工具攻击形式化为一个自进化、记忆增强的强化学习过程。Evo-Attacker构建动态攻击记忆，并采用深思熟虑的推理来检索对抗模式，在关键时刻规划修改干预。此外，我们引入Attack-Flow GRPO通过终端结果优化中间推理步骤，解决长期信用分配难题。全面实验表明，Evo-Attacker持续优于基线方法，凸显其泛化与进化能力，以及防御性工具保障的迫切需求。

## Abstract
While Large Language Model-based Multi-Agent Systems (LLM-MAS) demonstrate remarkable capabilities in solving complex tasks by orchestrating specialized agents and external tools, the implicit trust in tool outputs creates a critical attack surface. Existing tool attacks are limited by domain specificity or fixed and static templates. To address these challenges, we propose Evo-Attacker, which formulates the tool attack as a self-evolving, memory-augmented reinforcement learning process. Evo-Attacker constructs a dynamic attack memory and employs deliberative reasoning to retrieve adversarial patterns and strategize modifying interventions at critical moments. Furthermore, we introduce Attack-Flow GRPO to optimize intermediate reasoning steps via terminal outcomes, addressing the long-horizon credit assignment challenge. Comprehensive experiments demonstrate that Evo-Attacker consistently outperforms baselines, highlighting its generalization and evolutionary capabilities and the urgent need for defensive tool safeguards.

### 1. 核心问题与整体含义（研究动机和背景）

- **背景**：基于大语言模型的多智能体系统（LLM-MAS）通过编排专用智能体和外部工具（如搜索、代码执行、API）处理复杂任务。然而，系统对工具返回结果天然信任，而用户输入则受到怀疑和过滤，这导致工具通道成为关键攻击面：攻击者可以劫持网络传输或第三方服务，在工具返回中植入微小扰动，从而在绕过安全过滤器的情况下引发级联故障，导致整个系统失效。
- **现有局限**：现有工具攻击方法存在明显不足：
  - 单智能体方法（如InjecAgent、Forced Output）仅针对单智能体，忽略多智能体协作，易被下游智能体识别为上下文不一致而失败。

### 2. 方法论：核心思想、关键技术细节

- **核心思想**：将攻击形式化为一个自进化的、记忆增强的强化学习过程。攻击者维护一个动态攻击记忆库（Attack Memory），通过审慎推理（Deliberative Reasoning）从记忆库中检索相关对抗模式，结合当前上下文做出攻击决策（何时攻击、攻击哪个工具，如何修改返回值），并在决策中进行反思和调整。整个推理管道通过专门设计的 Attack-Flow GRPO 强化学习算法优化，解决长期信用分配问题，使早期决策受到最终全局失败信号的强化训练。

- **关键技术细节**：
  1. **攻击记忆生成（Attack Memory Generation）**：在初始探索阶段，攻击者零样本尝试攻击，每当一次执行轨迹导致系统失败（满足失败判断函数 J(o)=1），就将该完整轨迹封装为记忆条目 m(e) = ⟨X_ctx, T_trace⟩，其中 X_ctx 是任务上下文（图结构G、任务T、目标智能体a*），T_trace 是攻击交互序列。累积所有成功攻击经验形成动态记忆库 M_A。
  2. **记忆增强攻击（Memory-Augmented Attack）**：在每个回合 t，攻击者根据当前状态 x_t（包括当前工具调用和累积历史），执行三步推理：
     - **检索**：基于状态生成检索查询 q_t，从 M_A 中检索 top-k 最相似的记忆。
     - **反思**：评估检索到的记忆在当前情境下的可转移性和可行性，输出推理摘要 c_t 和决策 d_t（ATTACK、CONTINUE、NOOP）。若 CONTINUE，则更新上下文后重新检索；若 ATTACK，进入修改阶段；若 NOOP，则跳过该回合。
     - **修改**：选择特定工具调用索引 i_t 并生成具体修改指令 φ_{t,i_t}，应用该指令将原始返回值 r 修改为对抗值 r' 返回给目标智能体。
  3. **优化：Attack-Flow GRPO**：将攻击轨迹 ζ 视为一系列交织的攻击者动作和环境响应。定义复合奖励 R(ζ) = I(J(o_sys)=1) + λ·R_struct(ζ)，即攻击成功（系统失败）奖励加格式/预算约束惩罚。采用组相对优势（Group-Relative Advantage）：使用 G 条并行轨迹计算每个轨迹的优势 A_i，将其广播（Broadcast）到该轨迹中所有攻击者生成的 token 上。优化目标为带 KL 正则的 PPO 风格损失，仅对攻击者 token 计算梯度，环境 token 被屏蔽，从而让终端奖励信号有效传播到早期推理步骤，解决长期信用分配问题。
