## 摘要
随着组织将AI智能体投入生产部署——这些智能体执行非确定性工作流、维护有状态会话，并通常以特权访问内部服务——工程挑战从构建单个智能体转向在适当的隔离、治理和安全性下进行规模化运营。本文介绍Agyn，一个围绕智能体工作负载设计的开源平台，基于三个关键原则：基于Kubernetes的信号驱动、有状态无服务器运行时；用于智能体和工具链定义的Terraform提供程序；以及基于零信任和最小权限原则的安全模型。Agyn具有智能体无关、模型无关和云无关的特性。

## Abstract
As organizations move toward production deployments of AI agents, which execute non-deterministic workflows, maintain stateful sessions, and often operate with privileged access to internal services, the engineering challenge shifts from building individual agents to operating them at scale with proper isolation, governance, and security. In this paper we present Agyn, an open-source platform designed around three key principles tailored for agent workloads: a signal-driven, stateful serverless runtime on Kubernetes; a Terraform provider for agent and harness definition; and a security model grounded in zero-trust and least-privilege principles. Agyn is agent-agnostic, model-agnostic, and cloud-agnostic.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：组织在生产环境中部署AI智能体时面临三重挑战：工作流非确定性、有状态会话、以及对内部服务的特权访问。现有基础设施无法同时满足以下三个关键需求：  
  - **高效按需执行**：不能为每个智能体定义预分配资源，需仅在运行时分配并回收空闲资源。  
  - **智能体定义即代码（Agent as Code）**：智能体的提示词、工具、密钥、资源限制等需像基础设施一样进行版本控制、代码审查和回滚。  
  - **零信任访问内部服务**：访问必须基于身份、默认拒绝，且凭证绝不能暴露给模型。  
- **整体含义**：亟需一个集成上述三方面要求的开源平台，以支持大规模、安全、可治理的AI智能体运营。Agyn正是为此设计。

## 2. 方法论

### 核心思想
Agyn平台围绕三个核心原则构建，所有设计面向智能体工作负载，且智能体无关、模型无关、云无关。

### 关键技术细节
- **信号驱动的有状态无服务器运行时**  
  - 基于Kubernetes：每个智能体实例作为短期Pod运行，由会话消息触发创建（事件驱动），空闲后回收（默认5分钟空闲超时）。  
  - 持久化状态：每个实例挂载持久卷，状态跨冷启动保留。  
  - 可扩展性：无需预置资源，支持水平扩展。

- **智能体定义即代码（Terraform Provider）**  
  - 提供Terraform provider（`agynio/agyn`），将智能体定义及配套基础设施（MCP服务器、秘密、卷、网络身份等）声明为代码。  
  - 支持模块化复用，可打包通用工具链模式。  
  - 变更通过`terraform apply`应用，无需重启。

- **零信任与最小权限安全架构**  
  - **每容器隔离**：智能体主容器与MCP侧边容器分离，文件系统和进程隔离，秘密仅注入需要的容器。  
  - **零信任覆盖网络**：使用OpenZiti为每个智能体实例分配临时的x509身份，所有通信强制mTLS，实现基于身份的网络隔离。  
  - **关系型授权（ReBAC）**：基于OpenFGA（受Zanzibar启发），通过图遍历进行细粒度权限控制（所有者、维护者、参与者等角色）。
