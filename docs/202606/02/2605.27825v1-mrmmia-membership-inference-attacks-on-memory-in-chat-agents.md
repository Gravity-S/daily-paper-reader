## 摘要
成员推断攻击（MIAs）测试目标数据记录是否属于系统的私有数据，并已成为衡量机器学习系统中隐私泄露的标准工具。先前的工作主要关注训练语料库或检索数据库。然而，针对代理记忆的MIAs受到的关注较少，尽管这种记忆可能包含敏感的用户-代理交互、检索到的事实和用户偏好。因此，在这项工作中，我们专注于对话代理记忆MIAs，其中对手推断候选记忆单元是否属于对话代理的记忆存储。我们提出了多召回记忆MIA（MRMMIA），这是一种统一的攻击，利用对代理的多个召回探针来在黑盒、灰盒和白盒设置中提取成员信号。我们的实验表明，MRMMIA始终优于基线。我们的结果暴露了代理中的隐私风险，并为对话代理记忆系统中的成员泄漏提供了初步评估框架。

## Abstract
Membership inference attacks (MIAs) test whether a target data record belongs to a system's private data, and have become a standard tool to measure privacy leakage in machine learning systems. Prior work has primarily focused on training corpora or retrieval databases. However, MIAs against agent memory have received less attention, even though such memory can contain sensitive user-agent interactions, retrieved facts, and user preferences. Therefore, in this work, we focus on chat agent memory MIAs, where an adversary infers whether a candidate memory unit belongs to the chat agent's memory store. We propose Multi-Recall Memory MIA (MRMMIA), a unified attack that utilizes multiple recall probes to the agent to extract the membership signal across black-box, gray-box, and white-box settings. Our experiments demonstrate that MRMMIA consistently outperforms baselines. Our results expose the privacy risk in agents and provide an initial evaluation framework for membership leakage in chat-agent memory systems.

## 2. 论文提出的方法论

### 核心思想
- **Multi-Recall Memory MIA (MRMMIA)**：利用**多个召回探针（recall probes）** 向代理提问，从代理的响应中提取成员信号。其核心直觉是：若候选语句在记忆库中，代理能准确回答各项细节并提供合理来源；若不在，代理可能只能依靠推断或其他记忆回答部分问题，且回答会表现出不确定性或来源缺失。
- 覆盖三种访问设置：**黑盒**（仅观察输出文本）、**灰盒**（额外获得输出token对数概率）、**白盒**（可访问检索到的记忆单元）。

### 关键技术细节

#### 第一阶段：召回查询生成
- 使用辅助LLM（Qwen2.5-7B-Instruct）生成K个召回查询（默认K=5）。
- 设计原则：
  - **原子主题（Atomic Topic）**：每个查询聚焦候选语句的一个特定方面（如人物、地点、物品）。
  - **后续理由问题（Follow-up rationale questions）**：每个查询末尾附加“你如何知道？”等追问，迫使代理暴露信息来源。
  - **多样性（Diversity）**：覆盖候选语句的不同属性，若属性不足则通过改写填充。

#### 第二阶段：探针与评分
- **黑盒**：对每个查询，用辅助LLM评分模型`Mr`，根据响应是否支持预期关键词及理由是否与候选语句对齐，输出离散分数（1.0/0.67/0.33/0.0）。最终得分`sb = (1/K) Σ wr·Mr(x, qi, ri)`。
- **灰盒**：在响应得分基础上，加入输出token的平均对数概率`log P(ri|qi)`，乘以权重`wp`。
- **白盒**：进一步加入检索到的记忆单元与候选语句的相似度得分`max_ℓ Mm(x, miℓ)`，使用另一个辅助LLM评分模型`Mm`。得分函数为加权和。
- 最终将K个查询的得分平均，与阈值γ比较做出成员/非成员判定。

### 算法流程（文字说明）
1. 输入：目标代理F、候选语句x、查询生成器Gq、响应评分器Mr、记忆评分器Mm（白盒）、探针数K、权重wr、wp、wm、阈值γ、访问模式c。
2. Gq生成K个多样性召回查询。
3. 对每个查询qi：
   - 向代理发送qi，获得响应ri。
   - 计算响应得分si += wr·Mr(x, qi, ri)。
   - 若为灰盒/白盒，额外加入 log概率项。
   - 若为白盒，获取检索到的记忆单元，计算最大相似度得分并加入。
4. 计算平均得分s。
5. 若s ≥ γ，判为成员（1），否则为非成员（0）。

## 6. 论文的主要结论与发现
1. **MRMMIA在所有设置下持续优于基线**，尤其在低FPR（TPR@FPR1%）指标上提升显著。例如，灰盒LoCoMo数据集上，MRMMIA将TPR@FPR1%从IA的13.5%提升至55.9%（Mem0），从17.3%提升至63.4%（MemGPT）。
2. **现有LLM/RAG MIA方法在代理记忆场景下效果有限**，说明代理记忆MIA需要专门的攻击设计。
