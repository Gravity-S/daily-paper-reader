## 摘要
测试时技能演化被视为增强部署的智能体系统的新范式。现有工作主要依赖于硬编码的技能演化策略或需要下层大语言模型昂贵参数更新的参数化学习。在本文中，我们证明在不同下游场景下，对技能演化框架本身进行测试时细化对于智能体系统的持续改进是必要的，并且轻量级算法适应性是可行的。具体而言，我们提出HiSME，一种轻量级层次化技能元演化解决方案，通过从智能体的任务执行轨迹中学习元技能，联合优化技能和技能演化策略。在多种智能体基准上的实验表明，元演化能够产生比纯技能演化更高质量的技能库，并能针对不同场景推导出多样的元技能，从而促进未来的持续经验学习。我们的代码暂时公开在https://anonymous.4open.science/r/HiSME-BD45。

## Abstract
Test-time skill evolving is regarded as a new paradigm for enhancing deployed agentic systems. Existing works mainly focus on hard-coded skill evolving strategies or parametric learning that rely on expensive parameter updates in the underlying LLMs. In this paper, we demonstrate that test-time refinement of the skill evolving framework itself is necessary for continuous improvement of the agent systems in different downstream scenarios, and lightweight algorithmic adaptation is feasible. Specifically, we propose HiSME, a lightweight hierarchical skill meta-evolving solution that jointly optimizes skills and the skill evolving strategy by learning meta-skills from agents' task execution traces. Experiments on diverse agentic benchmarks show that meta-evolving can produce a higher-quality skill library than pure skill evolving and can derive diverse meta-skills for different scenarios, thereby facilitating future continual experience learning. Our code is temporarily public at https://anonymous.4open.science/r/HiSME-BD45.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有测试时技能进化（test-time skill evolving）方法中，技能演化策略本身是“静态”的——它不会在部署过程中积累证据来改进自身的提取、维护或细化过程。这导致两个实际限制：① 当技能生成算法存在系统性缺陷时，智能体系统会反复消耗维护成本修复同一缺陷模式；② 固定的演化策略无法适配不同目标领域（如数据处理任务与安全关键任务对技能的规格、范围和维护要求差异很大）。部分工作通过参数化学习构建技能演化框架，但会带来大量计算开销。
- **研究动机**：探索技能演化框架本身是否也能在测试时自动优化，实现轻量级、无需修改底层LLM参数的持续自适应改进。
- **整体含义**：提出层次化技能元演化（HiSME），不仅能构建可复用的技能库，还能使技能演化框架自身适应未来任务和领域，实现智能体系统的“更高层次”持续学习。

## 2. 方法论

### 核心思想
将技能演化建模为多级残差优化问题：
- **第0阶**：frozen executor产生轨迹的效用（trace utility）
- **第1阶**：技能演化（skill evolving）创建、修订或过滤技能，以优化轨迹效用
- **第2阶**：技能元演化（skill meta-evolving）创建、修订或过滤元技能，以优化技能效用

### 关键技术细节
1. **技能生成（Skill Generation）**：
   - **Extractor**：从单条轨迹中识别可复用片段或经验（基于人类工程师先验）
   - **Refactorer**：接收相似或相关轨迹/技能组，通过构建重叠图（Overlap Graph）——节点为轨迹片段，边权重由语义相似度和N-gram统计决定——提取团（clique），由LLM生成简洁可复用技能
   - 产出三种类型技能：可调用函数、工作流框架、领域知识

2. **技能评估与维护（Skill Evaluation & Maintenance）**：
   - **Bundle Tester**：为每个技能维护测试包（含单元测试和集成测试），检验技能是否忠实履行其契约
   - **Credit Assigner**：利用真实执行轨迹估计每个暴露的技能是有帮助、有害还是中性
   - **Refiner**：基于证据状态（测试包、信用事件、生命周期事件）修改技能，修改后需通过维护的测试包才能发布
   - **Filter**：基于有害/有益证据计数（公式14-16）标记有害技能为禁用状态

3. **元技能生成（Meta-skill Generation）**：
   - 为每个基于LLM的算法角色（EXT, RFC, RFN）维护角色特定的回放缓冲区，存储产出技能及其后期效用
   - 定期从缓冲区采样，调用META_EVO代理总结规则经验作为元技能（最多5条）
   - 元技能供人角色在后续演化中作为学到的后验知识使用
