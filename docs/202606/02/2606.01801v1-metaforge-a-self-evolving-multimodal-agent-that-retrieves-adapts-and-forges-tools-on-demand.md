## 摘要
多模态智能体通过工具使用在复杂推理任务上取得了显著进展，但仍受限于两个问题：静态预定义的工具库无法泛化到未见场景，而 indiscriminate 的工具调用会带来冗余成本和噪声引起的错误。我们提出 MetaForge，一种多模态智能体框架，它学习何时调用工具以及如何按需进化其工具集。MetaForge 将智能体行为分解为四个耦合阶段：判断（判断是否值得使用工具）、检索（选择合适工具）、适配（将工具参数接地于任务上下文中）和锻造（在线合成新技能并将其回收至工具库以供重用），形成一个判断-检索-适配-锻造-回收的闭环。统一编排策略使智能体能够在直接回答、重用现有工具或锻造新工具之间进行选择。我们通过强化学习联合优化调用必要性、检索准确性、执行效果和锻造技能的可重用性，并加入显式调用成本惩罚以抑制冗余调用。在12个基准测试中，MetaForge 在准确性、效率和泛化性上始终超越16个基线，验证了从静态工具库到按需自进化的范式转变。

## Abstract
Multimodal agents have achieved notable progress on complex reasoning tasks through tool use, yet remain limited by two issues: statically predefined tool inventories fail to generalize to unseen scenarios, and indiscriminate tool invocation incurs redundant cost and noise-induced errors. We propose MetaForge, a multimodal agent framework that learns when to invoke tools and how to evolve its toolset on demand. MetaForge factorizes agentic behavior into four coupled stages: Decide (judging whether tool use is warranted), Retrieve (selecting suitable tools), Adapt (grounding tool parameters in task context), and Forge (synthesizing new skills online and recycling them into the tool library for reuse), forming a closed judge-retrieve-adapt-forge-recycle loop. A unified orchestration policy enables the agent to choose among answering directly, reusing existing tools, or forging new ones. We jointly optimize invocation necessity, retrieval accuracy, execution effectiveness, and forged-skill reusability via reinforcement learning, with an explicit invocation-cost penalty discouraging redundant calls. Across 12 benchmarks, MetaForge consistently surpasses 16 baselines in accuracy, efficiency, and generalization, validating a paradigm shift from static tool inventories to on-demand self-evolution.

## 1. 核心问题与研究动机
- **问题背景**：多模态智能体通过调用外部工具（如OCR、图像处理模块）在复杂推理任务上取得了进展，但现有方法受限于两大核心缺陷：
    - **静态预定义工具库**：工具集在部署前固定，无法泛化到未见过的任务场景。
    - **不加区分的工具调用**：模型在不需要工具时也盲目调用，导致冗余计算成本和噪声引发的错误。
- **整体目标**：设计一个能够**按需自进化**的工具编排框架，使智能体学会判断何时需要工具、如何灵活调用已有工具，以及在能力缺口出现时动态锻造新工具并纳入工具库，从而实现从“静态工具库”到“按需自演化”的范式转变。

## 2. 方法论核心思想与关键技术细节
- **核心框架：MetaForge**，将智能体行为分解为四个耦合阶段，形成闭合循环：
    - **Decide（判断）**：判断当前任务是否需要使用外部工具，还是可以直接回答。
    - **Retrieve（检索）**：若需要工具，从动态工具库中检索合适的工具。
    - **Adapt（适配）**：根据任务上下文调整工具调用参数（如指定图像索引、提示词等）。
    - **Forge（锻造）**：当现有工具均无法满足需求时，在线合成新的可执行技能，通过验证后注册回工具库供未来复用。
- **关键设计**：
    - 工具池分为三类：**可执行工具**（如OCR、代码解释器）、**模型工具**（如专用视觉模型）、**锻造工具**（按需生成的Python脚本技能）。
    - 锻造过程分两阶段：**规划阶段**（生成结构化规范：名称、描述、参数接口、脚本列表）→ **实现阶段**（为每个脚本生成代码）。锻造后的工具需通过编译/执行验证和语义验证双重门控。
    - 训练使用**多轮GRPO**（Group Relative Policy Optimization），设计**复合奖励函数**：
        - **Retrieve/Adapt Reward**：答案正确性的二进制信号。
        - **Decide Reward**：在正确轨迹内，根据总工具调用延迟给出效率排名，鼓励简洁调用。
        - **Forge Reward**：在正确轨迹内，奖励成功锻造工具和成功复用已注册锻造工具。
        - **Format Reward**：强制结构化输出格式（合规+0.2，不合规-0.2）。
    - 引入**显式调用成本惩罚**（通过Decide Reward中的延迟归一化）抑制冗余调用。
    - **容量受限的工具池管理**：通过语义去重（合并功能重叠的锻造工具）和基于使用频率的保留策略（优先保留高频使用或新锻造的工具），控制工具池规模，避免上下文窗口膨胀。

## 6. 主要结论与发现
- **主要结论**：MetaForge 在 12 个基准的 IID 和 OOD 设置下均取得最佳平均准确率（IID: 88.14%，OOD: 88.98%），显著优于所有基线。综合收益来自：（1）判断是否调用工具的机制避免了冗余；（2）统一工具接口使模型能灵活调度未见过的工具；（3）在线锻造补足了能力缺口；（4）GRPO 复合奖励同步优化正确性、效率和工具复用。
- **具体发现**：
    - 固定工具使用策略（如 R1-Onevision-7B）当工具变化时表现下降，甚至低于基座模型，说明缺乏自适应能力。
    - 工具互补性重要：QwenVL 受益于 OOD 工具（包含 InternVL 等互补模型），InternVL 受益于 IID 工具（包含 QwenVL），交叉系列模型提供互补增益。
    - 锻造行为分析：基线和现有方法 FSR 低（20~55%），MetaForge 达到 64.7%（IID）和 91.7%（OOD），说明训练后模型能更可靠地锻造并注册有效工具。
    - 在 IID 工具设置下，模型几乎不在 IID 数据集上锻造工具，但在 OOD 数据集（如 WebSight）上会积极锻造；在 OOD 工具设置下，因工具覆盖更广，几乎不锻造。说明模型学会了按需锻造，而非盲目创建。
    - 消融实验表明，所有奖励组件均有用：移除 Format 奖励导致最大下降（工具调用语法崩溃），移除 Forge 奖励主要影响 OOD 任务，移除 Decide 奖励导致冗余调用和错误增加。

## 7. 优点
- **方法创新**：首次将工具使用建模为“判断-检索-适配-锻造-回收”闭环，实现了在线技能生成和工具库动态扩展，突破了静态工具集的限制。
- **训练范式**：使用多轮 GRPO 和复合奖励，同时优化正确性、效率、工具复用，且引入成本惩罚鼓励简洁调用。
- **泛化能力**：在 IID/OOD 数据和工具设置下均表现出色，骨干泛化实验证明方法不依赖特定模型架构或规模。
- **实验全面**：覆盖 12 个基准、16 个基线、双设置、消融、超参数扫描、错误分析、案例研究，提供了充分的证据支持。
- **效率可控**：通过 Decide Reward 和容量管理，在保持高准确率的同时，将平均交互轮次控制在 2.3 左右，推理成本较低。
- **部署友好**：锻造工具验证注册机制确保了新工具的质量，减少了运行时错误。

## 8. 不足与局限
- **工具形式限制**：当前锻造工具仅支持可执行脚本（Markdown 规范 + JSON schema + Python 脚本），适用于程序化任务（代码处理、图表分析、数值计算）。对于需要新模型或外部资源（如视频合成）的任务，仅靠当前锻造机制难以解决。未来需探索更强的锻造模型、更丰富的技能形式（例如调用外部API或新模型）。
- **评估范围局限**：实验集中于图像-文本任务，未覆盖音频、视频、时间推理或交互式环境（如机器人、Web 浏览）。跨模态泛化能力尚待验证。
- **效率目标单一**：目前效率优化只关注延迟，未考虑货币成本、上下文token消耗等其他资源约束。实际部署中可能需要平衡多个成本维度。
- **工具池容量管理**：虽然设计了去重和保留策略，但容量上限 K=50 的设置可能不足以应对长期积累；当任务种类极多时，如何自动判断工具冗余与优先级仍需进一步研究。
- **语言与领域偏向**：训练和评估主要为英文数据集，中文或其他语言场景未验证，可能存在语言和文化的偏向。
