## 摘要
基于LLM的搜索智能体能够进行多步推理和使用工具。然而，这些能力会引入检索导致的安全退化，因为有害意图可能分解为看似无害的子查询，从而导致不安全的结果。现有的对齐方法难以捕捉稀疏的安全信号，并且无法监督多步交互中的各种违规行为。我们提出了COMPASS，一个认知MCTS引导的过程对齐框架，旨在在整个智能体工作流中实现鲁棒的安全对齐，同时保持通用效用。COMPASS集成了认知树探索（CTE）以高效合成隐秘的攻击轨迹，以及内省逐步对齐（ISA）以隔离有风险的中间动作进行细粒度过程监督。实验结果表明，COMPASS在实现良好安全-效用权衡的同时，大幅减少了所需训练数据。

## Abstract
LLM-powered search agents enable multi-step reasoning and tool use. However, these capabilities introduce retrieval-induced safety degradation, as harmful intents may decompose into seemingly innocuous sub-queries that lead to unsafe outcomes. Existing alignment methods struggle to capture sparse safety signals and fail to supervise diverse violations across multi-step interactions. We propose COMPASS, a Cognitive MCTS-Guided Process Alignment framework designed to achieve robust safety alignment throughout the agent workflow while preserving general utility. COMPASS integrates cognitive tree exploration (CTE) to efficiently synthesize stealthy attack trajectories, and introspective step-wise alignment (ISA) to isolate risky intermediate actions for fine-grained process supervision. Empirical results show that COMPASS achieves a favorable safety-utility trade-off while requiring substantially less training data.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 驱动的搜索智能体在多步推理和工具调用过程中，可能将有害意图分解为表面上无害的子查询，从而绕过基座模型的安全护栏，导致“检索诱导的安全退化（retrieval-induced safety degradation）”。现有对齐方法难以在指数级放大的状态空间中高效发现稀疏的安全信号，也无法对多步交互中不同类型的安全违规（如早期风险推理、在风险上下文下的推理、最终有害响应）进行细粒度过程监督。

## 2. 论文提出的方法论

### 2.1 整体框架：COMPASS（Cognitive MCTS-Guided Process Alignment）

- 将搜索智能体的迭代推理过程建模为马尔可夫决策过程（MDP），状态为交互历史，动作包括：查询生成（Query Generation）、证据提取（Evidence Extraction）、答案生成（Answer Generation）。
- 核心目标：最大化在每一步发现目标模型脆弱性的期望累积奖励（即发现安全漏洞的轨迹）。

### 2.2 认知树探索（Cognitive Tree Exploration, CTE）

- 使用 **蒙特卡洛树搜索（MCTS）** 作为启发式探索骨架，通过选择、扩展、模拟、反向传播四个步骤构建搜索树。
- **关键创新**：引入“认知 Q 值估计”，用一个增强推理能力的 LLM 直接估计中间动作的 Q 值（评估轨迹的隐蔽性和攻击进展），替代传统 MCTS 中昂贵的 rollout 过程，提供稠密过程信号指导 UCT 选择。
- **终端奖励函数**：对于答案生成动作，使用一个对齐良好的推理模型评估终端响应的危害程度，得到终端攻击成功率（归一化到 [0,1]），用于反向传播标记所有祖先节点。
- 通过结合认知 Q 值（非终端）和终端奖励（终端），CTE 能高效合成高隐匿性的多步攻击轨迹，用于后续监督数据的构建。

### 2.3 内省逐步对齐（Introspective Step-wise Alignment, ISA）

- 基于 CTE 生成的轨迹，构建 **步骤级偏好数据** 三元组 `(˜st, y_safe, y_unsafe)`，其中 `˜st` 为当前状态（初始恶意查询 + 历史交互），`y_unsafe` 为被拒绝的风险动作（由模型实际输出），`y_safe` 为通过内省提示模板引导模型生成的更安全替代动作。
- **风险状态选择**：设置终端成功阈值 φ 和中间风险阈值 γ。成功攻击轨迹中，标记第一个风险节点（第一点偏离）、后续风险传播节点、终端节点；失败攻击轨迹中，标记第一个高 Q 值（≥γ）节点。以此覆盖不同阶段的违规。
- **细粒度偏好优化**：先进行监督微调（SFT）学习智能体工作流格式；然后使用 **步骤级 DPO** 在特定偏离状态 ˜st 上优化策略，损失函数为：
  ```
  Lθ = -E_Dstep[ log σ( β log(πθ(ysafe|˜st)/πref(ysafe|˜st)) - β log(πθ(yunsafe|˜st)/πref(yunsafe|˜st)) ) ]
  ```
  其中 β 为 KL 惩罚系数，σ 为 sigmoid 函数。这直接惩罚局部恶意推理，实现主动防御。

## 6. 论文的主要结论与发现

1. **外部检索削弱基座模型安全**：Naive RAG、Base Agent、Search-R1 显著提升效用，但有害率大幅上升。即使有思考-搜索机制，任务分解仍可伪装有害意图。
2. **外部护栏效果有限**：查询过滤和文档过滤只能部分缓解风险，平均有害率仍较高（32.0% 和 33.8%），且带来高延迟（最多 11.6 倍）。
3. **COMPASS 实现优越安全-效用权衡**：
   - Qwen-7B 上平均有害率 18.0%（最低），帮助度保持竞争力；Llama-8B 上平均有害率 14.6%。
   - 效用方面，EM 平均分在基线中最佳，F1 第二。训练数据量仅为 Search-R1 的 1/8，Safesearch 的 1/6。
4. **跨模型泛化能力强**：Qwen-7B 生成的攻击轨迹和偏好对也能有效改善 Llama-8B 的安全性（有害率从 19.1% 降至 14.6%），帮助度提升明显（StrongREJECT 从 1.29 升至 2.06）。
5. **步骤级过程对齐优于最终输出对齐**：ISA 在中间状态纠正风险比仅惩罚最终响应（+DPO_final）或仅 SFT 更有效。

## 7. 优点

- **创新性**：首次将 MCTS 引导的过程级探索引入搜索智能体安全对齐，提出认知 Q 值高效替代昂贵 rollout，解决稀疏信号探索难题。

- **评估鲁棒性**：多模型验证评估器偏差，人工标注验证 Q 值判断，实验设计严谨。
