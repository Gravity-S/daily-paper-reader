## 摘要
网络上丰富的程序性知识对于帮助智能体解决长期任务具有巨大潜力。然而，这类知识通常是多模态、异质、有噪声的，并且隐含地假设了人类执行者，因此难以直接用作智能体所需的技能。为了弥合面向人类的指南与智能体可执行技能之间的差距，我们将此问题形式化为指南到技能学习：将现实中的指南转化为可执行技能，并从智能体可观测到的轨迹中持续改进它们。为了评估现有智能体在此任务上的能力，我们推出了MMG2Skill-Bench，这是针对此问题的首个基准测试。我们进一步提出了MMG2Skill，一个闭环框架，它将指南编译为可编辑的技能，在执行过程中基于这些技能条件固定视觉语言模型（VLM）智能体，并从轨迹级根因反馈中修改技能，而不使用基准分数。在GUI控制、开放式游戏和策略卡牌游戏中，使用六个VLM骨干网络，MMG2Skill在每个模型领域设置中均优于普通基线智能体，跨骨干网络实现了宏平均增益12.8到25.3个百分点。消融研究表明，直接用原始指南提示智能体会降低性能，而结构化的技能构建和轨迹驱动的修订对于观察到的改进是必要的。在可推断成功的任务中，基于分析器的提前停止进一步防止了后期性能下降，并在成功信号适当校准的情况下节省了25%-53%的尝试次数。

## Abstract
Abundant procedural knowledge on the Web holds great potential for helping agents solve long-horizon tasks. However, such knowledge is often multimodal, heterogeneous, noisy, and implicitly assumes human executors, making it difficult to use directly as the skills required by agents. To bridge the gap between human-oriented guides and agent-executable skills, we formalize this problem as guide-to-skill learning: converting in-the-wild guides into executable skills and continuously improving them from trajectories observable to the agent. To evaluate the capability of existing agents on this task, we introduce MMG2Skill-Bench, the first benchmark designed for this problem. We further propose MMG2Skill, a closed-loop framework that compiles guides into editable skills, conditions a fixed vision-language model (VLM) agent on these skills during execution, and revises the skills from trajectory-level root-cause feedback without using benchmark scores. Across GUI control, open-ended gameplay, and strategic card play with six VLM backbones, MMG2Skill consistently outperforms vanilla baseline agents in every model-domain setting, achieving macro-average gains of +12.8 to +25.3 percentage points across backbones. Ablation studies show that directly prompting agents with raw guides can degrade performance, while both structured skill construction and trajectory-driven revision are necessary for the observed improvements. On success-inferable tasks, analyzer-based early stopping further prevents late-stage performance regressions and saves 25%-53% of attempts when the success signal is properly calibrated.

- **研究动机**：网络上存在大量面向人类的多模态过程性知识（教程、指南、wiki等），它们包含丰富的程序性知识，有助于智能体解决长期任务。但这些指南是异构、有噪声的，且隐式假设人类执行者，无法直接作为VLM智能体可执行、可编辑的技能。
- **核心问题**：能否将现实中的多模态指南提炼为可执行且能通过智能体自身轨迹不断进化的技能？作者将其形式化为“指南到技能学习”（guide-to-skill learning）问题。
- **整体含义**：弥合人类指南与智能体执行技能之间的鸿沟，使智能体能够持续利用公共网络知识自我改进，而不依赖人工标注或外部奖励信号，推动更自主、可扩展的智能体系统。

### 2. 方法论

**核心思想**：MMG2Skill是一个闭环框架，它（1）从多模态指南中提取结构化可编辑技能，（2）在固定VLM智能体执行时注入当前技能集，（3）从智能体可见的轨迹中生成诊断，进而修订技能，整个过程不依赖基准分数。

**关键技术细节**：

- **技能构建（Construct Skills）**：将指南（HTML+图片）通过VLM一次性转换为结构化SKILL.md文件，每个技能包含步骤（u）、适用条件（c）、预期状态视觉线索（v）、恢复知识（q）。技能数量由VLM自动决定。
- **技能条件化执行（Skill-conditioned Execution）**：在每次尝试开始时，将当前技能集Sk注入智能体上下文，智能体在任务指令I、近期历史ht和技能集Sk的条件下采样动作。
- **分析器（Analyzer）**：尝试结束后，分析器读取任务指令和完整的智能体可见轨迹（无基准分数、无隐藏状态），输出根因诊断ρk = (ek, rk)，其中ek描述工作/失败内容，rk为自评结果（likely_success / uncertain / likely_failure）。长轨迹通过分块读取（chunked analysis）保持上下文预算。
- **精炼器（Refiner）**：基于原始指南、当前技能和所有历史诊断ρ1:k，精炼器原地编辑技能文件，修正失败步骤、强化成功行为、避免撤销之前的修复。编辑强度由分析器输出值控制。

**算法流程**（Algorithm 1）：
1. S1 ← CONSTRUCT_SKILLS(G, I)
2. for k=1 to N do
3.   τk ← EXECUTE(I, Sk)
4.   ρk = (ek, rk) ← ANALYZE(I, τk)
5.   if rk = likely_success then return Sk, τk
6.   Sk+1 ← REFINE(G, I, Sk, ρ1:k)
7. end for
8. return SN, τN

### 6. 论文的主要结论与发现

1. **公开指南包含有用过程知识，但原始注入不可靠**：Raw Guide在GUI上与Vanilla持平，在Game和Strategy上反而降低性能（-1.67 pp和-1.53 pp）。
2. **结构化技能提取提供更安全过程先验**：仅技能构建（无修订）即超过Vanilla，在GUI上提升最显著（+8.34 pp），在Game和Strategy上接近持平。
3. **轨迹驱动修订修复指南-运行时偏差**：完整MMG2Skill在Game和Strategy上贡献了大部分增益（+22.22 pp和+13.75 pp相对无修订），在GUI上贡献较小但仍为正。
4. **增益是跨模型、跨领域一致的**：所有18个模型-领域组合均获得正提升，宏平均+12.8~+25.3 pp。
5. **修订增益非单调，早期停止优于始终取最后一次尝试**：在Game和Strategy上，早期停止分数显著高于全运行分数；早期停止也节省25%-53%的尝试和24%-42%的API调用。
6. **成功可推断任务中分析器信号可靠**：likely_success精度74%-87%，召回率71%-99%；但在私人信息任务（德州扑克）上精度仅为9.9%。
