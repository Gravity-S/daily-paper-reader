## 摘要
基于LLM的智能体越来越多地部署在高风险场景中，如电子邮件管理、金融交易和代码执行，它们通过工具调用与外部世界交互。在执行过程中，这些智能体必须读取攻击者可控制的外部数据源（电子邮件、网页、文件）；通过间接提示注入，攻击者将恶意指令嵌入这些数据中，操纵智能体执行未经授权的操作，例如将资金转移到攻击者控制的账户。现有防御要么执行工具调用级的值检查而不追踪参数值的来源，要么从单一角度分析执行轨迹，没有干净的授权基线进行比较。我们提出AuthGraph，一种双图对齐防御框架，构建两个互补图：一个注入推理图，从实际执行轨迹（包括可能被操纵的属性）中建模信息溯源；一个授权图，从用户意图在隔离的干净上下文中推导，该上下文在信息论上不可能受到注入影响；然后通过图对齐检查器对两个图进行结构比较，检测工具级和参数源级偏差。在AgentDojo上，AuthGraph将攻击成功率从40%降至1%，同时在GPT-4o上保持76%的任务完成率；在AgentDyn上，它将攻击成功率从39%降至2%，同时保留51%的效用，优于包括CaMeL、DRIFT和Progent在内的最先进防御。据我们所知，AuthGraph是首个在参数源级别上对授权规范与执行溯源进行结构比较的智能体安全防御方法，在不牺牲智能体灵活性的情况下实现了细粒度的注入检测。

## Abstract
LLM-based agents are increasingly deployed in high-stakes scenarios such as email management, financial transactions, and code execution, where they interact with the external world through tool calling. During execution, these agents must read external data sources (emails, webpages, files) that attackers can control; through indirect prompt injection, attackers embed malicious instructions in this data to manipulate agents into performing unauthorized operations such as transferring funds to attacker-controlled accounts. Existing defenses either perform tool-call-level value checking without tracking where parameter values originate, or analyze execution traces from a single perspective without a clean authorization baseline for comparison. We propose AuthGraph, a dual-graph alignment defense framework that constructs two complementary graphs: an injected reasoning graph that models information provenance from the actual execution trajectory (including potentially manipulated attributions), and an authorization graph derived from the user's intent in an isolated clean context that is information-theoretically impossible to be influenced by injection; a graph alignment checker then structurally compares the two graphs to detect both tool-level and parameter-source-level deviations. On AgentDojo, AuthGraph reduces the attack success rate from 40% to 1% while maintaining 76% task completion rate on GPT-4o; on AgentDyn, it reduces the attack success rate from 39% to 2% while preserving 51% utility, outperforming state-of-the-art defenses including CaMeL, DRIFT, and Progent. To our knowledge, AuthGraph is the first agent security defense to structurally compare authorization specifications against execution provenance at the parameter-source level, achieving fine-grained injection detection without sacrificing agent flexibility.

- **整体含义**：本文提出一种全新的双图对齐防御框架AuthGraph，通过结构比较“智能体实际做了什么”与“智能体应该做什么”来检测注入攻击，能够在参数来源级别实现细粒度检测，同时保持智能体的灵活性。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 构建两个互补图：
  - **注入推理图（IRG）**：从智能体实际执行轨迹中建模信息溯源，**故意暴露于注入内容**，忠实记录智能体的“主观视角”（包括被操纵的属性）。
  - **授权图（AG）**：在隔离的干净上下文中，仅使用用户提示和工具目录自动推导，**信息论上不可能受到注入影响**。
- 通过**图对齐检查器**对两个图进行结构比较，检测工具级和参数源级偏差。

### 关键技术细节
1. **注入推理图构建**：
   - 节点类型：input（用户指令，可信）、observation（工具返回值，不可信）、decision（工具选择与参数值）、intermediate（推理产物）。
   - 边类型：DERIVE（推理/转换）、EXTRACT（从observation提取数据）、INVOKE（调用工具，决策节点指向观察节点）。
   - 构建策略：多轮交互式构造（生成/扩展/细化），使用LLM（GPT-4o）作为图构建器。

2. **授权图构建**：
   - 使用Planner Agent（GPT-4o-mini）在无轨迹数据的干净上下文中生成**最小期望工具序列**和**参数溯源策略（ParamPolicy）**。
   - **ParamPolicy**为每个安全关键参数声明合法来源：`user_prompt`、`observation_direct`、`observation_nl`、`any`，并指定允许的源工具列表（source_tools），防止跨工具污染。
   - **动态授权（Replan）**：对于依赖运行时观察的任务（如“回复紧急邮件”），允许在运行时扩展授权，但受限允许工具白名单，遵循最小权限原则。

3. **图对齐检查器**：
   - 利用INVOKE结构签名从IRG中提取有序工具调用列表，与授权图序列进行线性对齐（双指针遍历，复杂度O(n)）。
   - **三层检测**：
     - 第一层（硬阻断）：当授权步骤耗尽但智能体继续调用不在授权集合S中的工具时，直接程序化阻断。
     - 第二层（工具名检查）：LLM判断工具调用是合法辅助步骤、跳过步骤还是注入驱动。
     - 第三层（参数源检查）：核实参数值是否来自声明的源工具。对于`observation_direct`，先进行字符串匹配，未命中则回退到LLM判断。匹配语料使用原始观察文本（而非图构建器的摘要），避免构建器被操纵后削弱检测。

## 7. 优点

1. **创新性**：首次在参数来源级别对授权规范与执行溯源进行结构比较，实现细粒度注入检测。
2. **双图架构**：分离“实际做了什么”和“应该做什么”，利用干净上下文生成不可伪造的授权基线，从根本上克服单图分析易被操纵的问题。
3. **三层检测**：从硬阻断（零LLM开销）到工具名检查到参数源检查，逐步深入，兼顾效率与准确率。
4. **动态授权（Replan）**：允许运行时扩展授权，限制最小权限，适应复杂任务。
5. **参数溯源策略（ParamPolicy）**：细粒度控制每个安全关键参数的合法来源，可检测跨工具污染等高级攻击。
6. **实用性强**：模型无关，可搭配不同LLM；开销可接受；实验验证充分。

## 8. 不足与局限

1. **同观察污染**：若攻击者直接污染权威工具的数据源（如篡改航班搜索后端），ParamPolicy无法检测。
2. **图构建器归因准确性**：注入推理图依赖于LLM的归因能力，归因错误可能导致误报或漏报。
3. **多智能体场景未覆盖**：当前设计仅针对单智能体执行，多智能体系统中跨智能体信息流追踪超出范围。
4. **重规划信任边界**：当用户授权动态行为时，重规划机制打开了一个受控信任边界，但复杂攻击者可能利用白名单工具进行有害操作。
