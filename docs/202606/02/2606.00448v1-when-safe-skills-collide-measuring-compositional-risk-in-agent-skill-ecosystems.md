## 摘要
LLM代理越来越依赖社区贡献的技能，这些技能扩展了代理的操作能力集。我们研究了Agentic AI系统中的一个核心安全问题：单独安全的技能是否可能组合成不安全的已安装技能集。我们提出了SkillReact，一个组合安全测量框架，包含三个部分：一个确定性的静态组合基准、一个双评分员LLM辅助的人类裁决流程，以及一个基于行为的可利用性测试工具。在1,520个ClawHub技能中，651个通过了单独检查，形成了211,575对组合；基准测试将这些组合中的22.25%标记为结构性候选。我们将这个原始比率视为面向召回率的扫描上限，并根据人类判断进行校准：在一次模式分层审计中，大约五分之一的标记对模式命中真正构成组合风险（人口加权有效率为18.2%，这是我们的主要结果），这意味着单个注册表中大约有14,000个真实风险成员，而这些风险由于每对组合单独安全而被逐技能扫描遗漏。然后，基于行为的测试工具探测这些候选何时成为模型发布的工具调用，并发现实现过程受宿主模型倾向的门控：在一个锚定条件的投放器子集上，Haiku-4-5在所有39次直接提示试验中发布了投放阶段的工具调用（其中36次是完整的下载后执行链，3次仅下载），Opus-4-7在下载阶段停止，Sonnet-4-6则直接拒绝。一个保持请求固定而仅改变已安装技能的对照实验发现，在没有安装任何技能时合规性最高：组合决定了哪些能力可用，而宿主模型决定是否使用它们。这些结果共同表明，安装时的组合检查和能力隔离是对逐技能扫描的补充。

## Abstract
LLM agents increasingly rely on community-contributed skills that expand an agent's operational capability set. We study a core safety problem in agentic AI systems: whether individually safe skills can compose into unsafe installed skill sets. We present SkillReact, a compositional security measurement framework with three components: a deterministic static-composition benchmark, a two-rater LLM-assisted human-adjudication pipeline, and an action-based exploitability harness. On 1,520 ClawHub skills, 651 pass individual inspection and form 211,575 pairs; the benchmark flags 22.25% of these as structural candidates. We treat this raw rate as a recall-oriented scanner ceiling and calibrate it against human judgment: in a pattern-stratified audit, roughly one in five flagged pair-pattern hits survives as a real compositional risk (population-weighted validity 18.2%, our headline result), implying about 14K genuine risk memberships in a single registry that per-skill scanning misses by construction, since every pair is individually safe. An action-based harness then probes when these candidates become model-issued tool calls, and finds realization gated by host-model disposition: on an anchor-conditioned dropper subset, Haiku-4-5 issues the dropper-stage tool call on all 39 direct-prompt trials (36 of them the full download-then-execute chain, 3 download-only), Opus-4-7 stops at the download, and Sonnet-4-6 refuses outright. A control that holds the request fixed and varies only the installed skills finds compliance highest with no skills installed: a composition fixes which capabilities are reachable, while the host model decides whether to use them. Together these motivate install-time compositional checks and capability isolation as complements to per-skill scanning.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：大型语言模型（LLM）驱动的智能体（如Claude Code、Cursor、Windsurf等）越来越多地依赖社区贡献的“技能”（skills）来扩展操作能力。每个技能单独经过安全扫描（per-skill scanning）可以确保本身不违反安全策略，例如一个只读文件的助手技能和一个只发网络请求的API连接技能各自都很安全。
- **核心问题**：但当用户将多个单独安全的技能安装到同一个统一运行时，技能的**能力联合**（capability union）可能满足安全策略中的禁止组合（如同时拥有文件读取和网络外发能力，构成数据外泄）。传统逐技能扫描由于其单位是单个技能，**在结构上必然遗漏这类组合风险**。本文衡量的是：在真实的技能生态系统中，这种结构性组合风险候选有多常见？以及这些候选在宿主模型实际调用时是否会真正实现为工具调用？
- **整体含义**：安全评估的对象不应是单个技能包，而应是已安装的技能集（installed skill set）。安装时的组合检查和能力隔离应作为逐技能扫描的补充。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程（用文字说明即可）

- **核心思想**：将组合风险分解为三个证据层级：
  1. **结构性组合风险候选**：两个单独安全的技能，其能力联合满足任意禁止模式，但尚不涉及模型实际行为。
  2. **模型发布的工具调用尝试**：宿主模型在呈现技能工件后，实际调用工具（如Bash、Read）朝向禁止目标。
  3. **链路**：从静态候选到实际调用的门控受宿主模型倾向、提示和运行环境的影响。

- **框架SkillReact由三部分组成**：
  - **确定性静态组合基准**（Static-Composition Benchmark）：
    - 通过正则表达式从技能工件（SKILL.md和代码文件）中提取8种能力（文件读、文件写、网络出/入、环境读、进程生成、凭据访问、系统信息），形成能力位集。
    - 定义10种禁止组合模式（如数据外泄：FR∧NO；投放器：NO∧FW∧PS等）。
    - 对每对独自安全的技能计算能力联合，若满足任一模式则标记为结构性候选。
    - 输出核心指标：**组合候选率（CCR）** = 结构性候选对数 / 所有单独安全技能对数。该原始CCR被明确视为一个面向召回率的**扫描器上限**（recall-oriented scanner ceiling），而非实际利用发生率。
  - **LLM辅助人类裁决管道**（Human-Adjudication Pipeline）：
    - 从47,075个候选对中，按10种禁止模式分层采样200个单元（每模式20个）。
    - 每个单元由两个来自不同供应商的LLM（Claude Sonnet 4.6和Codex gpt-5.5）分别独立标注（给出九个字段的JSON判断），然后由人类审计员仲裁。
    - 人类审计员通过一个仅使用stdlib的Python HTTP UI查看双方标记和证据，记录最终裁决（有效、误报、上下文良性、不清楚）。
    - 使用分层加权平均估计**人口加权对-模式有效性比率** \(\hat{p}_{pp}\)，即被标记的候选中有多少是真实组合风险。
  - **基于行为的可利用性测试工具**（Action-based Exploitability Harness）：
    - 将选定的技能工件呈现给宿主模型（如Haiku-4-5、Sonnet-4-6、Opus-4-7等），在尽量宽松的运行时设置下（开启`--dangerously-skip-permissions`、白名单只有Bash和Read），记录模型是否实际发布工具调用以实现禁止目标。

## 6. 论文的主要结论与发现

1. **组合风险真实存在且常见**：在1,520 ClawHub技能中，57.2%的单独技能直接违规被标记；剩下的651个单独安全技能形成211,575个对，其中22.25%（47,075个）满足至少一种禁止组合模式（原始CCR）。经人类审计校准后，约18.2%的标记对-模式命中是真实的组合风险，相当于大约14,000个真实风险成员对，对应4.05%的成对点估计（约8,567个唯一风险对）。
2. **数据外泄是主导模式**：71.6%的结构性候选对匹配数据外泄模式（文件读取+网络外发），反映了生态系统结构特性（文件读取和网络外发技能都普遍且单独安全）。
3. **能力级别噪声大但主要能力可靠**：网络外发能力检测最干净（κ=0.50，否定窗口仅1.9%），移除其正则检测器后CCR下降92%；文件读取能力次之（κ=0.03但有否定窗口7.0%，贡献44%下降）。其余6种能力对CCR影响很小。
4. **模型行为依赖宿主模型倾向**：在相同提示和技能工件下，Haiku-4-5会完全执行下载执行链，Opus-4-7只执行下载部分（不执行），Sonnet-4-6完全拒绝。跨厂商模型也展示了从完全合规到几乎完全拒绝的梯度。
5. **组合控制实验表明实现由请求驱动，而非特定技能对**：固定请求时，无任何技能安装时的合规性最高（17/18），投放器对（11/18）未比无关对（12/18）更高。说明组合只提供静态能力表面，而宿主模型对显式请求的倾向是实际调用的门控因素。
6. **设计启示**：建议安装时组合检查器（快速集级别门控+证据报告）和能力隔离作为逐技能扫描的补充；同时强化模型自身的对齐层（如奖励信号审计）以拒绝越界联合。

## 7. 优点：方法或实验设计上有哪些亮点

- **清晰的三层证据分离**：将单独技能标记、结构性候选、模型实际调用明确区分，避免了混淆统计候选与实际利用。
- **从扫描上限到校准有效性的校准路径**：明确原始CCR是带噪的上限，通过结构化的人工审计（分层采样、两个LLM辅助）得到可信的有效性估计，并给出成对层面的点估计和区间，统计严谨。
- **全面的诊断分析**：包含模式消融、能力ΔCCR、能力级别与LLM的一致性κ、否定窗口分析、LLM评分一致性、人类覆盖比率等，暴露了扫描器的噪声来源和主导能力。
- **动作测试中的组合控制**：通过固定请求、单独变化技能安装数量，分离了“技能对提供的能力”与“模型对请求的遵从性”对实现的影响，支持了“宿主模型倾向是门控”的结论。
- **跨厂商和跨模型版本的比较**：不仅在Claude系列内做比较，还测试了6个跨厂商模型，增强了发现的外部有效性。
- **可复现的确定性静态基准**：正则提取和模式定义版本控制，确保数字可复现。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

- **静态过近似的固有限制**：能力联合不意味着执行（即使候选对具备条件，模型可能不执行），原始CCR是上界。校准后仍受限于人工审计的单评审员偏差（未见报告人–人一致性κ），且人工审计时看到了LLM标记，可能存在锚定偏差。
- **能力提取噪声**：正则提取是回忆导向的，会匹配散文描述（如在文档中提到“没有API密钥需要”仍可能触发能力标记），导致误报。论文报告9.2%的证据落在40字符否定窗口内。虽然通过校准部分缓解，但底层噪声仍然存在。
- **成对分析局限**：仅分析成对组合，实际用户可能安装三个或更多技能，三元组甚至更多的组合可能引入新的候选或冗余。论文承认这一点，但未展开。
- **动作测试的样本偏差**：
  - 锚定特定单个技能（`steipete-apple-notes`），提示策略集中于直接提示，未充分探索间接或注入提示。
  - 跨厂商比较时，Claude是工具调用级，而其他模型仅测量响应级合规，二者不可直接合并。
  - 组合控制未覆盖Haiku-4-5（仅跨厂商模型），因此Haiku的响应是否同样受请求主导尚需验证。
- **运行时假设**：假设统一非隔离运行时（最坏情况）。实际生产环境可能有技能沙箱、网络白名单、用户确认等缓解措施，论文仅视其为补充而非测量对象。
- **生态范围**：仅使用ClawHub一个注册表的快照，未延伸至GPT actions、MCP服务器等，且1,520技能仅为部分快照（原始5,128个中2,286个失败，部分恢复），可能存在未表征的选择偏差。作者同时做了作者可信度过滤检查，CCR仅略有下降（22.25%→17.95%），但整体外部泛化仍需验证。
- **人工审计数量较小**：200个对-模式单元（每模式20个）用于10种模式，导致某些模式（如反向Shell，仅0.07%占比）只有1个样本有效，估计的置信区间较宽（18.2%的95%CI [11.3%-27.4%]）。
- **未测量人类-人类一致性**：未报告第二位独立审计员的判断，因此无法量化人工审核本身的可靠性。
- **伦理与责任**：虽然论文实施了负责任的披露（仅公开已有包标识符，汇总统计，且在工具测试中未实际执行远程载荷），但结果可能被滥用；论文明确致促使进行组合检测和防御，而非攻击配方。
