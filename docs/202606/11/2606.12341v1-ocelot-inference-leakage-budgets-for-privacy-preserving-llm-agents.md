## 摘要
大型语言模型（LLM）代理越来越多地代表用户行事——读取个人文件、调用工具、与外部服务交易——可能在每一步跨越信任边界泄露个人可识别信息（PII）。这里的隐私不是单个输出的属性，而是整个轨迹的属性，三个特性使其变得困难：泄露是累积的，因为单独无害的释放会随诚实但好奇或共谋的汇点累积成关于受保护秘密的推断；是双向的，因为恶意观察可以注入指令，使代理自身的推理模型转而对抗用户；且是任务依赖的，因为同一字段对某个接收者是必要的，对另一个则是多余的。每释放的情境完整性过滤器、信息流控制和后验泄露监控器各自解决了部分问题，但均无法在运行时控制基于累积推断的泄露。我们将代理隐私重新定义为后验风险控制，并提出OCELOT，一种运行时中介器，它预算对手关于秘密的信念在整个轨迹中的改善程度，而非过滤输出。其机制——见证验证的降密——将判断与信任分离：一个不受信任的、本地微调的防御模型检查每个候选释放并发出结构化证据——标记原子和提议的降密操作符——然后由确定性验证器审计，为所选变体收取经认证的最小熵成本，并在防篡改账本上记录基于汇点信任加权的预算下，授权最小泄露的有用释放。在多种代理基准和最近的防御措施中，OCELOT在更高任务效用下实现了显著更低的泄露，抵抗了自适应注入、越狱、累积推断和汇点共谋，且仅增加了适度开销。

## Abstract
Large language model (LLM) agents increasingly act on a user's behalf -- reading personal files, calling tools, transacting with external services -- possibly leaking personally identifiable information (PII) across trust boundaries at every step. Privacy here is a property not of a single output but of an entire trajectory, and three properties make it hard: leakage is cumulative, as individually innocuous releases accumulate across honest-but-curious or colluding sinks into inferences about a protected secret; bidirectional, as a malicious observation can inject instructions that turn the agent's own reasoning model against the user; and task-dependent, as the same field is necessary for one recipient yet gratuitous for another. Per-release contextual-integrity filters, information-flow controls, and posterior-leakage monitors each address part of this but none controls cumulative, inference-based leakage at runtime. We recast agent privacy as \emph{posterior-risk control} and present OCELOT, a runtime mediator that budgets how much an adversary's belief about a secret may improve across a trajectory, rather than filtering outputs. Its mechanism, \emph{Witness-Verified Declassification}, separates judgment from trust: an untrusted, locally fine-tuned defender model inspects each candidate release and emits structured evidence -- labeled atoms and proposed declassification operators -- which a deterministic verifier audits, charging a certified min-entropy cost for the chosen variant and authorizing the least-disclosing useful release under a sink-trust-weighted budget recorded on a tamper-evident ledger. Across diverse agent benchmarks and recent defenses, OCELOT attains significantly lower leakage at higher task utility, resists adaptive injection, jailbreak, cumulative inference, and sink collusion, and adds only modest overhead.

- **背景**：大型语言模型（LLM）代理在执行用户任务时（如订机票、调用工具、进行支付）需要跨越多个信任边界释放信息（PII）。隐私在此不再是单次输出的属性，而是整个**轨迹（trajectory）**的累积暴露。
- **核心问题**：现有防御（如每释放的上下文完整性过滤器、信息流控制、后验泄露监控）无法同时满足三个关键特性：
  - **P1（累积性）**：单独无害的释放可经累积或共谋还原出秘密。
  - **P2（双向性）**：信息不仅在外向动作中泄露，还在内向观察中泄露（注入指令导致代理后续泄露更多）。
  - **P3（任务依赖性）**：同一字段对不同接收者的必要性不同，无法有通用安全答案。
- **研究意义**：需要一种运行时机制，能定量、可组合地度量轨迹级别的后验风险，并在维持任务效用的前提下控制累积泄露。OCELOT首次实现了这一目标。

### 2. 方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：将代理隐私重新定义为**后验风险控制（posterior-risk control）**，通过预算对手关于秘密的信念（min-entropy）改善上限来替代逐条过滤。机制名为**见证验证降密（Witness-Verified Declassification, WVD）**，分离语义判断与授权决策。
- **系统架构**：
  - **防御者（Defender）**：本地微调LLM（信任但可被越狱），对候选释放x应用版本化的**规则集（rubric）**，输出结构化证据：原子（atoms）标签（字段、必要性、操作符）和降密操作序列σ。
  - **验证器（Verifier）**：确定性、受信任的组件，审计防御者的证据，分四个阶段：
    1. **Bind**：解析、检查哈希与账本链接。
    2. **Validate**：通过SMT求解器（Z3）检查覆盖性、必要性、操作符一致性等五项条件。
    3. **Cost**：使用预校准的每操作符-字段min-entropy成本，取max over可能字段，乘以信任类权重μ(z)（信任类：REQUIRED-SERVICE:1, SEMI-TRUSTED:2, ADVERSARIAL:4），得到释放成本cost。另运行credal后验审计作为经验性检查。
    4. **Commit**：检查累计成本Lcum + cost ≤ B（每轨迹基础预算），若超则沿操作符优先级（IDENTITY→GENERALIZE→SUBSTITUTE→REDACT→DROP）升级，直至找到可用变体或拒绝。成功则追加Merkle链账本条目。
- **操作符代数**：五种降密操作（IDENTITY, REDACT, GENERALIZE, SUBSTITUTE, DROP），每对(op, field)有校准的min-entropy成本C(op, f)，通过bootstrap UCB从代理对抗器池（校准池：Llama-3.3-70B, Qwen-3.5-27B；压力池：更强模型）获得。
- **训练**：防御者从Qwen-3.5-9B-Instruct初始化，两步微调：SFT（82,400样例，含对抗数据生成） + GRPO（奖励由验证器自身计算，优化授权结果）。

### 7. 优点

1. **分离信任与判断**：防御者（LLM）只提建议，验证器（确定性）执行最终授权，使得安全不依赖任何LLM的正确性，提供可证明的预算保证（Layer-1a）。
2. **细粒度降密操作**：支持IDENTIFY到DROP五种操作，可在任务效用与泄露之间精确权衡，避免二分式“允许/拒绝”导致过度拒绝。
3. **累积风险与多汇点共谋处理**：通过全局账本和信任权重，确保跨步骤、跨汇点的累积泄露被预算统一限制，解决P1和共谋问题。
4. **保守默认与防御训练**：未标记或错误标记的原子自动按最高风险计费，且越狱后仅导致回退而非沉默泄露；GRPO训练直接优化验证器奖励，泛化到全部规则集。
5. **实证评估严格**：包含校准覆盖率、压力池、真实轨迹验证；报告了置信区间、统计检验、所有消融和敏感性分析。

### 8. 不足与局限

1. **模型相对性**：语义隐私保证（Layer-2）依赖于校准的有效性。实验显示在压力对抗器下覆盖率从96.6%降至92.1%，仍有7.9%的违反（图6）。校准只能覆盖有限背景知识，无法推理承诺无条件安全。
2. **规则集的依赖**：规则集的完整性和正确性决定了系统效用，但规则集是人工+自动化构建的，可能存在未覆盖的泄露路径（虽然保守默认会导致回退而非泄露）。规则集验证出于实际考虑，不是形式化完备的。
3. **合成数据局限**：主要评估使用合成生成的轨迹（OCELOT-Traj-500），尽管通过真实轨迹验证，但真实部署中复杂性可能更高，标签噪声（necessity维度Krippendorff α=0.74-0.79）可能影响。
4. **防御者仍可被越狱**：虽然验证器会捕获错误标记（残差δ=1.7%），但仍有极低概率逃逸。此外对观测通道的注入主要通过包装（quoted-data wrapper）而非检测，但极端对手可能绕过。
5. **小信任基**：验证器和主程序本身假设系统完整性未被破坏（无内核漏洞、无密钥泄露）。实际中这是标准假设，但TEE等正交措施未讨论。
6. **开销虽可接受**：p50延迟204 ms，p99 736 ms，对实时交互可能有一定影响；需A100 GPU驱动防御者，经济成本不可忽视。
7. **未覆盖的攻击**：不包括物理侧信道、拒绝服务攻击、仅基于公共背景知识的推断等。
