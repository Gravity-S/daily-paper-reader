## 摘要
技能文档是指导大型语言模型（LLM）智能体的结构化自然语言指令，对现代智能体框架至关重要，但LLM难以写出真正有效的技能。在SkillsBench上，人类撰写的技能将通过率提高了16.2个百分点，而LLM撰写的技能没有带来可衡量的提升。我们提出了SkillAxe，一个完全无监督的框架，使LLM能够迭代地诊断和优化自身的技能。SkillAxe将技能质量分解为四个可解释的维度（质量影响、触发精确性、指令对故障归因的遵循度以及解决方案路径覆盖率），生成结构化改进简报，无需真实标签、测试套件或环境奖励。在SkillsBench上，相较于未改进的LLM技能，SkillAxe将通过率相对提高了28%，并缩小了与人类撰写技能之间47-67%的差距。我们在SpreadsheetBench上验证了该方法作为持续改进引擎的实际效果，使用仅22个技能，基于SkillAxe构建的技能库从过去的智能体轨迹中学习，将通过率从16.0%提升至52.0%。

## Abstract
Skill documents, structured natural-language instructions that guide Large Language Model (LLM) agents, are critical to modern agent frameworks, yet LLMs struggle to write skills that actually work. On SkillsBench, human-authored skills improve pass rates by 16.2 percentage points, while LLM-authored skills provide no measurable gain. We introduce SkillAxe, a fully unsupervised framework that enables LLMs to iteratively diagnose and refine their own skills. SkillAxe decomposes skill quality into four interpretable dimensions (quality impact, trigger precision, instruction compliance with fault attribution, and solution-path coverage), producing structured improvement briefs that require no ground-truth labels, test suites, or environment rewards. On SkillsBench, SkillAxe improves pass rates by 28\% relative over unimproved LLM skills and closes 47--67\% of the gap to human-authored skills. We validate the approach as a continuous improvement engine in the wild on SpreadsheetBench, where a SkillAxe-built skill library learns from past agent trajectories and raises pass rate from 16.0\% to 52.0\% using only 22 skills.

## 1. 核心问题与整体含义（研究动机与背景）
LLM驱动的智能体依赖“技能文档”（skill documents）——结构化自然语言指令，用于编码领域知识、API使用模式和过程策略。然而，LLM自行撰写的技能在实际任务中表现很差：在SkillsBench上，人类撰写的技能能将智能体通过率提高16.2个百分点，而LLM自写的技能**几乎没有带来可测量的提升**。问题根源在于缺乏可操作的诊断反馈——现有评估仅提供粗粒度的任务级通过/失败信号，无法定位技能失败的具体原因（触发错误、指令模糊、智能体未遵循等）。因此，需要一个**无监督的、可自动诊断并改进技能**的框架。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
SkillAxe是一个**评估引导的自我精炼框架**。它将技能质量分解为四个可解释的维度，通过对比智能体在有/无当前技能时的运行表现，生成结构化改进简报，再由LLM根据简报重写技能，实现迭代优化。

### 关键技术细节
1. **质量影响（Quality Impact）**  
   - 两阶段评估：先通过LLM法官比较有技能输出 \(r_s\) 与无技能输出 \(r_0\)，确定偏好方向（+1、0、-1）；再估计改进幅度 \(m\in[0,1]\)，得到综合评分 \(Q(s,\tau)=d\cdot m\in[-1,1]\)。

2. **触发精确性（Trigger Precision）**  
   - 从技能描述中提取正触发短语（应激活的任务）和负触发短语（不应激活的任务），使用上下文感知的向量嵌入（短语语义+句子上下文）构建激活区域和排除区域。  
   - 计算三个几何指标：**覆盖广度**（正触发多样性）、**负特异性**（排除短语与激活区域的距离）、**边界清晰度**（最近正负触发对的距离）。

3. **指令合规性与故障归因（Instruction Compliance with Fault Attribution）**  
   - 将技能分解为可评估的规则（含严重性权重 \(w_i\)）。  
   - 对每条规则 \(R_i\) 评估：相关性 \(\rho_i\)、遵循度 \(a_i\)、规则质量 \(g_i\)、技能故障程度 \(f_i\)（判断失败源于技能本身还是智能体执行错误）。  
   - 原始遵循度：\(\text{Compliance}=\frac{\sum w_i g_i a_i}{\sum w_i}\)；故障调整后的技能信用：\(c_i = a_i + (1-a_i)(1-f_i)\)，最终技能得分 \(\text{SkillScore}=\frac{\sum w_i g_i c_i}{\sum w_i}\)。  
   - 该分解防止因智能体执行差而错误修改正确规则。

4. **解决方案路径覆盖（Solution-Path Coverage）**  
   - 列举任务可能的多种解决路径 \(\{P_j\}\)，测量每个路径与技能块的最大余弦相似度，取平均：\(\text{PathCoverage}=\frac{1}{k}\sum_j \max_{c\in\text{chunks}} \cos(e(P_j), e(c))\)。  
   - 低覆盖说明技能只支持窄带行为，导致智能体采用其他有效策略时技能失效。

**整体流程**：对每个技能，在有/无技能条件下运行智能体（Phase 1）→ 计算上述四个诊断（Phase 2）→ LLM根据改进简报重写技能（Phase 3）→ 迭代直至收敛（Phase 4）。所有诊断无需真实标签、测试套件或环境奖励。

## 7. 优点
- **完全无监督**：无需真实标签、测试套件或环境奖励，仅需任务指令和智能体输出。
- **可解释的多维诊断**：将失败分解为触发、合规、覆盖等维度，便于定位问题。
- **分离技能缺陷与执行错误**：通过故障归因防止错误修改正确规则。
- **通用性强**：适用于不同领域（编程、电子表格、文档处理），且能作为生产环境中的持续改进引擎。
- **效率高**：在SpreadsheetBench上以较少技能达到同等性能，降低token成本和延迟。
- **公平评估**：引入公平评估器，揭示原生验证器的偏见（技能可能引导有效的替代方案）。

## 8. 不足与局限
