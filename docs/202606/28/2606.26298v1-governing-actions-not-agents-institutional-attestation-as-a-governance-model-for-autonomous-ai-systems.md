## 摘要
自主人工智能主体可能开始执行具有重大影响且不可逆的行为，例如临床处方和产品软件部署。本文观察到，人类机构在治理强大的自主行为体时，并非通过监控其推理过程，而是通过在关键时刻要求独立认证的证据。我们将这种机构模式形式化为一种适用于人工智能主体系统的计算治理模型。在该模型下，主体保留规划和推理的完全自主权，但对指定的高风险行为不拥有执行权限。执行取决于每个前提条件，这些条件均由独立的权威来源分别认证，以加密方式绑定到已声明的意图，并由确定性策略进行评估。决策记录在防篡改日志中，便于独立重新验证。我们提供了一个概念验证实现，并通过软件部署和临床处方的示例说明该模型。

## Abstract
Autonomous AI agents may begin to perform consequential, irreversible actions such as clinical prescribing and production software deployment. This paper observes that human institutions have governed powerful autonomous actors not by monitoring their reasoning but by requiring independently attested evidence at the point of consequential action. We formalise this institutional pattern as a computational governance model for AI agent systems. Under the proposed model, an agent retains full autonomy over planning and reasoning but holds no execution authority over designated high-risk actions. Execution is conditional on preconditions that are each independently attested by a separate authoritative source, cryptographically bound to a declared intent, and evaluated by a deterministic policy. Decisions are recorded in a tamper-evident log amenable to independent re-verification. We present a proof-of-concept implementation and illustrate the model with examples from software deployment and clinical prescribing.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **代理者（courier）模式**：agent保留规划和推理的完全自主权，但不对高风险动作持有执行权限。agent像信使一样收集认证，由治理中心（governance hub）决定是否执行。
- **多方可信认证**：每个前提条件由独立权威（称为oracle）签名认证，绑定到特定意图标识（intent ID），防止重放和替换。
- **确定性策略评估**：认证事实输入确定性策略语言（如Cedar），策略执行结果可复现。
- **防篡改审计**：决策记录在哈希链或Merkle树日志中，任何第三方可独立验证。

### 关键技术细节
1. **流程步骤**：
   - **意图声明（Intent Declaration）**：agent向治理中心请求授权特定动作，中心颁发密码学随机意图ID，返回所需认证列表。
   - **证据收集（Evidence Collection）**：agent联系各oracle，获取签名认证，每个认证包含意图ID、源标识、过期时间、负载（事实）。
   - **提交与评估（Submission & Evaluation）**：agent提交认证，治理中心验证签名、意图绑定、新鲜度、完整性，组装策略上下文，评估策略。
   - **条件授权（Conditional Authorization）**：若策略允许，中心执行动作或颁发窄范围能力令牌。决策追加到防篡改日志。

2. **加密绑定**：每个oracle使用Ed25519签名，认证信封包含`source_id`, `intent_id`, `expires_at`, `payload`。意图ID在签名内，无法修改或替换。

3. **确定性策略**：使用Cedar语言编写，例如：
   ```
   permit(principal, action == Action::"deploy_to_production", resource)
   when { context.tests_passed == true && context.coverage >= 80 ... };
   ```

4. **可验证计算**：若前提涉及计算（如剂量计算），agent可运行批准代码并提交TEE或零知识证明，治理中心仅接收验证后的输出。

5. **动作组合**：成功执行的签名收据可作为后续动作的前提，形成治理链。

6. **治理发现**：每个动作的技能合约（skill contract）发布所需认证列表，agent运行时读取，无需修改agent或中心逻辑。
