## 摘要
随着AI系统在代码生成中的广泛应用，一个核心安全问题浮现：当今的模型和编码代理在生成安全代码方面究竟能做什么？它们在哪里仍然失败？以及什么能推动该领域前进？现有工作已探索了用于安全代码生成的提示、微调、强化学习和智能体工作流，但该领域仍缺乏对这些技术如何提升安全性以及为何持续存在重大失败的系统性理解。在本系统化知识（SoK）中，我们系统化梳理了AI安全代码生成的进展、陷阱与未来路径。我们引入了一个三级框架，分别衡量模型对安全编码原则的自然语言理解、生成过程中对这些原则的代码级执行，以及两者之间的知识与执行差距。我们在基准测试中跨模型和编码代理实例化了该框架，覆盖了独立的函数级安全性和完整的Web应用程序安全性。我们的结果表明，安全编码原则理解是代码级结果（包括功能正确性、安全性以及联合功能-安全性正确性）在统计上的强预测因子。然而，显著的知识-执行差距仍然存在：模型能够识别相关的安全原则，但仍未能将其转化为安全且功能正确的代码。这些发现提供了一个以原则为中心的视角，揭示了AI安全代码生成的当前状态，并通过原则引导的生成、评估、基准测试和智能体工作流确定了具体的未来路径。

## Abstract
The increasing use of AI systems for code generation raises a central security question: what can today's models and coding agents actually do to produce secure code, where do they still fail, and what would move the field forward? Existing work has explored prompting, fine-tuning, reinforcement learning, and agentic workflows for secure code generation, but the field still lacks a systematic understanding of how these techniques improve security and why substantial failures persist. In this SoK, we systematize the progress, pitfalls, and paths forward for AI secure code generation. We introduce a three-level framework that measures models' natural-language understanding of secure coding principles, their code-level actuation of those principles during generation, and the knowledge--actuation gaps between the two. We instantiate this framework across models and coding agents on benchmarks covering both isolated function-level security and full web-application security. Our results show that secure-coding-principle understanding is a statistically strong predictor of code-level outcomes, including functional correctness, security, and joint functional-security correctness. Yet substantial knowledge--actuation gaps remain: models can recognize relevant security principles but still fail to translate them into secure and functional code. These findings offer a principle-centered account of where AI secure code generation stands today and identify concrete paths forward through principle-guided generation, evaluation, benchmarking, and agentic workflows.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：以**安全编码原则**（Secure Coding Principles, SCPs）为中心，将安全生成分解为三个可测量层次，从而诊断失败机制。
- **三级框架（KAUGE）**：
  - **Layer 1（知识）**：衡量模型对SCP的自然语言理解能力，涵盖四个推理维度：
    - 陈述性（规则是什么）
    - 因果性（违反为何导致漏洞）
    - 程序性（如何应用）
    - 情境性（何时适用）
    - 使用GLUE/SuperGLUE/CosmosQA/ReClor等任务格式构建6,382道固定答案题，覆盖OWASP和CERT标准。
  - **Layer 2（执行）**：衡量模型生成的代码是否**同时满足功能正确性和安全性**（可执行安全测试通过）。
  - **Layer 3（差距）**：通过**漏洞→SCP防御公式映射**（Disjunctive Normal Form, DNF）和**SCP合规性判断器**，分析知识到执行的差距，区分四种结果：
    - 原则成功（安全且合规）
    - 其他方式安全（安全但合规）
    - 合规但仍脆弱（合规但可被利用）
    - 执行失败（可利用且不合规）
- **关键技术**：
  - 漏洞→SCP防御公式：双模型（GPT-5-nano和Claude Haiku）独立生成最小充分防御子句，第三个模型裁定。
  - SCP合规判断器：相同双模型+裁定流程，对功能正确的代码检查是否实施了映射的SCP。
  - 所有构建物都经过人工抽样验证（n=100，有效性问题、引用答案正确性、映射充分性等）。

## 6. 论文的主要结论与发现

### 关键发现
1. **安全编码原则理解普遍存在**：所有模型在Layer 1上的分数远高于随机（OWASP平均93.7–96.5%，CERT 86.2–91.1%）。
2. **知识-执行差距显著**：知识（Layer 1）与执行（Layer 2）呈正相关（CWEval: ρ=0.71, p=0.025；BaxBench: ρ=0.63, p=0.056），但执行系统性低于知识（CWEval差47.9点，BaxBench差72.9点）。
3. **失败根源多样**：Layer 3诊断显示：
   - 原则成功（安全且合规）占主导，但对单一明确定义的漏洞类（如SQL注入）表现最好。
   - 其他方式安全（如代理实现其他防御）不罕见，但合规但不安全（如验证了错误的属性）是最具欺骗性的失败。
   - 直接执行失败（可运行但可被利用且无任何原则应用）在复杂任务中常见。
4. **推理级干预作用有限**：Security-Aware提示可恢复接近正确的代码，但RCI可能过度纠正并破坏功能。
5. **训练级干预未缩小差距**：SafeCoder和PurpCodeRL未系统性提升执行；PurpCodeRL甚至下降了功能正确率。
6. **代理脚手架最有效**：Codex Scaffold在CWEval上达到91.6% secure-pass@1，显著高于其他方法，因为它解决了软件构建问题而非仅仅安全问题。

### 核心总结
- 代码安全的瓶颈在于**将已知原则转化为特定边界上的安全实现**（知识-执行传递问题），而非缺乏原则知识。
- 未来方向应聚焦于：可执行反馈、接收点感知的原则指导、功能保持生成、因果-执行训练和原则/利用感知的评估。
