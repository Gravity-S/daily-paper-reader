## 摘要
大型语言模型（LLM）代理日益依赖长期记忆来支持复杂任务执行、用户个性化和领域适应。同时，针对LLM代理的访问控制机制正在被探索，以阻止违反策略的请求并防止滥用。我们揭示了代理记忆操作中出现的一种新攻击面：会触发访问控制的禁止内容可以被跨交互碎片化，以良性形式存储在长期记忆中，并通过记忆检索在最终用户查询中不显式出现的情况下随后重建。我们提出FragFuse，这是第一种利用长期记忆引入的时间通道使无特权用户能够绕过代理访问控制的攻击。FragFuse分三个阶段运行：（1）通过基于片段掩码的黑盒自适应查询识别拒绝响应的片段；（2）使用标记载体查询将这些片段注入记忆；（3）通过后续攻击查询检索并融合存储的片段。尽管FragFuse可以手动实例化用于单个代理，但我们进一步开发了一种基于代理的优化方案，该方案调整融合指令和标记设计，使其能够在违反攻击者威胁模型假设的情况下进行自动化攻击生成。我们在四种代表性代理设置和任务领域上评估FragFuse，涵盖三种最先进的代理访问控制机制。FragFuse在所有设置中平均绕过成功率为86.3%，平均端到端有害任务成功率为41.1%，与没有访问控制的配置相比，平均任务成功率仅下降4.4%。我们还表明，包括最先进的提示注入检测器和困惑度检测器在内的替代防御措施并不能有效应对这种攻击。

## Abstract
Large language model (LLM) agents increasingly rely on long-term memory to support complex task execution, user personalization, and domain adaptation. Meanwhile, emerging access-control mechanisms for LLM agents are being explored to block policy-violating requests and prevent misuse. We reveal a novel attack surface arising from agent memory operations: prohibited content that would trigger access control can be fragmented across interactions, stored in long-term memory in benign-appearing form, and later reconstructed through memory retrieval without appearing explicitly in the final user query. We propose FragFuse, the first attack that enables unprivileged users to bypass agent access control by exploiting this temporal channel introduced by long-term memory. FragFuse operates in three stages: (1) identifying rejection-responsive fragments via black-box adaptive querying with fragment masking; (2) injecting these fragments into memory using marker carrier queries; and (3) retrieving and fusing the stored fragments through a follow-up attack query. Although FragFuse can be instantiated manually for individual agents, we further develop a surrogate-based optimization scheme that tunes fusion instructions and marker designs, enabling automated attack generation without violating the attacker's threat-model assumptions. We evaluate FragFuse across four representative agent settings and task domains, covering three state-of-the-art agent access-control mechanisms. FragFuse achieves an average bypass success rate of 86.3% and an average end-to-end harmful task success rate of 41.1% across all settings, with only 4.4% average task-success degradation compared with configurations without access control. We also show that alternative defenses, including state-of-the-art prompt-injection detectors and perplexity detectors, do not effectively address this attack.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：大型语言模型（LLM）代理越来越多地配备长期记忆机制，以支持复杂任务、个性化和领域适应。与此同时，为了防止滥用，代理系统开始部署访问控制机制，拒绝违反策略的查询。现有访问控制仅检查当前查询，假设违规意图显式出现在当前输入中。
- **新攻击面**：本文揭示代理的记忆操作引入了一个**时序通道**——攻击者可以在早期交互中看似良性地将敏感内容碎片化写入记忆，然后在后续交互中通过记忆检索将其重构，使得最终攻击查询本身不包含违规内容，从而绕过访问控制。
- **整体含义**：这是首个利用长期记忆这一攻击面绕过LLM代理访问控制的工作，暴露了当前访问控制设计（只检查单次查询）的根本局限性，亟需专门防御。

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：通过三步利用记忆的时序通道，将原始违规查询 q 的敏感部分碎片化存储并后续融合，使攻击查询 q_att 本身无法触发访问控制，但代理在执行时能重建完整意图。
- **技术细节**：
  - **Stage 1: 敏感片段发现**  
    - 自动化黑盒方法：用 LLM 作为 **FRAG EXTRACTOR**，从原始禁止查询 q 中提取违反策略的片段 f_t，用标记 v_t（如`<>`）替换，得到掩码查询 q_mask。重复提交变体给访问控制，直到被接受或达到迭代上限（最多3次）。
  - **Stage 2: 载体构建与注入**  
    - 将发现的敏感片段在词级别拆分，用标记封装，附加在良性宿主查询 q_host 之后构成载体查询 q_car。提交该查询通过正常交互写入记忆。
  - **Stage 3: 片段检索与融合**  
    - 构造攻击查询 q_att，包含：相同的宿主查询（确保检索到载体记录）、融合指令 I_fuse、掩码查询 q_mask。代理执行时检索到载体记录，提取标记内容，按指令填充到掩码中完成重构。
  - **融合指令优化**：  
    - 针对黑盒威胁模型，提出**替代约束优化**框架。  
    - 定义损失：L_ret（促进检索）+ L_fus（促进融合成功），约束：L_coh（困惑度，避免异常）。  
    - 使用遗传算法在替代模型（GPT-4o）上离线优化 I_fuse，无需与目标代理交互。  
    - 优化过程包括初始化种子、候选生成（LLM改写、交叉、变异）、评估与选择、终止条件。

- **公式/算法**（文字说明）：  
  - 融合指令通过最小化 L_ret + L_fus 并满足 L_coh ≤ η_coh 获得。  
  - L_ret 计算攻击查询与宿主查询的语义距离（余弦相似度）；L_fus 计算替代模型生成的融合查询与原始禁止查询的语义距离；L_coh 为攻击查询的负对数似然（GPT-2计算）。  
  - 算法流程：初始化精英集 → 每代生成候选 → 评估合并约束 → 排序选择新精英 → 直到收敛。

## 6. 主要结论与发现

- **攻击有效**：FragFuse在所有设置中平均BSR为**86.3%**，平均E2E-SR为**41.1%**（基线仅5.7%）。TSR平均仅比无访问控制时下降**4.4%**。
- **查询效率高**：敏感片段发现平均仅需1.18–2.37次查询（表2）。
- **鲁棒性强**：对记忆大小、检索度量、标记选择、不同记忆机制均表现出稳定性。
- **现有防御无效**：提示注入检测器（PromptArmor、PromptGuard）和困惑度检测无法可靠区分载体/攻击查询与良性查询。
- **根本原因**：访问控制仅检查当前查询，未考虑通过记忆进行时序组合，导致漏洞。

## 7. 优点

- **新颖性高**：首个利用长期记忆作为攻击面绕过访问控制的研究，指出时序通道这一新威胁。
- **威胁模型实际**：攻击者为普通用户，无特权访问，仅通过正常交互实现，适用性广。
- **自动化程度高**：片段发现、指令优化均可自动化；替代优化允许离线完成，降低在线成本。
- **评估全面**：覆盖多种代理、任务域、访问控制、骨干LLM，消融深入，验证了鲁棒性。
- **实验设计公平**：对比基线（直接绕过指令）合理；使用自动化+人工验证确保TSR可靠性；公开代码和数据集促进复现。

## 8. 不足与局限

- **任务域覆盖有限**：评估限于购物、操作系统、网页导航、工具调用，未扩展到医疗、科学发现等，因缺少可测试的访问控制策略。
