## 摘要
代理系统日益通过第三方技能行动，使模型生成的决策能够影响文件、通信渠道和网络物理设备。这些技能通常包含自然语言规范，定义访问权限、披露限制、执行权限和必要前置条件。尽管此类规范描述了技能行为的预期边界，但它们本身并不提供可执行的运行时执行。执行这些规范带来了上下文粒度挑战：即使针对特定任务上下文编写了策略，监控器仍必须决定观察哪些事件、保留什么状态、在多大程度上推理执行过程以及在哪里干预。选择错误的粒度可能会阻塞良性执行，或遗漏仅跨多个动作才会出现的违规。然而，大多数现有执行机制假设固定的事件模型或执行点。

在这项工作中，我们提出了VIGIL，一个面向代理系统的端到端运行时执行框架。VIGIL根据来自技能规范、操作员定义约束以及跨多个技能的全局策略的行为策略，检查代理的实际执行轨迹。为了使此类策略可执行，VIGIL引入了一种策略语言，该语言捕捉代理-工具事件上的上下文特定执行需求，包括时间依赖、参数约束和值流条件。该语言与符号评估规则配对，将策略转化为有限轨迹上的SMT约束，使VIGIL能够检测依赖于事件顺序、参数关系或跨调用值流的违规，而不是依赖于固定的单调用过滤器。在涵盖办公文档、操作和工程任务的真实LLM代理运行中，VIGIL以超过95%的召回率和低于10%的误报率检测到策略违规。

## Abstract
Agentic systems increasingly act through third-party skills, allowing model-generated decisions to affect files, communication channels, and cyber-physical devices. These skills often include natural-language specifications that define access permissions, disclosure limits, execution privileges, and required preconditions. Although such specifications describe the intended boundaries of skill behavior, they do not by themselves provide executable runtime enforcement. Enforcing them raises a contextual granularity challenge: even when a policy is written for a particular task context, a monitor must still decide which events to observe, what state to retain, how far across the execution to reason, and where to intervene. Choosing the wrong granularity can either block benign executions or miss violations that emerge only across multiple actions. Most existing enforcement mechanisms, however, assume a fixed event model or enforcement point.   In this work, we present VIGIL, an end-to-end runtime enforcement framework for agentic systems. VIGIL checks an agent's actual execution trace against behavioral policies from skill specifications, operator-defined constraints, and global rules spanning multiple skills. To make such policies executable, VIGIL introduces a policy language that captures context-specific enforcement requirements over agent-tool events, including temporal dependencies, argument constraints, and value-flow conditions. The language is paired with symbolic evaluation rules that translate policies into SMT constraints over finite traces, allowing VIGIL to detect violations that depend on event order, argument relationships, or cross-call value flow rather than relying on fixed single-call filters. On real LLM-agent runs spanning office-document, operational, and engineering tasks, VIGIL detects policy violations with over 95% recall and a false-positive rate below 10%.

### 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：大型语言模型（LLM）代理越来越多地通过第三方“技能”（skills）执行操作，这些技能附有自然语言编写的行为规范（如权限、前置条件）。然而，当前部署中这些规范仅被记录，运行时仅依赖代理自身遵守，缺乏任何自动化的强制执行机制。这导致严重的信任鸿沟，已出现零点击提示注入、代理失控等实际安全事故。
- **根本挑战**：跨动作的违规具有“时间隐身性”——单次工具调用看似无害，但多个动作的组合可能违反规范（如要求先验证再消费某个工件）。现有防御（沙箱、单动作监控器、LLM事后判断）均无法在运行时实时捕捉这种跨时间/跨工件的违规。
- **整体含义**：需要一种能够将自然语言规范编译为可执行策略、在运行时对完整事件轨迹进行确定性检查的监控系统，以填补规范与执行之间的鸿沟，保障代理系统的安全可信。

### 2. 方法论：核心思想与关键技术细节
- **核心思想**：VIGIL将行为规范抽象为“有限轨迹上的安全属性”，通过严格分离规范的通用含义与运行时的具体实例化，实现上下文感知的强制执行。其决策流程分为四个步骤：
    - **轨迹抽象（Trace Abstraction）**：将原始工具调用流解析为带类型的结构化事件，记录动作、参数、状态、输出，形成有序事件序列和完整的事实库。
    - **策略实例化（Policy Grounding）**：将自然语言规范编译为基于轨迹签名（动作、参数名、值集合）的具体策略语句，使用LLM辅助翻译但限于以可验证的符号表达。
    - **SMT检查**：每个策略语句被编码为无量词的SMT约束，与事实库一起发送给SMT求解器（Z3）。若事实库与约束矛盾（unsat），则判定违规，并从不满足核心（unsatisfiable core）中提取违规事件作为精确证人。
    - **干预**：对违规调用进行阻止，并可触发回滚、重新规划或上报用户。
- **关键技术创新**：
    - **策略语言**：包含6种时序形式（absence, precedence, response, bounded response, resolve, until），支持变量绑定以关联同一工件在不同调用中的身份。
    - **符号编译规则**：将每个策略语句编译为对每个事件的条件公式（如`R_φ = ∧_e [ψ(e) ⇒ ∨_{e'≺e} ψ'(e')]`），保证查询的有限性和可判定性。
    - **证人定位**：直接利用求解器的unsat核心获取导致违规的具体事件，而非仅报告二元判决。

### 3. 实验设计

### 7. 优点
- **可执行的形式化基础**：将自然语言规范编译为有限轨迹上的SMT约束，决策具有确定性，不依赖代理自身推理。
- **跨动作的上下文感知**：通过变量绑定和时序形式，能捕捉单动作监控器无法看到的、涉及工件身份和顺序的违规。
- **精确的证人定位**：直接利用SMT求解器的unsat核心，输出具体违规事件，便于后续回滚或干预。
- **可扩展性**：策略语言覆盖六种常见时序模式，可通过Fallback（□Θ）表达任意组合。
- **低运行时开销**：SMT检查平均仅0.27秒（占端到端6.2%），主要开销来自可缓存的LLM编译。
- **真实世界有效性**：发现并报告了已部署生态系统的实际缺陷（NVIDIA已确认），展示了实际工业价值。

### 8. 不足与局限
- **轨迹可观察性边界**：若策略相关行为发生在不产生工具调用的脚本内部（如`secure_archive.py`中的文件操作），则无法被监控，导致假阴性。
- **编译保真度限制**：LLM在将自然语言编译为策略语句时可能丢失限定词（如“批量”修改 -> 误报单次修改），造成假阳性。
- **仅支持有限轨迹上的安全属性**：不适用于无限持续时间或需要实时回滚的活性属性。
- **依赖封闭世界假设**：只有工具调用层面的事件可被记录，若攻击者通过非标准通道（如网络、共享内存）绕过，则可能遗漏。
- **实验规模有限**：主数据集152个轨迹，虽然已覆盖多个任务类型，但真实世界技能包多样性尚未充分考察。
- **未报告跨模型/跨生态系统泛化**：仅使用claude-opus-4-6作为编译LLM，其他模型效果未知。
