## 摘要
在高风险领域，智能体安全需要形式化的策略执行，但现有方法要么依赖概率性护栏（精调分类器、基于提示的引导），这些方法不提供形式化保证；要么依赖手工编码的符号执行，但无法扩展到真实策略规范的广度。我们提出了一种自动形式化流程，利用基于LLM的生成器-批评者循环，将智能体提示、MCP工具描述和自然语言策略文档转化为经过形式化验证的策略。生成的策略采用Cedar策略语言编写。在MedAgentBench基准测试中，我们的自动形式化策略覆盖了源自然语言规范中比先前工作中手工编码的符号执行更广泛的部分。

## Abstract
Agent safety in high-stakes domains requires formal policy enforcement, but most existing approaches either rely on probabilistic guardrails (fine-tuned classifiers, prompt-based steering) that offer no formal guarantees, or on hand-coded symbolic enforcement that does not scale to the breadth of real policy specifications. We present an autoformalization pipeline that translates agent prompts, MCP tool descriptions, and natural language policy documents into formally verified policies using an LLM-based generator-critic loop. The resulting policies are written in the Cedar Policy Language. On the MedAgentBench benchmark, our autoformalized policies cover substantially more of the source natural-language specification than the hand-coded symbolic enforcement in prior work.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：LLM驱动的自主智能体在执行复杂任务时需要提升权限，但面临提示注入等安全风险。现有安全方法（如精调分类器、基于提示的引导）只提供概率性保障，缺乏形式化保证；而手工编写符号化策略虽能提供确定性，但难以扩展到真实场景中的大量策略规范。
- **研究目标**：提出一种自动形式化（autoformalization）管道，将自然语言的智能体指令、工具描述和策略文档自动转化为形式化验证的策略代码，实现可扩展且严格保证的智能体行为约束。

## 2. 方法论

- **核心思想**：“验证三明治”（Verification Sandwich）架构 + 生成器-批判者循环（Generator-Critic Loop），将离散的形式化推理与LLM的生成能力结合。
- **关键技术细节**：
  1. **底层（Grounding Layer）**：从智能体提示、MCP工具描述等中提取实体、工具模式，自动生成Cedar策略语言使用的模式（Schema），定义主体-资源-动作本体。
  2. **中间层（Model Layer）**：LLM（Gemini 3 Pro）基于输入（系统提示、工具定义、Schema、自然语言策略文档）生成候选Cedar策略集。
  3. **顶层（Safety Layer）**：双重批判器：
     - **硬批判**（Hard Critic）：确定性检查语法错误、Schema合规性、逻辑矛盾（如空策略、冲突规则），使用Cedar官方工具链。
     - **软批判**（Soft Critic）：LLM充当法官（Gemini 2.5 Flash），评估策略的语义对齐和定性问题（如是否符合原始意图），依据评分规则。
  4. 若策略未通过任意批判器，反馈给生成器进行迭代优化，直至达到质量阈值。
- **运行时**：最终策略集由外部确定性策略引擎在智能体调用工具前执行，采取“失败闭锁”原则（任何错误默认阻止动作）。

## 6. 主要结论与发现

- 自动形式化管道能够覆盖比手工编码更多的自然语言策略规则：Cedar拦截率始终高于Hong等人的不安全率，因为Cedar规则集更全面（覆盖了88条规则中的更多部分，而Hong仅实现了23条）。

- 将策略执行与LLM推理上下文解耦，可有效缓解提示注入和越狱攻击。
- 硬批判保证了策略的语法和逻辑正确性；软批判确保了语义对齐，两者结合使迭代生成稳定收敛。
- 形式化验证（Cedar语言）提供了人类可审计、可静态分析的安全保障。

## 7. 优点

1. **可扩展性**：首次实现从自然语言到形式化策略的自动化翻译，大幅降低了手工编写符号策略的人力成本。
2. **双重保障**：硬批判（确定性）与软批判（语义级）结合，同时保证语法正确和意图对齐。
3. **解耦安全**：策略引擎独立于LLM上下文，攻击者无法通过提示操纵安全规则，且失败闭锁原则提升了鲁棒性。
4. **可审计性**：Cedar策略人类可读，安全工程师可直接审核生成的策略代码。
5. **开源实现**：发布了Python绑定的Cedar智能体框架，便于社区复现和扩展。
