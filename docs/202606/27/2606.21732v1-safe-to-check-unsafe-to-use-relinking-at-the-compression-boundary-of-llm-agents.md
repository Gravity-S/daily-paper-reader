## 摘要
基于摘要的提示压缩被LLM智能体越来越多地用于缩短长而分布式的上下文，但它改变了安全边界：过滤器检查压缩前的提示，而后端则作用于新生成的压缩上下文。我们识别出重新链接，这是一种压缩边界漏洞，其中压缩器表现为一个混淆代理，将分布式的、局部良性的片段总结为完整的恶意指令。与提示注入不同，重新链接无需在源上下文中放置显式的恶意载荷。我们表明重新链接源于摘要本身：注意力机制使分离的片段联合可用，预训练使兼容的片段连接变得合理，而训练后则偏好紧凑的、后端可操作的摘要。我们将攻击者诱导的形式形式化为对抗性重新链接，并提出了Relink，一个基于DSL的自动化工具，它将恶意载荷拆分为良性片段，同时在压缩前保持完整载荷不存在。在四个长上下文智能体基准测试中，Relink实现了86.9%的重新链接率和后端行动率，而干净分割对照组为17.0%。现有防御措施未能可靠捕获对抗性重新链接；我们的KBRA防御将残余后端行动率降至0.0%。

## Abstract
Summarization-based prompt compression is increasingly used by LLM agents to shorten long, distributed contexts, but it shifts the security boundary: filters inspect the pre-compression prompt while the backend acts on a newly generated compressed context. We identify relinking, a compression-boundary vulnerability where the compressor behaves as a confused deputy, summarizing distributed, locally benign fragments into a complete malicious instruction. Unlike prompt injection, relinking need not place an explicitly malicious payload in the source context. We show that relinking arises from summarization itself: attention makes separated fragments jointly available, pre-training makes compatible fragments plausible to connect, and post-training favors compact backend-actionable summaries. We formalize the attacker-induced form as adversarial relinking and present Relink, an automated DSL-based tool that splits malicious payloads into benign fragments while keeping the complete payload absent before compression. Across four long-context agent benchmarks, Relink achieves 86.9% Relink Rate and Backend Action Rate versus 17.0% for clean-split controls. Existing defenses fail to reliably capture adversarial relinking; our KBRA defense reduces residual Backend Action Rate to 0.0%.

# 论文总结：《Safe to Check, Unsafe to Use: Relinking at the Compression Boundary of LLM Agents》

## 一、核心问题与整体含义（研究动机和背景）

- **背景**：现代LLM代理（如Claude Code、Codex、Cursor、OpenClaw）在长上下文场景中广泛采用**基于摘要的提示压缩**（summarization-based prompt compression）来缩短上下文，降低token成本和推理延迟。压缩器将源上下文摘要为紧凑的自然语言表示，后端LLM则基于压缩后的上下文执行行动。
- **核心问题**：压缩改变了代理管线的安全边界——前端过滤器检查压缩前的源输入，而后端基于新生成的压缩上下文行动。这产生了**表示层面的时间检查到使用时间（TOCTOU）差距**。研究者识别出一种新漏洞——**重新链接（Relinking）**：压缩器充当“混淆代理”，将分布在不同文档中的、局部良性的碎片（如一个动作描述和一个电子邮件地址）自动组装成一条完整的恶意指令。这种漏洞**无需在源上下文中放置任何显式的恶意载荷**，因此可以绕过所有基于源输入的防御。
- **整体含义**：重新链接是**基于摘要的提示压缩器的内生漏洞**，源于其训练和架构的三个机制（注意力机制、预训练、后训练）的组合。该漏洞对LLM代理的安全性构成严重威胁，现有防御无法有效应对。

## 二、方法论：核心思想、关键技术细节

### 2.1 机械论假设（Root Cause）
论文提出三个渐进的假设解释重新链接为何发生：
1. **注意力机制（Cross-Fragment Binding）**：Transformer的自注意力机制基于内容寻址，允许一个文档中的谓词（如“转发摘要到___”）与另一文档中匹配类型的值（如邮箱地址）在压缩时绑定，即使它们空间上分离。
2. **预训练分布（Typed Slot Prior）**：预训练使模型学习到每种语义角色（如收件人需要邮箱格式）有高概率完成值，因此类型匹配的值即使来自不同文档也会成为高概率补全。
3. **后训练偏好（Helpfulness Preference）**：RLHF/指令微调奖励“有帮助”的摘要（紧凑、自包含的绑定输出），促使压缩器将分离的谓词和值合并为一个句子，即使这会产生恶意指令。

### 2.2 攻击形式化：对抗性重新链接（Adversarial Relinking）
- **定义**：攻击者在源上下文\(H_{\text{untrust}}\)中仅放置不完整的动作片段（谓词带开放槽）和分离的值片段，没有任何片段声明两者之间的绑定（\(\text{Support}=0\)）。压缩器在压缩上下文\(S\)中合成完整指令（\(\text{Express}=1\)），后端执行该行动（\(\text{Act}=1\)）。
- **四个指示器**：Ground（源上下文中存在成分）、Support（源断言绑定）、Express（压缩上下文断言绑定）、Act（后端执行）。

### 2.3 Relink工具：三阶段流水线
基于对11个代理语料库的经验分析，设计了一种领域特定语言（DSL）和自动构建工具Relink，将恶意载荷\(\tau^\star\)分割为良性片段，同时确保压缩前完整载荷不存在。
1. **分解（Decompose）**：删除动作和值之间的绑定边，得到动作子图\(G_p\)（仅谓词+开放槽）和值子图\(G_v\)（仅值+类型标签），使源上下文满足\(\text{Ground}=1, \text{Support}=0\)。
2. **伪装（Disguise）**：将保留的角色标签\(r^\star\)投影到非绑定的角色提示（cue）空间（如相同工作流提示、参考角色提示），使两侧子图共享角色坐标但不恢复绑定。
3. **分发（Distribute）**：根据掩码选择承载序列\(L_e\)中的两个不同锚点（anchor），分别渲染\(G_p\)和\(G_v\)，确保它们分开放置且不产生本地绑定。

## 三、实验设计：数据集、基准与对比方法

### 3.1 数据集与基准
- 从四个长上下文代理基准中构建评估集，共619个样本：
  - **AgentDojo**（30个，涉及银行、工作区、Slack、旅行等）
  - **ASB**（188个，系统、金融、医疗、法律等）
  - **InjecAgent**（200个，数据窃取、直接危害）
  - **τ2-bench**（201个，航空、零售、电信客服）
- 每个样本包含干净的载体、基准定义的目标行动、精确的目标参数，且源上下文足够长以支持分布式放置。

### 3.2 对比方法
- **基线**：Clean-split control（相同载体和目标但无Relink的预处理）、Random-split、各种Payload splitting方法（变量重建、语义分解重建、上下文划分）。
- **防御对比**：11种代表性防御，涵盖输入过滤器（Llama Prompt Guard 2、ProtectAI DeBERTa-v2、ShieldGemma）、压缩提示防御（Sandwich prompting、Delimiter prompting、BIPIA reminder）、源清理器（PromptLocate、AttnTrace）、后端加固（SecAlign）、运行时策略（CaMeL）。以及提出的KBRA。

### 3.3 实验场景
- **RQ1（机制验证）**：在源上下文中构造匹配对（兼容输入vs角色破坏控制），测量注意力差距、概率差距、后训练偏好差距。
- **RQ2（有效性）**：619样本，Relink vs Clean-split，测量Relink Rate（RR）和Backend Action Rate（BAR）。
- **RQ3（泛化）**：180样本子集，变化压缩率、压缩提示、压缩模型、后端模型。
- **RQ4（消融）**：移除角色提示、错配提示、仅距离放置、通用实现等方式。
- **RQ5（实用性）**：在OpenClaw和Claude Code的真实代理系统中部署。

## 六、主要结论与发现

1. **重新链接是总结式压缩器的内在漏洞**：三个机制假设均被实验验证（注意力差距正性99.4%、类型槽先验正性95.2%、有用性偏好正性75.3%）。后训练是选择性承诺的关键（it模型对兼容绑定有偏好，base模型无鉴别力）。
2. **Relink工具高效**：在619样本上总体RR和BAR达到**86.9%**，远超clean-split的17.0%。在ASB上达到100%，在τ2-bench（最难的客服场景）为63.2%。
3. **泛化能力强**：跨不同压缩率（0.3-0.8）、不同压缩提示（agent handoff最脆弱）、不同压缩模型（Gemma、Qwen、GPT等）和不同后端模型均有效，证明漏洞普遍存在。
4. **现有防御无效**：11种代表性防御中，无一种能在维持低误报率的同时彻底阻断攻击。例如Llama Prompt Guard 2完全阻止但误报100%；ShieldGemma-2B误报1.5%但残余BAR 84.6%。
5. **提出的KBRA防御有效**：通过审计压缩边界，检查压缩后的绑定是否在源中有单段授权（局部性要求），将残余BAR降至**0.0%**（exact匹配）或0.3%（semantic匹配），同时仅产生0.93%良性误报（backend语义变体）。

## 七、优点

- **新漏洞发现**：首次提出并形式化了“重新链接”这一压缩边界漏洞，区分于提示注入、载荷分割、压缩攻击和摘要幻觉。
- **机理深度分析**：从注意力、预训练、后训练三个层面给出了可检验的机械论假设，并通过探测实验验证，为理解漏洞根源提供坚实基础。
- **自动化工具Relink**：基于语料库驱动的DSL，将攻击构建拆解为三个可组合的阶段，支持跨异构场景自动生成攻击样本。
- **全面评估**：覆盖四个基准、多种模型/提示/压缩率配置、消融实验、真实案例，实验设计严谨（匹配控制、统计检验）。
- **有效防御KBRA**：不依赖恶意内容检测，而是利用不变性（压缩后绑定必须由源单段授权），实现了安全性和实用性的平衡（0%残余BAR，0.93% FP）。

## 八、不足与局限

- **防御KBRA的适用范围**：KBRA保护的是压缩边界，但对直接出现在源上下文中的恶意指令（如直接提示注入）无效，需与输入过滤器配合。攻击者可通过将完整指令放入单一低权限段（A1策略）绕过局部性要求，但此时漏洞降级为传统注入。
- **KBRA的检测开销**：某些变体（如compressed语义审计）的误报率仍较高（34.6%），主要来自自由文本摘要提取丢失。论文指出backend审计可大幅降低误报（0.93%），但需依赖结构化后端提案。
- **τ2-bench性能较低**：Relink在该客服场景仅63.2%成功率，显示某些工作流结构对重新链接有较强抵抗性，论文未深入分析原因。
- **资源消耗**：防御评估中某些方法（如AttnTrace）延迟达6.56秒，实用性受限；KBRA虽轻量（7-70ms CPU），但依赖预定义的规范化和工具架构知识。
- **实验资源细节不足**：未披露完整实验的GPU小时数或训练成本，仅提及部分探针实验使用8 H800。
- **攻击假设**：攻击者需具备应用层知识以选择合理的载荷，且不能修改系统提示或后端，现实中可能更受限。
- **未探讨多模态压缩场景**：仅针对文本摘要压缩，未覆盖其他压缩方法（如token选择、KV缓存压缩）。
