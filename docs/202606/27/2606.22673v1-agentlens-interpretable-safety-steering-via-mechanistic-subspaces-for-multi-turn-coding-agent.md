## 摘要
基于大语言模型（LLM）的编程智能体展现出显著的自主能力，但在与外部环境的多轮交互中也引入了重大的安全与滥用风险。现有安全机制主要依赖外部护栏，在执行过程中进行细粒度行为控制的能力有限。与此同时，近期用于LLM安全的机制可解释性方法大多局限于单轮或越狱式问答场景，难以捕捉多轮智能体执行过程中不断变化的风险动态。本文从内部视角研究多轮编程智能体的安全性。我们提出AgentLens（机制子空间干预与操控），一种白盒防御框架，可为编程智能体执行运行时安全检测与表示级缓解。与传统的智能体护栏不同，AgentLens通过步骤级隐藏表示检测有害执行状态，并通过干预单层内的10维子空间来缓解不安全行为。为支持本研究，我们引入了机制智能体安全（MAS）基准，包含使用LLaMA-3.1-8B、Qwen-2.5-7B和Gemma-2-9B在194个任务上构建的全面注释的多轮执行轨迹。大量实验表明，AgentLens实现了强大的安全检测性能，为前瞻性风险预测提供了初步证据，并显著减少了编程智能体的有害行为，为将机制可解释性应用于动态LLM智能体安全奠定了基础。代码地址：https://github.com/EddyLuo1232/AgentLens

## Abstract
Coding agents based on large language models (LLMs) demonstrate remarkable autonomous capabilities, but they also introduce significant safety and misuse risks during multi-turn interactions with external environments. Existing safety mechanisms mainly rely on external guardrails, which have a limited ability to perform fine-grained behavioral control during execution. Meanwhile, recent mechanistic interpretability methods for LLM safety are mostly confined to single-turn or jailbreak-style QA settings, limiting their ability to capture the evolving risk dynamics of multi-turn agent execution. In this paper, we investigate the safety of multi-turn coding agents from an internal perspective. We propose AgentLens (Mechanistic Subspace Intervention and Steering), a white-box defense framework that performs runtime safety detection and representation-level mitigation for coding agents. Unlike conventional agent guardrails, AgentLens detect harmful execution states from step-level hidden representations and mitigate unsafe behavior by intervening in a 10-dimensional subspace within a single layer. To support this research, we introduce the Mechanistic Agent Safety (MAS) benchmark, comprising comprehensively annotated multi-turn execution trajectories across 194 tasks using LLaMA-3.1-8B, Qwen-2.5-7B, and Gemma-2-9B. Extensive experiments show that AgentLens achieves strong safety detection performance, provides preliminary evidence for lookahead risk anticipation, and substantially reduces harmful actions of the coding agent, establishing a foundation for applying mechanistic interpretability to dynamic LLM agent safety. ## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
AgentLens 是一个白盒防御框架，包含两个阶段：
- **阶段一：安全检测（Safety Detection）**：在每一步交互中，从 LLM 的中间隐藏层提取最后 token 的表示，使用训练好的线性探针（linear probe）预测当前步骤是否为有害执行状态。
- **阶段二：风险缓解（Risk Mitigation）**：当检测到有害步骤时，通过构建一个稀疏的转向向量（steering vector），在相同的目标层对隐藏表示进行干预，引导智能体生成安全的行为。

### 2.2 关键技术细节

- **线性探针**：在冻结的 LLM 的某一层 $ℓ$，提取最后 token 的隐藏表示 $h^{(ℓ)}(s) \in R^d$，训练一个 logistic 回归分类器：$g_\phi(h) = \sigma(w^\top h + b)$，预测步骤是否有害（1 表示有害，0 表示良性）。损失为二元交叉熵。
- **检测目标**：
  - **当前步骤安全检测**：训练数据包括所有良性步骤（标签 0）和恶意任务中的关键有害执行步骤（标签 1）以及安全拒绝步骤（标签 0）。目的是实现步骤级干预，而非任务级意图分类。
  - **前瞻性安全检测（Lookahead）**：使用有害步骤前一步的隐藏表示，预测下一步是否将有有害动作（标签 1），作为早期预警信号。
- **风险缓解**：
  - 从探针的权重矩阵 $W \in R^{2 \times d}$ 中计算每个维度的判别性分数：$|W_{1,k} - W_{0,k}|$，选取前 K 个维度（K=10）组成转向子空间。
  - 构建稀疏转向向量 $v \in R^d$：$v_k = 1$ 若 $k \in K$，否则 $0$。
  - 将转向向量广播到目标层所有 token 位置：$\tilde{H}_t^{(ℓ)}(\alpha) = H_t^{(ℓ)} + \alpha 1_{n_t} v^\top$，其中 $\alpha$ 是转向强度。
  - 使用 LLM 控制器（DeepSeek-V4-Pro）自适应搜索最优 $\alpha$，最大化安全与效用的加权组合（$\lambda_s=0.6, \lambda_u=0.4$），避免过度转向导致生成崩溃。

### 2.3 算法流程（文字说明）

1. **检测阶段**：给定步骤 $t$ 的上下文 $c_{t-1}$，提取目标层隐藏表示 $z_t^{(ℓ)}$，输入探针得到预测标签 $\hat{y}_t$。若为 0，继续执行；若为 1，触发缓解。
2. **缓解阶段**：基于探针权重计算 K 个关键维度，构建转向向量 $v$。LLM 控制器通过 adjust-steer-evaluate 循环搜索最优 $\alpha$，生成最终的安全动作。

## 6. 论文的主要结论与发现

- **可线性解码性**：在三个模型的中早期层，安全相关信息可以从步骤级隐藏表示中通过线性探针高精度解码（最高准确率：LLaMA-3.1-8B 99.15%，Qwen-2.5-7B 97.46%，Gemma-2-9B 95.35%），证明安全信号在这些中间层已充分涌现。
- **检测性能**：AgentLens 在当前步骤检测上优于所有 LLM-as-a-judge 和护栏基线，平均准确率提升 2.99%–5.82%。在前瞻性检测上也表现出比基线更稳定的性能（平均 94.54%）。
- **缓解性能**：AgentLens 显著降低攻击成功率，平均从 85.99% 降至 13.36%（降低 72.63 个百分点），优于 RepE 和 Self Reminder。但引入了一定崩溃率（平均 18.54%）。
- **因果控制**：反向转向实验显示，在自然拒绝状态上施加反向方向可将安全拒绝转为有害执行（ASR 达 100% 或 63%），证实发现的子空间具有因果控制作用。
- **OOD 泛化提示**：在恶意提示注入场景中，即使检测未触发，转向方向仍能有效降低 ASR（从 86.7% 降至 6.7%），表明表示级控制可能具有跨分布迁移能力。

## 7. 优点

- **创新性**：首个从内部表示角度研究多轮编程智能体安全的工作，将机制可解释性从单轮扩展到多轮动态交互场景。
- **框架设计**：检测与缓解均基于同一层同一探针导出的子空间，轻量高效；自适应 α 搜索平衡了安全与效用。
- **基准构建**：MAS 基准提供了步骤级注释的多轮轨迹，支持细粒度评估，并通过反事实增强减少伪相关。
- **前瞻性检测**：初步展示了隐藏表示可包含未来有害动作的早期信号，为主动防御提供可能。
- **因果验证**：通过反向转向提供了子空间因果性的强证据，而不仅仅是相关性。
- **开源可复现**：代码已公开，实验使用了可复现的 Docker 环境和确定性解码。

## 8. 不足与局限

- **白盒依赖**：AgentLens 需要访问模型内部隐藏表示，不适用于黑盒 API 或闭源模型，限制了部署范围。
- **数据规模与生成困难**：当前开源白盒模型在长序列智能体任务上能力有限，难以生成足够多的恶意轨迹，导致 MAS 基准规模较小（总步骤数约 1156 步），且前瞻性子集稀疏、不平衡。论文自身承认“lookahead 评估是最数据受限的部分”。
- **步骤级标注的歧义性**：恶意任务中的预备步骤标记为良性可能低估其轨迹级风险；直接有害步骤的判定有主观性。虽然采取了反事实增强，但标签噪声仍可能存在。
