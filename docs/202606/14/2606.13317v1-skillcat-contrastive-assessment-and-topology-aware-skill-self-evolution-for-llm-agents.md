## 摘要
面向LLM智能体的技能自我进化方法旨在将执行轨迹转化为可复用的技能文档，但当前流程通常从每个任务学习一条轨迹，在检查之前合并候选技能补丁，并在推理前加载完整技能语料库。我们提出SkillCAT，一个无需训练的框架，将这个过程分为三个阶段。对比因果提取（CCE）为每个任务采样多条轨迹，并比较同任务成功/失败对，以识别解释结果差异的证据。评估增强进化（AAE）在源任务克隆上回放每个候选补丁，并在层次化技能补丁合并前仅保留改善或保持任务结果的补丁。拓扑感知任务执行（TTE）将进化后的技能编译为可路由的子技能拓扑，因此推理仅加载与任务相关的能力节点。我们在常见智能体基准上评估SkillCAT，包括SpreadsheetBench、WikiTableQuestions和DocVQA，并进一步测试跨模型和分布外泛化。在这些设置中，SkillCAT将基线平均得分提升高达40.40%，展示了无需模型训练的可靠技能进化。

## Abstract
Skill self-evolution methods for LLM agents aim to turn execution trajectories into reusable skill documents, but current pipelines typically learn from one trajectory per task, merge candidate skill patches before checking them, and load the full skill corpus before inference. We propose SkillCAT, a training-free framework that separates this process into three stages. Contrastive Causal Extraction (CCE) samples multiple trajectories for each task and compares same-task success/failure pairs to identify evidence that explains outcome differences. Assessment-Augmented Evolution (AAE) replays each candidate patch on source-task clones and keeps only patches that improve or preserve task outcomes before hierarchical skill patch merging. Topology-Aware Task Execution (TTE) compiles the evolved skills into a routable sub-skill topology, so inference loads only the capability nodes relevant to the task. We evaluate SkillCAT on common agent benchmarks, including SpreadsheetBench, WikiTableQuestions, and DocVQA, and further test cross-model and out-of-distribution generalization. Across these settings, SkillCAT raises the average score over baselines by up to 40.40%, demonstrating reliable skill evolution without model training.

## 1. 核心问题与整体含义（研究动机和背景）

现有基于LLM的智能体（Agent）在执行任务时通常依赖外部技能文档（如结构化指令、工具使用规则、任务经验等）。为了自动从轨迹中提取和精炼技能，研究者提出了“技能自我进化”方法。然而当前主流方法（如Trace2Skill）存在三个根本性缺陷：

- **单轨迹偏差**：每个任务只学习一条轨迹，成功轨迹可能包含偶然策略，失败轨迹难以定位根本原因。
- **未经验证的合并**：候选补丁直接合并，未独立检查其对源任务的影响，低质量或有害补丁可能混入技能库。
- **上下文过载**：技能库不断膨胀，推理时加载全部技能导致提示过长，干扰智能体对任务相关内容的关注。

针对这些问题，论文提出**SkillCAT**框架，将技能自进化过程解耦为对比因果提取、评估增强进化和拓扑感知任务执行三个阶段，实现无需模型训练、可靠且高效的技能进化。

## 2. 方法论核心思想与关键技术细节

### 2.1 整体框架
SkillCAT将技能生命周期分为三个可观测阶段：
- **对比因果提取 (CCE)**：离线学习阶段，从多轨迹中提取因果证据。
- **评估增强进化 (AAE)**：离线学习阶段，通过源任务重放筛选并合并补丁。
- **拓扑感知任务执行 (TTE)**：在线部署阶段，路由到最小相关技能。

### 2.2 对比因果提取 (CCE)
- 对每个任务采样K条轨迹（默认5条种子），形成同任务成功/失败对比对。
- 定位**因果分水岭**：成功与失败轨迹动作序列首次出现差异的步骤。
- 提取以该点为中心的候选经验记录（包括局部证据、失败原因、可编辑教训）。
- **后备机制**：若所有轨迹全成功或全失败，则使用单轨迹提取器，或跳过该任务。

### 2.3 评估增强进化 (AAE)
- 对每个候选补丁，在源任务克隆上重放，记录基线结果和重放结果。
- 根据结果转移情况赋予分数（式2）：
  - 失败→成功：3.0（最强证据）
  - 成功→成功：2.0（保持行为）
  - 失败→失败：1.0（低权重）
  - 成功→失败：0.0（拒绝）
- 分数筛选：仅保留分数≥θ（默认2.0）的补丁。
- 层次化分级合并：按分数等级由低到高逐步合并，保留更多已验证补丁。

### 2.4 拓扑感知任务执行 (TTE)
- 离线编译：将进化后的技能解析为**核心技能**（始终加载）和**可路由节点集**（含标题、关键词、摘要、依赖关系）。
- 在线路由：给定任务描述和预算k，LLM路由器从拓扑摘要中选择相关节点，再经确定性扩展（召回锚点、添加依赖节点、基础节点）。
- 组装路由技能：核心技能 + 所选节点原始正文，仅加载最小上下文。

## 6. 主要结论与发现

1. **SkillCAT显著优于现有方法**：在Qwen3.5-35B-A3B Skill Deepening下，Vrf达55.50%，比Trace2Skill +Combined高25.83个百分点；平均分数提升最高达40.40%。
2. **CCE、AAE、TTE各自贡献显著**：消融实验表明移除任一模块均导致性能下降，其中移除AAE甚至低于Trace2Skill基线。
3. **源任务重放是可靠过滤器**：分数校准显示重放结果与下游性能单调相关（失败→成功最佳，成功→失败最差）。
4. **TTE可大幅减少推理上下文**：在保持甚至提升性能的同时，上下文减少41.6%。
5. **跨模型泛化有效但不稳定**：SkillCAT技能可被gemma-4和gpt-5.4-mini复用，但提升幅度因用户模型而异（gemma受益明显，gpt受益较小）。
6. **多模态迁移成功**：在DocVQA上，SkillCAT达到0.9159 ANLS（+0.232），而Trace2Skill反而低于无技能基线。

## 7. 优点

- **训练无关**：完全无需模型微调，即可实现技能自进化，计算成本低，易于部署。
- **系统化设计**：将技能生命周期分为证据提取、补丁验证、上下文控制三阶段，每个阶段针对现有方法的局限提出明确解决方案。
- **对比因果提取**：利用同任务成功/失败对比，有效识别因果分水岭，避免单轨迹偏差。
- **重放验证机制**：在合并前对每个补丁在源任务克隆上评估，防止有害补丁进入全局技能库。
- **拓扑感知路由**：首次将技能组织为可路由拓扑，实现推理时仅加载相关子技能，兼顾效率和准确性。
- **多维度实验**：覆盖同模型、跨模型、分布外、多模态等场景，消融和参数分析深入，结论可靠。
