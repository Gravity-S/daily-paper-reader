## 摘要
检索增强生成（RAG）代理越来越多地运行具有跨用户会话累积的持久内存。这创造了一个新的攻击面：仅通过正常渠道交互的对手可以注入精心制作的内存，一旦被检索，就会引导代理对未来用户的响应，而无需触及模型权重或代码。我们将此称为多会话内存投毒（MSMP），并表明现有防御无法对此进行认证；静态语料库防御（RobustRAG、ReliabilityRAG）假设固定的知识库，启发式过滤器则被流畅的企业风格文本绕过。我们提出带签名内存和平滑检索（SMSR），这是首个针对此场景具有认证鲁棒性边界的防御。组件1在写入时添加HMAC-SHA256来源，阻止未签名注入。组件2在查询时应用基于判决多数投票的随机内存消融，限制已认证对手的影响。我们证明无来源的检索时过滤器无法认证针对自适应注入，推导出组件2的超几何证书，并形式化一致少数效应，即一致对抗性答案作为数值少数赢得基于字符串的投票，而基于判决的投票将其移除。在15个企业场景（3,150次重复试验）中，组件1将攻击成功率从93-100%降至0%（针对所有未签名变体）。对于单次注入的已认证对手，组件2将成功率控制在8.0%（95% CI [5.8, 10.9]，n=450），低于认证最坏情况。在端到端仅查询攻击中（代理自身编写投毒而非预植入），SMSR在实时代理堆栈上将成功率从65.3%降至5.3%（n=150，非重叠CI）。干净查询效用为90%（组件1）和85%（组合）。

## Abstract
Retrieval-augmented generation (RAG) agents increasingly run with persistent memory that accumulates across user sessions. This creates a new attack surface: an adversary interacting only through normal channels can inject crafted memories that, once retrieved, steer the agent's responses for future users, without touching model weights or code. We call this Multi-Session Memory Poisoning (MSMP) and show that no existing defence certifies against it; static-corpus defences (RobustRAG, ReliabilityRAG) assume a fixed knowledge base, and heuristic filters are bypassed by fluent enterprise-style text. We present Signed Memory with Smoothed Retrieval (SMSR), the first defence with a certified robustness bound for this setting. Component 1 adds HMAC-SHA256 provenance at write time, blocking unsigned injection. Component 2 applies randomised memory ablation with verdict-based majority voting at query time, bounding the influence of authenticated adversaries. We prove that no provenance-free retrieval-time filter can certify against adaptive injection, derive a hypergeometric certificate for Component 2, and formalise the Consistent Minority Effect, whereby a consistent adversarial answer wins string-based voting as a numerical minority while verdict-based voting removes it. Across 15 enterprise scenarios (3,150 repeated trials), Component 1 cuts attack success from 93-100% to 0% for all unsigned variants. For an authenticated adversary with a single injection, Component 2 holds success to 8.0% (95% CI [5.8, 10.9], n=450), below the certified worst case. In an end-to-end query-only attack where the agent itself writes the poison rather than it being pre-seeded, SMSR reduces success from 65.3% to 5.3% (n=150, non-overlapping CIs) on a live agent stack. Clean-query utility is 90% (Component 1) and 85% (combined).

### 1. 核心问题与整体含义（研究动机和背景）

- **背景**：检索增强生成（RAG）代理系统在持续运行中积累跨用户会话的持久内存，这产生了一个新的攻击面——多会话内存投毒（MSMP）。攻击者仅通过正常交互（如员工、客户）即可注入精心构造的文本，这些文本被存储后，会在未来用户查询时被检索到，从而引导代理做出错误响应（例如给出假的政策限额），而无需修改模型权重或应用程序代码。
- **现有防御不足**：静态语料库防御（RobustRAG、ReliabilityRAG）假设知识库固定，不适用于动态的运行时内存写入；启发式过滤器（关键词黑名单、困惑度、语义异常检测）可以被流畅的企业风格文本完全绕过（实验显示100% ASR）。没有任何现有防御能提供形式化的安全证书（certified robustness bound）。
- **本文目标**：提出首个针对MSMP场景提供认证鲁棒性边界的防御——SMSR（带签名内存和平滑检索）。

### 2. 方法论

#### 核心思想
SMSR由两个互补的组件构成：
- **组件1**：写入时HMAC-SHA256来源标记（provenance tagging），创建密码学硬边界阻止未签名注入。
- **组件2**：查询时随机记忆消融（randomised memory ablation）+ 基于裁决的多数投票（verdict-based majority aggregation），限制已认证对手（legitimate user）的注入影响。

#### 关键技术细节
- **Component 1**：每个合法内存写入通过一个受信任的服务端签名预言机（oracle）附加HMAC-SHA256标签：`τ = HMAC_K(content || session_id || timestamp)`。检索时只接受标签有效的条目。安全性：若攻击者不知道密钥K，产生有效标签的概率≤2⁻²⁵⁶（Proposition 1）。
- **Component 2**：给定查询q，先检索top-m个已验证候选（m > k）。然后进行n_runs次独立试验，每次从这m个中均匀无放回采样k个条目，用LLM生成响应，再通过一个LLM法官（judge）将每个响应判为“correct”、“malicious”或“neither”。最终输出由多数裁决决定（malicious counts >一半则失败）。
- **关键理论贡献**：
  - **Theorem 1（不可能性结果）**：任何仅依赖内容（provenance-free）的确定性的检索时过滤器无法对自适应注入实现非平凡（非退化）的安全证书。证明利用了嵌入空间的稠密性。
  - **Theorem 2（Component 2的超几何证书）**：假设候选池中有N个条目，最多t个对抗注入，检索到的top-m中最多t'个对抗。单次消融无对抗的概率p_clean = C(m-t', k)/C(m, k)。多裁决为malicious的概率δ ≤ Σ_{i=⌈n/2⌉}^{n} C(n, i) (1-p_clean)^i p_clean^(n-i)。系统因此实现(t, δ)-SMSR安全。
  - **Consistent Minority Effect (CME)**：形式化并量化了一致少数效应——对抗性响应因文本一致（如总是说“限额$50,000”）在字符串级别多数投票中获胜，即使数量上占少数；而基于裁决的投票将相同内容的多种表述归为相同裁决，从而消除该效应（Proposition 2）。

#### 算法流程（文字说明）
1. 写入路径：所有内存写入都经过签名预言机产生HMAC标签；未签名写入被拒绝。
2. 查询路径：
   - 从内存中检索所有标签有效的条目。
   - 用余弦相似度获取top-m候选。
   - 重复n_runs次：无放回采样k个条目 → 送入LLM生成响应 → LLM法官给出裁决。
   - 多数裁决决定最终输出。

### 6. 主要结论与发现

1. **Component 1（HMAC来源）**：所有未签名注入变体（包括启发式绕过）的ASR从93-100%降至0%。
2. **Component 2（随机消融+裁决投票）**：
   - 生产规模（20种子，m=20，n_runs=5）：t=1已认证注入ASR = 8.0%（95% CI [5.8, 10.9]），低于证书上界δ=10.4%。若n_runs=7可收紧至7.1%。
   - 小存储评估（10种子，m’=11）：t=1直接注入ASR = 37.8%（CI [33.4, 42.3]），低于δ=41.5%；泛洪注入ASR=43.1%，CI跨δ（证书紧致）。
   - t=2、3时直接注入ASR均低于证书；泛洪变体接近证书。
3. **一致少数效应**：同一组15次消融运行，字符串投票导致93.3%ASR，而裁决投票仅13.3%（单次运行点估计），验证了CME的存在及裁决投票的修复效果。多重复后（n=450）裁决投票ASR为37.8%，仍比字符串投票约低2.5倍。
4. **端到端查询仅攻击（代理自己写入毒害）**：SMSR将ASR从65.3%降至5.3%（n=150，非重叠CI）。
5. **模型泛化性**：Sonnet在同样生产规模下ASR也是8.0%（与Haiku一致），证明防御不依赖特定模型。
6. **法官可靠性**：Haiku与Sonnet法官在84个样本上Cohen’s κ = 0.955（97.6%一致），可高度信赖。
7. **效用**：Component 1保持90%效用（与基线相同），Component 2降到85%（因随机子采样丢失部分上下文）。

### 7. 优点

- **首次提供认证鲁棒性**：针对动态内存投毒场景给出形式化安全证书，而非仅经验性评估。
- **理论贡献扎实**：证明了无来源检索时过滤器的不可能性定理；推导了组件2的超几何证书；形式化了一致少数效应并给出解决方案。
- **两个组件互补**：Component 1从根本上消除未签名注入，Component 2处理已认证注入；即使对手有签名权限，仍能被限制在一定概率内。
- **实验非常全面**：覆盖多类攻击、多个t、多种存储大小、不同模型、法官可靠性、端到端真实堆栈验证；所有报告带置信区间和证书对比。
- **部署友好**：作为drop-in wrapper，无需修改底层LLM或嵌入模型；HMAC验证开销微乎其微；API调用成本低（10× agent+judge调用，但与业务损失相比可忽略）。
- **实用性**：干净查询效用85-90%，在安全性和可用性之间取得合理平衡。

### 8. 不足与局限

- **证书对大t的局限性**：当t ≥ m/2时，对手控制候选池多数，证书δ接近1，防御没有保证。需通过部署控制（如每用户写入配额、去重、日志审计）使t保持在较小范围。
- **依赖HMAC密钥安全**：若攻击者能获取签名预言机或密钥，可伪造标签；建议短密钥轮换（≤7天）以缩小暴露窗口。
- **评估场景限制**：所有场景为合成的企业知识库（Nexora Corp），真实部署可能更长、更模糊的记忆条目，且可能存在更复杂的对抗响应（如欺骗法官）。未收集人类标签（不过法官可靠性已通过跨模型验证）。
- **可能受自适应对手影响**：认知对手可观察法官裁决并尝试构造被法官判为“correct”但仍有害的响应。论文建议轮换法官模型及随机化其系统提示作为缓解，但未做实验验证。
- **清洁查询效用下降5%**：由于随机子采样，在干净上下文中信息不完整可能导致错误反馈。对于某些对准确性极其敏感的应用，这可能不可接受。
- **单层代理**：未扩展到多代理场景（一个代理写入另一个代理的内存），论文提到这是未来工作。
- **仅评估Claude系列模型**：未测试其他基础模型（如GPT、开源模型）下的表现。不过理论不依赖模型特定属性，但实证泛化性有限。
