## 摘要
随着组织将AI代理（执行非确定性工作流、维护有状态会话并且通常具有对内部服务的特权访问）向生产部署推进，工程挑战从构建单个代理转向以适当的隔离、治理和安全性大规模运行它们。在本文中，我们介绍Agyn，这是一个开源平台，围绕针对代理工作负载的三个关键原则设计：在Kubernetes上基于信号驱动的有状态无服务器运行时；用于代理和流程定义的Terraform提供程序；以及基于零信任和最小权限原则的安全模型。Agyn是代理无关、模型无关和云无关的。

## Abstract
As organizations move toward production deployments of AI agents, which execute non-deterministic workflows, maintain stateful sessions, and often operate with privileged access to internal services, the engineering challenge shifts from building individual agents to operating them at scale with proper isolation, governance, and security. In this paper we present Agyn, an open-source platform designed around three key principles tailored for agent workloads: a signal-driven, stateful serverless runtime on Kubernetes; a Terraform provider for agent and harness definition; and a security model grounded in zero-trust and least-privilege principles. Agyn is agent-agnostic, model-agnostic, and cloud-agnostic.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：围绕三个关键原则设计：
  1. 信号驱动的有状态无服务器运行时（基于Kubernetes）；
  2. 通过Terraform Provider实现代理和工具定义即代码；
  3. 基于零信任和最小权限的安全模型。
- **关键技术细节**：
  - **运行时**：
    - **信号驱动启动**：当用户在线程中发送消息时，通知服务发布事件，编排器订阅事件，解析代理配置、请求OpenZiti身份，创建Kubernetes Pod并挂载持久卷（保留会话状态）。
    - **空闲回收**：代理运行时每10秒发送保活信号，超过空闲超时（默认5分钟）后编排器停止Pod并删除网络身份，状态和线程历史保留，下一条消息重新启动实例并恢复状态。
  - **配置**：
    - 使用Terraform Provider声明完整代理定义（容器镜像、工作区配置、系统提示、模型、密钥、MCP服务器、持久卷等）及其基础设施“束具”（harness）。
    - 支持预构建代理镜像（Claude Code、OpenAI Codex、Agyn自有代理循环），也支持自定义容器。
    - 通过Terraform模块复用常见模式，确保一致性。
  - **安全**：
    - **每容器隔离**：主容器运行代理进程，边车容器运行MCP服务器，各有独立文件系统和进程树，仅共享回环接口；密钥仅注入需要它的容器（通常是MCP边车），绝不暴露给代理LLM。
    - **零信任覆盖网络**：使用OpenZiti为每个代理实例分配临时的x509身份，在mTLS握手阶段识别调用者，并通过ABAC策略限制每个代理只能访问明确授权的服务。
    - **关系驱动授权（ReBAC）**：基于OpenFGA（类似Google Zanzibar）实施细粒度图遍历权限，定义代理所有者、维护者、参与者等角色，线程访问限定于参与者。
- **算法/流程**：无特定数学公式，主要描述事件驱动、状态持久化、按需创建与销毁Pod的编排逻辑。

## 6. 主要结论与发现

- Agyn是首个同时满足以下三个要求的开源平台：高效的按需执行、代理定义即代码、零信任访问。
- 通过信号驱动的有状态无服务器运行时，结合Terraform定义和OpenZiti零信任覆盖网络，实现了代理的弹性伸缩、治理合规和最小权限安全。
- Agyn是代理无关、模型无关、云无关的，支持自定义代理镜像和MCP服务器，可部署在任何Kubernetes集群上。
- 对比现有平台（AgentCore、Claude Code、Google AX、kagent），Agyn在功能完整性上领先。

## 7. 优点

- **设计系统性**：针对代理生产部署的三大痛点，提出了集成化的解决方案，而非拼凑现有工具。
- **定义即代码**：使用Terraform而非YAML或控制台，使得代理定义可审查、可版本化、可复用，符合基础设施即代码最佳实践。
- **安全模型严谨**：从容器隔离、临时身份、覆盖网络到关系授权，层层递进，确保即使LLM被攻破也无法泄露凭证或越权访问。
- **开放性与可扩展性**：开源（AGPL-3.0）、与代理/模型/云无关，支持自定义代理容器和MCP服务器，社区可贡献扩展。
- **状态保持的无服务器**：在FaaS的弹性与代理会话连续性之间取得平衡，通过持久卷和信号驱动实现“状态随实例恢复”。
