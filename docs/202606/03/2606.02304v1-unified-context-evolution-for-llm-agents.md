## 摘要
基于LLM的智能体能够通过将推理与环境反馈相结合来解决多步交互任务，然而每个回合都从相同的固定上下文开始，任务结束后，任何在此过程中发现的有用策略都会丢失。现有方法要么将学习限制在当前任务上，要么将所有经验汇集到单个未分类的存储中，不区分知识类型、不通过使用追踪质量，也不平衡库中仍然缺失的内容。我们提出了统一上下文进化（UCE），这是一个无需梯度的框架，将智能体的经验外化为一个不断演化的类型化可进化上下文单元（ECU）库。UCE将经验分解为四种互补类型（记忆、策略、工作流和技能），每种类型由轨迹在特定类型条件下生成，在决策时被检索，通过重复使用结果进行评分，并在不再有价值时被剪枝。调度模块将每个周期的生成预算分配给库中最弱的类型。在两个交互基准测试中，UCE将ALFWorld的成功率从75.4%提升到96.3%，将WebShop的任务得分从45.1%提升到61.3%，并且积累的库无需重新训练即可迁移到其他智能体骨干网络上。

## Abstract
LLM-based agents can solve multi-step interactive tasks by combining reasoning with environment feedback, yet each episode starts from the same fixed context and any useful strategy discovered along the way is lost once the task ends. Existing approaches either limit learning to the current task or pool all experience into a single untyped store, without distinguishing knowledge types, tracking quality through use, or balancing what the library still lacks. We introduce Unified Context Evolution (UCE), a gradient-free framework that externalizes agent experience into an evolving library of typed Evolvable Context Units (ECUs). UCE decomposes experience into four complementary types (Memory, Strategy, Workflow, and Skill), each generated from trajectories under type-specific conditions, retrieved at decision time, scored through repeated usage outcomes, and pruned when no longer valuable. A scheduling module allocates each cycle's generation budget toward the types where the library is weakest. Across two interactive benchmarks, UCE raises ALFWorld success from 75.4% to 96.3% and WebShop task score from 45.1% to 61.3%, and the accumulated library transfers to alternative actor backbones without retraining.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：基于LLM的智能体在解决多步交互任务时，每个episode都从相同的固定上下文（系统指令、少样本示例、工具模式）开始，任务中发现的任何有用策略（如环境机制、决策规则、工作流程）在任务结束后立即丢失。这使得智能体无法跨任务积累和复用经验。
- **现有方法局限**：
  - 强化学习方法需梯度访问和高训练成本，不适用于闭源模型。
  - 任务内自适应（如Reflexion、Self-Refine）仅改进当前任务，无法跨episode持久化。
  - 跨任务知识库方法（如ExpeL、Voyager）要么将所有知识混合存入无类型池，要么只支持单一知识形式，且通常一次性提取而不跟踪使用质量，也无资源分配策略。
- **意义**：提出一种无需修改模型权重的上下文学习机制，使智能体能够持续积累并动态优化跨任务知识，显著提升交互任务性能，且所学知识可迁移到不同骨干模型。

## 2. 方法论

### 2.1 核心思想
- 将智能体经验外部化为一个**类型化的、可进化的上下文单元（ECU）库**，库在多个任务周期中不断更新，不更新模型权重。
- 定义四种功能互补的ECU类型：**Memory（记忆）**、**Strategy（策略）**、**Workflow（工作流）**、**Skill（技能）**，每种有独立的生成条件、检索边界和淘汰规则。
- 每个ECU表示为五元组：`(类型, 任务类型, 检索条件, 知识内容, 适应度分数)`。检索条件与知识内容分离，便于优化相关性。
- 通过**知识收益调度（KYS）** 动态分配每个周期的生成预算，优先补充库中最弱的知识类型。

### 2.2 关键技术细节
- **四种ECU类型定义**（见表1）：
  - Memory：环境机制的事实性陈述，由单个轨迹生成。
  - Strategy：IF-THEN形式的决策规则，由同一任务类型的多个轨迹生成。
  - Workflow：任务类型的标准执行序列，由同一任务类型的成功轨迹生成。
  - Skill：跨任务类型的可复用子操作，由多种任务类型的轨迹生成。
- **检索公式**：`score(e, t) = sim(h_we, h_di) + β·1[τ_e=τ_i] + λ·(f_τi(e)−0.5)`，其中包含语义相似度、任务类型匹配奖励、质量偏差。
- **适应度分数**：Laplace平滑公式 `f_e = (s_e+1)/(u_e+2)`，其中u_e为使用次数，s_e为累积成功信用。新ECU初始为0.5。
- **进化周期**（每个周期5个阶段）：
  1. **评估（Evaluate）**：在held-out评估任务上运行智能体，记录成功率，不更新适应度。
  2. **收集（Collect）**：在不相交的收集任务集上运行，存储轨迹并更新适应度分数。
  3. **调度（Schedule）**：KYS根据库状态和收集成功率，为各类型分配生成预算。
  4. **生成（Generate）**：按类型特定提示从轨迹中提炼候选ECU，包含质量反馈示例和重洗测试（拒绝实例特定的观察）。
  5. **清理（Cleanup）**：移除低适应度或冗余的ECU，每个（类型，任务类型）组只保留最高适应度ECU。
- **KYS三层**：
  - 第1层（可行性过滤）：若某类型已存在足够ECU覆盖所有任务类型，则分配零预算。
  - 第2层（需求评估）：根据覆盖缺口和质量缺口计算需求d。
  - 第3层（收益追踪）：使用指数移动平均追踪各类型的历史收益，并融合得分分配预算（设置最小份额和比例分配）。

## 3. 实验设计

- **数据集/场景**：
  - **ALFWorld**：文本家庭环境，134个任务，6种任务类型（Pick、Look、Clean、Heat、Cool、Pick2）。评估成功率。
  - **WebShop**：文本在线购物环境，100个任务，5个子类别（beauty, clothing, electronics, food, home）。评估任务得分和购买成功率。
- **Benchmark**：在每个基准上运行多个周期（ALFWorld 5周期，WebShop 11周期），周期0为ReAct基线。
- **对比方法**：
  - 跨范式基线：ExpeL（无类型经验池）、Reflexion（多试次自反思）。
  - 不同actor骨干：NoThinking、Plan-and-Act、ReflAct（均使用相同LLM骨干）。
  - 所有方法在相同actor模型、生成器/洞察模型、任务集和步骤预算下复现。
- **模型**：agent使用gpt-4.1-mini（T=0.7），生成器使用gpt-5.2（T=0.4），嵌入使用all-MiniLM-L6-v2。

## 6. 主要结论与发现

- **性能提升**：
  - ALFWorld：成功率从75.4%（ReAct基线）提升到峰值96.3%（+20.9点）。最弱任务类型（Look从50%到100%，Pick2从52.9%到82.4%）。
  - WebShop：购买成功率从30%提升到42%（+12点），任务得分从45.1%提升到61.3%（+16.2点）。
- **跨骨干迁移**：UCE库能有效转移到不同actor骨干（NoThinking、Plan-and-Act、ReflAct），无需重新训练。Plan-and-Act+UCE在ALFWorld上达到最高97.8%。
- **单类型消融**：Workflow类型对ALFWorld贡献最大（97.0%），但对WebShop不足（仅34%），全类型组合在WebShop上效果显著高于任何单一类型。
- **调度有效性**：KYS动态分配预算，优先补充薄弱类型，使库在多个周期中持续改善。
- **错误分析启示**：WebShop失败主要源于属性选择不精确（50%），反映了提示级知识在精细操作上的天然边界，需要强化学习方法补充。

## 7. 优点

- **无需梯度更新**：完全基于提示注入和检索，适用于闭源LLM模型，无需访问模型权重或训练基础设施。
- **类型化知识管理**：四种ECU类型覆盖环境事实、决策规则、工作流程、可复用技能，功能互补，避免单一知识形式不足。
- **使用驱动的质量评估**：通过基于实际使用结果的适应度分数（Laplace平滑）动态跟踪每个ECU的价值，而非一次性LLM评估。
- **动态资源分配**：KYS根据库的薄弱环节分配生成预算，避免饱和类型的浪费，加快覆盖缺口。
- **可迁移性**：积累的ECU库可作为独立上下文模块，直接注入不同推理骨架，无需重新训练。
- **实验充分、统计显著**：在多个基准、多个骨干上验证，消融、案例、显著性检验完备。

## 8. 不足与局限

- **环境限制**：仅在两个文本交互环境（离散动作空间）上评估，未测试多模态观测、连续动作空间或更长规划Horizon（如AndroidWorld, VisualWebArena）。
- **模型单一**：仅使用一组闭源模型对（gpt-4.1-mini为agent，gpt-5.2为生成器），未独立变化生成器模型，无法分离框架设计、生成器能力和环境规律性的贡献。
- **任务池规模较小**：ALFWorld 134任务，WebShop 100任务，KYS的调度信号估计自有限数据，在更大任务集或更长进化周期下是否仍可靠、库是否持续改善或最终退化仍不明确。
- **ECU类型手工设计**：四种类型是手工定义的，未来可探索自动类型发现机制以适应新领域。
- **未与参数更新结合**：当前未耦合轻量级参数更新，未来可将适应度驱动生命周期与权重更新结合，实现“学什么”和“怎么做”的协同学习。
