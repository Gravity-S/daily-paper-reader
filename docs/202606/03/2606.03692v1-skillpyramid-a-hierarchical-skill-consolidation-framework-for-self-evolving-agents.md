## 摘要
近期的人工智能智能体能够灵活调用技能解决复杂任务，但其长期改进从根本上受限于缺乏系统性的技能构建、积累与迁移。特别是，缺乏统一的技能巩固框架时，智能体倾向于在不同任务中冗余构建相似能力，无法有效将经验转化为可复用资产，并且难以将任务特定技能泛化到新场景。为解决这一局限，我们提出了SkillPyramid，一种通过复用已有技能经验实现更广泛任务泛化的技能巩固框架。SkillPyramid在分层技能拓扑结构上运行，进一步引入了自进化机制，使智能体能够在任务执行过程中组合、验证并整合新技能。在ALFWorld、WebShop和ScienceWorld上基于四种骨干模型的实验表明，SkillPyramid将平均奖励显著提升了38.0%，并将执行步骤减少了27.7%。总体而言，我们的方法将技能集合从静态资源池转变为动态进化系统。

## Abstract
Recent AI agents can flexibly invoke skills to solve complex tasks, but their long-term improvement is fundamentally constrained by a lack of systematic skill construction, accumulation, and transfer. In particular, without a unified framework for skill consolidation, agents tend to redundantly construct similar capabilities across different tasks, are unable to effectively transform experience into reusable assets, and struggle to generalize task-specific skills to novel scenarios. To address this limitation, we propose SkillPyramid, a skill consolidation framework that reuses existing skill experience for broader task generalization. Operating on a hierarchical skill topology, SkillPyramid further introduces a self-evolution mechanism that enables agents to compose, validate, and incorporate new skills during task execution. Experiments on ALFWorld, WebShop, and ScienceWorld across four backbone models show that SkillPyramid substantially increases the average reward by 38.0% and reduces execution steps by 27.7%. Overall, our method transforms a skill collection from a static resource pool into a dynamic evolution system.

###  SkillPyramid: 面向自进化智能体的分层技能巩固框架 - 论文总结

#### 1. 核心问题与研究动机
- **核心问题**：当前的AI智能体虽然在调用技能完成复杂任务方面取得了进展，但缺乏系统性的技能构建、积累和迁移机制。这导致：(1) **冗余构建**：不同任务中重复构建相似能力；(2) **经验固化**：无法将完成任务的经验有效转化为可复用的资产（技能）；(3) **泛化困难**：任务特定的技能难以迁移到新场景。
- **研究动因**：手动维护的技能库会迅速过时，智能体需要具备持续的技能进化能力，将孤立的、任务特定的解决方案，转化为可复用、可组合的知识。现有的方法（如ReAct、Reflexion、ExpeL、ReAct+Skills）要么未充分利用技能内部的可迁移经验，要么无法将新技能从旧经验中组合生成，更多只是检索、精炼或追加，而非从先验技能中进行“组合”。

#### 2. 方法论：SkillPyramid
- **核心思想**：将扁平化的技能集合（Flat skill library）组织成多级“金字塔”（Hierarchy）结构。底层持有细粒度、可复用的原子能力（Atomic skills），上层归纳出抽象的、问题求解的模式（Abstract skills）。通过显式的复用和依赖关系，新任务可以通过“组合”已有原子和抽象技能来解决，而非“重建”。
- **关键技术细节**：一个多智能体框架，包含Relation Analyzer和Relation Builder。
    - **金字塔构建 (Construction)**：
        - **粗粒度分组**：Relation Analyzer首先根据技能名称和描述对技能进行粗略分组。
        - **细粒度分析**：对每个粗分组内的技能进行详细分析，提取可复用关系（如共享子程序、抽象模式）。
        - **向下原子提取 (Downward Atomic Extraction)**：Relation Builder从共享相似执行步骤的技能组中提取出“原子技能”（如“find object → take object”）。原子技能被放置在金字塔的**更低层**。原技能会被重写以引用（reuse）这个新原子技能。
        - **向上抽象归纳 (Upward Abstract Induction)**：从共享相似高层面问题解决模式的技能组中归纳出“抽象技能”（如“acquire → transform → place”）。抽象技能被放置在金字塔的**更高层**，描述如何组织下层技能。
    - **技能生成 (Skill Creation)**：面对新任务时，Skill Creator智能体**不从头生成**技能，而是：
        1.  **检索**：从金字塔中检索相关技能。
        2.  **框架构建**：优先使用高层（抽象）技能来规划任务结构和解决逻辑。
        3.  **细节填充**：再使用低层（原子）技能来填充具体的可执行操作。
    - **自进化 (Self-Evolution)**：每次执行任务并生成新技能后，系统不会重建整个金字塔，而是**增量式更新**。新技能作为查询，由Relation Analyzer找出其与现有相关技能的关系，再由Relation Builder构建新的复用或依赖关系，并将其插入金字塔中。
- **算法流程**：这是一个迭代的闭环过程：现有技能库 → Relation Analyzer + Relation Builder构建金字塔 → 新任务 → Skill Creator检索+组合生成新技能 → 自进化机制更新金字塔 → 等待下一个新任务。

#### 3. 实验设计
- **基准/数据**：
    1.  **ALFWorld**：文本化的家居环境，需要完成长视距任务（如烹饪、清洁）。分为 **Seen** (140个任务) 和 **Unseen** (134个任务) 两个子集。
    2.  **WebShop**：模拟电商网站，智能体需搜索、比较、购买商品。使用了200个随机采样的任务。
    3.  **ScienceWorld**：基于文本的互动科学实验环境。分为 **Seen** (194个任务) 和 **Unseen** (211个任务) 两个子集。
- **对比方法**：
    1.  **ReAct**：基础方法，无外部技能。
    2.  **Reflexion**：增加经验反思，但无结构化技能。
    3.  **ExpeL**：积累自然语言经验教训，但无层级结构。
