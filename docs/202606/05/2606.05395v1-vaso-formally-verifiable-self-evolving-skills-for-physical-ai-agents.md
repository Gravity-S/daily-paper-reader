## 摘要
可复用的机器人技能正在成为具身智能体将开放式指令转化为长时域物理行为的基本单元。我们认为，虽然基础模型大幅降低了创建这些技能的成本，但信任它们的成本并未降低。现有的技能进化循环通过执行反馈、单元测试、环境奖励或LLM自我批评来完善技能，但这些信号仅提供轨迹级证据：它们表明技能在采样执行中有效，而不是技能引发的规划在未测试条件下满足时间安全合约。我们提出VASO，一个面向LLM生成的机器人技能合约的验证引导自进化框架。在VASO中，每个技能被表示为具有两个耦合接口的语义合约：一个形式接口，将机器人状态、观测和控制命令与用于模型检测的逻辑命题对齐；一个面向规划器的接口，指导可执行行为生成。模型检测器首先过滤逻辑不一致的技能合约，然后验证由技能引发的规划是否符合全局和局部时间规范。当验证失败时，VASO将反例轨迹转化为文本梯度，更新可复用的技能合约，同时冻结基础模型权重。在Clearpath Jackal和PX4四旋翼任务上，VASO使用不到100个优化样本即达到97.2%的形式规范合规性，优于执行反馈、提示优化和微调基线。据我们所知，VASO是首个为物理AI智能体实现形式验证与自进化LLM生成技能闭环的框架：形式反例成为可复用机器人技能合约的优化反馈，而不仅仅是验证一次性规划、调整规划器提示或微调模型权重。

## Abstract
Reusable robot skills are becoming the basic units through which embodied agents turn open-ended instructions into long-horizon physical behavior. We argue that, while foundation models have collapsed the cost of creating these skills, the cost of trusting them has not. Existing skill-evolution loops refine skills through execution feedback, unit tests, environment reward, or LLM self-critique, but these signals provide only trace-level evidence: they show that a skill worked on sampled executions, not that skill-induced plans satisfy temporal safety contracts under untested conditions. We introduce VASO, a framework for verification-guided self-evolution of LLM-generated robot skill contracts. In VASO, each skill is represented as a semantic contract with two coupled interfaces: a formal interface that aligns robot states, observations, and control commands with logical propositions for model checking, and a planner-facing interface that guides executable behavior generation. A model checker first filters logically inconsistent skill contracts, then verifies plans induced by the skill against global and local temporal specifications. When verification fails, VASO translates the counterexample trace into a textual gradient that updates the reusable skill contract while keeping foundation-model weights frozen. On Clearpath Jackal and PX4 quadcopter tasks, VASO reaches 97.2% formal-specification compliance using fewer than 100 optimization samples, outperforming execution-feedback, prompt-optimization, and fine-tuning baselines. To our knowledge, VASO is the first framework that closes the loop between formal verification and self-evolving LLM-generated skills for physical AI agents: formal counterexamples become optimization feedback for reusable robot skill contracts, rather than merely verifying one-off plans, tuning planner prompts, or fine-tuning model weights.

- **核心问题**：物理AI智能体（如地面机器人和无人机）的可复用技能虽可通过大语言模型低成本生成，但其**信任成本高昂**。现有技能进化方法（如执行反馈、单元测试、环境奖励、LLM自我批评）仅提供**轨迹级证据**，只能证明技能在采样执行中有效，**无法保证**技能生成的规划在**未经测试的条件下**（如新场景、罕见故障）满足时序安全合约（如“检测到人时必须停止”）。
- **研究背景**：基础模型（如GPT）大幅降低了技能创建门槛，但在机器人等安全关键领域，如何确保技能可**验证、可复用、可提升**成为瓶颈。现有方法要么聚焦单次规划验证（仅拒绝不安全规划），要么将反馈吸收到提示或模型参数中（难解释、难复用）。**缺乏一种将形式化反例直接转化为技能自身优化信号的机制。**

## 2. 方法论：核心思想、技术细节与算法流程

### 核心思想
- **提出“可验证自进化技能表示”**：每个技能被表示为一个**语义契约**，包含两个耦合接口：
  - **形式接口**：将机器人状态、观测、控制命令与**逻辑命题**对齐，用于模型检测（Model Checking）。
  - **规划器接口**：以自然语言或代码模板形式引导LLM规划器生成可执行行为。
- **闭环优化**：形式验证器（NuSMV）对技能生成的规划进行查验，如果失败，将**反例轨迹**转化为**文本梯度**，反馈给技能本身进行迭代更新（冻结基础模型权重，不调参）。

### 关键技术细节
1. **技能形式定义**：技能 `sk = (G, ψ_sk, C_sk)`，其中：
   - `G`：全局规范（LTL公式，如“总是当人出现时下一时刻停止”）。
   - `ψ_sk`：局部规则（技能专属的LTL公式）。
   - `C_sk = (L_sk, Cp_sk)`：语义契约，包括**命题对齐标签函数**`L_sk`（将执行状态映射到命题集）和**规划模板**`Cp_sk`（文本引导）。

2. **两级验证**：
   - **技能级验证**：检查全局规范与局部规则是否逻辑一致（通过构建全转移系统 `M`，检查 `M |= ¬(G ∧ ψ_sk)`，若成立则技能不可行，需重新生成）。
   - **规划级验证**：将LLM生成的规划 `π` 编译为**自动机** `A_π`（转化为NuSMV文件），验证其是否满足 `G ∧ ψ_sk`。

3. **语义契约优化（VASO核心）**：
   - **前向传播**：当前契约 → 生成规划 → 编译自动机 → 模型检测。
   - **后向传播**：若验证失败，反例轨迹被转换为**文本梯度**（如“违反规范：G(人检测 -> X停止)”）。
   - **迭代更新**：使用 `F_skill` 模型（如GPT-5-nano）根据当前契约和梯度生成新契约 `C^(k+1)_sk = F_skill(C^(k)_sk, g, τ)`，持续优化直到满足所有规范。

4. **算法流程**：
   - 输入：任务提示τ、系统命令集S、全局规范G。
   - 初始化：`F_skill` 生成初始技能 `sk`。
   - 技能级筛选：检查 `sk` 逻辑可行性。
   - 循环：生成规划 → 规划级验证 → 若通过则结束；若失败则提取反例 → 反例转文本梯度 → 更新 `C_sk` → 重复。

## 8. 不足与局限

1. **命题对齐函数的可靠性**：自动生成的标签函数**未经验证**，可能存在误差（如文中案例：无人机忽略竖直速度分量），这会**直接破坏形式化验证的保证**。**高漏报风险**：低层误差可能使不安全规划通过检查。
2. **仅支持顺序并发性**：框架假设技能按顺序执行，**无法处理重叠或交错执行**的技能，限制了在复杂多任务场景中的应用。
3. **依赖LLM的推理能力**：若`F_skill`或`F_plan`在反例理解或契约修改上产生误判，优化可能会走偏。**无显式错误恢复机制**。
4. **符号抽象假设**：将连续的机器人状态（如速度、位置）简化为离散命题，可能丢失细节，影响验证的保真度。
