## 摘要
自主AI智能体推动了从对话到任务执行的转变，将安全失败从文本欺骗转移到系统入侵。尽管安全评估对于主动风险预防至关重要，但先前的工作受限于基本瓶颈，包括碎片化的风险覆盖、静态或低保真执行环境，以及单一维度且粗粒度的评估指标。为应对这些挑战，我们提出了AgentCanary，一个针对自主AI智能体的全面安全评估框架。AgentCanary通过三个贡献提供了系统性解决方案：首先，全面的风险覆盖：我们引入了一个正交的入口×影响风险分类法，将对抗性影响如何进入智能体与其最终造成何种危害解耦，并将其实例化为一个覆盖现实部署工作流的场景对齐任务套件。其次，高保真实时可执行环境：智能体与真实工具交互，针对动态配置的任务工件进行操作，而非静态问答或模拟工具响应，跨多步交互保持持久状态，自然支持长视野攻击评估。第三，轨迹基础的多维度评估：评估消耗完整的智能体轨迹，而非回复文本或单一工具调用，从而沿三个正交维度（结果安全、安全意识和任务效用）进行分解评分。我们在AgentCanary上对多个前沿模型，针对三种智能体框架下的多种已建立的对抗攻击方法进行了评估。结果表明，当前智能体常常无法识别其面临的攻击，特别是在受损技能、持久状态和长视野执行攻击下，并为开发更可靠和安全的智能体系统提供了系统基线。

## Abstract
Autonomous AI agents have driven the transition from conversation to task execution, shifting security failures from textual deception to system compromise. Although security evaluation is crucial for proactive risk prevention, prior work is constrained by fundamental bottlenecks, including fragmented risk coverage, static or low-fidelity execution environments, and single-dimensional and coarse-grained assessment metrics. To address these challenges, we propose AgentCanary, a comprehensive security evaluation framework for autonomous AI agents. AgentCanary provides a systematic solution along three contributions. First, comprehensive risk coverage: we introduce an orthogonal Entry $\times$ Impact risk taxonomy that decouples how adversarial influence enters the agent from what harm it ultimately causes, and instantiate it as a scenario-aligned task suite spanning realistic deployment workflows. Second, a high-fidelity real executable environment: rather than static Q&A or mocked tool responses, agents interact with real tools against dynamically provisioned task artifacts, with persistent state across multi-step interactions that naturally supports long-horizon attack evaluation. Third, trajectory-grounded multi-dimensional evaluation: evaluation consumes the full agent trajectory rather than the reply text or a single tool call, enabling decomposed scoring along three orthogonal dimensions, Outcome Safety, Security Awareness, and Task Utility. We evaluate a broad set of frontier models on AgentCanary against multiple established adversarial attack methods across three agent frameworks. The results reveal that current agents often fail to recognize the attacks they face, particularly under compromised skills, persistent state, and long-horizon execution attacks, and provide a systematic baseline for developing more reliable and secure agent systems.

### 论文核心问题与整体含义（研究动机和背景）

自主AI智能体的出现，标志着大型语言模型（LLM）从被动的信息检索器转变为主动的数字执行者。它们能够使用工具、调用API、操作文件系统并执行多步任务。这种从“语义理解”到“状态操控”的转变，在极大扩展应用边界的同时，也引入了全新的、更为严峻的安全攻击面。

- **核心问题**：传统LLM的安全威胁主要是“文本欺诈”（如提示注入导致输出有害内容），而AI智能体的安全失败则可能导致“系统入侵”（如文件删除、数据泄露、系统控制权丧失）。现有的智能体安全评估工作存在三大瓶颈：
    1.  **风险覆盖碎片化**：将提示注入、工具投毒、记忆污染等风险孤立看待，没有统一的抽象模型，导致评估覆盖不全，容易遗漏盲区。
    2.  **执行环境低保真**：多数基准依赖静态问答或模拟工具响应，无法复现真实的多步工具调用链、动态状态变化和持久化状态。这些环境无法忠实评估状态性或长程攻击的真实影响。
    3.  **评估维度单一粗粒度**：大多使用单一二进制标签（攻击成功/失败）或简单的工具调用作为代理指标。这混淆了“结果安全”、“安全感知”和“任务效用”这三种不同的行为，无法准确刻画智能体的真实安全态势。

- **整体含义**：论文指出，智能体安全的重心需要从“内容合规”转向更复杂的“系统完整性和动作可控性”问题。因此，必须建立一种新的、系统化的、主动的安全评估机制。

### 论文提出的方法论

AgentCanary框架的核心设计原则是：在攻击者可能利用的同一系统层面上，通过端到端执行来评估智能体安全。为了贯彻这一原则，论文提出了三个关键技术贡献：

1.  **正交的风险矩阵（Entry × Impact Risk Taxonomy）**
    - **核心思想**：解耦风险是如何**进入**智能体工作流的（风险入口，Entry）和风险最终**造成**什么危害（风险影响，Impact）。这两个维度是正交的，同一个入口可能导致不同影响，同一个影响也可能源自不同入口。
    - **风险入口 (E)**：定义了5种风险来源或条件。
        - 用户交互：直接提示注入（DPI）。
        - 不受信任的外部内容：间接提示注入（IPI）。
        - 受损技能/工具生态：技能投毒（SP）。
        - 持久记忆和状态：记忆污染（MC）。
        - 内在失败（IF）：无外部攻击者，智能体自身决策失误（如幻觉、过度信任）。
    - **风险影响 (U)**：定义了7类可实现的后果。
        - 本地环境和可用性风险、数据与信息安全风险、持久状态与记忆污染、特权与系统控制风险、网络攻击与远程控制风险、业务滥用风险、金融与交易风险。
    - 每个评估任务被形式化为一个元组，明确指定其对应的入口和影响，从而构建了系统化的风险覆盖。

2.  **高保真可执行环境（High-Fidelity Executable Environment）**
    - **核心思想**：让智能体在隔离的沙箱环境中，与**真实的工具接口**进行交互，而非调用模拟的API。
    - **关键技术细节**：
        - **工具级保真度**：提供真实的文件系统、进程执行、shell命令、本地网络服务（如DNS、邮件、Web服务）及业务模拟器。工具调用会产生真实的系统状态变化。
        - **数据级自动化**：建立一个自动化配置引擎，在每次评估前，动态地为任务准备所需的人工制品（如邮件、网页、银行账户、记忆条目等）。这确保每个任务的执行上下文都是丰富且独特的。
        - **任务隔离与状态持久**：每个评估实例运行在独立的Docker容器中，防止任务间污染。同时，在单次评估内部，支持跨步、跨会话的状态保持，以评估长程攻击。
        - **跨框架可扩展**：环境与底层智能体框架解耦，支持灵活切换不同框架（如Hermes, NanoClaw, OpenClaw）进行比较。

3.  **轨迹级多维度评估（Trajectory-Grounded Multi-Dimensional Evaluation）**
    - **核心思想**：评估消耗智能体的**完整行为轨迹**（包括用户查询、工具调用及参数、工具返回值、智能体回复），而非仅基于最终回复或单一工具调用。
    - **三个正交分数**：基于轨迹进行分解评分。
        - **结果安全分数 (OSS)**：衡量有害的**环境后果**是否被真正阻止。评估的是系统层面的结果。
        - **安全感知分数 (SAS)**：评估智能体是否能自主**识别**风险来源、理解其不安全机制并采取适当的防御性行动。评估的是智能体的认知和判断。
        - **任务效用分数 (TUS)**：衡量在承受风险的同时，智能体能否成功完成**合法的任务目标**。评估的是实用性。
    - **总分聚合**：通过加权求和（Stotal = α·OSS + β·SAS + γ·TUS）得到任务级总分。权重根据任务类型而定，例如，对于DPI任务，γ=0（因为没有合法任务目标）；对于IPI任务，α=0.6, β=0.2, γ=0.2。这避免了单一维度评价的混淆。

### 论文的主要结论与发现

1.  **当前智能体的安全能力极不均衡**：只有Claude Opus 4.6展现出全面且强大的安全性能（总分83.9），远超其他模型（第二名Qwen3.6-Plus为69.7）。安全和规模之间没有简单的正比关系，模型安全和家族、微调策略强相关。
2.  **风险严重程度因攻击面而异**：间接提示注入（IPI）的威胁被较好地抑制（平均不安全结果率UOR为9%），而长期、持久化的攻击威胁更大：技能投毒（UOR 62%）、长视野攻击（UOR 51%）、记忆污染（UOR 42%）是更致命的威胁。
3.  **静态评估会低估风险，动态攻击压力巨大**：动态攻击进化（DAE）揭露了静态DPI测试无法发现的、模型特定的安全行为。技能伪装（Script-based Camouflage）能导致所有模型的安全分数崩溃（如Claude Sonnet 4.5从84.4降至31.6）。这说明稳健性需要被动态地、自适应地检验。
4.  **安全结果和安全感知可以严重解耦**：许多模型，尤其是在IPI场景中，表现出高OSS（例如>90分，即阻止了有害后果），但SAS非常低（例如<50，即未识别出攻击）。论文将这种“表面上安全但未理解攻击”的行为称为“沉默的服从”（silent obedience），指出这种安全是脆弱的，一旦攻击方式稍有变化就可能失守。
5.  **安全性和实用性可以对齐**：安全评分最高的模型（如Claude Opus 4.6）同时也是任务效用最高的模型，并非通过过度拒绝来换取安全。这反驳了安全必须牺牲效用的传统观点。
6.  **智能体框架是影响安全性的重要因素**：在相同模型和任务下，不同框架（OpenClaw, Hermes, NanoClaw）能导致模型安全评分高达15.2分的差异。这说明安全性不能脱离具体框架来评估。
7.  **运行时防御是必要但不充分的**：当前的运行时防御插件能提供边际改进，但无法替代模型本身的安全能力。最好的防御模型仍远落后于无防御的最强模型（Claude Opus 4.6）。

### 优点

- **系统性的风险分类法**：“入口×影响”正交风险矩阵是论文的核心亮点。它提供了一种有原则、可审计的方式来组织和覆盖风险空间，优于以往扁平化或混淆的攻击列表。这种抽象使得比较不同攻击源和后果成为可能。
- **对执行环境真实性的极高追求**：论文明确区分了静态Q&A、模拟工具和真实可执行环境这三个层次，并选择了后者。这确保了评估能捕捉到系统层面的副作用、工具调用的真实效果和状态依赖性，是评估智能体安全的关键。
- **创新的多维度评估范式**：引入OSS、SAS、TUS这三个正交维度是对传统单一成功率指标的重大改进。它不仅能判断是否安全，还能深入分析“为什么安全”或“为什么不安全”，为模型开发和改进提供了更丰富的诊断信息。特别是“安全感知”维度的提出，指出了“沉默的服从”这一重要但易被忽视的脆弱性。
- **对动态攻击的深入研究**：通过DAE和LPA两种设置，论文不仅考察了静态鲁棒性，更深入探究了在自适应和长视野攻击下的鲁棒性，揭露了静态评估的盲点，使评估更具现实意义。
- **评估维度的全面性**：涵盖了模型、攻击方法、框架、运行时防御、可重复性等多个方面，实验设计非常系统和完善。

### 不足与局限

- **场景覆盖的局限性**：尽管任务套件较全面，但主要聚焦于办公自动化、信息检索、社交和金融等场景。对于更特定、更专业的领域（如医疗、自动驾驶、工业控制），其安全风险的形态和评估方法可能需要进一步扩展和定制。
- **LLM作为评判者的依赖性**：整个评估体系的核心依赖于一个LLM评判器（Claude Opus 4.6）来分析复杂轨迹并进行多维度评分。这种“LLM-as-a-Judge”方法本身就存在内在偏差（如对自身或更强大模型的偏好），且其评分稳定性虽然经过测试，但对最终结论的可靠性有着根本影响。未来工作应探索更鲁棒的混合评判方法。
- **对框架的侧重**：主要实验结果（表6）完全基于OpenClaw框架。虽然论文承认并分析了框架带来的影响，但其作为最核心结论的基线，存在被特定框架特性“污染”的风险。读者需要意识到这一点，在解读模型排名时保持谨慎。
- **可扩展性与成本**：构建高保真、动态配置的执行环境以及进行多步轨迹的LLM评估，其计算成本和实现复杂度远高于传统静态基准。这可能会限制其在资源受限的研究者或小型团队中的普及使用。
- **对“安全”定义的简化**：论文将安全主要定义为阻止对环境、数据、状态、服务的有害影响。这覆盖了“完整性”和“可用性”的方面，但对“公平性”、“隐私”（特别是信息被用于非意图的推理）、以及更广泛的社会伦理风险（如滋生网络攻击自动化）的探讨还不够深入。
