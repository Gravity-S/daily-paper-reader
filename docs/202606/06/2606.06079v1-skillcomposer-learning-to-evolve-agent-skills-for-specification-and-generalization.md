## 摘要
智能体技能由可复用的策略组成，这些策略指导智能体的推理与行动，在推理时提升模型能力方面展现出巨大潜力。然而，当前的技能构建方法将问题视为一次性提取，忽视了一个基本的矛盾：为特定任务定制的技能难以迁移，而抽象化的技能往往提供不足的指导。我们将这种脆弱性归因于缺乏明确的技能特化与泛化机制。为弥补这一不足，我们提出了SkillComposer，一个将技能构建分解为三个可学习操作的框架：创建、改进与合并。通过系统化的拒绝采样方法训练，SkillComposer使语言模型能够在推理时自我演化技能，并支持三种部署模式：离线模式用于构建通用库，在线模式用于任务特定优化，混合模式结合两者。在τ^2-Bench、LiveCodeBench v6和AppWorld上的全面实验表明，SkillComposer持续优于基线方法。我们的SkillComposer-4B在智能体任务上提升27B执行器性能高达+4.5，在代码任务上提升+3.4，同时泛化到训练中未见过的领域和任务类型。分析表明，合并与改进操作分别处理正交的质量维度，且技能组合是一种可迁移的元能力，为技能增强推理提供了实用方案。

## Abstract
Agent skills, which consist of reusable strategies that guide agent reasoning and action, have shown strong potential for improving model capability at inference time. However, current skill construction methods treat the problem as one-shot extraction, overlooking a fundamental tension: a skill tailored to the specific task fails to transfer, while the abstracted skill often provides insufficient guidance. We attribute this fragility to the absence of explicit mechanisms for skill specification and generalization. To address this gap, we introduce SkillComposer, a framework that decomposes skill construction into three learnable operations: create, improve, and merge. Trained via systematic rejection sampling recipe, SkillComposer enables language models to self-evolve skills at inference time and supports three deployment modes: offline for building generalized libraries, online for task-specific refinement, and hybrid for combining both. Comprehensive experiments on $τ^2$-Bench, LiveCodeBench v6, and AppWorld show that SkillComposer consistently outperforms baselines. Our SkillComposer-4B improves a 27B executor by up to +4.5 on agent tasks and +3.4 on code tasks, while generalizing across domains and task types unseen during training. Analysis reveals that merge and improve address orthogonal quality dimensions and that skill composition is a transferable meta-ability, providing a practical recipe for skill-augmented inference.

# SkillComposer 论文详细总结

## 1. 核心问题与整体含义（研究动机和背景）

当前基于大语言模型的智能体系统通过“技能”（skills）来增强推理能力：技能是可重用的自然语言指令，包含名称、触发条件和过程体。然而，现有技能构建方法存在根本性矛盾：
- **任务特化**：为特定任务定制的技能无法迁移到其他任务；
- **泛化抽象**：过度抽象的技能又缺乏对具体任务的充分指导。
现有方法（如一次性提取、依赖人工编写或成功轨迹）缺乏显式的**特化（specification）** 与**泛化（generalization）** 机制，导致技能质量不可控，甚至出现负面效果。

**论文整体目标**：提出SkillComposer框架，将技能构建分解为三个可学习的操作（创建、改进、合并），使语言模型能在推理时自主演化技能，无需地面真实监督，从而同时满足特化与泛化需求。

## 2. 方法论核心思想与关键技术细节

### 核心思想
将技能的生命周期建模为三个正交的可学习能力：
1. **Skill Create**：从原始执行轨迹中提取可重用过程知识，生成初始技能。
2. **Skill Improve**：通过新执行经验细化现有技能，驱动**特化**。
3. **Skill Merge**：合并语义相似的技能对，整合成更广泛、更可迁移的技能，驱动**泛化**。

### 关键技术细节
- **技能表示**：每个技能为三元组 `(name, description, body)`，以 markdown 文档形式存储。
- **相似度计算**：多视图相似度，对名称、描述、主体分别计算语义相似度后平均，阈值为 δ=0.8。
- **训练方法——拒绝采样监督微调（SFT）**：
  - 使用 **delta pass rate**（候选技能相对于基线的 pass@1 变化）作为质量信号。
  - 只有满足 delta ≥ ϵ（ϵc=0.4, ϵm=0.4, ϵi=0.4）的样本才被保留。
  - 对于 create：比较带技能 vs 无技能的 pass@1 在同一任务上。
  - 对于 merge：比较合并后技能 vs 两个原始技能各自在源任务上的平均 pass@1 提升。
  - 对于 improve：比较改进后技能 vs 原始技能在源任务和目标任务上的平均 pass@1 提升。
- **部署模式**（算法1）：
  - **离线模式**：先用 create 从所有任务轨迹中提取技能，再迭代 merge 直到无相似对，构建通用库；推理时检索+自选技能。
  - **在线模式**：对每个新任务，先 create 初始化技能，再迭代 improve T 次（实验中 T=5）。
  - **混合模式**：离线圈初始化后，在线 refine。

## 6. 主要结论与发现

1. **SkillComposer 一致优于所有基线**：4B 模型提升 27B 执行器在 agent 任务上 +4.5，代码任务 +3.4。
2. **混合模式综合最优**：在大多数场景中优于离线或在线单独使用，尤其对困难任务增益显著。
3. **Merge 驱动泛化，Improve 驱动特化**：两者正交互补，离线库构建中 Merge 贡献最大。
4. **技能组合是**可迁移的元能力**：agent-only 训练也能提升代码任务性能（+1.5）。
5. **迭代演化优于同预算暴力重复采样**：随轮数增加差距扩大，说明结构化技能演化更有效。
6. **跨模型泛化良好**：小模型产生的技能对大执行器依然有效。
7. **跨域泛化**：从 Retail 学到的技能可以有效迁移到 Airline/Telecom。

## 7. 优点

- **创新性**：首次显式将技能构建分解为创建、改进、合并三大操作，并赋予语言模型自主演化能力。
- **实用性**：支持三种部署模式，覆盖从离线训练到在线实时适应各种场景。
- **训练效率**：使用 delta pass rate 作为质量信号，通过拒绝采样获得高质量监督数据，无需人工标注。
- **强泛化性**：跨模型规模、跨域、跨任务类型均表现优异，且分析表明技能组合是元能力。
- **可解释性**：通过可视化合并前后技能分布、操作消融等揭示机制。
- **效率提升**：技能演化后 token 消耗下降，性能上升。
