## 摘要
内存是AI智能体的核心组件，使其能够在交互中积累知识并提升性能。然而，持久化内存引入了内存投毒的风险，单个对抗性内存写入即可对智能体行为产生长期影响。我们对基于LLM的智能体中的内存投毒进行了系统性研究。我们识别出四种内存写入通道以及模型能力、系统提示设计和智能体系统架构中的九种结构性漏洞，这些漏洞使得上述通道可被利用。基于这些漏洞，我们构建了六类内存投毒攻击的分类体系。此外，我们设计了MPBench——用于评估内存投毒攻击的基准测试，并表明设计得更积极写入和检索内存的智能体更容易被利用。我们还发现现有的提示注入防御措施无法覆盖内存投毒攻击。我们的发现为理解和缓解针对AI智能体的内存投毒攻击提供了基础。

## Abstract
Memory is a core component of AI agents, enabling them to accumulate knowledge across interactions and improve performance. However, persistent memory introduces the risk of memory poisoning, where a single adversarial memory write can exert long-term influence over agent behavior. We present a systematic study of memory poisoning in LLM-based agents. We identify four memory write channels and nine structural vulnerabilities in model capabilities, system prompt design, and agent system architecture that make these channels exploitable. Based on these vulnerabilities, we develop a taxonomy of six classes of memory poisoning attacks. Furthermore, we design MPBench -- a benchmark for evaluating memory poisoning attacks, and show that agents designed to write and retrieve memory more aggressively are more exploitable. We also show that existing prompt injection defenses fail to cover memory poisoning attacks. Our findings provide a foundation for understanding and mitigating memory poisoning attacks against AI agents.

- **核心问题**：当智能体从不可信的外部环境（如网页、文档、邮件、工具输出等）获取信息时，可能被诱导将恶意内容写入持久化内存。由于写入内存的内容在后续会话中被视为“可信知识”，一次成功的恶意写入就能对智能体行为产生**持久、长期**的操控影响。这被称为**内存投毒攻击**。
- **研究动机**：现有研究缺乏系统性，要么攻击模型假设过强（如要求直接访问内存），要么只孤立评估个别攻击。本文旨在填补这一空白，建立一个系统性的漏洞分析与攻击分类框架，并指出现有的**提示注入防御**无法有效应对内存投毒这一新型威胁。

#### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：提出一个系统化的分析框架，将内存投毒攻击分解为四个分析维度：**内存写入通道**（Where）、**结构漏洞**（Why）、**攻击分类**（How）和**评估基准**（Measure）。通过连接这四者，使得攻击分析不再是孤立技术的堆砌，而是基于底层原理的体系化研究。
- **关键技术细节**：
    - **内存写入通道（4种）**：定义了智能体写入长时记忆的四个通道，基于触发机制和写入内容决策权。
        - **C1 显式指令执行写入**：直接由外部输入的显式写存指令触发。
        - **C2 系统提示驱动写入**：由系统提示中的模糊策略（如“保存相关信息”）触发，写入内容由模型自行判断。
        - **C3 压缩驱动写入**：在上下文窗口即将溢出时，对交互历史进行压缩、总结并写入。
        - **C4 经验到程序写入**：将成功的任务执行流程自动提炼为可复用的技能程序，写入程序化记忆。
    - **结构漏洞（9个）**：从**模型能力、系统提示设计、系统架构**三个层面识别了9个漏洞。
        - **模型层**：V-M1 指令-数据边界盲点、V-M2 多源上下文中的来源归因失败。
        - **提示层**：V-P1 内存写入策略规定模糊、V-P2 压缩时缺乏源过滤机制。
        - **系统层**：V-S1 写入路径无验证、V-S2 共享多源上下文、V-S3 可操控的压缩触发器、V-S4 技能创建无验证、V-S5 自主改进机制放大攻击效果。
    - **攻击分类（6类）**：基于被利用的漏洞，将攻击分为**强信号**和**弱信号**两大类。
        - **强信号攻击**（在输入层面有可检测模式）：
            - **显式命令插入**：直接嵌入“保存”、“记住”等指令。
            - **条件命令插入**：当用户做出特定回应（如“是”）时才触发写入。
            - **显著性驱动压缩投毒**：在输入中重复恶意内容，使其在压缩时被当作重要信息而保留。
        - **弱信号攻击**（与合法内容语义上不可区分）：
            - **策略一致事实注入**：将恶意信息伪装成合理的领域知识或用户偏好。
            - **虚假先例插入**：伪造一个成功任务的记录，让代理在将来模仿。
            - **技能程序插入**：引导代理将含有恶意步骤的任务流程提炼为技能程序。
    - **MPBench基准**：首次专门评估内存投毒攻击的基准。通过两步评估：
        - **内存写入阶段**：检测恶意载荷是否被成功写入了持久化内存（攻击成功率，ASR）。
        - **内存检索阶段**：在后续的新会话中，检测写入的恶意指令是否能被检索并影响行为（检索成功率，RSR）。

#### 6. 论文的主要结论与发现

- **攻击高度有效且持久**：内存投毒攻击在OpenClaw和HERMES上的平均成功率达 **50.46%**，检索成功率达 **41.05%**。
- **代理的内存设计越积极，越容易被利用**：HERMES拥有更宽松的写入策略和更低的压缩阈值，其ASR（**66.67%**）和RSR（**64.70%**）显著高于策略更保守的OpenClaw（ASR 34.25%, RSR 17.40%）。这揭示了**提升内存性能与确保安全性之间的矛盾**。
- **现有提示注入防御基本无效**：所有四种防御在未经过针对性训练时，均无法有效检测内存投毒攻击。它们的总体TPR（最好的是PromptArmor，**67.67%**）不理想，且对**弱信号攻击的检测能力极差**。
- **防御需要从根本上转变思路**：防御不应仅停留在输入边界（这是提示注入防御的思路），而应深入到内存写入路径，例如通过收紧写入策略、引入来源追踪、增加写后验证等手段来防御。

#### 7. 优点

- **系统性强**：是首项对LLM Agent内存投毒进行系统性分析的工作，建立了“写入通道-结构漏洞-攻击分类”的统一框架，极具开创性。
- **基础性贡献明确**：清晰阐述了内存投毒与提示注入的根本区别（前者攻击的是持久化状态，后者攻击的是当前上下文），并基于此分析了现有防御失败的根本原因。
- **实用性强**：设计了专用的MPBench基准，并提供了有价值的经验实验结果（如Agent设计越激进越脆弱），对实际Agent系统的安全设计和红队测试具有直接的指导意义。
- **分析深入**：对9个结构性漏洞的分析深入具体，直击系统本质，为后续防御研究提供了清晰可行的方向。

#### 8. 不足与局限

- **模型单一性**：所有实验仅基于一个模型（GPT-OSS-120B），无法评估攻击在其他模型（如更小参数量的模型或不同架构的模型）上的泛化能力。
- **场景简化**：部分测试用例通过“静态上下文”提供恶意负载，简化了真实复杂工具调用链中负载传递的过程，可能低估了攻击的复杂性或高估了攻击的成功率。
- **威胁模型范围有限**：作者明确将“多用户共享记忆”等更具挑战性的场景排除在外（Out of Scope），这限制了结论的适用范围。
- **对C4通道评估的局限性**：虽然HERMES支持，但对“技能程序插入”（C4）攻击的评估细节相对较少，尤其是在被毒化技能被“自我改进”循环持续放大影响方面的量化数据还不够丰富。
- **未探索对抗性防御**：论文主要聚焦于分析攻击和现有防御的失效，但并未提出或验证任何针对性的新型防御方案，对读者而言，这更像是一个需要立即解决的开放性问题。
