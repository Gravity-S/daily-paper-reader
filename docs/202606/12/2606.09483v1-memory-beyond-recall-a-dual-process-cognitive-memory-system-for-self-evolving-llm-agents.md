## 摘要
LLM智能体的长期记忆不仅仅是在正确的时间检索正确的段落。当前的记忆系统将信念修正、因果耦合和跨领域抽象压缩到一个为表面回忆优化的单一检索平面，因此在需要推理用户如何演变的隐式个性化任务中表现不佳。我们提出DCPM，它沿着认知能力层级重新组织智能体记忆，从原始输入和原子事实，通过历时信念轨迹和身份，到领域模式、潜在意图和跨领域模式。该层级由继承双过程理论架构分割的两个过程驱动：一个同步的日间写入器（系统1），记录信念修正为双向链接的替代链；一个异步的夜间引擎（系统2），归纳模式和意图，并扫描跨领域冲突，抽象为更高级别的核心模式。在LongMemEval、PersonaMem和PersonaMem-v2上，当基准测试奖励隐式跨会话推理时（PersonaMem-v2上最多提升5.20），启用系统2贡献最大，而在跨度回忆上贡献最小，这与架构预测一致。

## Abstract
Long-term memory for an LLM agent is more than retrieving the right passage at the right time. Current memory systems collapse belief revision, causal coupling, and cross-domain abstraction into a single retrieval surface tuned for surface recall, and consequently struggle on implicit personalisation that requires reasoning over how a user has evolved. We propose DCPM, which reorganises agent memory along a cognitive capability hierarchy ascending from raw inputs and atomic facts, through diachronic belief trajectories and identity, to domain schemas, latent intentions and cross-domain patterns. The hierarchy is driven by two processes inheriting the architectural split of dual-process theory: a synchronous daytime writer (System1) that records belief revisions as doubly linked supersedes chains, and an asynchronous nighttime engine (System2) that induces schemas and intentions and sweeps for cross-domain collisions abstracted into higher-level core schemas. On LongMemEval, PersonaMem and PersonaMem-v2, enabling System2 contributes most where the benchmark rewards implicit cross-session inference (up to +5.20 on PersonaMem-v2) and least on span recall, matching the architectural prediction.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有LLM长期记忆系统仅将记忆视为“存储-检索”问题，使用向量存储、时序知识图谱等机制，侧重于表面事实的召回，却无法处理用户信念随时间的演化、偏好改变的因果链路，以及跨生活领域的隐式关联。这导致智能体在面对需要推理用户如何演变的隐式个性化查询时表现糟糕。
- **背景**：作者指出，真正了解用户的智能体需要建模的不仅是用户说过什么，更要追踪信念如何被修正、偏好为何转变、以及不同领域行为背后的潜在模式。当前系统将这些维度全部压缩到单一检索平面，丢失了过渡状态、因果链接和高阶模式。
- **含义**：本文主张记忆应从“存储”范式转向“认知”范式，沿着认知能力层级组织记忆，并基于双过程理论（快思考与慢思考）分离在线编码与离线抽象，从而支持智能体的自演化。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 提出DCPM（Dual-Process Cognitive Memory），将记忆按认知能力层级组织，分为五层：原始输入与原子事实 → 身份项与历时替代链 → 域内行为模式 → 潜在意图 → 跨领域核心模式。
- 采用双过程架构：**System 1**（同步日间写入器）实时处理新交互，记录信念修正链；**System 2**（异步夜间引擎）在空闲期归纳模式、意图并检测跨领域冲突，生成高阶核心模式。

### 关键技术细节
#### System 1：同步日间写入器
- **触发方式**：仅当用户或应用显式调用`add_memory`时执行，而非每个对话轮次。
- **流程**：
  1. **原始持久化**：原始输入写入向量存储，保证零丢失。
  2. **提取**：调用提取LLM（`Me`）从当前轮次及近期历史中提取身份项和原子事实，附带类型、主题、显著性和时间戳。
  3. **调解**：调解LLM（`Mr`）将每个新项与向量存储中top-k最近项比较，判断三种操作：**ADD**（全新）、**SUPERSEDE**（矛盾或演化）、**UPDATE**（补充或扩展）。重复项静默丢弃。
  4. **批量嵌入与替代链**：对保留项批量嵌入并更新到向量存储。对于SUPERSEDE操作，添加双向指针：`n_new.supersedes = n_old`，`n_old.superseded_by = n_new`，形成双链表。旧项不被删除，保留完整演化轨迹。

#### System 2：异步夜间引擎
- **运行时机**：空闲或夜间窗口，延迟预算宽松。
- **三个阶段**：
  1. **预处理（无LLM）**：筛选尚未纳入任何模式或意图节点的事实，使用两阶段DBSCAN聚类（先语义嵌入粗聚类，再细分大簇）。每个簇计算质心嵌入，检索邻近的现有图节点。
  2. **LLM代理归纳**：为每个簇调用归纳代理`Ma`，提供四个工具：`CREATE_SCHEMA`（创建域内行为模式节点）、`CREATE_INTENTION`（创建潜在意图节点）、`ADD_EVIDENCE`（将事实ID附加到已有节点）、`ADD_EDGE`（写节点间结构边）。所有操作按序写入图`G`。
  3. **跨域扫描器**：
     - 行为抽象：每个模式节点生成简短行为描述，编码到行为-认知空间`E_beh`；同时语义嵌入`E_sem`用句子编码器生成。
     - 冲突扫描：枚举跨域模式对，若满足`cos(E_beh(si), E_beh(sj)) > θ_beh`且`cos(E_sem(si), E_sem(sj)) < θ_sem`，则标记为冲突。
     - 核心归纳：对每个冲突簇，调用核心模式LLM`Mc`归纳生成更高阶核心模式，写入图并关联成员模式。

#### 读取路径
- 推理时：查询嵌入通过向量相似度命中事实/身份存储，遍历替代链，再检索图`G`中通过`evidence_vdb_ref`边关联的模式、意图和核心模式节点。全程不调用LLM，延迟由向量搜索主导。

### 算法流程（文字说明）
- System 1写入流程（算法1）：接收`add_memory`请求 → 原始持久化 → 提取身份项和事实 → 对每个项从向量存储检索最近邻 → 调解LLM决定ADD/SUPERSEDE/UPDATE → 批量嵌入更新 → 对SUPERSEDE项写双向指针。
- System 2引擎（算法2）：预处理：筛选新事实 → 两阶段DBSCAN聚类 → 为每个簇检索上下文 → LLM归纳代理执行创建/添加/关联操作 → 跨域扫描器：对每个模式节点生成行为描述并编码 → 枚举跨域冲突 → 对冲突簇调用核心模式LLM写入。

## 6. 主要结论与发现

1. **双过程架构预测得到验证**：System 2在隐式跨会话推理任务（PersonaMem-v2）上带来显著提升（kimi-2.5下+5.20），而在简单事实回忆（LongMemEval）上提升很小（+0.33~0.44），符合“快思考用于在线记录，慢思考用于离线抽象”的理论。
2. **替代链是关键设计**：78%的正确知识更新查询触及替代链，未使用替代链的系统（如Mem0）在知识更新任务上低31%。
3. **跨域核心模式贡献最大**：仅占节点1.3%的核心模式在PersonaMem-v2上贡献约4.1个准确率点（消融实验），说明少量高阶认知结构具有高杠杆作用。
4. **DCPM full超越了长上下文基线**：在kimi-2.5上PersonaMem-v2准确率59.30%，高于长上下文基线的58.46%；在LongMemEval kimi上86.14% vs 80.61%，原因是避免了长上下文的“中间迷失”问题。
5. **读取路径无LLM调用**延迟低，与纯检索系统一致。

## 7. 优点

- **理论驱动设计**：基于发展心理学中的Theory-of-Mind和双过程理论，构建了清晰的认知能力层级，有坚实的理论依据。
- **新颖的替代链机制**：保留信念演化轨迹而非覆盖，使得读取时可恢复完整因果链，解决了更新-原地系统的核心缺陷。
- **异步System 2设计**：与System 1解耦，不增加在线延迟；通过双空间冲突检测发现表面不相似但行为模式相同的跨域关联，具有高创新性。
- **实验设计严谨**：消融实验层次分明；分析实验验证了架构的预测（技能收益不对称）；定性案例直观展示了替代链和跨域冲突的实际效果。
- **可解释性**：所有记忆节点（事实、模式、意图、核心模式）均以自然语言呈现，便于用户审查和理解。
- **伦理考虑完善**：提出了逐层同意、硬删除路径等机制，关注用户隐私和知情权。

## 8. 不足与局限

- **实验覆盖狭窄**：仅限英语聊天助手场景，未涉及跨语言迁移、具身智能体或用户身份不确定的场景。
- **算力成本未量化**：未报告System 2离线运行的具体硬件需求、运行时间，对实际部署的能力评估不够透明。

- **领域多样性假设**：假设用户行为受少量重复认知模式控制，对于单领域用户跨域核心模式收益有限；高领域多样性时二次冲突扫描成为瓶颈。
