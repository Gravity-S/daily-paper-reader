## 摘要
AI智能体依赖技能、工具和工作流组成的工具链来解决复杂问题。持续改进这一工具链对于适应新任务至关重要。然而，现有的优化方法通常需要真实标注的验证集，但在实际部署场景中，这类标签数据难以获取。为解决这一问题，我们提出回顾性工具链优化（RHO），一种仅利用过去轨迹的自监督方法。具体而言，RHO从过去的轨迹中选取多样化的具有挑战性的核心任务集，并并行重新求解。智能体通过自验证和自一致性分析这些重放轨迹，生成候选工具链更新，并通过自身的成对自偏好选择最有效的更新。我们在软件工程、技术工作和知识工作三个不同领域评估了RHO。值得注意的是，单次优化轮次使SWE-Bench Pro上的通过率从59%提升至78%，而无需任何外部评分。此外，我们的分析表明RHO有效针对先前的失败模式。因此，优化后的工具链改变了智能体的行为模式，并在长周期会话中保持更高的准确性。

## Abstract
AI agents rely on a harness of skills, tools, and workflows to solve complex problems. Continually improving this harness is essential for adapting to new tasks. However, existing optimization methods typically require ground-truth validation sets, yet such labeled data is difficult to acquire in practical deployment settings. To address this problem, we introduce Retrospective Harness Optimization (RHO), a self-supervised method that optimizes the agent harness using only past trajectories. Specifically, RHO selects a diverse coreset of challenging tasks from past trajectories and re-solves them in parallel. The agent analyzes these rollouts using self-validation and self-consistency, then generates candidate harness updates and selects the most effective one by its own pairwise self-preference. We evaluate RHO across three diverse domains, spanning software engineering, technical work, and knowledge work. Notably, a single optimization round improves the pass rate on SWE-Bench Pro from 59% to 78% without any external grading. Furthermore, our analysis demonstrates that RHO effectively targets prior failure modes. As a result, the optimized harness alters the agent's behavior patterns and sustains higher accuracy during long-horizon sessions.

## 1. 论文的核心问题与整体含义（研究动机和背景）

AI agent 在执行复杂任务时依赖于一套“工具链”（harness），包括技能、工具和工作流程。持续改进这一工具链对于适应新任务至关重要。然而，现有的优化方法通常需要带有真实标签的验证集来指导改进，但在实际部署环境中，获取这样的标注数据非常困难。另一方面，agent 在日常运行中自然会积累大量过去的任务执行轨迹（trajectories）。核心研究问题：**能否仅利用过去的轨迹（无外部标签）来优化 agent 的工具链，从而提升未来的性能？**

论文提出了一种自监督方法 RHO（Retrospective Harness Optimization），通过回顾性分析历史轨迹，利用 agent 自身的偏好来引导优化，无需任何人工标注或外部评分。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
RHO 将工具链优化视为一个三阶段的自监督流程：从历史轨迹中选择一个代表性的核心任务子集 → 对这些任务进行多路并行重放，提取自验证和自一致性诊断信号 → 根据诊断信号生成多个候选工具链更新，并通过 agent 自身的成对偏好选择最优更新。

### 关键技术细节
- **阶段一：核心集选择（Coreset Selection）**  
  使用一个语言模型（LLM）作为难度判断器，对每个历史轨迹评估难度分数 \(r_i\) 并生成摘要指纹（文本描述）。然后构建一个**行列式点过程（DPP）**核函数 \(K = \operatorname{diag}(e^{\tilde{r}}) S \operatorname{diag}(e^{\tilde{r}})\)，其中 \(S\) 是轨迹描述嵌入的余弦相似度矩阵，\(\tilde{r}\) 是归一化的难度分数。DPP 通过参数 \(\theta\) 平衡难度与多样性，选择 \(k\) 个既难又多样化的任务构成核心集 \(D_{\text{core}}\)。

- **阶段二：组重放（Group Rollout）**  
  对核心集中每个任务，并行运行 \(G\) 次 agent 求解，得到一组轨迹。然后 agent 从两个维度进行自偏好分析：
  - **自验证（self-validation）**：检查每条轨迹内部的正确性，标记错误工具调用、错误假设等。
  - **自一致性（self-consistency）**：比较不同轨迹之间是否存在矛盾，识别不一致的计划、工具序列或最终答案。
  两个维度共同生成结构化的改进指令 \(I_t\)，合并后用于后续优化。

- **阶段三：最佳 N 个工具链提案（Best-of-N Harness Proposal）**  
  基于改进指令，并行生成 \(N\) 个候选工具链 \(h_1,...,h_N\)。然后用每个候选工具链重新求解核心集任务，得到新轨迹。对每个任务，使用 agent 成对偏好评分（将新轨迹与原始工具链的轨迹比较），得到相对优势分数 \(S_j\)。选择分数最高且严格大于0的候选工具链作为最终更新。若没有候选得分>0，则保留原工具链。

## 3. 实验设计：数据集、基准、对比方法

### 数据集与场景（涵盖三个领域）
- **SWE-Bench Pro**：长期软件工程任务，需仓库级推理和多文件编辑。划分前100个任务为训练池（用于生成历史轨迹），后100个为测试池。
- **Terminal-Bench 2**：命令行任务，带可执行评分器。共89个任务，前30个为训练池，后59个为测试池。
- **GAIA-2**：动态异步环境下的知识工作。共200个场景，前100个训练池，后100个测试池。

### 基准方法
#### 无反馈基线（无需验证标签）
- **Dynamic Cheatsheet**：维护一个可重用的策略和代码片段记忆。
- **ReasoningBank**：存储可重用的推理模式，推理时检索。
- **Sleep-time Compute**：离线预处理历史轨迹成紧凑笔记，预置到 agent 上下文中。
#### 有验证反馈基线
- **Meta-Harness**：需要标注验证集，通过迭代搜索候选工具链并用验证集评分选择最佳更新。

- RHO 在所有三个基准上均取得一致提升：SWE-Bench Pro 从 0.59 提升至 0.78（+0.19）；Terminal-Bench 2 从 0.71 提升至 0.76（+0.05）；GAIA-2 从 0.29 提升至 0.37（+0.08）。

## 6. 论文的主要结论与发现

- RHO 作为一种完全自监督的方法，仅依赖历史轨迹就能显著提升 agent 工具链的性能，无需任何外部评分或验证标签。
- 优化后的工具链会新增针对历史失败模式的技能和工具（如图3所示），从而改变 agent 的行为模式——在 SWE-Bench Pro 上更频繁地验证结果，在 Terminal-Bench 2 和 GAIA-2 上更积极地执行新工具。
- 核心集选择需要同时平衡难度和多样性，单一维度无效。
- 自验证和自一致性两个诊断信号缺一不可，且显式诊断优于直接给优化器看原始轨迹。
- RHO 在长期任务上的改进尤为显著，帮助 agent 在长周期会话中保持更高的准确率。

## 7. 优点：方法或实验设计上的亮点

- **完全自监督**：不依赖任何标注验证集，解决了实际部署中的关键瓶颈。
- **全工具链优化**：RHO 可以更新指令、技能、可执行工具等多个部分，而不仅仅是记忆或提示文本。
- **利用自偏好选择**：通过 agent 自身的成对偏好来筛选候选，避免单次优化随机性的影响，提升稳定性。
- **精细的诊断信号**：自验证（检查轨迹内部正确性）和自一致性（跨轨迹对比）两个维度互补，提供具体可操作的改进指令。
- **实验设计严密**：进行了多种消融、行为分析、与有监督方法的公平比较，且代码和日志开源以确保可复现性。

## 8. 不足与局限

- **依赖可重置环境**：组重放阶段需要每个任务能多次干净地重新执行，不适合一次性或不可逆任务。
- **假设工具链可编辑**：方法要求 agent 的能力主要通过可编辑的提示、技能和工具文件体现，在某些领域可能不适用。
- **可能放大模型偏好**：RHO 使用自身判断作为优化信号，若模型存在系统性偏见或错误，可能会固化这些模式。论文在伦理声明中强调需要审计日志和人工审批。
- **计算成本**：虽然比有监督迭代搜索更高效，但对每个核心任务进行 \(G\) 次重放和候选测试，仍需要较多 agent 调用（SWE-Bench Pro 优化阶段约 103 次调用）。
- **未测试跨域泛化**：实验仅在三个独立基准上进行，未验证优化后的工具链是否能泛化到完全不同分布的任务。
- **未深入分析成功率提升与工具链内容之间的因果关系**：虽然展示了新增技能和工具，但未量化每个新增组件的独立贡献。
