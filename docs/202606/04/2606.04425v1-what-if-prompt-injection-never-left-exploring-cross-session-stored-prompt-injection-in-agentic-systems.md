## 摘要
现代智能体系统将大语言模型从会话限制的助手转变为有状态系统，通过记忆、文件系统、工具和其他长期存在的上下文制品，跨会话持久化并演化共享的世界状态。这一转变从根本上扩大了提示注入的攻击面。然而，先前关于提示注入的研究主要关注单个会话内的模型级威胁，忽视了跨会话的持久系统状态如何从根本上改变智能体系统的系统级风险。受Web系统中存储式跨站脚本的启发，我们引入了跨会话存储式提示注入，其中一次成功的注入可以持久存在于智能体系统状态中，并在原始攻击者交互结束后很长时间内静默地影响未来的执行。为了系统研究这一威胁，我们形式化了存储式提示注入，并开发了一个关于对抗性内容如何跨会话持久化并影响智能体系统的分类法。我们进一步开发了一个基准测试和沙盒工具包，用于评估存储式提示注入的风险，从而能够跨不同模型、攻击目标和持久性渠道对攻击成功率进行定量分析。我们的发现强调，持久性将提示注入从短暂的模型级威胁转变为嵌入智能体执行状态中的长期系统级漏洞。我们希望这项工作能引起对这种新兴威胁的更广泛关注，并激励社区系统性地研究和减轻智能体系统中持久性带来的系统风险。

## Abstract
Modern agentic systems transform LLMs from session-bounded assistants into stateful systems that persist and evolve shared world state across sessions through memories, filesystems, tools, and other long-lived contextual artifacts. This shift fundamentally expands the attack surface of prompt injection. However, prior works on prompt injection have largely focused on model-level threats within a single session, overlooking how cross-session persistent system state fundamentally changes the system-level risk of agentic systems. Inspired by stored cross-site scripting in web systems, we introduce cross-session stored prompt injection, where a successful injection can persist within agentic system state and silently influence future executions long after the original attacker interaction has ended. To systematically study this threat, we formalize stored prompt injection and develop a taxonomy of how adversarial content persists and affects agentic systems across sessions. We further develop a benchmark and sandbox toolkit to evaluate the risks of stored prompt injection, enabling quantitative analysis of attack success across different models, attack goals, and persistence channels. Our findings highlight that persistence transforms prompt injection from an ephemeral model-level threat into a long-lived system-level vulnerability embedded within agent execution state. We hope this work draws broader attention to this emerging threat and motivates the community to systematically study and mitigate system risks arising from persistence in agentic systems.

## 2. 论文提出的方法论：核心思想、关键技术细节
### 核心思想
- 将SPI的生命周期分为两个阶段：**注入阶段**（恶意内容写入持久化状态）和**激活阶段**（持久化状态在未来会话中被重新加载并影响Agent行为），二者在时间和会话上解耦。
- 形式化定义：存储式提示注入是指对抗性内容`p`写入或修改持久化上下文`P`，并在后续执行中被上下文构造函数`A`纳入Agent输入`x_t = A(q_t, ·, P)`，从而被模型`M`解释并改变Agent行为。

### 关键技术细节
1. **系统模型**：将Agent系统建模为上下文构造管道，输入由多个通道组成：
   - 用户查询上下文`q_t`、指令上下文`I`、会话历史`H_t`、工具接口上下文`T`、检索上下文`R_t`、文件支持的上下文`S_t`。
   - 按持久性和整合可能性分为三类：**瞬时上下文**、**条件持久上下文**、**强持久上下文**。

2. **威胁模型**：
   - 攻击者能力：通过用户输入、外部消费内容、供应链工具等写入原语，将恶意内容写入持久化上下文通道。
   - 攻击者目标：建立持久影响力，而非单次劫持。
   - 攻击者约束：不能直接修改模型、上下文构造逻辑或强制加载。

3. **分类法（Taxonomy）**：三个维度：
   - **注入源**：用户提供内容、外部消费内容、供应链可安装工具。
   - **持久化通道**：Agent记忆（工作记忆/档案记忆）、工具可见状态（工具描述/工具响应结果）、文件支持上下文（约定文件如AGENTS.md/普通工作空间文件）。
   - **上下文整合机制**：直接加载（无条件包含，如系统提示、AGENTS.md）与条件加载（需检索或主动访问）。
   - **攻击危害类别**：事实操纵、偏好操纵、行动范围操纵。

4. **评估框架**：定义阶段式成功率指标：
   - 写入成功率（WSR）= 写入持久化通道次数 / 总尝试次数
   - 整合率（IR）= 被纳入Agent上下文的次数 / 写入次数
   - 激活率（AR）= 产生攻击者意图结果的次数 / 被纳入次数
   - 端到端攻击成功率（E2E-ASR）= WS × IR × AR

## 6. 论文的主要结论与发现
1. **SPI普遍有效**：三个模型E2E-ASR在32%–42%之间，显示跨会话存储式提示注入是严重威胁。
2. **三个阶段解耦**：写入成功率高不保证整合和激活成功，模型在不同阶段表现各异（如MiniMax-M2.7写入率86.4%但激活率72.5%；GLM-5.1写入率64.2%但激活率86.1%）。
3. **事实操纵最易成功**：AR达到100%，注入的事实被模型无条件接受；偏好操纵最困难（E2E-ASR近0%），因为模型优先遵循显式用户约束；行动范围操纵因模型而异（AR 56%–92%）。
4. **直接加载通道风险最大**：工作记忆和文件支持上下文（如AGENTS.md）因无条件整合，E2E-ASR高于条件加载通道（档案记忆）。
5. **上下文伪装提高写入成功率**：将payload伪装为合法策略比朴素注入更易被写入持久化状态，但对后续整合和激活帮助有限，瓶颈主要在写入阶段。
6. **场景差异有限但存在**：GPT-5-mini在金融场景中表现显著更低，暗示某些模型在敏感领域可能存在更强的安全机制。
