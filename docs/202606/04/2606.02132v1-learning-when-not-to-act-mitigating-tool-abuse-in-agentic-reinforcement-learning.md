## 摘要
智能体强化学习可能引发工具滥用，即模型过度使用外部工具，即使对于可通过内部推理解决的查询也是如此。现有方法通过统一的工具使用惩罚或硬性限制来缓解此问题，这虽然减少了工具使用频率，但也可能抑制有用的工具辅助探索。我们提出了EAPO——一种高效的智能体策略优化框架，能够学习选择性使用工具。EAPO在每个批次中引入无工具轨迹，应用难度感知的奖励塑造来主要对简单查询的冗余工具调用进行惩罚，并使用置信度感知的令牌重加权来改进策略学习。在九个数学和知识密集型推理基准测试中，EAPO在Qwen2.5-3B、Qwen2.5-7B和Llama3.1-8B上持续改善了准确率-效率权衡。与GRPO相比，EAPO平均性能分别提升了10.45%、7.27%和9.69%，同时平均工具调用次数分别减少了18.33%、18.33%和24.59%。这些结果表明，智能体能够学会在何时不使用工具，而不会损害工具集成推理。

## Abstract
Agentic reinforcement learning can induce tool abuse, where models overuse external tools even for queries solvable by internal reasoning. Existing approaches mitigate this issue with uniform tool-use penalties or hard limits, which reduce tool frequency but may also suppress useful tool-assisted exploration. We propose EAPO, an Efficient Agentic Policy Optimization framework that learns selective tool use. EAPO introduces tool-free trajectories into each rollout group, applies difficulty-aware reward shaping to penalize redundant tool calls mainly on easier queries, and uses confidence-aware token reweighting to improve policy learning. Across nine mathematical and knowledge-intensive reasoning benchmarks, EAPO consistently improves the accuracy efficiency trade-off on Qwen2.5-3B, Qwen2.5-7B, and Llama3.1-8B. Compared with GRPO, EAPO improves average performance by 10.45%, 7.27%, and 9.69%, while reducing average tool calls by 18.33%, 18.33%, and 24.59%, respectively. These results show that agents can learn when not to use tools without compromising tool-integrated reasoning.

# 论文总结：Learning When Not to Act: Mitigating Tool Abuse in Agentic Reinforcement Learning

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：在智能体强化学习中，模型倾向于过度使用外部工具（工具滥用），即使对简单查询（如1+1=？）也调用计算器或搜索引擎，导致不必要的计算成本、延迟和部署开销。
- **现有方法局限**：统一惩罚工具调用次数（如OTC）或硬限制次数，会抑制模型对困难探索的有用工具使用，难以区分“必要”与“滥用”。
- **本文目标**：让智能体学习**何时不使用工具**，而非减少工具使用总量，平衡准确性与效率。

## 2. 方法论（EAPO框架）
### 核心思想
在策略优化过程中，通过**对比有无工具的经验轨迹**，让模型学会选择性调用工具。引入三个组件：

- **Efficiency-Aware Rollout（效率感知轨迹生成）**  
  每组 rollout 中，确保至少 n 条轨迹**禁止使用工具**（通过修改提示和禁用工具执行）。这使得模型可以观察自身内部推理能否解决该查询，从而为区分必要与滥用提供直接信号。

- **Difficulty-Aware Reward Shaping（难度感知奖励塑造）**  
  定义查询难度 $d(q) = 1 - \frac{1}{M}\sum_{i=1}^M \mathbb{I}(T_i)$ （正确轨迹比例补数）。调整每条轨迹奖励：  
  $\tilde{R}_i = [d(q) + (1-d(q))\cdot \varepsilon(T_i)] \cdot R_i$  
  其中效率系数 $\varepsilon(T_i)=\exp(-\beta(c_i - c_{\min}))$，$c_i$ 为工具调用数，$c_{\min}$ 为正确轨迹中最小调用次数。  
  ⟹ 在简单查询（$d(q)$小）上加强工具惩罚，在困难查询（$d(q)$大）上保留探索空间。

- **Confidence-Aware Advantage Reweighting（置信度感知优势重加权）**  
  对每个参与策略学习的 token，根据模型生成置信度赋权：  
  - 正确轨迹中，低置信度 token（低于 $\rho$ 分位数）给予上权重 $W_{\text{pos}}$，促使模型关注不确定部分。  
  - 错误轨迹中，高置信度 token（高于 $1-\rho$ 分位数）给予下权重 $W_{\text{neg}}$，纠正过度自信。  
  重加权后优势 $A_{i,u,t} = \omega_{i,u,t} \cdot A_i$，代入策略梯度优化。

### 算法流程
1. 对每个查询，生成 n 条无工具轨迹和 M-n 条有工具轨迹。
2. 计算每条轨迹的正确性、工具调用数、原始奖励。
3. 计算难度 $d(q)$ 和 $c_{\min}$，按公式调整奖励。
4. 计算组内优势。
5. 对每个 token 计算置信度，按规则赋权，得到重加权优势。
6. 结合 KL 散度正则化更新策略参数。

## 6. 主要结论与发现
- **EAPO 显著提升准确率-效率权衡**：相比 GRPO，平均性能提升 10.45%（3B）/7.27%（7B）/9.69%（8B），工具调用减少 18.33%/18.33%/24.59%。
- **简单提示（TIR）不足以有效多工具推理**，甚至可能不如直接推理。
- **效率增益在小模型上较小**，因小模型本身冗余工具使用较少。
- **难度感知惩罚有效**：在训练初期（模型准确率低时）工具惩罚影响小，避免过早压制探索。
- **置信度重加权适度提升性能**，但过度权重大会损害优化。

## 7. 优点
- **方法创新**：通过强制插入无工具轨迹提供直接对比信号，而非仅惩罚次数，从而区分必要与滥用。
- **难度适应**：奖励塑造自动根据查询难度调整惩罚强度，避免“一刀切”。
- **信息聚焦**：置信度重加权关注信息密度高的 token，提升学习效率。
- **实验充分**：覆盖多种模型规模、多任务、多基线，消融清晰，结果可复现性强。
- **实用价值高**：在保持甚至提升准确率的同时大幅减少工具调用，适用于成本敏感部署。
