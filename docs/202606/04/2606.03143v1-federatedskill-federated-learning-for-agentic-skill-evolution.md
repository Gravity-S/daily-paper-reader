## 摘要
现代LLM智能体日益依赖技能库来处理复杂任务，使得技能演进成为自我提升的主要驱动力。然而，孤立的单用户任务流缺乏构建全面技能所需的多样性。尽管跨用户协作可以克服这一数据瓶颈，但当前的轨迹共享方法会损害用户隐私，并强加一个统一的全局库，无法适应客户端异质性。我们提出了FederatedSkill，一种用于协作智能体演进的隐私保护框架。超越原始轨迹共享，FederatedSkill利用语义技能差异（对本地库的结构化补丁）作为通信的基本单元。在服务器端，一个演进智能体聚合这些补丁，动态建模客户端特定的能力边界，促进严格个性化的技能演进，而非次优的全局平均。在20个不同的智能体任务族上评估，FederatedSkill相对于自我演进基线取得了显著提升，成功率最高提升44.4%，计算成本降低37.5%。

## Abstract
Modern LLM agents increasingly rely on skill libraries to handle complex tasks, making skill evolution a primary driver of self-improvement. However, isolated single-user task streams lack the diversity required to build comprehensive skills. While cross-user collaboration can overcome this data bottleneck, current trajectory-sharing approaches compromise user privacy and impose a uniform global library that fails to accommodate client heterogeneity. We introduce FederatedSkill, a privacy-preserving framework for collaborative agent evolution. Moving beyond raw trajectory sharing, FederatedSkill utilizes semantic skill diffs, structured patches over local libraries, as the fundamental unit of communication. On the server side, an evolution agent aggregates these patches to dynamically model client-specific capability boundaries, facilitating strictly personalized skill evolution rather than a suboptimal global average. Evaluated across 20 distinct agent task families, FederatedSkill demonstrates substantial gains over self-evolving baselines, achieving up to a 44.4% increase in success rate and a 37.5% reduction in computational cost.

### 1. 核心问题与整体含义（研究动机与背景）

-   **核心问题**：现代大型语言模型（LLM）智能体（Agent）广泛依赖“技能库”来处理复杂任务，技能库的持续演进（添加、修改、删除技能）是智能体自我提升的关键。然而，单用户的本地任务流数据多样性不足，导致技能库覆盖不全面，形成“数据瓶颈”。
-   **现有方案的缺陷**：
    -   **隐私风险**：现有的跨用户协作演进方法（如共享原始轨迹）会导致敏感信息泄露。
    -   **异质性挑战**：简单聚合所有用户的轨迹生成一个“全局技能库”是次优的。不同用户使用的模型、框架和任务分布差异巨大（即客户端异质性），一个全局库无法满足所有人的个性化需求。
-   **研究意义**：本文旨在解决一个核心问题：**如何在保护用户隐私的前提下，让多个用户协作演进各自的技能库，并最终为每个用户生成高度个性化的、而非通用的技能库？** 这填补了隐私保护和个性化在智能体协作演进领域的空白。

### 2. 方法论：FederatedSkill框架

-   **核心思想**：借鉴联邦学习（Federated Learning）的思想，不直接传输原始数据（轨迹），而是让客户端本地生成并只上传“语义技能差异（skill patch）”，服务器端则负责基于这些补丁，为每个客户端模型定制个性化的演进方案。
-   **关键技术细节**：
    1.  **客户端：本地执行与补丁蒸馏**
        -   **本地执行**：每个客户端基于自身模型（`ρi`）和当前技能库（`Lti`）执行本地任务，收集轨迹批次（`Bti`）。
        -   **补丁生成**：一个“本地打补丁器（patcher）”将原始轨迹蒸馏成一个结构化的**技能补丁（δti）**。补丁仅包含对技能库的操作，如 `ADD`、`EDIT`、`DELETE`，而非轨迹细节。
        -   **隐私保障**：补丁被设计为“语义级”的，明确被提示要泛化任务通用经验、省略特定数值，且所有路径都经过安全校验。原始轨迹、任务输入等敏感数据永远留在本地，从设计上避免了数据泄露。
    2.  **服务器端：个性化演进**
        -   服务器端的**演进智能体**在部分可观测马尔可夫决策过程（POMDP）框架下工作，它只能看到补丁，必须从中推断每个客户端的能力边界。
        -   **第一阶段：演进计划生成**
            -   **能力矩阵**：根据所有客户端的补丁，构建一个任务工作流(workflow) × 客户端的矩阵，标记每个工作流在每个客户端上的状态（覆盖、吸收、损坏、空白）。
            -   **双层能力记忆**：
                -   **高层次（共享）**：记录整个家族中所有客户端共同面临的、仍未解决的普遍性问题。
                -   **低层次（私有）**：针对每个客户端，记录其特定的失败模式（如模型类型导致的特定工具误用）。
            -   **演进指令**：基于能力矩阵和记忆，生成针对每个客户端的、具体可操作的演进指令。
        -   **第二阶段：个性化库更新**
            -   根据计划，演进智能体为每个客户端执行指令，将其他客户端成功但不可直接复制的技能进行“改写”，以适应目标客户端的模型和框架，最终生成个性化更新（`∆ti`），并更新本地库（`Lt+1i`）。

### 3. 实验设计

-   **数据集与基准**：使用 **SkillFlow** 基准测试平台。该平台包含 20 个多样化的任务族，每个族中的任务按难度递增排序，专门用于评测智能体的终身技能发现与演化能力。这是当前最适合评测该问题的基准之一。
-   **对比方法**：
    -   **自我演进（Self-Evolution, SE）**：这是主要的基线方法，即每个客户端完全不进行任何交流，仅依赖自身轨迹进行技能库的本地演进。
    -   **消融基线（Global）**：为了验证个性化的重要性，作者设立了一个变体。该变体保留补丁共享协议，但服务器端为所有客户端生成一个统一的、非个性化的全局技能库。
-   **实验场景**：
    -   **同质场景**：所有客户端使用相同的模型和框架 (GLM-5 + Claude Code)。
    -   **异质骨干网络**：客户端使用不同的骨干模型（Qwen, GLM-5, Kimi K2.5），但框架相同。
    -   **异质骨干网络与框架**：客户端不仅在模型上不同，还在代理框架上不同（Qwen Coder, Claude Code, Kimi CLI），这是最极端的异质场景。

### 6. 主要结论与发现

1.  **显著性能提升**：相比“自我演进”基线，FederatedSkill在所有场景下都取得了一致性、且显著的性能提升。在同质场景下平均成功率提升8.4个百分点；在最复杂的“异质骨干与框架”场景下，平均提升4.82个百分点。单任务族最高提升44.4%。
2.  **显著的效率提升**：在所有客户端上，每试错（trial）成本和每解决任务成本均有显著下降。在“异质骨干与框架”场景下，Kimi K2.5的每任务成本降低了37.5%。
3.  **个性化的必要性**：通过消融实验（全局库 vs 个性化库）证明，提供个性化的技能库是取得优异表现的关键。个性化方案比全局库方案平均高出9.8-12.2个百分点。
4.  **有效的隐私保护**：设计了一套严格的隐私审计流程，证明FederatedSkill的补丁传输方式将敏感实体泄露率从直接传输摘要的52%降低到只有0.09%（广播回客户端的补丁），几乎与随机选择的无关补丁无异。
5.  **技能库更优形态**：FederatedSkill能生成更紧凑、高效的技能库，避免了自我演进中出现的“库膨胀”（生成过多冗余技能）或“库停滞”（技能增长缓慢）问题。

### 7. 优点

-   **创新的范式**：首次将联邦学习的思想成功应用于智能体技能演进领域，提出了用“语义技能补丁”替代“原始共享”的隐私保护框架，这是一个重要的范式创新。
-   **解决核心矛盾**：同时优雅地解决了协作技能演进中“隐私”和“个性化”两大核心矛盾，并进行了量化验证。
-   **高度可解释性与审计性**：传输的是可读的纯文本补丁和JSON文件，决策过程有详细的日志记录（如`DECISIONS.md`），整个过程透明、可审计，不同于传统联邦学习中的梯度黑箱。
-   **极低通信成本**：与动辄成百上千兆字节的模型参数传输相比，整个20个任务家族的8轮演进过程，通信总量仅为约3.49MB，实用性极强。

### 8. 不足与局限

-   **可扩展性未知**：实验仅涉及3个客户端，框架面临数十、数百个客户端时的协调、融合效率及涌现行为尚不明确，存在严重的信息“去重”和“聚合”挑战。
-   **对抗性环境风险**：当前框架假设环境是可信的。没有考虑恶意客户端上传虚假或有害补丁的情况，这在实际部署中是一个重大的安全风险。
-   **性能退化风险**：实验中发现，在特定任务上（如Kimi K2.5在处理新增子域OCR时），其他客户端的贡献会因其硬编码的经验而“锚定”住接收方，导致性能退步。这表明框架在处理新颖任务时可能对已有经验过度依赖。
-   **系统级优化缺失**：论文自称是一项概念验证工作，未对补丁传输的带宽效率、存储等实际工程问题进行优化，未来可以在这一方向上进行改进。
