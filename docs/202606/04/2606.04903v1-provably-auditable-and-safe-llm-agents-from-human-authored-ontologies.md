## 摘要
我们提出了LLM智能体架构Agentic Redux，旨在用于需要线性可审计性的非平凡问题领域。利用类型化lambda演算，我们证明，在适当的领域上运行时，Agentic Redux的执行在语义上保证正确，所有决策记录在仅追加账本中。我们提出了两个生产级适用领域：医疗账单合规性和安全漏洞披露。在支持代码仓库中提供了在两个领域上运行Agentic Redux的工作代码。我们还引入了本体优先智能体设计，这是一种在问题领域上创建智能体框架的方法，其中人类专家使用基本形式本体对问题领域进行本体化，然后分配LLM以推导出智能体和环中人可以扮演的角色，以便处理该领域中的问题。

## Abstract
We introduce the LLM agent architecture Agentic Redux, intended for use with nontrivial problem domains that require linear auditability. Using the typed lambda calculus, we prove that, run on appropriate domains, Agentic Redux executions are semantically guaranteed to be correct, with all decisions recorded in an append-only ledger. We present two production-grade appropriate domains, in healthcare billing compliance, and security vulnerability disclosure. Working code for Agentic Redux run on both domains is available in a supporting code repository. We also introduce Ontology-First Agent Design, a methodology for creation of agent frameworks on a problem domain, in which a human expert ontologizes the problem domain with Basic Formal Ontology, and then assigns an LLM to derive roles that agents and humans-in-the-loop can fill, in order to work the problems in the domain.

#### 2. 论文提出的方法论

论文的核心贡献是提出了**Agentic Redux架构**和**Ontology-First Agent Design（本体优先智能体设计）** 方法论。

- **核心思想：Agentic Redux 架构**
    - **灵感来源**：受前端状态管理器Redux启发。核心是“智能容器，哑组件”，将状态变更的决定权集中化。
    - **架构组成**：系统包含一个**元智能体**和多个**子智能体**。
        - **子智能体**：每个子智能体只能看到全局状态的一个“切片”（state slice）。它们独立地评估自己的任务，并向元智能体**提议**新的动作。
        - **元智能体**：拥有完整的全局状态视图，是系统中唯一有权限修改全局状态的实体。它负责裁决（adjudicate）所有子智能体的提议。
    - **关键技术细节：不变量（Invariant）**
        - 不变量是在系统整个生命周期中必须永远为真的规则，例如“系统总花费不得超过10万美元”。
        - 元智能体的裁决函数被编写成**必须保留所有不变量**。一个提议只有通过所有不变量的检查，才会被批准并执行。
        - **类型化lambda演算证明**：论文使用类型化lambda演算形式化定义了Agentic Redux，并证明了**定理1：不变量保留**，即系统在任何执行路径下的所有可达状态都满足所有不变量。这提供了语义上的安全保障。
    - **线性可审计性**：
        - 元智能体维护一个**仅追加（append-only）的账本**。每次裁决（批准或拒绝）都会生成一条日志条目。

- **方法流程：Ontology-First Agent Design**
    1.  **本体化问题域**：人类专家使用**基本形式本体**对目标问题域进行建模，定义其中的实体、过程和角色。
    2.  **评估领域指纹**：让LLM评估本体是否存在“写入偏斜（Write Skew）”和“可审计性”等关键特征。
    3.  **转换为域模块**：如果符合，让LLM根据本体自动生成代码，即“域模块”。包括定义不变量、子智能体的提议函数和元智能体的裁决函数。
    4.  **运行与完善**：在Agentic Redux内核上运行域模块，人类专家审查结果并迭代优化本体。
