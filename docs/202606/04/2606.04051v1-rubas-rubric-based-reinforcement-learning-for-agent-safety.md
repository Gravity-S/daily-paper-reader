## 摘要
大型语言模型演变为具备工具能力的智能体，这带来了一类与真实世界执行相关的新安全挑战，而不仅仅是简单的文本生成。现有的对齐方法通常依赖于粗粒度的拒绝信号或静态监督，使得在多样化的智能体风险中难以平衡安全与有用的工具执行。我们提出了RUBAS，一种基于评分准则的强化学习框架，用于智能体安全。RUBAS将智能体行为分解为四个维度：工具使用安全、参数安全、响应安全和有用性。这些结构化的评分准则在完整的智能体轨迹上提供细粒度和可解释的奖励，使得强化学习能够在保持任务完成的同时优化安全工具使用。在多个智能体安全基准和模型上的广泛实验表明，RUBAS在安全性上优于标准对齐基线，减少了工具相关的幻觉，并保持了有竞争力的效用。我们的结果表明，多维评分准则奖励为在安全关键的工具使用环境中对齐语言智能体提供了有效的训练信号。

## Abstract
The evolution of LLMs into tool-enabled agents creates a new class of safety challenges associated with real-world execution rather than simple text generation. Existing alignment methods often rely on coarse refusal signals or static supervision, making it difficult to balance safety with useful tool execution across diverse agentic risks. We introduce RUBAS, a rubric-based reinforcement learning framework for agent safety. RUBAS decomposes agent behavior into four dimensions: tool-use safety, argument safety, response safety, and helpfulness. These structured rubrics provide fine-grained and interpretable rewards over complete agent trajectories, enabling reinforcement learning to optimize safe tool use while preserving task completion. Extensive experiments across multiple agent safety benchmarks and models show that RUBAS improves safety over standard alignment baselines, reduces tool-grounded hallucinations, and maintains competitive utility. Our results suggest that multi-dimensional rubric rewards provide an effective training signal for aligning LLM agents in safety-critical tool-use settings.

- **整体含义**：需要一种能够对智能体完整的执行轨迹进行多维度、细粒度、可解释评估的奖励机制，并通过在线强化学习优化行为，以同时提升安全性和有用性。

## 2. 论文提出的方法论

### 核心思想
- **RUBAS**：基于评分准则（Rubric）的强化学习框架。将智能体行为分解为四个可验证的维度，每个维度包含一组二值准则。这些准则组装成轨迹级别的奖励信号，用于GRPO（组相对策略优化）在线训练。

### 关键技术细节
- **四维评分准则**：
  - **工具使用安全**：检查是否调用了禁止工具、是否遵守调用顺序、是否在合理次数内等。
  - **参数安全**：确保传入工具的参数不包含敏感或违规内容、参数值在有效范围内。
  - **响应安全**：评估最终用户输出是否包含有害内容，以及在必要时是否拒绝。
  - **有用性**：衡量任务完成程度和回复质量（仅用于良性或敏感场景）。
- **准则实现**：除“拒绝判断”使用Qwen3Guard模型外，其他准则均为程序化确定性检查（如正则匹配、工具名列表对比）。
- **奖励函数**：
  - 完整性奖励 \( R_c \)：轨迹是否在token预算内正常结束。
  - 推理奖励 \( R_r \)：防止奖励作弊（省略推理过程）。
  - 评分准则奖励 \( R_{rubric} \)：各维度加权归一化后求和。
  - 最终奖励：完整轨迹则 \( R_{rubric} + R_r \)，不完整则 –0.5。
- **训练算法**：GRPO（组相对策略优化），每个prompt采样8条轨迹，将得分转化为组内相对优势更新策略。

## 6. 论文的主要结论与发现

1. **RUBAS显著提升安全性**：在Qwen3-8B上平均风险从52.7%降至15.9%，优于所有非退化基线；Qwen3-14B和GLM-4.7-Flash上也取得最佳或次佳安全-效用权衡。
2. **保持效用**：RUBAS在BFCL上得分与基线相当甚至更高（如GLM上71.0%为最高），未因过度保守而牺牲工具调用能力。
3. **减少幻觉**：在ToolBeHonest上RUBAS得分最高（Qwen3-8B：61.6），表明细粒度奖励鼓励更谨慎的工具使用和结果核实。
4. **多维奖励与在线RL缺一不可**：粗粒度奖励（Rule/GuardModel）监督不足，离线方法（SFT/DPO）泛化差或易退化；两者结合才能实现最佳安全对齐。
5. **各维度均有贡献**：移除任意维度均带来特定影响，尤其响应安全和工具使用安全是安全性的关键防线，参数安全调节安全-效用平衡，有用性防止过度拒绝。

## 7. 优点

- **结构化可解释性**：评分准则将抽象安全要求转化为可检查的二进制项，奖励计算透明、可审计。
- **轨迹级细粒度监督**：不仅监督最终响应，还监督中间工具选择、参数构造和行为序列。
- **无需人类偏好标注**：通过LLM生成评分准则并程序化验证，可大规模扩展。
- **在线自适应**：GRPO使策略在探索中不断优化，适应新出现的错误模式。
- **通用性**：在多个模型族（Qwen、GLM）上均有效，且能兼容非思维链模型（未验证但设计通用）。

## 8. 不足与局限

- **合成数据依赖**：训练数据完全基于GPT-5.1生成，可能无法全面反映真实用户意图、动态环境和领域特定策略。需要进一步用真实工具和人工指令验证。
- **评分准则质量依赖生成器**：奖励信号质量直接受GPT-5.1的能力、校准和偏见影响。尽管做了可靠性验证，但模型偏差可能传递到策略。
- **仅限思维模式模型**：实验均在启用思考模式（thinking mode）的模型上进行，未验证无结构化推理的模型是否同样有效。
- **对抗泛化有限**：可能无法防御训练分类之外的新对抗攻击。需要结合运行时监控、人工确认等外部安全措施。
- **计算开销**：在线RL需要多次与环境交互，对资源要求较高（8卡A800，3.5小时/模型）。
- **环境仿真简化**：工具环境是模拟的，未考虑真实网络延迟、错误反馈、并发等复杂因素。
