## 摘要
LLM智能体通过一个共同的生成通道处理可信指令、检索记录和工具观察结果。这混淆了数据流与权限：即使没有应用策略授权，一个不可信的字符串也可能影响包含秘密的响应或行动提议。我们提出AgentSecBench，作为该问题的一个形式化安全框架的经验性实例化。该框架定义了三个博弈——指令完整性、检索保密性和能力完整性——基于一个共同的意图到执行无干扰概念，并允许泄露。它将应用策略表示为对授权观察和能力的投影，区分提示注释与强制执行投影，并同时衡量对抗优势以及防御是否在生成前关闭了相关的模型可见通道。精确标记实验有意作为博弈的一个可观测实例，而非完整的语义安全声明：它们通过明确的地面真实值来测试泄露和禁止动作的区分。我们使用Qwen3-0.6B和Qwen3-1.7B在成对的对抗性和良性控制执行上评估了六种防御类别。测量结果显示了风险降低何时遵循通道关闭，以及何时模型可见的对抗能力仍然可被利用。结果是一种面向安全的评估方法：提示文本可以描述一个边界，而来源投影、能力限制和输出验证可以强制执行一个边界。

## Abstract
LLM agents process trusted instructions, retrieved records, and tool observations through a common generative channel. This conflates data flow with authority: an untrusted string can affect a secret-bearing response or an action proposal even when no application policy authorizes that influence. We introduce AgentSecBench as an empirical instantiation of a formal security framework for this problem. The framework defines three games-instruction-integrity, retrieval-confidentiality, and capability-integrity-under a common notion of intent-to-execution noninterference with permitted leakage. It represents an application policy as a projection onto authorized observations and capabilities, distinguishes prompt annotations from enforcing projections, and measures both adversarial advantage and whether a defense closes the relevant model-visible channel before generation. The exact-marker experiments are intentionally one observable instantiation of the games rather than a complete semantic security claim: they test disclosure and forbidden-action distinguishers with unambiguous ground truth. We evaluate six defense classes with Qwen3-0.6B and Qwen3-1.7B on paired adversarial and benign-control executions. The measurements show when risk reduction follows channel closure and when a model-visible adversarial capability remains exploitable. The result is a security-oriented evaluation method: prompt text can describe a boundary, whereas provenance projection, capability restriction, and output validation can enforce one.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程（用文字说明）

**核心思想**：将智能体安全视为非干扰特性——在固定可信指令和允许泄露的情况下，未授权观察不应改变受保护的输出谓词。

**关键技术细节**：

## 6. 论文的主要结论与发现

**主要结论**：
1. **通道闭合至关重要**：组合投影防御（来源门控+能力投影+过滤）闭合了所有测量通道，攻击成功率为0。分隔符强化（纯注释）闭合率为0，攻击成功率高于基线（0.438 vs 0.375）。
2. **行为抵抗 ≠ 安全**：过滤器和最小权限防御部分闭合通道并降低攻击成功率，但残留风险仍存在。纯注释防御可以改变模型行为但未闭合通道，其安全依赖于模型行为而非机制。
3. **区分游戏**：在检索保密性游戏中，来源投影完全消除了金丝雀泄露（泄漏率0），而分隔符留下泄露率0.562。在工具完整性游戏中，能力投影将攻击优势从0.938降至0。
4. **区分注释与强制执行**：论文展示了提示注释（分隔符）和投影视（来源/能力门控）在通道闭合上的根本差异。前者的安全声明依赖于模型行为，后者是可审计的机制。
5. **风险分解**：当通道闭合时（组合栈），残留风险为0；当通道开放时（无防御、分隔符），风险来自模型对可见未授权标记的响应。

**关键发现**：
- 在无防御基线中，工具完整性博弈的攻击优势高达0.938（模型几乎总是执行未授权动作）。
- 分隔符强化在某些博弈中反而增加了攻击成功率（指令完整性从0到0.062，检索保密性从0.188到0.562），可能因为标记更显眼。
- 组合栈在保持中等效用（0.335）的同时实现了零风险。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

- **实验规模小**：每个游戏仅8个实例，模型仅两个小模型，统计效力有限。结果可能不能推广到更大模型或更复杂场景。
- **精确标记限制**：使用精确匹配作为保护谓词，无法检测同义替换、意译或编码变体。这低估了实际风险，且非发射不证明安全。
- **单一攻击模型**：仅测试单轮、单次观察的直接注入。未覆盖多轮自适应攻击、链式工具调用、隐藏编码（如unicode攻击）或语义变体。
- **缺乏语义效用评估**：效用是简单的词汇代理（任务词出现次数），不捕捉生成质量或任务完成度。可能高估或低估真实效用损失。
- **未测试更大的模型或随机解码**：Qwen3-0.6B/1.7B规模小，仅使用贪婪解码。指令微调、模型规模、采样策略可能显著改变p_open和p_closed。
- **投影实现理想化**：来源门控假设干净租户标签和显式能力名称。实际检索系统可能跨租户混合块、误标或排名注入段落。能力名称可能被模糊引用（如“发送邮件”而非精确标记）。
- **未覆盖后生成验证**：论文仅评估提议的动作（文本输出），未执行动作。对于能力完整性，真正的安全需要后生成验证器（validator）。论文在理论中提及r因子，但未实验验证。
- **偏差风险**：控制实例由评估者生成，可能与真实场景分布不同。攻击模板是透明的，可能有利于模式匹配过滤。未测试对抗学习或适应性对手。
- **可重复性**：没有说明随机种子（虽然确定贪婪解码），也没有提供代码仓库或数据（仅在前言中描述）。资源消耗未知。

总体而言，AgentSecBench提出了一个有价值的评估框架和概念区分，但其实验作为概念验证，还不足以作为实际系统安全性的唯一依据。
