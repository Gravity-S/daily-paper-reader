## 摘要
自我进化的多智能体系统（MAS）已成为让LLM智能体从经验中持续改进的一种有前景的途径，其核心是持久性记忆。然而，现有设计几乎都采用跨智能体共享的集中式存储库，这带来了通信和协调开销，引发了隐私问题，并削弱了智能体的多样性。我们提出DecentMem，一种去中心化记忆框架，其中每个智能体维护自己的双池记忆——一个由整合的过去轨迹构成的利用池，以及一个由LLM生成的针对未见上下文的候选探索池。这两个池基于来自LLM作为评判者的阶段性反馈进行在线重新加权。理论上，我们证明该设计保证了解决方案空间的全局可达性，并实现了$O(\log T)$的累积遗憾，与随机赌臂下界常数阶匹配。在实践中，跨越三个MAS框架（AutoGen、DyLAN、AgentNet）、三个Qwen3骨干网络（4B/8B/14B）、两个Gemma4骨干网络（E2B/E4B）以及涵盖数学、代码、问答和具身任务的五个基准测试，DecentMem在最强集中式记忆基线上平均准确率提升高达23.8%，在无记忆基线上提升高达52.5%，同时将令牌使用量减少高达49%。

## Abstract
Self-evolving multi-agent systems (MAS) have emerged as a promising route to LLM agents that continually improve from experience, with persistent memory at their foundation. However, existing designs almost exclusively adopt a centralized repository shared across agents, incurring communication and coordination overhead, raising privacy concerns, and collapsing agent diversity. We propose DecentMem, a decentralized memory framework in which each agent maintains its own dual-pool memory -- an exploitation pool of consolidated past trajectories and an exploration pool of LLM-generated candidates for unseen contexts. The two pools are reweighted online based on stage-wise feedback from an LLM-as-a-judge. Theoretically, we prove that this design guarantees global reachability of the solution space and achieves $O(\log T)$ cumulative regret, matching the stochastic bandit lower bound up to constants. In practice, across three MAS frameworks (AutoGen, DyLAN, AgentNet), three Qwen3 backbones (4B/8B/14B), two Gemma4 backbones (E2B/E4B) and five benchmarks spanning math, code, QA, and embodied tasks, DecentMem improves average accuracy by up to 23.8% over the strongest centralized memory baseline and by up to 52.5% over the no-memory baseline, while reducing token usage by up to 49%.

- **整体含义**：本文质疑了集中式记忆作为MAS默认架构的合理性，提出了一种全新的、更优的默认设计范式——**去中心化记忆**。

## 2. 方法论：DecentMem
### 2.1 核心思想
DecentMem 提出，为每个智能体维护一个**私有**的**双池记忆**，通过在线路由机制动态平衡“利用”与“探索”，使每个智能体能够独立地从自身经验中学习和进化，从而保持多样性并提升整体性能。

### 2.2 关键技术细节
- **双池记忆架构**：
    - **利用池 (E-pool)**：存储来自过去任务的成功轨迹。用于从历史相似经验中进行检索和复用（局部搜索）。
    - **探索池 (X-pool)**：由LLM在当前未见上下文中动态生成的候选策略。用于探索全新策略，打破局部最优（启发式传送）。
    - 每个记忆片段不仅包含动作，还包含协作轨迹，作为动作和协调先验。

- **在线路由机制**：
    - 在每个执行阶段，智能体的路由器根据当前E-pool和X-pool的权重 (`w_E`, `w_X`) 决定从哪个池中提取信息。选择E-pool的概率为 `α = w_E / (w_E + w_X)`。
    - E-pool的检索基于语义相似度，仅在超过相似度阈值时使用，否则自动回退到X-pool。
    - 权重更新基于LLM评判者的分阶段反馈。如果选择E-pool带来了阶段性提升 (`Δ_t > 0`)，则增加其权重；反之则衰减。X-pool的权重固定为1.0，但通过E-pool权重的变化来间接调控探索与利用的比例。

- **记忆更新**：
    - 任务完成后，所有X-pool中的记忆被合并到E-pool中，X-pool被清空，从而完成经验的沉淀，同时为下一轮探索预留空间。

### 2.3 算法流程（文字说明）
1.  **任务获取**：系统接收新任务，并分派给活跃智能体。
2.  **池选择**：智能体的在线路由器根据当前权重 (`α`) 概率性地选择E-pool或X-pool。
3.  **信息检索/生成**：
    - 选中E-pool：通过语义检索获取相似的历史记忆。
    - 选中X-pool：由LLM为当前上下文生成全新的、不依赖历史的行动方案。
4.  **动作生成**：智能体将获取到的记忆或新方案作为上下文，生成具体动作。
5.  **聚合与评估**：所有智能体输出被聚合，得出最终答案。之后，LLM评判者对每个阶段的输出进行评分。
6.  **在线更新**：根据阶段评分的变化（成功/失败），更新智能体E-pool的权重。
7.  **记忆迁移**：任务结束后，将X-pool中的所有经验转移到E-pool。
