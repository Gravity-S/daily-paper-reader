## 摘要
现代云和企业系统依赖基于身份的授权，假设持有有效凭证的调用者执行命令是安全的。自主AI智能体的出现推翻了这一假设：智能体可以生成语法有效但语义不安全的操作，使得静态权限成为重大操作风险。这种风险在主权的AI系统中尤为严重，其中自主智能体可能与云基础设施、受监管数据、金融工作流以及国家级数字服务交互。受治理的变更基板通过干预智能体操作来降低这一风险：智能体提交意图，基础设施评估上下文和策略，并中介执行。然而，这转移了信任边界：如何使授权意图的决策变得可验证、可分布式和可重放？
我们引入了一个分布式信任框架（DTF），这是一个用于受治理变更系统的验证框架，它从结构化、可验证的工件中计算执行权限。DTF引入了理由证明（Justification Proof）来编码操作的可接受性基础，一种用于独立评估的共识模型，基于已批准证明衍生的临时执行身份（Execution Identity），以及一个仅追加的证据链（Evidence Chain）来保留授权生命周期。在所述的基板假设下，该架构强制执行一个紧凑的授权不变性：没有证据对象就没有高风险执行，没有共识就没有衍生权限，没有脱离证据的有效变更。
我们定义了该模型，在基于OpenKedge的受治理变更基板上实例化，并展示了它如何映射到云原生环境。通过将授权从静态身份转变为证明衍生权限，DTF为在主权的AI部署中使智能体执行可治理、可审计和有边界提供了基础设施基础。

## Abstract
Modern cloud and enterprise systems rely on identity-centric authorization, assuming that callers possessing valid credentials are safe to execute commands. The emergence of autonomous AI agents invalidates this assumption: agents can generate syntactically valid but semantically unsafe actions, making standing privileges a significant operational risk. This risk becomes especially acute in sovereign AI systems, where autonomous agents may interact with cloud infrastructure, regulated data, financial workflows, and national-scale digital services. Governed mutation substrates reduce this risk by interposing on agent actions: agents submit intents, infrastructure evaluates context and policy, and execution is mediated. However, this shifts the trust boundary: how can the decision to authorize an intent be made verifiable, distributed, and replayable?   We introduce a Distributed Trust Framework (DTF), a verification framework for governed mutation systems that computes execution authority from structured, verifiable artifacts. DTF introduces a Justification Proof to encode the admissibility basis of an action, a consensus model for independent evaluation, an ephemeral Execution Identity derived from the approved proof, and an append-only Evidence Chain that preserves the authorization lifecycle. Under stated substrate assumptions, this architecture enforces a compact authorization invariant: no high-stakes execution without a proof object, no derived authority without consensus, and no valid mutation detached from evidence.   We define the model, instantiate it over an OpenKedge-based governed mutation substrate, and show how it maps onto cloud-native environments. By shifting authorization from standing identity to proof-derived authority, DTF provides an infrastructure foundation for making agentic execution governable, auditable, and bounded in sovereign AI deployments.

- **本文目标**：提出**分布式信任框架（DTF）**，将授权从静态身份转变为**证明衍生的临时权限**，使授权决策变为结构化、可独立验证、可重放的完整生命周期。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 授权不再是主体（智能体）的属性，而是**决策生命周期的结果**。任何高风险执行必须经过：**构造理由证明 → 独立评估者共识 → 派发临时执行身份 → 执行并记录证据链**。
- 关键不变性：无证明不执行，无共识不授权，无证据不认可任何有效变更。

### 关键技术细节

#### （1）理由证明（Justification Proof, JP）
- 结构：`JP_t = (M_t, S_t, Π_t, R_t, B_t)`
  - `M_t`：突变规范（具体动作和资源）
  - `S_t`：绑定的状态和上下文快照
  - `Π_t`：策略基础（应用的策略及版本）
  - `R_t`：风险与可接受性评估
  - `B_t = (μ_t, ρ_t, τ_t, Ω_t)`：最大执行边界（动作、资源、时间窗口、义务）
- 特性：**显式性**（所有授权相关假设在证明中或引用不可变工件中）、**可重评估性**（审计者可基于存储的证明重新判断，无需重建智能体内部推理）、**边界可推导**（`Boundary(JP_t) = B_t`，授权不能超出该边界）。

#### （2）共识验证（Consensus Validation）
- 评估者集合 `V_t` 中的每个独立评估者 `v_i` 对同一 `JP_t` 发出可归因的证明 `at_i`（含 approve/reject/abstain 及可选异议）。
- 共识函数 `q: A^n × G → D` （决策空间为{approve, reject, escalate}）根据治理元数据 `Γ_t`（包含法定人数阈值、否决权、所需评估者类别多样性等）计算最终决定。
- 关键点：**并不运行复制状态机的拜占庭共识**；而是分布式授权信任——防止任一单一评估者失败模式产生执行权限。

#### （3）临时执行身份（Execution Identity, EI）
- 若 `D_t = approve`，系统从批准的证明、证言和治理元数据派生 `EI_t = h(JP_t, A_t, Γ_t)`。
- EI是**临时的、不可转移的、作用域受限的**能力令牌，包含允许的动作类、资源集、有效时间、执行义务，并关联到原始证明和共识记录。
- 执行代理必须验证 `Scope(EI_t) ⪯ B_t` （作用域不超过证明边界）以及 `Valid(EI_t, X_t)` 才能执行突变。

#### （4）证据链（Evidence Chain, EC）
- 每个受治理的突变尝试都记录为一条完整的生命周期记录：`EC_t = (I_t, C_t, P_t, JP_t, A_t, Γ_t, D_t, EI_t, X_t, O_t)`
- 仅追加、防篡改（例如使用AWS QLDB或哈希链），确保授权依据永远可重放审计。

### 算法流程（Algorithm 1）
1. 构造证明 `JP_t = f(I_t, C_t, P_t)`
2. 各评估者独立生成证言 `at_i`
3. 聚合证言 `A_t`，执行共识 `D_t = q(A_t, Γ_t)`
4. 若批准，推导 `EI_t`；若拒绝/升级，`EI_t = ∅`
5. 执行前检查 `Scope(EI_t) ⪯ B_t` 和 `Valid(EI_t, X_t)`
6. 记录完整的证据链 `EC_t`

### 与OpenKedge的关系
- OpenKedge提供意图治理基板（意图标准化、策略评估、执行合同、任务导向身份、IEEC日志）。
- DTF在上面添加通用验证层：将任务身份变为**证明衍生的执行身份**，将IEEC变为**记录完整授权生命周期的证据链**。

## 6. 主要结论与发现

- **安全性**：DTF在所有2,400个不安全变体上实现了100%拦截或升级（相比B2允许336次不安全突变通过，通过率14%）；无单一评估者能独自派发权限。
- **权限约束**：EI将每个请求的平均可变资源从450个（基线）减少到1个，资源缩减因子99.7%（均值）和99.9%（95百分位）；300次边界漂移测试全部被拒绝。
- **可审计性**：证据链实现100%生命周期字段完整（证明、证言、决策、身份、执行、结果）；重放成功率99.9%（B1仅23%，B2仅58.7%）。
- **延迟开销**：端到端决策延迟均值58.3ms，中位数52.6ms，95百分位112.4ms，99百分位171.3ms——对于高风险基础设施操作可接受。
- **消融实验**：去除共识 → 61/200异常评估者案例可能通过；去除EI → 所有边界漂移可执行；去除证据链 → 重放成功率降至52.1%。

## 7. 优点

1. **问题定义清晰**：准确识别了自主AI智能体使身份授权失效的根源，并定义了验证授权决策本身的新需求。
2. **框架设计新颖且系统化**：将证明构建、独立共识、临时身份、证据链作为一个完整生命周期闭环，替代了传统的单一策略引擎或事后日志。
3. **可验证性和可审计性**：从“信任主体”转向“信任决策生命周期”，审计员可以重放授权依据而不依赖智能体的内部状态。

## 8. 不足与局限

1. **实验场景有限**：仅在合成负载和已知突变类型上测试，未涉及真实世界智能体产生的未知边缘情况或偶然性漏洞。
2. **依赖上下文正确性**：如果状态快照、依赖图、策略版本**不准确或不完整**，则证明可能编码错误的基础，框架不能保证授权正确性。
3. **评估者多样性质量未深入验证**：如果评估者共享相同数据源或失效模式，名义上的多样性可能虚假；实验未评估真实世界中评估者独立性。
4. **不处理完全妥协**：假设TCB（DTF层和基板）未被攻破；不防御评估者全集合、经纪人或证据存储的全泄密。
5. **粒度限制**：某些云基板只暴露粗粒度权限，可能无法精确执行边界，需要额外中介层。
6. **未解决智能体内部安全**：框架不能防止提示注入或智能体产生危险意图（只确保执行前必须经过证明和共识）。
7. **性能评估粒度**：延迟数据来自受控环境，真实负载下高并发或大规模评估者集可能引入更大延迟或成本。
8. **未讨论策略设计正确性**：假设策略本身正确；框架只负责确保授权基于策略，不负责策略逻辑的完备性。
