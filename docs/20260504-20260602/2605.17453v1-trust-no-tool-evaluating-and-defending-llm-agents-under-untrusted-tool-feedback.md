## 摘要
使用工具的LLM智能体日益依赖外部工具做出关键决策，然而现有的大多数智能体安全基准和防御措施都隐含地假设，一旦工具被选中，其反馈就是可信的。我们研究了一种不同的故障模式——认知污染，其中恶意工具在探索阶段表现合理，通过看似良性的反馈积累信任，并且仅当隐藏状态条件与最终可执行动作对齐时才变得有害。为了研究这一场景，我们构建了TRUST-Bench，一个包含1970个隐藏触发工具妥协事件的任务条件化基准，并配有匹配的安全对照组；引入了一个非对称惩罚度量GuardedJoint，以更好地反映实际部署风险；并提出了VISTA-Guard，一个与骨干无关的最终动作风险评分框架。核心思想是将多步工具交互抽象为结构化的环境变量，编码信任形成动态，然后从这种轨迹条件化表示中对最终可执行动作的风险进行评分。实验表明，以提示为中心的启发式方法、标量化特征和零样本判断在该场景下均失败，而轨迹感知的最终动作评分在域内具有强区分能力，并在平衡的分布外迁移下保持有效。在GuardedJoint下，VISTA-Guard在域内达到84.2，在平衡的分布外评估中达到56.9，而仅优化安全性-效用权衡中一面的方法则崩溃至零。这些发现支持了一个更广泛的黑盒工具生态系统中智能体安全的观点：决定性的防御目标不仅是局部提示文本或工具描述，而是信任在整个交互轨迹中形成并通过最终动作承诺的方式。

## Abstract
Tool-using LLM agents increasingly rely on external tools to make consequential decisions, yet most existing agent-security benchmarks and defenses implicitly assume that tool feedback is trustworthy once a tool has been selected. We study a different failure mode, cognitive poisoning, in which a malicious tool behaves plausibly during exploration, accumulates trust through benign-looking feedback, and becomes harmful only when hidden state conditions align with the final executable action. To study this setting, we construct TRUST-Bench, a task-conditioned benchmark of 1,970 hidden-trigger tool-compromise episodes with matched safe controls, introduce an asymmetric penalty metric, GuardedJoint, to better reflect real deployment risk, and present VISTA-Guard, a backbone-agnostic framework for final-action risk scoring. The core idea is to abstract multi-step tool interaction into structured environment variables that encode trust-formation dynamics and then score the risk of the final executable action from this trajectory-conditioned representation. Experiments show that prompt-centric heuristics, scalarized features, and zero-shot judges fail in this regime, whereas trajectory-aware final-action scoring yields strong in-domain discrimination and remains effective under balanced out-of-distribution transfer. Under GuardedJoint, VISTA-Guard reaches $84.2$ in-domain and $56.9$ on balanced out-of-distribution evaluation, while methods that optimize only one side of the safety--utility tradeoff collapse to zero. These findings support a broader view of agent security in black-box tool ecosystems: the decisive defense target is not local prompt text or tool descriptors alone, but the way trust is formed across the interaction trajectory and committed through the final action.

## 1. 核心问题与整体含义（研究动机与背景）

当前使用工具的 LLM 智能体严重依赖外部工具做出关键决策，但现有的大多数智能体安全基准和防御隐式假设：一旦工具被选中，其反馈就是可信的、本地可见的。然而在实际的黑盒工具生态中，智能体无法直接验证工具的内部可信性，只能通过反复交互来推断信任。这创造了一个独特的攻击面——**认知中毒（cognitive poisoning）**。在这种威胁下，一个恶意工具在探索阶段表现得与良性工具无异，通过若干看似正常的反馈积累信任，直到最终可执行动作满足一个由隐藏状态条件（如特定参数组合）组成的触发规则时才变得有害。攻击成功不是因为某条消息明显恶意，而是整个轨迹塑造了智能体将危险最终动作视为常规的习惯。论文将此问题定义为**状态条件化工具妥协**，并指出防御目标不应是孤立检查提示文本或工具描述，而应是**跨交互轨迹的信任形成方式**以及最终动作的承诺。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
将多步工具交互抽象为结构化环境变量，编码信任形成动态，然后基于轨迹条件化表示对最终可执行动作进行风险评分。

### VISTA-Guard 框架（Variable-state Inference for Safe Tool Actions）
- **轨迹到三元组抽象**：每个 episode 表示为 `(z_traj, z_param, y)`。
  - **轨迹状态视图（z_traj）**：从每次探索轮次的工具反馈中提取无标签环境信号，包括：triggered_count（触发阶段数）、probe_detect_count（探测检测数）、high_mismatch_count（高不匹配数）、id_drift_count（标识符漂移）、warning_count（警告数），以及由此计算的异常密度、重复一致性异常标志、强信号标志和辅助的 `state_risk` 值。
  - **最终动作参数视图（z_param）**：包含工具名称、payload 键/令牌、high_impact_count、bypass_count、safety_count 等紧凑标记计数，以及可选的 `param_risk` 值（由这些计数线性组合并裁剪而得）。
- **风险评分与训练**：骨干模型（如 Mistral-7B）将序列化的三元组作为输入，输出两个标签后缀 `low_risk`（执行）和 `high_risk`（拒绝）的对数概率。通过 softmax 计算拒绝概率 `r(x)`，并与阈值 `τ_f` 比较决定是否拒绝。阈值在每个训练折内通过最大化 GuardedJoint 指标校准。
- **训练目标**：标签后缀的负对数似然 `L(θ) = -Σ log P_θ(y_i | x_i)`，采用全参数 BF16 微调。
- **关键设计**：不使用原始工具 ID 或渲染的调用字符串；所有轨迹信号均为无标签的自由证据，构成模型输入而非决策规则。

## 3. 实验设计

### 数据集 / 场景
- **TRUST-Bench 基准**：从五个上游工具使用和安全套件构造：OpenAgentSafety、RedTeamCUA、MCP-SafetyBench、MCPMark、Toolathlon。初始产生 1,753 个源任务、7,012 个恶意/良性变体（见表1），经过两人审计后保留 1,970 个 episode（包含匹配的隐藏触发恶意事件和良性对照）。
- **OOD 测试集**：来自 ToolEmu 和 SafeToolBench 的 9,216 个 episode，覆盖 17 个未见过的工具家族，进行平衡的分布外迁移测试。
- **直接外部攻击迁移**：1,040 个外部有害 episode，仅包含恶意样本，用于评估有害事件召回率。

### 对比方法
- **VISTA-Guard 变体**：使用 6 种 LLM 骨干：Mistral-7B-v0.3、Qwen3-8B、Qwen2.5-7B、OLMo-2-7B、Llama-3.1-8B、Meta CWM-32B。
- **非 LLM 文本基线**：TF-IDF + Logistic Regression / SVM / MLP；Gradient Boosting（10个标量特征）；BERT-base。
- **提示中心/零样本基线**：GPT-5.4 Judge（全轨迹和仅特征两种模式）、ToolShield、GuardAgent、LLM-as-Judge、Keyword Heuristic。

### 评估指标
- **GuardedJoint**：非对称安全性效用指标 `G_ρ = max(0, 100 - (ρ+0.5)AMR - 1.5RNR)`，主实验取 ρ=1.5。同时报告 AMR（接受恶意率）和 RNR（拒绝正常率）。
- 实验采用分组 5 折交叉验证，确保同一源任务的恶意/良性对在同一折内，阈值仅从训练折校准，避免泄漏。

## 6. 主要结论与发现

1. **认知中毒攻击难以被提示中心或标量化方法防御**：ToolShield、GuardAgent、LLM-as-Judge、GPT-5.4 Judge 等均崩溃至 0 GuardedJoint，因为它们无法捕获轨迹层面的信任形成。
2. **轨迹感知的最终动作评分有效**：VISTA-Guard 在域内 GuardedJoint 达 84.2（Mistral-7B），平衡 OOD 达 56.9，显著优于所有基线。
3. **结构化表示是关键**：相同输入隔离显示，给非 LLM 文本模型提供结构化输入可提升性能（TF-IDF+LogReg 从 44.6 升至 55.2），但仍落后 VISTA-Guard 的 LLM 骨干。表示消融（表5）表明，完整结构化三元组优于原始文本、去掉自适应状态评分或仅使用参数都会大幅下降。
4. **骨干选择影响性能但非决定性**：6 种 LLM 骨干均获得正 GuardedJoint，Mistral 最高但方差大，Qwen3 最稳定。核心增益来自轨迹抽象本身。
5. **OOD 迁移仍是挑战**：域内到 OOD 的 GuardedJoint 下降 20-39 分，说明对未见工具家族泛化仍有难度。
6. **防御关键位置是最终动作风险评分**，而非孤立检查提示文本或工具描述。

## 7. 优点

- **问题新颖**：提出认知中毒攻击模型，不同于间接提示注入和传统 MCP 中毒，攻击面是探索性信任形成过程本身。
- **基准构建严谨**：TRUST-Bench 基于真实工具使用任务，经红蓝构建和人工审计，确保匹配对照、无泄漏、标签正确。
- **评估指标合理**：GuardedJoint 非对称惩罚，同时惩罚接受恶意和拒绝良性，避免单一指标偏差。
- **方法骨干无关**：VISTA-Guard 可搭配任意自回归 LM，且设计上避免使用原始 ID 或渲染字符串，增强泛化性。
- **实验设计严密**：分组分割防止泄漏、训练折阈值校准、hardened input、same-input isolation，均提升结论可靠性。
- **充分消融与分析**：包含表示消融、骨干消融、OOD 迁移、敏感性分析、鲁棒性测试、攻击配置压力测试等。

## 8. 不足与局限

- **基准构造性质**：TRUST-Bench 是构造的而非真实部署收集，尽管基于现有基准，仍有仿真偏差。
- **探索预算固定**：标准化的三步探索是“最早信息点”，但更长的交互（4-6 步）可提升性能，也增加成本。论文未研究自适应攻击者动态调整探索长度的情况。
- **动作空间有限**：仅执行/拒绝二元决策，未包含多动作补救策略（如要求复述、降级执行等）。
- **状态建模简化**：轨迹证据被总结为离散环境变量，而非完整的潜在状态建模；某些微妙的跨步模式可能丢失。
- **OOD 迁移仍有较大下降**：最强方法在 OOD 上仅 56.9，显示对全新工具家族的稳健泛化仍需改进。
- **计算资源未详细报告**：未说明 GPU 数量和训练时长，影响可复现性评估。
- **未测试自适应攻击者**：假设攻击者固定，未考虑攻击者观察防御信号后调整策略的对抗性场景。
- **人类审计规模有限**：审计样本量及跨语言/领域覆盖未详细讨论。
