## 摘要
现代AI智能体通过工具调用（如文件操作、Shell命令、HTTP请求和数据库查询）执行实际副作用。单个不安全操作（包括意外删除、凭证泄露或数据窃取）可造成不可逆的损害。现有防御机制存在不足：事后基准测试衡量执行后的行为，静态防护栏无法应对混淆和多步上下文，而基础设施沙盒仅限制代码运行位置却不理解操作含义。我们提出AgentTrust——一个运行时安全层，在智能体工具调用执行前进行拦截，并返回结构化裁决：允许、警告、拦截或审查。AgentTrust结合了Shell反混淆归一化器、用于更安全替代方案的SafeFix建议、用于多步攻击链检测的RiskChain以及用于模糊输入的缓存感知型LLM-as-Judge。我们发布了涵盖六类风险场景的300个场景基准测试，以及额外630个独立构建的真实世界对抗场景。在内部基准上，仅生产环境规则集实现了95.0%的裁决准确率和73.7%的风险等级准确率，端到端延迟为毫秒级。在630个场景基准测试中（采用修补规则集评估，不声称零样本），AgentTrust实现了96.7%的裁决准确率，其中对Shell混淆负载的准确率约为93%。AgentTrust以AGPL-3.0许可证发布，并为MCP兼容智能体提供模型上下文协议服务器。

## Abstract
Modern AI agents execute real-world side effects through tool calls such as file operations, shell commands, HTTP requests, and database queries. A single unsafe action, including accidental deletion, credential exposure, or data exfiltration, can cause irreversible harm. Existing defenses are incomplete: post-hoc benchmarks measure behavior after execution, static guardrails miss obfuscation and multi-step context, and infrastructure sandboxes constrain where code runs without understanding what an action means.   We present AgentTrust, a runtime safety layer that intercepts agent tool calls before execution and returns a structured verdict: allow, warn, block, or review. AgentTrust combines a shell deobfuscation normalizer, SafeFix suggestions for safer alternatives, RiskChain detection for multi-step attack chains, and a cache-aware LLM-as-Judge for ambiguous inputs.   We release a 300-scenario benchmark across six risk categories and an additional 630 independently constructed real-world adversarial scenarios. On the internal benchmark, the production-only ruleset achieves 95.0% verdict accuracy and 73.7% risk-level accuracy at low-millisecond end-to-end latency. On the 630-scenario benchmark, evaluated under a patched ruleset and not claimed as zero-shot, AgentTrust achieves 96.7% verdict accuracy, including about 93% on shell-obfuscated payloads. AgentTrust is released under the AGPL-3.0 license and provides a Model Context Protocol server for MCP-compatible agents.

### 1. 论文的核心问题与整体含义（研究动机和背景）

*   **核心问题**：现代AI智能体（如 Claude Code、Cursor）在通过工具调用执行文件操作、Shell命令等具有真实世界副作用的任务时，缺乏有效的运行时安全防护。一个错误的操作（如 `rm -rf /`）可能导致不可逆的损害。
*   **现有防御不足**：
    *   **事后基准测试**：只能评估行为，无法在运行时阻止伤害发生。
    *   **静态规则**：容易被混淆技术绕过，且无法理解语义和多步攻击链的上下文。
    *   **基础设施沙箱**：仅限制代码执行位置，但不理解操作的意图。例如，一个在被允许的HTTP出口上窃取数据库的智能体，其行为在沙箱内部是合法的。
*   **论文目标**：提出一个全新的、实时的、语义感知的安全拦截框架（**AgentTrust**），在智能体执行任何工具调用**之前**进行评估并返回结构化裁决，从而在动作层面阻断风险。

### 2. 论文提出的方法论：核心思想、关键技术细节

AgentTrust 的核心是一个由八个组件构成的管道，在智能体与其工具之间工作。

*   **核心思想**：实现一个实时的、语义感知的“安全层”，通过多维度分析（反混淆、规则、多步行为、LLM语义理解）生成一个四值裁决（允许/警告/拦截/审查），而非简单的“通过/拒绝”。

*   **关键技术细节**：
    1.  **动作模型与裁决定义**：定义了结构化的 `Action` 记录和五级风险格（无/低/中/高/严重）以及一个四值裁决集合`{allow, warn, block, review}`。`warn` 和 `allow` 是可执行的，`review` 需要人工干预。
    2.  **ShellNormalizer（Shell反混淆归一化器）**：包含9种纯文本重写策略（如变量扩展、十六进制/八进制转义解码、命令替换、ANSI-C 引用等），用于揭示隐藏的危险命令。它不直接执行任何代码，保证了自身安全性。
    3.  **ActionAnalyzer & PolicyEngine（动作分析器与策略引擎）**：分析器通过42个正则表达式提取风险特征；策略引擎包含170条可配置的 YAML 规则，用于裁决。系统采用**最大严重性聚合**，确保任何反混淆变体都能触发最高风险。
    4.  **SafeFix（安全修复引擎）**：在产生负面裁决（block, warn, review）时，提供37条修复规则，给出更安全的替代方案（如将 `chmod 777` 建议为 `chmod 755`），而非简单阻止。
    5.  **RiskChain（风险链 - 会话跟踪器）**：利用**顺序感知的贪心匹配**算法，检测由多个“单独看来无害”的动作组成的多步攻击链（如读取凭据 -> 编码 -> 外传），并触发最高等级的拦截。
    6.  **Cache-Aware LLM-as-Judge（缓存感知型LLM法官）**：当规则引擎无法决断时，调用 LLM。其核心创新在于**基于块哈希的增量检测**机制（灵感来自 `rsync`/`git`）。它只计算与上次评估相比发生变化的部分（通常是会话尾部新增内容），从而在长会话中大幅节省标记（Token）成本。该组件有三种策略：缓存命中、增量评估和完整重新评估。
    7.  **安全合约（Safety Contracts）**：通过13个回归测试，定义了四个故障安全不变量，例如：**LLM法官不可达 => 返回 `review`**；**拦截器内部异常 => 返回 `review`**。这确保了系统本身不会静默地“开放”而导致危险。

### 6. 论文的主要结论与发现

*   **AgentTrust 的规则路径是最高效和准确的**：在内部基准上，生产级规则路径（无基准兼容规则）以**1.72毫秒**的中位延迟实现了**95.0%** 的裁决准确率，远优于简单规则、静态护栏和通用LLM。

*   **混合模式提供可权衡的精度**：规则+LLM法官混合模式能将假阴性率（漏过攻击）降至极低水平，但会以假阳性率（错误阻止）为代价。高安全场景可选此路径。

### 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

*   **静态分析的天花板**：规则路径本质上是基于文本的正则或模式匹配，无法对运行时数据流（如管道中的文件内容）进行推理，这可能导致复杂场景下漏判。
*   **Shell反混淆不完备**：当前只支持9种纯文本策略，无法处理恶意构造的**函数体、继承控制流**等，需要引入AST分析才能进一步解决。
*   **仅覆盖英文模式**：正则表达式和LLM法官的Prompt均以英语为中心，无法有效处理非英语命令。
*   **风险分级的主观性**：“高”和“严重”风险级别之间的界限是主观的，内部基准中的约35个分歧（共73个风险降级分歧）都集中在这一点上。裁决准确率才是更可靠的指标。
*   **进程内部署的依赖**：系统的运行基于智能体运行时环境中，代理和系统位于同一进程，如果是恶意代理程序本身绕过的话，模型将无法起作用。
*   **SessionTracker的价值在当前基准中无法验证**：虽然这是论文诚实承认的不足，但这确实影响了对其价值的评估，它需要合适的多步轨迹数据来验证其实际的检测能力。
