## 摘要
本文提出了一种面向线性时序逻辑（LTL）规范的多智能体系统的安全构建式规划与控制框架。该框架保护敏感信息免受被动入侵者的攻击，入侵者只能部分观测智能体的运动。多智能体协调中的安全性通过两个概念来体现，即防止入侵者推断出秘密任务是否已被执行，以及识别出负责执行该任务的智能体。所提出的框架通过构建一个安全的有限转移系统，移除所有违反这些约束的路径，将安全约束直接纳入LTL合成过程中。然后对安全抽象应用标准LTL合成以生成离散计划，再将其细化为动态可行的连续轨迹。此合成过程提供了形式化保证，确保多智能体系统的最终行为同时满足全局LTL规范和安全约束。通过一个双无人机案例研究验证了所提出框架的有效性。

## Abstract
This paper presents a secure-by-construction planning and control framework for multi-agent systems subject to linear temporal logic (LTL) specifications. The framework protects sensitive information from a passive intruder with partial observations of the agents' motion. Security in multi-agent coordination is captured by two notions that prevent the intruder from inferring whether a secret task has been executed and from identifying the agent responsible for its execution. The proposed framework incorporates the security constraints directly into the LTL synthesis procedure by constructing a secure finite transition system that removes all paths violating these constraints. Standard LTL synthesis is then applied to this secure abstraction to generate discrete plans, which are then refined into dynamically feasible continuous trajectories. This synthesis procedure provides formal guarantees that the resulting behavior of the multi-agent system satisfies both the global LTL specification and the security constraints. The effectiveness of the proposed framework is demonstrated through a two-drone case study.

### 1. 核心问题与整体含义（研究动机和背景）
- **背景**：多智能体系统（如无人机集群）在复杂任务中常使用线性时序逻辑（LTL）进行高层次规划，但现有研究大多忽略信息安全性。在存在被动入侵者（仅能部分观测智能体运动）的场景下，如何防止入侵者推断“秘密任务是否被执行”以及“由哪个智能体执行”，是一个关键挑战。
- **动机**：设计一种**安全构建式（secure-by-construction）**的规划与控制框架，使多智能体系统在执行LTL任务的同时，能够形式化地满足规定的安全约束（不泄露秘密信息）。
- **问题**：针对具有连续仿射动力学、输入约束的多智能体系统，在分区工作空间中合成一条连续轨迹，使其同时满足（1）LTL规范；（2）类型A和/或类型B安全。

### 2. 方法论：核心思想、关键技术细节与算法流程
- **核心思想**：将安全约束直接融入到离散抽象（加权迁移系统WTS）的构造中，通过**删除所有违反安全约束的状态和转换**，得到一个安全的全局迁移系统，然后在此安全抽象上应用标准LTL合成与轨迹细化。
- **关键技术细节**：
  1. **安全定义**：
     - **类型A安全**：防止入侵者推断任何智能体是否访问过秘密状态。要求对每段真实路径，存在一条观测相同但所有智能体在秘密时刻都处于非秘密状态的“副本路径”。
     - **类型B安全**：防止入侵者识别出是哪个智能体执行了秘密任务。要求当某智能体在秘密状态时，至少存在另一个智能体在同一时刻产生相同的观测。
  2. **安全抽象构造**：
     - 先构建全局加权迁移系统（gWTS）\(T_g\)。
     - 对于类型A：构建**孪生gWTS（Twin-gWTS）** \(V\) 和**安全孪生gWTS** \(V_s\)，剔除所有真实和副本状态均为秘密的孪生状态对。
     - 对于类型B：构建**类型B gWTS** \(T_g^B\)，剔除所有违反类型B安全条件的全局状态。
     - 同时满足两者：先构造\(T_g^B\)，再在其上构造\(V_s\)。
  3. **过渡可行性检查**：通过求解集中式二次规划（QP）验证每个离散转移是否满足动力学、控制输入界限、多面体正向不变性（使用控制障碍函数CBF约束）以及退出面条件。若QP无解，则丢弃该转移。
  4. **LTL合成**：将安全抽象\(V_s\)与LTL公式的NBA进行乘积构造得到PB自动机，然后寻找最优的前缀-后缀路径（通过Dijkstra算法），最小化加权成本。
  5. **轨迹细化**：将离散路径对应的每个转移的连续轨迹段（由QP解出）拼接，得到完整的连续轨迹与控制输入。

1. **实验不足**：仅一个双无人机案例，缺乏多智能体规模扩展性测试、不同任务LTL的测试、与现有方法的对比实验。
