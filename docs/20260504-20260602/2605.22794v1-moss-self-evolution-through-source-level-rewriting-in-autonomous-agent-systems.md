## 摘要
自主代理系统在部署后基本处于静态：它们无法从用户交互中学习，重复出现的失败一直持续到下一次人工驱动的更新发布修复为止。作为回应，出现了自我进化的代理，但所有进化都局限于文本可变的工件——技能文件、提示配置、记忆模式、工作流图——而代理框架则保持不变。由于路由、钩子排序、状态不变量和调度存在于代码中而非任何文本工件中，整个类别的结构故障从文本层在物理上是不可达的。我们认为，源码级适应是一种本质上更通用的媒介：它是图灵完备的，是每个文本可变范围的严格超集，能够确定性地生效而不依赖于基础模型的遵从性，并且不会因长上下文漂移而退化。我们提出了MOSS，一个在生产级代理基底上执行源码级自我重写的系统。每一次进化都锚定在一批自动策划的生产失败证据上，并通过一个确定性的多阶段流水线进行；代码修改委托给一个可插拔的外部编码代理CLI，而MOSS保留阶段排序和判决。候选代码通过在临时试验工作器中针对候选镜像重放这批证据来进行验证，然后通过用户同意门控、原地容器交换（带有健康探测门控的回滚）进行推送。在OpenClaw上，MOSS在单次循环中将四个任务的平均评分从0.25提升到0.61，无需人工干预。

## Abstract
Autonomous agentic systems are largely static after deployment: they do not learn from user interactions, and recurring failures persist until the next human-driven update ships a fix. Self-evolving agents have emerged in response, but all confine evolution to text-mutable artifacts -- skill files, prompt configurations, memory schemas, workflow graphs -- and leave the agent harness untouched. Since routing, hook ordering, state invariants, and dispatch live in code rather than in any text artifact, an entire class of structural failure is physically unreachable from the text layer. We argue that source-level adaptation is a fundamentally more general medium: it is Turing-complete, a strict superset of every text-mutable scope, takes effect deterministically rather than through base-model compliance, and does not erode under long-context drift. We present MOSS, a system that performs self-rewriting at the source level on production agentic substrates. Each evolution is anchored to an automatically curated batch of production-failure evidence and proceeds through a deterministic multi-stage pipeline; code modification is delegated to a pluggable external coding-agent CLI while MOSS retains stage ordering and verdicts. Candidates are verified by replaying the batch against the candidate image in ephemeral trial workers, then promoted via user-consent-gated, in-place container swap with health-probe-gated rollback. On OpenClaw, MOSS lifts a four-task mean grader score from 0.25 to 0.61 in a single cycle without human intervention.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有自主代理系统（如 OpenClaw、Hermes Agent）部署后处于静态，无法从用户交互中学习，重复故障需等待人工更新修复。已有“自我进化”方案仅能修改文本可变工件（提示、技能文件、记忆模式、工作流图），而代理框架（harness，包含路由、钩子排序、状态管理、调度等逻辑）以代码形式存在，文本编辑无法触及，导致结构性故障（如消息路由错误、钩子执行顺序乱序、状态不一致）无法自动修复。
- **整体含义**：作者提出源码级适应是更通用的进化媒介，具有四个优势：图灵完备（代码空间是文本空间的严格超集）、确定性生效（不依赖基础模型对文本的理解）、不因长上下文漂移退化、能覆盖所有文本可变范围。MOSS 系统首次将源码级自我重写应用于生产级代理基底，实现了无需人工干预的框架层自动修复。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：MOSS 将自我进化锚定于生产失败证据（自动扫描会话 + 用户标记），通过确定性多阶段流水线，委托外部编码代理 CLI 修改源码，经运行时验证后，通过原地容器热交换推送至生产。
- **关键技术细节**：
  - **证据收集**：两种渠道——后台定期 `moss evo catch-up` 扫描会话 JSONL，自动识别失败片段；用户通过 `moss evo flag` 标记不满意的对话轮次。两类证据合并成批次（batch），每个对话维护一个开放批次，达阈值（默认 8 个块）后密封。
  - **进化循环**：单次进化由迭代组成，每个迭代运行七阶段流水线；循环受边界条件约束（批次修复成功、模型能力上限、架构限制或平台停滞）。支持三种深度（light/standard/deep）调节预算。
  - **七阶段流水线**：
    1. **Locate**：读取基线追踪和批次故障，输出诊断（不提出修复）。
    2. **Plan**：识别根因，指定修复方案（哪些文件、加什么逻辑）。
    3. **Plan-Review**：质量门控，批准/拒绝/要求修改，与 Plan 轮替直到通过。
    4. **Implement**：写入代码，形成单次 git 提交。
    5. **Code-Review**：质量门控，审查 diff 是否匹配计划，与 Implement 轮替直到通过。
    6. **Task-Evaluate**：在构建和试验后，按选定关键点评分每个任务（强/足够/弱/缺失），锁定关键点集。
    7. **Verdict**：综合评估后输出四种判决之一：CONVERGED、NEED_MORE_WORK、FUNDAMENTAL_LIMIT_MODEL、FUNDAMENTAL_LIMIT_ARCHITECTURE。
  - **代码修改委托**：MOSS 自身不直接写代码，通过 `host-daemon` 调用外部编码代理 CLI（如 Claude Code、OpenAI Codex），抽象为四方法运行器接口，支持可插拔。
  - **运行时验证**：使用临时试验工作器（ephemeral trial workers），基于候选镜像启动 N 个容器，自主处理批次任务（每条任务重复多次以暴露不稳定性），隔离于生产网络和挂载。验证是运行时、生产等效的，比单元测试更真实。
  - **原地容器热交换**：收敛后不自动推送，需用户通过对话界面 `moss evo apply` 授权。host-daemon 检测到交换请求后重启主容器，使用候选镜像，进行 90 秒健康探测（每 5 秒一次，四项检查）。若三次连续通过则提交，否则回滚到上一个已知良好镜像。用户状态卷保持不变。
  - **拓扑**：主容器（代理 + 进化服务 + moss CLI）、host-daemon（asyncio 进程，管理 RPC、交换监督、自动扫描）、外部编码 CLI（按阶段生成）、试验工作器（按迭代生成）。

- **主要结论**：源码级自我进化在**生产级代理系统**上是可行且有效的，能够自动修复框架层（harness）的结构性缺陷，而先前的文本级进化无法做到。
- **具体发现**：
  - 在 OpenClaw 上单次进化循环（一次迭代）后，四个任务的平均 claweval 评分从 0.2526 提升到 0.6100（+0.3574）。
  - 最大提升出现在 T138（restock_chain_check，英文版），从 0.2090 跃升至 0.9049，三次试验均超过 0.75 通过线。
