## 摘要
现有针对基于大语言模型的智能体的后门攻击仍然是无状态的，其执行固定行为且仅限于单一会话中。我们提出了一种有状态智能体后门，该攻击能够在权限隔离下跨多个会话扩展攻击生命周期。攻击通过持久化组件维护状态，从而在一次性触发注入后，实现跨会话的自主、增量式执行。形式上，我们将该攻击建模为米利型有限状态机，并推导出一个分解框架，使得每个状态转换的数据构建可以独立进行。我们通过一个主要攻击及两种可扩展性变体来实例化该框架。主要实例化在四个模型上实现了80%–95%的攻击成功率，对每个状态转换的分析验证了分解的有效性。采用不同拓扑与持久化组件的可扩展性变体展现出一致的有效性。代码与数据可在 https://anonymous.4open.science/r/stateful_agent_backdoor-E89F 获取。

## Abstract
Existing backdoor attacks on Large Language Model-based agents remain stateless, executing fixed behaviors confined to a single session. We propose a stateful agent backdoor that extends the attack lifecycle across multiple sessions under permission isolation. The attack maintains state through persistent components, enabling autonomous, incremental execution across sessions following a one-time trigger injection. Formally, we model the attack as a Mealy machine and derive a decomposition framework that enables independent per-transition data construction. We instantiate this framework with a primary attack and two extensibility variants. The primary instantiation achieves an attack success rate of 80\%--95\% across four models, with per-transition analysis demonstrating the effectiveness of the decomposition. Extensibility variants with alternative topologies and persistent components demonstrate consistent effectiveness. Code and data are available at https://anonymous.4open.science/r/stateful_agent_backdoor-E89F.

# Stateful Agent Backdoor 论文总结

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：现有针对大语言模型（LLM）智能体的后门攻击均是无状态的，攻击行为固定且仅限单个会话（session）内执行。这导致攻击无法在跨会话的权限隔离场景中完成——例如，若隐私收集需要文件系统权限，而数据外传需要网络权限，且两者在不同会话中分别可用，则无状态后门将失败，因为攻击无法将状态和结果传递到下一个会话。
- **整体含义**：智能体框架通过持久化组件（如内存、笔记工具）形成了真正的有状态系统，而现有攻击却忽略了这一点。本文首次提出**有状态智能体后门**，利用持久化组件维护攻击状态，实现一次性触发后跨多个会话自主、增量式执行，从而突破会话级权限隔离。

## 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：将攻击建模为**米利型有限状态机（Mealy machine）**，其输出同时依赖于当前攻击状态和当前环境观测（如可用工具集）。攻击行为不再是固定的，而是由（状态, 输入）共同决定。
- **关键技术细节**：
  1. **状态机形式化**：定义 M = (S, Σ, Λ, δ, λ, s_init)，其中 S 包含四个状态：s_init（未触发）、s_collect（等待文件工具）、s_exfil（等待网络工具）、s_acc（完成）；输入 Σ 包括触发词存在性、文件工具可用性、网络工具可用性等布尔谓词；输出 Λ 包括 initiate（初始化）、collect（收集）、exfil（外传）、wait（等待）等行为模式。
  2. **分解框架**：一个完整的有状态后门可分解为多个子后门，每个子后门对应状态机的一个转移，其复合触发条件为 (当前攻击状态, 环境状态)。子后门之间通过状态更新因果链连接。训练时，可独立地为每个转移生成单会话轨迹数据，然后混合进行有监督微调（SFT）。
  3. **持久化利用**：攻击利用智能体的持久化组件（如内存模块或笔记工具）完成两件事：存储/恢复攻击状态和中间结果；使触发词持续存在，实现后续会话的自我激活。
  4. **可扩展性**：框架支持任意状态机拓扑（如分支-合并）和不同的持久化组件（如用笔记工具替代内存），只需相应调整数据构建即可。

## 3. 实验设计：使用了哪些数据集/场景、benchmark、对比方法
- **模型**：Llama-3.1-8B、Qwen2.5-7B、Qwen2.5-14B、Ministral-3-14B（四个指令微调变体）。
- **场景**：隐私外传场景，包含三个阶段：初始化（写入触发词和状态）、收集隐私数据、外传数据。每个会话只能访问部分工具（文件系统、网络、电子邮件、内存等），模拟权限隔离。
- **数据集构造**：生成1000条非冗余轨迹（主要实例化），包括5种攻击类型和1种良性类型。每条攻击轨迹通过将固定攻击模板以ReAct风格添加到良性轨迹前构建。触发词为随机五字符字符串。
- **对比方法**：无。作者指出现有后门攻击均为单会话无状态，无法应用于跨会话权限隔离场景，因此不进行直接对比。
- **Benchmark**：MMLU用于评估通用能力变化；FSR（Follow Step Ratio）评估格式遵从性。主要评价指标：
  - ASR（攻击成功率）：完成完整外传的episode比例。
  - 逐步骤保留率 R。
  - 逐转移成功率。
  - FPR（假阳性率）：无触发注入时错误执行攻击的比例。

## 6. 论文的主要结论与发现
- **可行性**：有状态后门在跨会话多阶段攻击中可行，主要实例化ASR达80%–95%（四个模型）。
- **分解有效性**：每个子后门在独立评价中成功率≥90%，且各转移成功率的乘积与端到端ASR接近，验证了分解框架的合理性。
- **可扩展性**：分支-合并拓扑和笔记工具实例化同样有效，ASR分别为70%–100%和80%–90%，FPR_exf ≤5%。
- **安全启示**：持久化组件可被主动利用作为跨隔离通道；四个主流智能体框架（LangGraph、CrewAI、MAF、OpenAI Agents SDK）均存在此类共享通道，威胁模型假设具有现实基础。
- **影响**：首次将有状态概念引入智能体后门攻击，扩展了攻击生命周期，并指出跨会话共享读写通道是设计安全智能体系统时需警惕的风险。

- **威胁模型现实性**：对主流框架进行详细调查，验证了隔离单元和持久化共享通道的存在，增强了攻击的实操可行性。

## 8. 不足与局限
- **实验局限性**：
  - 无现有方法对比（但跨场景限制了可比性）。
  - 工具输出由LLM模拟，非真实环境；真实部署中工具返回值可能不同，影响攻击鲁棒性。
  - 每模型只有20个episode，样本量不大（不过已用置信区间量化不确定性）。
- **方法局限**：
  - 持久化组件必须满足读/写、跨会话一致性、跨工具集可访问三个条件，意味着无法实现完全的工具隔离。
