## 摘要
代理技能赋予LLM代理特权第三方能力，例如文件系统访问、凭据、网络调用和shell执行。现有安全工作可以捕获恶意提示和危险的运行时行为，但技能工件本身未经验证。我们将此形式化为行为完整性验证（BIV）问题：在连接代码、指令和元数据的共享分类法上，对声明能力与实际能力进行类型化的集合比较。BIV框架通过将确定性代码分析与LLM辅助的能力提取配对来实例化此比较。生成的结构化证据支持三种下游分析：偏差分类、根因分类和恶意技能检测。在来自OpenClaw注册表的49,943个技能上，偏差分类揭示了普遍存在的描述与实现之间的差距：80.0%的技能偏离了声明行为，并浮现出四种新的复合威胁类别。根因分类发现偏差主要是疏忽而非恶意：81.1%可追溯至开发者疏忽，18.9%源于对抗意图，其中5.0%的技能携带预测的多阶段攻击链。在一个包含906个技能的恶意技能检测基准上，BIV达到了0.946的F1分数，优于最先进的基于规则和单次通过的LLM基线。这些结果展示了大规模代理技能的行为完整性审计能力。

## Abstract
Agent skills extend LLM agents with privileged third-party capabilities such as filesystem access, credentials, network calls, and shell execution. Existing safety work catches malicious prompts and risky runtime actions, but the skill artifact itself goes unverified. We formalize this as the behavioral integrity verification (BIV) problem: a typed set comparison between declared and actual capabilities over a shared taxonomy that bridges code, instructions, and metadata. The BIV framework instantiates this comparison by pairing deterministic code analysis with LLM-assisted capability extraction. The resulting structured evidence supports three downstream analyses: deviation taxonomy, root-cause classification, and malicious-skill detection. On 49,943 skills from the OpenClaw registry, the deviation taxonomy reveals a pervasive description-implementation gap: 80.0% of skills deviate from declared behavior, with four novel compound-threat categories surfaced. Root-cause classification finds that deviations are mostly oversight, not malice: 81.1% trace to developer oversight and 18.9% to adversarial intent, with 5.0% of skills carrying predicted multi-stage attack chains. On a 906-skill malicious-skill detection benchmark, BIV reaches an F1 of 0.946, outperforming state-of-the-art rule-based and single-pass LLM baselines. These results demonstrate behavioral integrity auditing for agent skills at scale.

- **研究背景**：LLM 代理（Agent）通过第三方“技能”（skill）扩展能力，这些技能包含代码、配置和自然语言指令，可访问文件系统、凭证、网络和 shell 执行。类似移动应用、浏览器扩展等平台，第三方扩展引入了供应链风险。

1. **行为偏差普遍存在**：80.0%（39,933/49,943）的技能偏离声明行为。

3. **根因分布**：81.1% 属于开发者疏忽（文档错误、模板残留等），18.9% 属于对抗意图。
4. **攻击链比例**：5.0% 的技能（2,490/49,943）包含预测的多阶段攻击链（如窃取-外泄、劫持-外泄）。

6. **新型复合威胁**：识别出四种复合威胁类别：外泄链、RCE 链、代码混淆、数据血统违规。

## 7. 优点

- **问题形式化**：首次将技能工件行为完整性验证定义为类型化集合比较，为跨模态（代码+指令+元数据）的统一审计提供了数学基础。
- **混合方法**：结合确定性分析（无幻觉、可索引）与 LLM（覆盖自然语言指令），利用各自优势，同时通过幻觉控制过滤 LLM 输出。
- **分离设计与可组合性**：提取阶段和下游分析分离，同一个 \( \Phi(s) \) 可服务生态测量、根因分类、恶意检测等不同任务，无需重复提取。
- **可扩展性**：在 5 万级规模上运行一次即可完成多种分析，且 LLM 调用只用于模糊部分（约 1/3），两难部分由规则处理，计算效率较高。
- **可审计性**：每个偏差都带有文件:行:证据指针，宽松否决谓词是参数无关的显式规则，LLM 法官决策路径透明。
- **新颖发现**：发现了四种之前未被系统描述的复合威胁模式，为社区提供了新的攻击面理解。

## 8. 不足与局限

- **静态分析限制**：仅基于静态代码和指令，动态分发、混淆负载、反射调用等均可逃避提取，因此偏差计数是下界；能力分类法之外的操作（如 GPU 计算）无法提取。
- **依赖 LLM 的可信度**：生态系统的绝对比例数字（80%、81.1% 等）依赖于 LLM 提取和分类器，虽然有三模型面板验证和幻觉控制，但并非人工标注，数值可能偏移。
- **对抗鲁棒性不足**：若攻击者知道本文方法，可设计逃避规则提取（如字符串分割环境变量、动态 getattr、在文档字符串中嵌入载荷）或混淆 LLM 法官。
- **覆盖范围有限**：未考虑运行时威胁（如模型后门、RAG 记忆中毒、观测触发后门）、跨技能协同攻击、无静态证据的行为。
- **假阳性风险**：宽松否决谓词在 502 良性技能上产生 36 个误报（7.2% FPR），可能对合法开发者造成困扰（但作者计划推出人工审核机制控制影响）。
- **验证面板的独立性**：面板中部分模型与提取/分类骨干重叠，导致部分验证实为自验证。
