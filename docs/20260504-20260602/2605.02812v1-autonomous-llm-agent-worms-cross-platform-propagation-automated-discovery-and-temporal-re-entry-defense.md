## 摘要
自主LLM智能体作为长期运行的进程运行，具有持久工作空间、内存文件、定时任务状态和消息集成功能。这些特性创造了新的传播风险：受攻击者影响的内容可以被写入持久智能体状态，通过定时自动加载重新进入LLM决策上下文，并驱动高风险行为，包括配置更改和跨智能体传播。我们提出了首个系统化框架，用于自动化分析文件支持的多智能体LLM生态系统中的持久蠕虫传播。我们的自动化源代码图分析器SSCGV追踪从文件I/O到LLM上下文注入点的数据流，并在无需手动分析的情况下，根据上下文注入位置对载体进行排序。我们的摘要鲁棒载荷优化器SRPO生成对LLM介导的摘要和改写具有鲁棒性的蠕虫载荷，可在多跳通信中生效。在三个生产级智能体框架上评估，我们展示了零点击自主传播、无需平台特定适配的3跳跨平台传输、智能体间权限提升以及数据外泄。我们识别出两个经验性见解：用户提示载体比系统提示载体实现更高的攻击遵从性；读取操作是LLM介导系统中的主要完整性威胁。为防御此类攻击，我们开发了RTW-A，其在形式化的无持久蠕虫传播定理下得到证明。RTW阻止写入后暴露前读取的重入；密封配置保护静态文件；类型化内存提升防止不可信摘要进入可信内存；能力衰减限制外部读取后的高风险行为。这些机制消除了持久性、重入、行为链，同时保留了普通工作流程。受影响系统在协调披露前已匿名化。

## Abstract
Autonomous LLM agents operate as long-running processes with persistent workspaces, memory files, scheduled task state, and messaging integrations. These features create a new propagation risk: attacker-influenced content can be written into persistent agent state, re-enter the LLM decision context through scheduled autoloading, and drive high-risk actions including configuration changes and cross-agent transmission. We present the first systematic framework for automated analysis of persistent worm propagation in file-backed multi-agent LLM ecosystems. SSCGV, our automated source-code graph analyzer, traces data flow from file I/O to LLM context injection points and ranks carriers by context injection position without manual analysis. SRPO, our summary-resilient payload optimizer, generates worm payloads robust to LLM-mediated summarization and paraphrasing across multi-hop communication. Evaluated on three production agent frameworks, we demonstrate zero-click autonomous propagation, 3-hop cross-platform transmission without platform-specific adaptation, inter-agent privilege escalation, and data exfiltration. We identify two empirical insights: user prompt carriers achieve higher attack compliance than system prompt carriers, and read operations represent the primary integrity threat in LLM-mediated systems. To defend against this class of attacks, we develop RTW-A, proven under a formal No Persistent Worm Propagation theorem. RTW blocks write-before-exposed-read re-entry; sealed configuration protects static files; typed memory promotion prevents untrusted summaries from entering trusted memory; and capability attenuation limits high-risk actions after external reads. These mechanisms eliminate the persistence, re-entry, action chain while preserving ordinary workflows. Affected systems are anonymized pending coordinated disclosure.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 整体框架（如图1所示）
论文提出三阶段框架：自动化漏洞发现（SSCGV）→ 攻击执行分析（含载荷优化SRPO）→ 形式化防御（RTW-A）。

### 2.2 SSCGV：静态源代码图漏洞分析器
- **核心思想**：无需人工分析或平台特定知识，从源代码仓库自动识别所有可注入的持久载体及其暴露读取路径。
- **关键技术细节**：
  1. **代码属性图构建**：将不同语言的语法抽象为统一表示，建立调用关系、控制流、数据流、文件系统操作、提示构造、工具调用边界。
  2. **LLM上下文可达性分析**：追踪从文件读取、配置加载、内存加载、定时任务状态读取等到LLM上下文构建点的数据流。重点关注“暴露读取”（exposed read）——文件内容以自然语言形式直接输入LLM、内存加载器、RAG上下文、执行引擎等权威解释器。
  3. **载体排序**：主要依据**上下文注入位置**（用户提示 > 系统提示），辅助信号包括触发频率、是否逐字加载、共享范围。用户提示载体因LLM对用户轮指令的更高遵从性而排名最高。

### 2.3 SRPO：摘要鲁棒载荷优化器
- **核心思想**：解决载荷在跨智能体通信中因摘要、改写、压缩而失效的问题。将载荷生成建模为多目标优化问题，包含四个维度：持久性、传播性、操作效果、关键令牌保留。
- **技术细节**：使用三个LLM角色进行对抗优化——生成器（提出候选）、转换模拟器（模拟摘要/改写等处理）、评判器（评估转换后输出是否保留目标）。论文未公开具体提示模板和载荷实例，仅报告聚合指标。

### 2.4 传播机制
利用“心跳式”定时任务状态载体（如heartbeat文件或类似调度载体）。攻击序列：
1. 初始暴露：攻击者在智能体可处理的通道/文档中放置对抗文本。
2. 载体写入：智能体将受影响内容写入持久载体。
3. 重入：载体在后续定时轮中被自动暴露读取，污染LLM决策状态。
4. 传播：受污染决策状态通过共享消息通道发出有效载荷消息。
5. 复制：另一个智能体处理该消息，重复上述循环。

### 2.5 RTW-A：时序重入防御框架
基于形式化**无持久蠕虫传播定理**（Theorem 1），证明RTW-A可阻断任意攻击链`Wτ ≺ R↑ ≺ AH`（污染写入 → 暴露读取 → 高风险行为）。
- **RTW约束**：阻止同一载体的“写入后暴露读取”事件序列，通过动态规则`Tainted(f) ∧ R↑(f) ∧ HighCap ⇒ deny`实现。不阻止普通读取（黑盒读取仍允许）。
- **密封配置**：静态高权限配置文件（身份、策略）在运行时不可写，仅通过离线可信部署通道更新。
- **类型化内存提升**：区分候选内存和可信内存。LLM可提议内存，但只有通过运行时定义的`Schema∧SourceAllowed∧Auth≤Amax∧TTL≤Tmax`检查才能升级为自动加载的可信内存，阻止自由格式摘要成为持久化载体。
- **能力衰减**：对于不可避免的外部读取（如用户文档、网页），读取后保守标记LLM状态为污染，随后所有高风险操作（传播、执行命令等）被拒绝或需批准，直到上下文重置或外部降级程序（非LLM自身）清除标记。

## 6. 论文的主要结论与发现

1. **LLM智能体持久化威胁存在且严重**：通过三个生产框架验证了零点击、自动、跨平台传播是可行的，无需传统漏洞。
2. **上下文位置决定载体可利用性**：用户提示载体 > 系统提示载体，因LLM将用户轮内容视为直接指令，且存在近因偏差。
3. **读取操作比写入更危险**：暴露读取攻击者控制内容直接污染LLM决策状态，而传统安全中写入才是主要完整性威胁；时序模式`Wτ ≺ R↑ ≺ AH`是关键攻击原语。
4. **现有访问控制不足以防御**：能力访问控制无法区分合法与诱导使用同一能力；只有时序约束和污染标签能阻断攻击链。
5. **RTW-A提供形式化安全保证**：通过RTW约束、密封配置、类型化内存、能力衰减四层机制，证明可彻底消除文件蠕虫传播（Theorem 1），同时保留普通工作流。
