## 摘要
我们认为，智能体安全必须作为系统问题来处理：驱动智能体的人工智能模型应被视为不可信组件，安全不变量必须在系统层面强制执行。通过这一视角，提高模型鲁棒性（社区的主流观点）的努力本身是不够的。相反，我们必须用来自系统安全领域的技术补充现有工作。基于我们在操作系统、网络、形式化方法和对抗机器学习领域的网络安全研究经验，我们阐述了一套植根于数十年系统安全研究的核心原则，为设计具有可预测保证的智能体系统提供了基础。作为证据，我们分析了十一个针对智能体的代表性现实世界攻击，并讨论了这些系统原则如果得以实现，本可以如何阻止这些攻击。我们还指出了在智能体中实施这些原则所面临的研究挑战。

## Abstract
We take the position that agent security must be approached as a systems problem: the AI model powering the agent must be treated as an untrusted component, and security invariants must be enforced at the system level. Through this lens, efforts to increase model robustness (the dominant viewpoint in the community) are insufficient on their own. Instead, we must complement existing efforts with techniques from the systems security domain. Based on our experience as cybersecurity researchers in operating systems, networks, formal methods, and adversarial machine learning, we articulate a set of core principles, grounded in decades of systems security research, that provide a foundation for designing agentic systems with predictable guarantees. As evidence, we analyze eleven representative real-world attacks on agents and discuss how systems principles, if realized, could have prevented these attacks. We also identify the research challenges that stand in the way of implementing these principles in agents.

### 1. 论文的核心问题与整体含义（研究动机和背景）

*   **核心问题：** 当前主流 AI 智能体（Agent）安全研究过度聚焦于提升模型自身的鲁棒性（即模型安全），但这种做法存在根本性局限。该论文的核心论断是：**智能体安全本质上是一个“系统”问题，而非单纯的“模型”问题。**

### 2. 论文提出的方法论：核心思想、关键技术细节

论文的核心思想是借鉴计算机系统安全领域数十年的成熟原则，将驱动智能体的 AI 模型视为一个 **“不可信”** 组件，并在其所在的**整个系统层面**强制执行安全不变量。具体提出了三大核心机制：

1.  **指令与数据的可证明分离 (Provable Separation of Instructions and Data)**
    *   **核心思想：** 类似于操作系统的 W⊕X（写入与执行互斥）机制，防止未信任的数据被当作指令执行。在智能体场景中，需要从根本上区分 LLM 上下文窗口中的（可信）指令和（不可信）数据，以防御提示注入攻击。
    *   **关键技术细节：** 需要寻找一种形式化、可准确实现的分离定义，并设计一个能够**确定性**地判断何时遵循新指令是安全的机制。当前的标签分隔（如 `<instruction>` 和 `<data>` 标签）已被证明可以被轻易绕过。这需要研究如何为智能体系统建立类似“控制流完整性 (CFI)”的策略。

2.  **基于最小权限的可验证策略生成 (Least-privilege Sandboxing with Verifiable Policy Generation)**
    *   **核心思想：** 遵循最小权限原则，将智能体“沙盒化”，仅赋予其完成任务所必需的最少权限。核心挑战在于将用户模糊的自然语言任务描述，自动、可靠地转化为形式化的、可被确定性引用监视器（Reference Monitor）强制执行的策略。
    *   **关键技术细节：** 关键在于解决“语义鸿沟”（Semantic Gap）。智能体的任务动态变化，其高层意图与底层工具调用之间存在抽象层次差距。因此，需要一个“翻译管道”，将自然语言策略转化为形式化策略，并且这个翻译过程必须是 **“可验证”** 的，即翻译结果（形式化策略 P）必须与原始自然语言意图（N）在逻辑上等价或近似（Mutual Entailment）。这需要解决策略的**底层基础定义（Grounding）、可检查性（Checkability）和合成过程（Synthesis）** 三大研究挑战。

3.  **信息流控制 (Information Flow Control, IFC)**
    *   **核心思想：** 沙盒化仅控制访问权限，而 IFC 进一步解决了信息在系统中长距离流动的问题。通过对数据打上来源标签，并追踪其在系统（包括模型内部）中的传播，来防止敏感数据泄露到不可信区域。
    *   **关键技术细节：** 在 LLM 中进行标签传播是核心难题。由于模型会混合不同来源的输入，输出标签会合并，导致“标签爆炸”（所有数据都被标记为“一切”）。论文提出了三个潜在方向：1) 使用**软标签（软分布标签）**：用概率量化输出携带了多少各输入数据的信息；2) **因果干预视角**：通过扰动或移除某些输入，观察其因果效应；3) **机械可解释性**：深入模型内部，获得忠诚的算法代理，以便应用 IFC。
