## 摘要
LLM智能体在多服务交互中发布私有数据。基于度量差分隐私的现有提示清理器将每次发布视为独立，因此跨轮次组合发布的对手可以恢复私有属性；每次发布都会导致隐私退化。这种退化是根本性的：当私有属性是计算图的“根”时，独立地对派生值加噪会使得根的可区分性放大至多等于派生函数的Lipschitz常数L，该常数在医疗和金融工作流中的非线性函数中可能远超名义隐私参数。RootGuard对根值进行一次清理，并随后从加噪后的根确定性计算后续发布。根据后处理定理，隐私保证仅依赖于初始根清理，无论对手的函数或轮次数量如何，派生值以零边际成本继承隐私。RootGuard进一步利用结构性领域知识（例如，根据身高和体重计算BMI，或已知目标函数）在根之间分配预算，改善隐私-效用权衡。一个迫使t轮次的最坏情况对手将总预算增加为B = t·ε。RootGuard将这个更大的预算分配到各根，而独立加噪每次发布花费ε，并给对手t个观测值通过MAP重建组合。这产生了一个“双重不对称”：更多轮次有助于RootGuard，同时削弱独立加噪。在八个NHANES医学诊断模板上，RootGuard在ε = 0.1时实现了比独立加噪低2.3至3.0倍的目标误差（在B = (2k+1)ε时，wMAPE为7.6%对比17.1%）。在MAP重建下，更多查询增强了对独立加噪的攻击，而RootGuard保持不变。

## Abstract
LLM agents release private data across multi-service interactions. Existing prompt sanitizers based on metric differential privacy treat each release independently, so adversaries combining releases across turns can recover private attributes; privacy degrades with every release.   This degradation is fundamental: when private attributes are the \emph{roots} of a computation graph, independently noising a derived value amplifies the root's distinguishability by up to the deriving function's Lipschitz constant $L$, which can far exceed the nominal privacy parameter for nonlinear functions in medical and financial workflows.   RootGuard sanitizes root values once and computes subsequent releases deterministically from the noised roots. By the post-processing theorem, the privacy guarantee depends only on the initial root sanitization, regardless of the adversary's functions or number of turns, and derived values inherit privacy at zero marginal cost. RootGuard further exploits structural domain knowledge (e.g., BMI from height and weight, or a known target function) to allocate budget across roots, improving the privacy-utility tradeoff.   A worst-case adversary forcing $t$ turns increases the total budget $B = t \cdot \varepsilon$. RootGuard distributes this larger budget across roots, while independent noising spends $\varepsilon$ per release and gives the adversary $t$ observations to combine via MAP reconstruction. This yields a \emph{double asymmetry}: more turns aid RootGuard while weakening independent noising.   On eight NHANES medical diagnostic templates, RootGuard achieves $2.3$--$3.0\times$ lower target error than independent noising at $\varepsilon = 0.1$ (7.6\% vs.\ 17.1\% wMAPE at $B = (2k{+}1)\varepsilon$). Under MAP reconstruction, more queries strengthen attacks against independent noising while RootGuard remains invariant.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题场景**：基于LLM的智能体在医疗、金融等敏感领域执行多轮多服务任务（如诊断、用药建议），每个服务都可能获取用户的私有数据（如实验室检测值）。现有隐私保护方法（如Prϵϵmpt、CAPE）基于度量差分隐私，对每次释放独立加噪。
- **关键缺陷**：在多轮交互中，独立加噪会导致隐私随轮次数量线性退化。攻击者可以通过组合多次独立观测（如反复请求同一根值或请求非线性函数派生值）来恢复用户的私有属性。该退化是根本性的：对派生值独立加噪会使得根属性的可区分性放大至多等于派生函数的Lipschitz常数L，对于常见医疗公式（如1/x，BMI计算）L可达数百倍，远超名义隐私参数ε。
- **研究动机**：现有方法无状态、无依赖感知，无法在多轮场景下保持隐私保证。因此需要一种依赖感知的机制，使隐私消耗不随交互轮次增长。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：RootGuard在初始化时对用户私有的**根属性**（如身高、体重、血糖等）进行一次加噪，并将加噪后的值缓存。后续所有来自服务的查询（包括对根值的重复请求、对派生函数的计算、最终任务计算）都**确定性**地从缓存值中派生，不再引入新噪声。
- **关键技术细节**：
  - **图结构建模**：将交互过程建模为有向无环图（DAG），根节点为私有值，派生节点为函数计算结果。
  - **加噪机制**：使用度量差分隐私（mDP），支持三种机制：Discrete Exponential、Bounded Laplace、Staircase。所有值映射到统一的索引空间（0 ~ m-1）以标准化敏感度。
  - **后处理定理保证**：根据差分隐私的后处理定理，所有派生值的隐私保护级别由初始根加噪的参数决定，与攻击者选择的函数或交互轮次无关。派生值以**零边际成本**继承隐私。
  - **领域知识利用**：
    - **Level 1：根间依赖**：如果某些根属性（如BMI）可从其他根（身高、体重）计算，则只需对独立根加噪，依赖根通过后处理得到，节省预算。
    - **Level 2：目标函数敏感度**：若已知任务中间函数g（如FIB-4公式），可在根间进行敏感度加权预算分配：εᵢ ∝ √(|hᵢ|·Dᵢ)，其中hᵢ为g对根i的偏导数（在群体均值处评估），Dᵢ为根域宽度。这能使目标误差最小化。
  - **预算分配公式**：在Laplace连续近似下，最优分配为εᵢ ∝ √(|hᵢ|·δᵢ)，满足总预算B = Σεᵢ。对于离散机制，通过数值优化求解。
- **双重不对称性**：当攻击者迫使t轮交互时，总隐私预算B = t·ε。RootGuard将该更大预算分配到k个根上（每个根噪声减小，效用提升），而独立加噪每次释放花费ε，且每个新观测给攻击者提供MAP重建的新信息。因此，更多轮次同时改善RootGuard的效用并加剧独立加噪的隐私泄露。

## 3. 实验设计

- **数据集**：使用美国CDC的NHANES 2017-2018公开数据集，包括身体测量、血压、血液学、胆固醇、血糖、胰岛素等。选取8个医学诊断模板（ANEMIA, FIB4, AIP, CONICITY, VASCULAR, TYG, HOMA, NLR），每个模板定义k个根属性和一个目标临床指数函数g。每个模板抽取200名成年男性患者用于RQ1-RQ3，前100名用于RQ4部署评估。
- **Benchmark**：对比三种配置：
  - **M-All**（独立加噪基线）：每次释放（包括根值和派生值）都独立添加噪声，每次释放消耗ε预算。
  - **M-Roots**（根一次性加噪，均匀分配预算）：只有根值加噪一次，后续所有派生值确定性计算，预算在k个根上均匀分配。
  - **M-Opt**（完整RootGuard）：根一次性加噪，预算按敏感度加权分配。
- **对比方法**：还包含文本DP方法CAPE-F作为参考，但不直接可比。
- **威胁模型**：攻击者控制t轮交互，前t-1轮可请求任意函数（重复根、非线性派生），第t轮请求所有根以完成目标计算。攻击者知道所有噪声机制和参数，可进行最大后验（MAP）重建。
- **评估指标**：加权平均绝对百分比误差（wMAPE）和风险类别误差（RCE），后者直接反映临床决策改变的比例。

## 6. 主要结论与发现

- RootGuard在目标效用上显著优于独立加噪：在ε=0.1时，wMAPE降低2.6~3.8倍（取决于预算级别），在更强隐私（ε=0.01）下优势更大。

- 对零敏感度根（如BMI中的RBC），RootGuard自动收回其预算分配给其他根，进一步提高效用。
- 部署实验完全复制了机制级结果，说明方法在实际LLM工具调用环境中同样有效。

## 7. 优点

- **零边际成本隐私继承**：通过后处理定理，所有派生值自动获得与根相同的隐私保障，不消耗额外预算。
- **双重不对称性**：更多攻击轮次反而帮助RootGuard（预算集中，效用提升）并伤害独立加噪（观测更多，隐私退化）。这一性质天然增强了对多轮攻击的鲁棒性。
- **利用公共领域知识**：通过识别根间依赖和目标函数敏感度，可进一步优化预算分配，而不影响隐私保证（因为知识是公开的）。
- **形式化隐私保障**：提供了完整的DAG隐私游戏和定理1-3，证明了RootGuard的隐私绑定仅依赖于根加噪参数，与攻击者函数选择无关。
- **全面的实验评估**：覆盖多种机制、攻击策略、临床模板，并在真实LLM部署中验证，结果具有强说服力。

## 8. 不足与局限

- **假设完美NER**：论文假设能完美识别出所有敏感根（如姓名、实验室值），这是其他清理器（Prϵϵmpt）的共同假设，但实际中NER可能有误，目前未将NER错误纳入隐私分析。
- **仅限于结构化数值**：方法设计针对数值型的根值（如身高、体重、检测值），未扩展到自由文本或非结构化数据。如果用户的根包含自由文本描述（如症状描述），则无法直接应用。
- **单用户会话**：实验仅涉及单个用户与一个服务之间的多轮交互，未考虑多用户共享数据、服务之间串通等更复杂场景。
- **已知目标函数假设**：Level 2优化要求已知中间函数g（如临床公式），若g完全未知或服务采用黑盒模型，则敏感度分配不可行，只能退回Level 1或均匀分配。论文对此有讨论。
- **非线性派生函数攻击未充分探索**：在重建实验中使用的是直接根查询，未测试非线性派生函数查询（理论上可进一步放大攻击），但作者指出这对RootGuard不构成问题，且使M-All的下界更宽松。
- **算力与成本未报告**：尽管部署实验调用GPT-5.4 nano，但未提供详细的计算资源消耗，限制了可复现性。
