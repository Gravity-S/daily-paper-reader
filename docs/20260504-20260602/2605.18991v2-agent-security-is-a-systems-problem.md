## 摘要
我们认为，智能体安全必须作为一个系统问题来对待：驱动智能体的人工智能模型必须被视为不可信组件，并且安全不变性必须在系统级别强制执行。从这个角度来看，提高模型鲁棒性的努力（社区的主流观点）本身是不够的。相反，我们必须用系统安全领域的技术来补充现有的努力。基于我们在操作系统、网络、形式化方法和对抗性机器学习方面的网络安全研究经验，我们阐述了一套以数十年系统安全研究为基础的核心原则，为设计具有可预测保证的智能体系统提供了基础。作为证据，我们分析了十一个具有代表性的针对智能体的真实世界攻击，并讨论了如果系统原则得以实现，这些攻击如何能够被阻止。我们还指出了在智能体中实施这些原则所面临的研究挑战。

## Abstract
We take the position that agent security must be approached as a systems problem: the AI model powering the agent must be treated as an untrusted component, and security invariants must be enforced at the system level. Through this lens, efforts to increase model robustness (the dominant viewpoint in the community) are insufficient on their own. Instead, we must complement existing efforts with techniques from the systems security domain. Based on our experience as cybersecurity researchers in operating systems, networks, formal methods, and adversarial machine learning, we articulate a set of core principles, grounded in decades of systems security research, that provide a foundation for designing agentic systems with predictable guarantees. As evidence, we analyze eleven representative real-world attacks on agents and discuss how systems principles, if realized, could have prevented these attacks. We also identify the research challenges that stand in the way of implementing these principles in agents.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：当前智能体安全领域的主流观点是专注于提升AI模型本身的鲁棒性（如通过对齐训练防御提示注入），但作者认为这种“模型中心”方法存在根本局限：经典对抗性机器学习已反复证明模型级防御易被攻击者绕过，且LLM时代同样出现绕过现象，攻击者无需付出更高成本。智能体系统远比视觉分类器复杂——LLM作为更大系统（智能体脚手架、环境、工具）的一部分工作，这为引入系统安全技术提供了独特机会。
- **核心问题**：如何设计能够提供可预测安全保障（predictable guarantees）的智能体系统？作者主张将智能体安全视为一个**系统问题**：将AI模型视为不可信组件，在**系统层面**强制执行安全不变式，而非仅依赖模型内在鲁棒性。
- **背景**：计算机安全领域数十年研究已形成从不可靠组件构建安全系统的原则（最小权限、可信计算基防篡改、完全中介、安全信息流、考虑人类弱点）。作者将这些原则迁移至智能体场景，并通过分析11个真实攻击案例验证其有效性。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **将AI模型视为不可信组件**：类比操作系统将进程视为不可信，智能体安全应在模型外部、在系统层面（例如智能体框架、工具调用、数据流）实施安全策略。
- **三个关键系统级安全机制**：

#### (1) 指令与数据可证明分离（Provable Separation of Instructions and Data）
- **问题类比**：类似CPU的W⊕X（写和执行互斥）、数据库的预编译语句（prepared statements）防止SQL注入。LLM将可信指令与不可信数据混合在同一上下文窗口，使得恶意数据可被解释为指令（提示注入）。
- **核心思路**：需要在系统层面强制分离，而非依赖模型内部区分。当前方法（如Instruction Hierarchy、结构化查询）已被证明可被自适应攻击绕过。作者推测模型级分离防御永远无法抵抗持久攻击者，需采用深度防御：通过后两个机制补充。
- **开放挑战**：①精确定义什么是指令/数据分离；②能否构建可证明遵循此分离的LLM（实用、规模、成本）；③当前定义是否覆盖所有提示注入变种；④如何处理动态行为（智能体需要从数据中学习新指令，如跟随网页链接）。

#### (2) 最小权限沙箱与可验证策略生成（Least-Privilege Sandboxing with Verifiable Policy Generation）
- **问题类比**：传统系统中沙箱隔离不可信组件，用最小权限策略限制资源访问。但智能体任务非固定，策略需用模糊自然语言描述且动态变化。
- **核心思路**：构建将自然语言策略自动翻译为形式化、可强制执行的最小权限策略的管道，并**配对形式化验证**以确保策略正确性。策略需在一个固定动作空间上表达（如系统调用、API端点）。
- **开放挑战**：①**基础层（Grounding）**：需要编码智能体动作空间及策略引用的概念（如“敏感”、“可信”）；②**可检查性（Checkability）**：即使基础层固定，语义蕴含检查仍是概率性的，需找到结构约束使检查可靠；③**合成过程（Synthesis Procedure）**：翻译器需提供可衡量的偏离界限，支持增量细化。

#### (3) 信息流控制（Information Flow Control, IFC）
- **问题类比**：访问控制只决定能否访问资源，不控制资源使用后的传播。IFC通过标签跟踪数据流动并强制策略（如敏感数据不可流向不可信输出）。
- **核心思路**：标记数据源并传播标签。但在LLM中，多源数据拼接后输入模型，当前只能假设输出标签是所有输入标签的并集，导致标签爆炸。
- **开放挑战**：①**标签传播**：需要更精细方法——软/分布式标签（定量信息流）、因果干预（量化输入因果效应）、机理可解释性（构建忠实算法代理）；②**降级（Downgrading）**：解分类（declassification）和认可（endorsement）解决标签爆炸，需结合人类参与和最小侵入；③**标签粒度**：传统IFC有自然抽象层（变量/文件/进程级），LLM输入为无结构token序列，需利用现有结构（如对话中的system/user/tool标签或web origin）。

## 6. 论文的主要结论与发现

- **核心结论**：智能体安全必须作为系统问题处理，模型鲁棒性本身不可靠，需用系统安全原则互补。三个关键技术（指令/数据分离、可验证策略生成、信息流控制）若实现，可消除大量攻击。
- **关键发现**：①所有11个攻击都违反了至少两个安全原则，多违反安全信息流和最小权限；②智能体系统缺乏传统系统的分层抽象边界，导致安全策略动态不确定、语义鸿沟（semantic gap）；③直接使用LLM作为参考监视器不可靠，因为其非确定性且可被对抗输入欺骗。
- **对社区的号召**：应转向系统安全视角，攻克提出的研究挑战（如何分离指令/数据、如何可靠地将自然语言策略转化为形式化策略、如何实现LLM中的标签传播）。

## 7. 优点

- **创新视角**：打破“模型中心”主流，将数十年系统安全成熟原则系统性地引入智能体安全领域，提供了新的解决框架。
- **务实性**：承认完全可靠的模型级分离无法实现，提倡深度防御和异构强制（不同抽象层采用不同机制），增加攻击者成本。
- **案例丰富性**：分析11个真实攻击，具体展示每个原则如何被违反，使得原则的实用价值直观。
- **指出开放挑战**：不仅提出原则，还明确列出实现这些原则所面临的具体研究问题（标签传播、策略翻译验证等），为后续研究提供路线图。
- **考虑人类弱点**：将“人类弱链接”纳入原则，强调安全机制需考虑用户和开发者的失败模式，全面性较好。
