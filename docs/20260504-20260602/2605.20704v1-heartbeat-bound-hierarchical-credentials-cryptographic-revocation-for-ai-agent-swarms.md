## 摘要
产生子代理集群的自主AI代理存在安全漏洞：现有凭证撤销机制（OAuth 2.0内省、OCSP和W3C状态列表）需要与中央权威机构建立网络连接，导致操作者关闭后，“僵尸代理”仍可在数分钟至数小时内执行特权操作。我们提出心跳绑定分层凭证（HBHC），这是一种将凭证有效性绑定到定期父节点活跃性证明的密码学协议。验证者仅使用缓存的公钥和本地时钟即可强制执行新鲜度，无需网络往返。当心跳生成停止时，所有后代凭证将在确定性有界窗口W_z ≤ W_max + Δ_h + ε内变得不可用，前提是时钟偏差有界且父节点密钥存储在安全飞地中。在协议层和基于真实LLM的代理集群（GPT-4o-mini）上的评估表明，与OAuth 2.0相比，僵尸窗口减少了90倍，Rust中完整认证仅需0.26毫秒，并发HTTP负载下每秒可完成18,000次以上验证，代理数量从10到10,000时单次验证延迟保持稳定。真实代理实验显示，工具调用的端到端开销为0.71%，在绕过应用层防护的提示注入攻击下，撤销后零次工具调用，并在理论界限内实现了49个代理四层层次结构的级联撤销。

## Abstract
Autonomous AI agents that spawn sub-agent swarms create a safety gap: existing credential revocation mechanisms, OAuth~2.0 introspection, OCSP, and W3C Status Lists, require network connectivity to a central authority, leaving ``zombie agents'' executing privileged operations for minutes to hours after operator shutdown. We present Heartbeat-Bound Hierarchical Credentials (HBHC), a cryptographic protocol that binds credential validity to periodic parent liveness proofs. Verifiers enforce freshness using only a cached public key and local clock; no network round-trip is required. When heartbeat generation ceases, all descendant credentials become unusable within a deterministically bounded window $W_z \le W_{\max} + Δ_h + ε$, conditional on bounded clock skew and parent keys held in secure enclaves. Evaluation at the protocol layer and with real LLM-backed agent swarms (GPT-4o-mini) demonstrates a 90$\times$ reduction in the zombie window over OAuth~2.0, 0.26~ms full authentication in Rust, 18,000+ verifications per second under concurrent HTTP load, and stable per-verification latency from 10 to 10,000 agents. Real-agent experiments show 0.71\% end-to-end overhead on tool calls, zero post-revocation tool calls under prompt injection that bypasses application-layer guardrails, and cascading revocation across a 49-agent four-level hierarchy within the theoretical bound.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：AI 代理（agent）会动态生成子代理集群，父代理关闭后，子代理持有的凭证（如 OAuth 2.0 令牌、X.509 证书）仍可在分钟至小时内继续执行特权操作，形成“僵尸代理”安全漏洞。
- **现有机制缺陷**：OAuth 2.0 内省、OCSP、W3C 位串状态列表等均需网络连接中央权威节点，在分布式部署或网络分区下无法保证即时撤销；短生命周期证书依赖 CA 续签，仍有分钟级僵尸窗口。
- **背景**：欧盟 AI 法案要求高风险系统具备人类监督与关闭机制，而现有身份层撤销手段无法满足离线确定性终止需求。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **凭证有效性绑定到父代理的周期性心跳签名**，子代理必须携带新鲜父节点心跳才能通过认证；当父代理被关闭后，心跳生成停止，所有后代凭证自动过期，无需撤销消息到达验证者。

### 关键技术细节
1. **密钥层次**：
   - 每个代理拥有三组密钥：身份密钥对（sk_i, pk_i）、心跳密钥对（hsk_i, hpk_i，从身份密钥硬化派生）、子派生密钥（cdk_i，BIP-32 链码）。
   - 子代理的身份密钥由父代理的 cdk 和子 ID 确定性派生。
   - 子凭证包含心跳绑定哈希 `H(hpk_p || id_c)`，将子代理束缚到特定父节点的心跳公钥。

2. **心跳生成**：
   - 父代理每间隔 Δ_h 生成一个签名心跳（包括 epoch、承诺、签名），epoch = floor(t / Δ_h)。
   - 承诺 `H(hpk_p || epoch)` 防止心跳移植与重放。

3. **认证与验证**：
   - 子代理构造认证证明：`(cred_c, epoch, σ_h, σ_c)`，其中 σ_c 是对挑战+心跳的签名。
   - 验证者（verifier）仅使用本地缓存的父节点心跳公钥、本地时钟和挑战进行四步验证：心跳新鲜度检查、心跳签名验证、绑定哈希匹配、子签名验证。
   - 无网络往返即可判断凭证是否有效。

4. **僵尸窗口理论界限**：`W_z ≤ W_max + Δ_h + ε`，其中 W_max 为可配置最大心跳年龄，Δ_h 心跳间隔，ε 时钟偏差。

5. **撤销方式**：
   - **隐式撤销**：父停止心跳 → 所有后代自然过期（主要机制）。
   - **显式撤销**：父广播“撤销心跳”通知子代理主动关闭。
   - **选择性撤销**：父停止向特定子代理分发心跳。

6. **可用性缓解**：预计算缓存、可配置宽限期、多路径心跳分发，将假阳性拒绝率降至 0.01%（极端丢包下）。
