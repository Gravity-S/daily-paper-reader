## 摘要
长程LLM智能体留下的轨迹可以成为可重用的经验，但原始轨迹嘈杂且难以治理。我们将智能体技能视为一种经验模式，它将可执行脚本与关于程序的非可执行指导相结合。然而，开放的技能生态系统包含冗余、不均衡、对环境敏感的工件，且不加区分的更新可能会污染未来的上下文。我们提出SkillsVote，一个从收集、推荐到演化的智能体技能生命周期治理框架。SkillsVote对百万规模的开源语料库进行环境需求、质量和可验证性分析，然后合成为可验证技能的任务。在执行前，SkillsVote在结构化技能库上执行智能体库搜索以暴露指导性技能上下文。执行后，它将轨迹分解为与技能相关的子任务，将结果归因于技能使用、智能体探索、环境和结果信号，并且仅允许成功的可重用发现进入证据门控的更新。在我们的评估中，离线演化使GPT-5.2在Terminal-Bench 2.0上提高了最多7.9个百分点，而在线演化使SWE-Bench Pro提高了最多2.6个百分点。总体而言，当系统控制暴露、归因和保存时，受治理的外部技能库可以改进冻结的智能体而无需模型更新。

## Abstract
Long-horizon LLM agents leave traces that could become reusable experience, but raw trajectories are noisy and hard to govern. We treat Agent Skills as an experience schema that couples executable scripts, with non-executable guidance on procedures. Yet open skill ecosystems contain redundant, uneven, environment-sensitive artifacts, and indiscriminate updates can pollute future context. We present SkillsVote, a lifecycle-governance framework for Agent Skills from collection and recommendation to evolution. SkillsVote profiles a million-scale open-source corpus for environment requirements, quality, and verifiability, then synthesizes tasks for verifiable skills. Before execution, SkillsVote performs agentic library search over structured skill library to expose instructional skill context. After execution, it decomposes trajectories into skill-linked subtasks, attributes outcomes to skill use, agent exploration, environment, and result signals, and admits only successful reusable discoveries to evidence-gated updates. In our evaluation, offline evolution improves GPT-5.2 on Terminal-Bench 2.0 by up to 7.9 pp, while online evolution improves SWE-Bench Pro by up to 2.6 pp. Overall, governed external skill libraries can improve frozen agents without model updates when systems control exposure, credit, and preservation.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将智能体技能定义为结构化的经验模式（耦合可执行脚本与非可执行过程指导），并对技能的整个生命周期（收集、分析、推荐、执行、归因、演化）进行治理。核心闭环：预任务推荐 → 任务内执行 → 后任务归因 → 受控库演化。
- **关键技术细节**：
  - **开放技能语料收集与概要分析**：从GitHub收集百万级SKILL.md文件，每个技能视为一个目录包（含SKILL.md、scripts/、references/、assets/等）。对每个技能进行三维概要：运行时需求（OS、权限、网络等）、质量（一致性、完整性、任务导向）、可验证性（是否存在低歧义成功条件、可复现沙箱环境）。
  - **可验证任务合成**：对通过可验证性分析的技能，从技能本身合成任务（含清晰指令、可复现环境、可执行验证器），运行真实智能体并记录成功率、成本、轨迹等。
  - **技能推荐：智能体库搜索**：基于任务条件，在结构化技能库中进行多步搜索（使用Glob、Grep等文件系统原生工具，类似DCI、CodeScout中的交互式检索），输出一组精选技能及简洁的使用指南，而非暴露整个库或仅依赖元数据。
  - **轨迹分解与子任务级归因**：执行后将轨迹分解为语义完整的子任务（subtask），每个子任务有独立目标、一个主要评估信号、至多一个关联技能。归因结果分为成功类（技能使用+额外探索、未用技能、未见技能）和失败/不确定类，并记录可复用增量（缺失步骤、前提条件、恢复模式等）。
  - **基于证据的受控演化**：仅当子任务成功且包含可复用探索时，才允许触发库更新。更新前进行聚合（合并支持同一改进的证据），然后路由到编辑（修复错误、添加知识、添加前提条件）或创建新技能。编辑保持最小改动，创建技能需独立可复用且避免任务特定常数。

## 6. 论文的主要结论与发现

- **离线演化**：从48个历史任务轨迹中蒸馏出的冻结技能库，在Terminal-Bench 2.0上显著提升GPT-5.2（+7.9 pp）、GPT-5.4 mini（+5.8 pp），表明轨迹衍生的技能可以形成有用的冷启动库，而非仅拟合源任务。
- **在线演化**：从空库开始，在线演化在Terminal-Bench 2.0上提升GPT-5.2（+2.7 pp）、GPT-5.4 mini（+1.1 pp），在SWE-Bench Pro上提升GPT-5.2（+2.6 pp）、GPT-5.4 mini（+2.1 pp）。增益虽小但一致，表明在线技能积累有用但受早期任务证据影响。
- **推荐效果**：推荐主要作为噪声过滤器，在在线早期防止不相关或低质量技能进入上下文；在离线设置中进一步提升正贡献占比（+15.3% vs +11.3%），降低负贡献（−2.0% vs −3.3%）。
- **迁移本质**：技能转移的是可复用的执行不变量（如持久化服务配置、端到端验证），而非任务特定常数或答案。

## 7. 优点：方法或实验设计上的亮点

- **全生命周期治理**：首次系统地将技能收集、概要、推荐、归因、演化整合为一个闭环，控制技能“输入”和“输出”两个易污染环节。
- **子任务级归因粒度**：在完整轨迹和单步之间引入子任务作为演化的最小单位，既能够定位责任，又保留可复用的过程知识。
- **证据门控演化**：通过可复用探索、成功结果、关联技能等条件严格限制哪些经验能进入库，避免环境噪声或试错被误记为技能。
- **启发式推荐**：使用文件系统原生工具（Glob、Grep）进行多步搜索，而非单纯依赖语义相似度，更贴近实际工程搜索模式。
- **可迁移性验证**：分离源任务性能与库迁移性能，证明演化积累的是可复用过程而非源任务过拟合。
- **开源生态构建**：构建并公开百万级技能语料库和评估平台（skills.vote），促进社区研究。
