## 摘要
随着大型模型从对话助手演变为自主代理，长期决策、工具使用和真实环境交互带来的挑战日益增加。现有的自主基础设施在评估、数据管理和代理进化方面仍然分散，使得系统性地发现风险并持续闭环改进模型变得困难。在本报告中，我们提出了Safactory，一种用于可信赖自主智能的可扩展代理工厂。Safactory集成三个紧密耦合的平台：用于轨迹生成的并行仿真平台、用于轨迹存储和经验提取的可信数据平台，以及用于异步强化学习和策略蒸馏的自主进化平台。据我们所知，Safactory是第一个为下一代可信赖自主智能提出统一进化管线的框架。

## Abstract
As large models evolve from conversational assistants into autonomous agents, challenges increasingly arise from long-horizon decision making, tool use, and real environment interaction. Existing agenticinfrastructure remain fragmented across evaluation, data management, and agent evolution, making it difficult to discover risks systematically and improve models in a continuous closed loop. In this report, we present \textbf{Safactory}, a scalable agent factory for trustworthy autonomous intelligence. Safactory integrates three tightly coupled platforms: a \textbf{Parallel Simulation Platform} for trajectory generation, a \textbf{Trustworthy Data Platform} for trajectory storage and experience extraction, and an \textbf{Autonomous Evolution Platform} for asynchronous reinforcement learning and on-policy distillation. As far as we know, Safactory is the first framework to propose a unified evolutionary pipeline for next-generation trustworthy autonomous intelligence.

## 1. 核心问题与研究动机
- **背景**：大语言模型从“对话助手”演变为“自主智能体”，承担长程决策、工具调用、真实环境交互等任务，安全风险从有害输出扩展到“感知→决策→执行→反馈”全过程累积的隐蔽问题（如工具误用、状态漂移、级联错误等）。
- **问题**：现有智能体基础设施在**评估、数据管理、模型进化**三方面碎片化：评估以静态问答或单轮对抗为主，无法捕获多步交互中的风险；数据仅作为运行时副产物，缺乏结构化积累与闭环反馈；训练、评估、部署脱节，无法持续修复和强化能力。
- **目标**：构建一个可扩展、闭环的智能体工厂 **Safactory**，实现“执行中发现风险→数据中沉淀证据→进化中完成修复→再执行验证效果”的持续迭代。

## 2. 方法论
Safactory 包含三个紧密耦合的平台：

### 2.1 并行仿真平台（Parallel Simulation Platform）
- **核心思想**：通过预温环境池、异步调度、“回溯沙箱”（类似 Git 版本管理）实现大规模高并发轨迹生成与多路径安全评估。
- **关键技术**：
  - 环境池预热 + ActorPool 并发创建 Agent 实例。
  - 弹性大模型网关统一处理请求、动态扩缩容。
  - “回溯沙箱”自动生成检查点，支持决策分支回滚与多路径探索。
  - 支持轻量级 Agent（Python 源码挂载）和复杂 Agent（Docker 镜像挂载）。
- **闭环**：所有交互轨迹自动记录为可复用的数据资产。

### 2.2 可信数据平台（Trustworthy Data Platform）
- **核心思想**：以数据资产为核心，通过意图驱动的执行引擎 **DataElf** 实现数据任务的自然语言描述→动态管线编排→执行→审计→工具进化。
- **关键技术**：
  - 双层架构：数据基础设施层（LanceDB 多模态湖仓，支持 PB 级存储与多层缓存） + DataElf 引擎。
  - DataElf 包含 Planner、Optimizer、Judge 智能体，支持“监督模式”（单次规划执行）和“自主驾驶模式”（持续试错、自主衍生新工具）。
  - 内置 30+ 工具集：数据价值评估（9 种评分器 + 聚类选择）、科学数据处理、安全审计（23 个检测器覆盖 13 类风险）、轨迹分析（经验蒸馏）。
- **安全机制**：隔离执行沙箱，数据“可用不可见”。

### 2.3 自主进化平台（Autonomous Evolution Platform）
- **核心思想**：以强化学习训练循环为核心，解耦 Rollout 与 Training，引入版本元数据、组一致性约束和轨迹陈旧度过滤，支持完全异步采样与策略更新。
- **关键技术**：
  - **Bridge 模块**：LLM Proxy + Rollout Manager，统一推理入口，记录轨迹与模型版本元数据，支持数据陈旧度过滤（可配置允许的版本差异）。
  - **Trajectory Transformation 模块**：增量编码、前缀匹配、Mask Painter，将原始轨迹转换为符合训练格式的样本。
  - **Buffer Server**：异步出队，平衡生产与消费速率。
  - **OPD（On-policy Distillation）**：可选白盒、token 级策略蒸馏，在优势驱动更新中加入反向 KL 正则项，提升训练稳定性。
- **流程**：Rollout Manager 持续生成轨迹 → Bridge 写入 Buffer Server → Training Endpoint 按需取数据进行更新。

## 6. 主要结论与发现
- **安全分层**：不同环境下能力与安全的关系不同——桌面 OS 呈“权衡”，Android 呈“正相关”，Embodied 整体瓶颈，OpenClaw 能力高但安全低。
- **数据价值**：DataElf 复合评分在 AlpacaEval 2.0、GSM8K、MT-Bench 上均优于单一评分器，且以 1/5 数据超越全量基线。
- **安全审计**：DataElf 通过规则+模型+LLM+启发式四类检测器，平均召回率（80.46%）远超最佳基线（DeepEval 48.62%），对数据投毒类风险（如标签翻转、后门）有独特优势。
- **轨迹技能**：经验注入能显著提升 OS 环境安全率（从 32.17% 到 71.08%）。
- **RL 异步**：Safactory 的完全异步架构相较 Slime-native，在训练步时间相近下，能生成更多轨迹（平均 1277.78 vs 512），减少推理侧闲置。

## 7. 优点
1. **系统集成度高**：首次将评估、数据、进化三大环节统一为闭环基础设施，避免碎片化。
2. **可扩展性强**：模块化设计支持低成本添加新环境（仅需环境适配器）、新工具（标准化接口+一键测试）。
3. **数据驱动闭环**：所有执行轨迹结构化存储为可复用资产，支持 SFT、RL 和故障分析。
4. **安全工具丰富**：内置 23 个安全检测器覆盖 13 类风险，数据价值评估支持 9 种评分器+聚类选择。
5. **高效异步训练**：通过陈旧度过滤和完全异步架构，在复杂 agent 任务中提升资源利用率。
6. **开源贡献**：发布 SATraj-OS 数据集（10,496 条轨迹，覆盖能力与安全任务），促进社区研究。

## 8. 不足与局限
1. **环境覆盖有限**：当前主要支持 OS、Android、Minecraft、Robotics 等，未覆盖高价值工业、医疗、金融等场景。
2. **自动评判困难**：长程任务中自动评判安全与效用仍困难，论文未提供强化的安全度量或人工监督机制。
3. **训练收益不稳定**：OPD 和异步 RL 策略在复杂信用分配和分布偏移下的稳定性未深入讨论，仅一个实验验证。

7. **技能注入实验局限**：仅在 Kimi K2.5 上验证，未推广到其他模型，且安全率提升可能因模型已有良好对齐而高估。
